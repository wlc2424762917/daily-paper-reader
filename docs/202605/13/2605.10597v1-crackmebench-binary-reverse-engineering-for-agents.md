## 摘要
针对编程智能体的基准测试越来越多地衡量源码级软件修复，而网络安全基准测试则越来越多地衡量广泛的夺旗赛（CTF）表现。经典的二进制逆向工程在定义上仍不够精确：在仅提供可执行文件的情况下，智能体能否恢复验证逻辑并生成程序所接受的输入、序列号、工件或密钥生成器？我们推出了 CrackMeBench，这是一个用于在教育类 CrackMe 风格逆向工程任务上评估语言模型智能体的基准测试。CrackMeBench 专注于具有可执行 Oracle、缺少符号表的二进制文件、显式本地工具访问权限以及外部评分提交（而非自由形式的解释）的确定性二进制验证问题。v0 版本基准测试结合了 8 个公开的校准 CrackMe 任务和 12 个基于 C、Rust 和 Go 模板生成的评分任务；智能体在无网络的 Linux Docker 沙箱中通过统一的 shell 界面运行，并配备标准逆向工程工具。在包含三个模型、五分钟预算且每个任务有三次评分提交机会的评估中，GPT-5.5 在生成任务集上的 pass@3 为 11/12 (92%)，Claude Opus 4.7 为 7/12 (58%)，Kimi K2 为 5/12 (42%)。难度较高的生成任务部分更显著地拉开了模型间的差距，pass@3 分别为 5/6、2/6 和 1/6；在包含 8 个任务的公开校准集上，pass@3 分别为 3/8、2/8 和 1/8。CrackMeBench 记录了 pass@1 和 pass@3、评分提交、实际运行时间、命令追踪、工具类别、供应商报告的 token 使用量、预估成本以及定性失败标签，为衡量从源代码推理向自主二进制分析的进展提供了一个可重复的测试平台，同时将范围限制在教育用途的专用程序内。

## Abstract
Benchmarks for coding agents increasingly measure source-level software repair, and cybersecurity benchmarks increasingly measure broad capture-the-flag performance. Classical binary reverse engineering remains less precisely specified: given only an executable, can an agent recover validation logic and produce an input, serial, artifact, or key generator accepted by the program? We introduce CrackMeBench, a benchmark for evaluating language-model agents on educational CrackMe-style reverse-engineering tasks. CrackMeBench focuses on deterministic binary validation problems with executable oracles, symbol-poor binaries, explicit local tool access, and externally scored submissions rather than free-form explanations. The v0 benchmark combines eight public calibration CrackMes with twelve generated main-score tasks built from seeded C, Rust, and Go templates, and agents run through an equal shell interface in a no-network Linux Docker sandbox with standard reverse-engineering tools. In a three-model evaluation with a five-minute budget and three scored submissions per task, pass@3 on the generated split is 11/12 tasks (92%) for GPT-5.5, 7/12 (58%) for Claude Opus 4.7, and 5/12 (42%) for Kimi K2. The harder generated half separates the models more sharply, with pass@3 of 5/6, 2/6, and 1/6, respectively; on the eight-task public calibration split, pass@3 is 3/8, 2/8, and 1/8. CrackMeBench records pass@1 and pass@3, scored submissions, wall-clock time, command traces, tool categories, provider-reported token usage, estimated cost, and qualitative failure labels, providing a reproducible testbed for measuring progress from source-code reasoning toward autonomous binary analysis while restricting scope to educational, purpose-built programs.

#### 1. 论文的核心问题与整体含义（研究动机和背景）
论文指出，现有的 AI 智能体基准测试主要集中在源代码级别的修复（如 SWE-bench）或广泛的网络安全夺旗赛（CTF）任务（如 Cybench）。然而，**经典的二进制逆向工程（Reverse Engineering, RE）**能力尚未得到精确衡量。逆向工程要求智能体在没有源代码的情况下，仅通过分析编译后的可执行文件来恢复验证逻辑，并生成正确的输入、序列号或密钥生成器。这一过程挑战了智能体对底层架构的理解、对剥离符号表代码的推理以及对反编译输出的批判性分析能力。

#### 2. 论文提出的方法论
论文提出了 **CrackMeBench**，一个专门用于评估语言模型智能体在二进制验证任务上表现的基准测试。
*   **核心思想：** 采用“可执行 Oracle（Executable Oracle）”机制。评价标准不是智能体给出的文字解释，而是其提交的答案（密码、工件或脚本）能否通过原始程序的验证。
*   **关键技术细节：**
    *   **沙盒环境：** 智能体在无网络的 Linux x86-64 Docker 容器中运行，确保安全性和可重复性。
    *   **显式工具访问：** 系统会向智能体提供一份明确的已安装工具清单（如 Ghidra, radare2, angr, gdb, Python 逆向库等），避免智能体盲目猜测环境。
    *   **任务抽象：** 任务分为三类提交形式：`password`（固定字符串）、`artifact`（二进制文件）和 `keygen`（生成序列号的脚本）。
    *   **交互协议：** 智能体通过 shell 命令与环境交互，最终提交一个包含答案的 JSON 对象。

#### 3. 实验设计
*   **数据集/场景：** 包含 20 个任务，分为两个部分：
    *   **公开校准集（8个）：** 选自 Crackmes.one 的真实教育任务，涵盖 C++、Rust、汇编等。
    *   **生成任务集（12个）：** 由作者基于 C、Rust 和 Go 模板生成，包含不同难度（核心级与困难级），涉及混淆、不透明谓词、简单虚拟机、加密逻辑等。
*   **对比方法（评估模型）：**
    *   **GPT-5.5** (Azure OpenAI, xhigh reasoning effort)
    *   **Claude Opus 4.7** (Anthropic, adaptive thinking)
    *   **Kimi K2** (Moonshot, kimi-k2-0711-preview)
*   **Benchmark 指标：** 主要衡量 `pass@1` 和 `pass@3`（在 3 次提交机会内的成功率），同时记录运行时间、Token 消耗和预估成本。

#### 4. 资源与算力
*   **训练算力：** 论文未涉及模型训练，仅评估现有的闭源大模型 API。
*   **推理资源：** 实验在配备 Docker 的 Linux 环境中运行。每个任务为智能体提供 **5 分钟** 的墙钟时间预算。

#### 5. 实验数量与充分性
*   **实验规模：** 针对 3 个模型在 20 个任务上进行了完整的矩阵测试，共计 60 组独立运行记录。
*   **充分性与公平性：** 实验设计较为客观。所有模型使用完全相同的系统提示词、工具集和沙盒环境。通过引入“生成任务集”有效降低了模型在训练数据中见过公开 CrackMe 题目（数据泄露）的风险。虽然 20 个任务的规模不算巨大，但覆盖了多种编程语言和逆向技术点，足以展示当前顶尖模型的能力阶梯。

#### 6. 论文的主要结论与发现
*   **模型排名：** **GPT-5.5 表现最强**，在生成任务集上达到了 92% 的 pass@3 成功率；Claude Opus 4.7 位居第二（58%）；Kimi K2 位居第三（42%）。
*   **难度差异：** 困难级别的生成任务（如涉及 Rust/Go 运行时噪音或复杂校验算法）能显著拉开模型差距。
*   **公开任务更难：** 所有模型在公开校准集上的表现均远逊于生成集（最高仅 38% 成功率），说明真实世界的逆向任务在输入约定和实现复杂性上对 AI 仍是巨大挑战。

#### 7. 优点
*   **客观性强：** 采用可执行 Oracle 评分，消除了人工评估文字说明的主观性。
*   **环境真实：** 提供了完整的逆向工程工具链，模拟了人类分析师的真实工作流。
*   **可重复性高：** 提供了详细的日志记录、Token 统计和 Docker 镜像配置，便于后续研究复现。
*   **多语言覆盖：** 任务不仅限于 C 语言，还包含了现代语言（Rust, Go）的二进制分析。

#### 8. 不足与局限
*   **任务规模有限：** 20 个任务对于全面衡量复杂的逆向工程能力来说样本量较小。
*   **时间限制较紧：** 5 分钟的限制可能导致模型无法完成需要深度静态分析的任务，更倾向于考察快速分类和简单逻辑恢复。
*   **平台单一：** 目前仅限于 Linux x86-64 ELF 文件，未涉及 Windows PE、macOS Mach-O 或移动端/固件二进制文件。
*   **交互模式：** 仅支持命令行交互，无法评估智能体使用图形化逆向工具（如 Ghidra GUI）的能力。
