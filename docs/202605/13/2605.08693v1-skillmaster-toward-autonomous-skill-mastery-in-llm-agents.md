## 摘要
技能为提升大语言模型（LLM）智能体处理复杂任务的能力提供了一种有效机制，但在现有的智能体框架中，技能的创建、优化和选择通常由外部教师、人工设计的规则或辅助模块控制。因此，技能仍然是被调用的外部资源，而非智能体能够通过经验开发、调整和内化的能力。为了赋予 LLM 智能体自主掌握技能的能力，我们提出了 SkillMaster，这是一个训练框架，旨在教导智能体在解决任务的过程中创建新技能、优化现有技能并选择积累的技能。这一能力通过三个关键设计实现。首先，我们通过基于轨迹信息的技能回顾来训练智能体，教导其根据已完成回合的证据来提议、更新或保留技能。其次，每个候选技能编辑都设计为通过其在相关探测任务上的反事实效用进行评估，从而为训练技能编辑决策提供直接的学习信号。第三，我们引入了 DualAdv-GRPO，它分别估计任务解决动作和技能编辑决策的优势值，从而稳定了任务解决与技能管理之间的联合训练。在 ALFWorld 和 WebShop 上的实验表明，SkillMaster 的总成功率分别比最先进的基准方法提高了 8.8% 和 9.3%，在所有对比方法中表现最佳。进一步的分析揭示了智能体能力的显著转变：使用 SkillMaster 训练的智能体能够识别技能失败，从轨迹证据中提炼程序性知识，并以有限的技能库编辑将改进迁移到未来的任务中。总之，SkillMaster 使 LLM 智能体超越了单纯的技能使用，向能够开发、调整和应用自身技能库的自我进化智能体迈进。

## Abstract
Skills provide an effective mechanism for improving LLM agents on complex tasks, yet in existing agent frameworks, their creation, refinement, and selection are typically governed by external teachers, hand-designed rules, or auxiliary modules. As a result, skills remain external resources to be invoked, rather than capabilities that agents can develop, adapt, and internalize through experience. To endow LLM agents with autonomous skill mastery, we propose SkillMaster, a training framework that teaches agents to create new skills, refine existing skills, and select accumulated skills during task solving. This capability is achieved through three key designs. First, we train agents through trajectory-informed skill review, teaching agents to propose, update, or retain skills based on evidence from completed episodes. Second, each candidate skill edit is designed to be evaluated by its counterfactual utility on related probe tasks, providing a direct learning signal for training skill-editing decisions. Third, we introduce DualAdv-GRPO, which separately estimates advantages for task-solving actions and skill-editing decisions, stabilizing joint training across task solving and skill management. Experiments on ALFWorld and WebShop show that SkillMaster improves the overall success rate over state-of-the-art baselines by 8.8% and 9.3%, respectively, achieving the best performance among all compared methods. Further analysis reveals a marked shift in agent capability: agents trained with SkillMaster can identify skill failures, refine procedural knowledge from trajectory evidence, and transfer improvements to future tasks with limited skill-bank edits. Overall, SkillMaster moves LLM agents beyond mere skill use toward self-improving agents capable of developing, adapting, and applying their own skill repertoires.

### 1. 核心问题与整体含义（研究动机和背景）
现有的 LLM 智能体虽然引入了“技能（Skills）”概念来处理复杂任务，但技能的创建、精炼和选择通常依赖于外部教师（如更强的模型）、预设规则或辅助模块。这导致技能只是被调用的“外部资源”，而非智能体通过自身经验发展、调整并内化的“内在能力”。
**核心问题：** 如何让智能体实现**自主技能掌握（Autonomous Skill Mastery）**，即让智能体能够根据自身执行任务的轨迹，自主决定何时增加新技能、如何优化旧技能，并将其内化到策略中。

### 2. 方法论
SkillMaster 框架通过将技能管理整合进强化学习（RL）循环，实现了任务执行与技能进化的闭环。
*   **轨迹驱动的技能评审（Trajectory-informed Skill Review）：** 在任务执行阶段（Acting Phase）之后增加一个技能掌握阶段（Skill-Mastery Phase）。智能体通过工具调用（`propose_skill`、`update_skill`、`keep_skill`）来评审刚完成的轨迹，决定是否修改技能库。
*   **反事实技能效用奖励（Counterfactual Skill Utility Reward）：** 为了解决技能质量难以评估的问题，该方法引入了“探测任务（Probe Tasks）”。当智能体提议修改技能时，系统会在相关探测任务上进行反事实对比：比较修改前后的技能库在成功率和步数效率上的差异，以此作为强化学习的直接奖励信号。
*   **DualAdv-GRPO 算法：** 针对任务解决动作（环境奖励）与技能编辑决策（效用奖励）目标不一致的问题，提出了双流优势估计。分别对动作和技能编辑的优势值进行归一化，并通过可调权重 $\gamma$ 进行融合，确保联合训练的稳定性。

    *   **WebShop：** 在线购物环境，要求智能体根据需求搜索并购买商品。
*   **对比方法：** 
    *   闭源模型（GPT-4o, Gemini-2.5-Pro）。
    *   基于提示词/记忆的方法（ReAct, Reflexion, Mem0, ExpeL）。

*   **消融实验：** 针对效用奖励、优势归一化方式、探测任务选择策略、冷启动 SFT 等关键组件进行了详细的消融分析。
*   **深度分析：** 额外进行了技能内化分析（测试不带技能库时的表现）、对初始技能库质量的鲁棒性测试、以及计算开销分析。
*   **评价：** 实验设计全面，对比了从零样本到最先进 RL 方法的各类基准，且通过控制变量验证了自主管理相比外部管理的优越性，实验结果客观且具有说服力。

### 6. 主要结论与发现
*   **性能突破：** SkillMaster 在 ALFWorld 和 WebShop 上的成功率分别比最强基准（SkillRL）高出 **8.8%** 和 **9.3%**。
*   **技能内化：** 训练后的智能体即使在测试时不检索技能库，其表现也接近带库运行，说明技能知识已通过训练内化到了模型参数中。
*   **自主进化：** 智能体能够识别轨迹中的失败原因，并提出如“不要搜索无效区域”等具有泛化价值的程序性规则。
*   **效率提升：** 通过反事实评估，智能体不仅学会了如何成功，还学会了如何通过更少的步骤完成任务。

### 7. 优点
*   **去中心化管理：** 摆脱了对外部教师模型的依赖，实现了真正的“自进化”。
*   **显式质量信号：** 引入反事实探测任务，为离散的技能编辑动作提供了连续且高质量的奖励反馈。
*   **训练稳定性：** DualAdv-GRPO 有效解决了异构任务（动作 vs 决策）在同一策略下联合优化的干扰问题。

### 8. 不足与局限
*   **计算成本：** 引入探测任务意味着每次技能修改都需要额外的 Rollout，增加了训练的时间和算力成本。
*   **探测任务依赖：** 目前依赖预定义的任务分类（如 ALFWorld 的任务族）来选择探测任务，在完全无结构的开放环境中可能需要更复杂的相似度度量。
*   **场景覆盖：** 虽然在两个基准上表现优异，但在更开放、长程的任务（如软件工程、复杂科研辅助）中的表现尚待验证。
