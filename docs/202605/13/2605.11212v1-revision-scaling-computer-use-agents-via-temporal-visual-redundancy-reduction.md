## 摘要
计算机使用智能体 (CUAs) 依赖于图形用户界面的视觉观测，每张截图都会被编码为大量的视觉 token。随着交互轨迹的增长，token 开销迅速增加，限制了在固定上下文和计算预算下可纳入的历史信息量。与其他领域不同，这导致在使用历史信息时性能提升极小甚至没有提升。我们通过引入 ReVision 来解决这一低效问题，该方法用于在轨迹上训练多模态语言模型，利用学习到的补丁选择器移除冗余视觉补丁。该选择器通过比较连续截图之间的补丁表示来识别冗余，同时保留模型所需的空间结构。在 OSWorld、WebTailBench 和 AgentNetBench 三个基准测试中，当使用 Qwen2.5-VL-7B 处理包含 5 张历史截图的轨迹时，ReVision 平均减少了约 46% 的 token 使用量，且成功率比不丢弃补丁的基线提高了 3%。这确立了显著的效率优势，使智能体能够以更少的 token 处理更长的轨迹。凭借这种效率的提升，我们重新审视了历史信息在 CUA 中的作用，并发现当冗余被移除后，性能会随着更多过去观测结果的加入而持续提升。这表明，通常观察到的视觉历史性能饱和并非由于过去信息的效用有限，而是 token 表示效率低下的结果。

## Abstract
Computer-use agents~(CUAs) rely on visual observations of graphical user interfaces, where each screenshot is encoded into a large number of visual tokens. As interaction trajectories grow, the token cost increases rapidly, limiting the amount of history that can be incorporated under fixed context and compute budgets. This has resulted in no or very limited improvement in the performance when using history unlike other domains. We address this inefficiency by introducing ReVision, which is used to train multimodal language models on trajectories where redundant visual patches are removed using a learned patch selector that compares patch representations across consecutive screenshots while preserving spatial structure required by the model. Across three benchmarks, OSWorld, WebTailBench, and AgentNetBench, when processing trajectories with 5 history screenshots using Qwen2.5-VL-7B, ReVision reduces token usage by approximately 46% on average while improving success rate by 3% over the no drop baseline. This establishes a clear efficiency gain, enabling agents to process longer trajectories with fewer tokens. With this improved efficiency, we revisit the role of history in CUAs and find that performance continues to improve as more past observations are incorporated when redundancy is removed. This suggests that the commonly observed saturation in visual history is not due to limited usefulness of past information, but rather a consequence of inefficient token representations.

### 1. 核心问题与整体含义（研究动机和背景）
论文关注的是**计算机使用智能体（Computer-Use Agents, CUAs）**在处理长程任务时的效率与性能瓶颈。
*   **研究动机**：当前的 GUI 智能体依赖多模态大模型（MLLM）处理屏幕截图。每张截图都会产生数千个视觉 Token，随着交互步骤增加，历史截图会迅速耗尽模型的上下文窗口和计算资源。
*   **核心矛盾**：尽管增加历史信息对复杂任务至关重要，但由于连续的 GUI 截图之间存在极高的**时间视觉冗余**（例如，点击一个按钮后，屏幕 90% 的区域保持不变），简单堆叠历史图片会导致模型处理大量重复信息，产生“性能饱和”现象，即增加历史反而不提升甚至降低成功率。

### 2. 方法论：ReVision 框架
ReVision 的核心思想是**在保留空间结构的前提下，动态移除连续截图间的冗余补丁**。
*   **RTS (ReVision Token Selection) 模块**：这是一个轻量级的三层 MLP 分类器。它接收连续两帧（$I_{t-1}$ 和 $I_t$）中对应位置补丁的特征向量，预测当前补丁是否为冗余。
*   **训练策略**：
    *   **监督信号生成**：利用 OmniParserV2 对截图进行语义区域分割，通过计算区域重叠（IoU）来自动标注哪些补丁是冗余的。
    *   **模型微调**：在过滤后的轨迹数据上对 MLLM（如 Qwen2.5-VL）进行微调。这使模型学会从之前的完整图像中“找回”当前帧被剔除的视觉信息，从而在极低 Token 开销下理解长序列。
*   **空间结构保留**：被保留的 Token 维持其原始的空间位置 ID（Position IDs），确保模型能够正确识别 UI 元素的坐标，不破坏空间推理能力。

### 6. 主要结论与发现
*   **冗余度惊人**：在主流基准测试中，连续截图间的视觉冗余度高达 **36% 至 56%**。
*   **效率与性能双增**：在 Qwen2.5-VL-7B 上，ReVision 平均减少了 **46% 的 Token 使用量**，同时将任务成功率提升了 **3%**。
*   **打破饱和假象**：研究发现，以往观察到的“历史信息收益饱和”其实是由于 Token 表示效率低下导致的。通过 ReVision 减少冗余后，**性能会随着历史长度的增加而持续提升**（直至 11 帧以上才开始平台化）。
*   **决策更高效**：ReVision 智能体倾向于用更少的交互步数完成任务。

### 7. 优点与亮点
*   **即插即用**：不需要修改 MLLM 的底层架构，仅作为 Token 过滤层存在。
*   **低延迟**：RTS 模块推理仅需约 22ms，远低于基于区域解析（如 OmniParser）的方法（>550ms）。
*   **揭示本质**：通过实验证明了视觉 Token 的“质”比“量”更重要，为长上下文多模态研究提供了新思路。

### 8. 不足与局限
*   **细微变化风险**：虽然 RTS 经过训练，但仍存在误删细微但关键视觉变化（如微小的状态指示灯闪烁）的风险。
*   **训练依赖**：模型需要针对特定的过滤分布进行微调，未经微调的模型直接使用 RTS 可能会出现性能下降。
*   **空间冗余未触及**：目前主要解决的是时间维度的冗余，单张截图内部的空间冗余（如大面积空白背景）仍有进一步优化的空间。
