## 摘要
大语言模型（LLM）智能体日益推动了服务应用的发展，例如预订机票。然而，这些服务智能体在长程任务中存在不可靠性，因为它们经常产生策略违规、工具幻觉和动作失配，这极大地阻碍了它们的实际部署。为了应对这些挑战，我们提出了 NOD（Navigator-Operator-Director），这是一种用于服务智能体的异构多智能体架构。与以往工作中在对话上下文中隐式维护任务状态不同，我们将结构化的全局状态（Global State）外部化，以实现显式的任务状态跟踪，并由导航员（Navigator）做出一致的决策。此外，我们在关键动作之前引入了选择性的外部监督，允许独立的指导员（Director）智能体验证执行情况并在必要时进行干预。因此，NOD 有效地缓解了长程任务中的错误传播和不安全行为。在 $τ^2$-Bench 上的实验表明，NOD 相比基准模型实现了更高的任务成功率和关键动作精度。更重要的是，NOD 通过减少策略违规、工具幻觉和用户意图失配，提高了服务智能体的可靠性。

## Abstract
Large language model (LLM) agents have increasingly advanced service applications, such as booking flight tickets. However, these service agents suffer from unreliability in long-horizon tasks, as they often produce policy violations, tool hallucinations, and misaligned actions, which greatly impedes their real-world deployment. To address these challenges, we propose NOD (Navigator-Operator-Director), a heterogeneous multi-agent architecture for service agents. Instead of maintaining task state implicitly in dialogue context as in prior work, we externalize a structured Global State to enable explicit task state tracking and consistent decision-making by the Navigator. Besides, we introduce selective external oversight before critical actions, allowing an independent Director agent to verify execution and intervene when necessary. As such, NOD effectively mitigates error propagation and unsafe behavior in long-horizon tasks. Experiments on $τ^2$-Bench demonstrate that NOD achieves higher task success rates and critical action precision over baselines. More importantly, NOD improves the reliability of service agents by reducing policy violations, tool hallucinations, and user-intent misalignment.

### 2. 方法论：NOD 架构
NOD 架构由三个异构角色组成，核心思想是“**无 NOD（Navigator-Operator-Director）不行动**”：
*   **Navigator（导航员）**：负责维护一个显式的、结构化的**全局状态（Global State）**。该状态以 JSON 格式存储，包含任务目标、约束、缺失信息、实体信息及子任务进度。它将隐式的对话历史转化为显式的执行蓝图。
*   **Operator（操作员）**：负责日常对话和提出动作建议。它不再直接依赖原始对话历史，而是基于 Navigator 提供的全局状态来决定下一步。
*   **Director（指导员）**：作为安全闸门，仅在**关键动作（Critical Actions）**（即会改变外部环境的操作，如扣费、下单）执行前介入。
    *   **决策逻辑**：Director 审核动作是否符合策略和意图，返回三种指令：`PASS`（通过）、`REVISE`（拦截并要求 Navigator 修正状态后重试）、`ABORT`（彻底终止不安全动作）。
*   **异构性**：通常使用较小的本地模型担任 Navigator 和 Operator，而使用更强大的前沿模型（如 GPT-5.1）担任 Director，实现成本与可靠性的平衡。

### 6. 主要结论与发现
*   **性能提升**：NOD 在所有骨干模型上均显著提升了 SR 和 CAP。相比 Vanilla 基准，SR 最高提升了 23.7 个百分点，CAP 提升了 25.8 个百分点。
*   **错误减少**：NOD 大幅减少了策略违规（Retail 减少 55.3%，Airline 减少 66.8%）和工具幻觉（减少约 75%-80%）。
*   **分工明确**：实验发现 `REVISE` 机制主要负责提升任务成功率（纠偏），而 `ABORT` 机制主要负责保障动作安全性（拦截）。
*   **稀疏干预**：Director 仅在约 10%-20% 的关键轮次介入，证明了该架构在提升可靠性的同时，并未过度依赖高成本模型。

### 7. 优点
*   **架构优雅**：将复杂的长程控制问题转化为“状态追踪+选择性验证”的结构化控制问题。
*   **显式状态管理**：通过外部化全局状态，解决了 LLM 难以处理超长上下文状态丢失的顽疾。
*   **高可靠性**：引入 Director 作为独立审计者，有效打破了单模型自我修正的局限性。
*   **实用性强**：支持异构模型组合，允许开发者用低成本模型处理常规对话，用高能力模型保障关键环节。

### 8. 不足与局限
*   **Token 成本**：Navigator 每轮都需要重新生成完整的 JSON 状态，这在极长对话中会显著增加 Token 消耗和延迟。
*   **状态更新的脆弱性**：目前 Navigator 采用全量重写状态的方式，如果某轮生成出错，可能导致后续状态的连锁反应（虽然有 Director 缓解）。
*   **应用限制**：该架构高度依赖于预定义的“关键动作”列表和结构化的状态 Schema，对于极度开放、无固定工具集的场景可能需要大量适配工作。
