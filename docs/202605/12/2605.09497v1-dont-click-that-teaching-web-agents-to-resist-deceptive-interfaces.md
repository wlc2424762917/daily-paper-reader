## 摘要
基于视觉语言模型 (VLM) 的 Web 智能体展示了令人印象深刻的自主 GUI 交互能力，但在欺骗性界面元素面前仍然十分脆弱。现有方法要么在不结合任务的情况下检测欺骗，要么仅记录攻击而不提出防御措施。我们对具备欺骗感知能力的 Web 智能体防御进行了形式化定义，并提出了 DUDE（欺骗性 UI 检测器与评估器），这是一个结合了非对称惩罚的混合奖励学习与经验总结的两阶段框架，旨在将失败模式提炼为可迁移的指导建议。我们引入了 RUC（真实 UI 点击框），这是一个包含跨越四个领域和欺骗类别的 1,407 个场景的基准测试。实验表明，DUDE 在保持任务性能的同时，将欺骗易感性降低了 53.8%，为稳健的 Web 智能体部署奠定了有效基础。

## Abstract
Vision-language model (VLM) based web agents demonstrate impressive autonomous GUI interaction but remain vulnerable to deceptive interface elements. Existing approaches either detect deception without task integration or document attacks without proposing defenses. We formalize deception-aware web agent defense and propose DUDE (Deceptive UI Detector & Evaluator), a two-stage framework combining hybrid-reward learning with asymmetric penalties and experience summarization to distill failure patterns into transferable guidance. We introduce RUC (Real UI Clickboxes), a benchmark of 1,407 scenarios spanning four domains and deception categories. Experiments show DUDE reduces deception susceptibility by 53.8% while maintaining task performance, establishing an effective foundation for robust web agent deployment.

## 1. 核心问题与研究背景
*   **核心问题**：现有的视觉语言模型（VLM）Web 智能体虽然在 GUI 交互上表现出色，但极易受到“暗黑模式”（Dark Patterns）的欺骗，如虚假下载按钮、诱导性弹窗和伪装广告。
*   **研究动机**：研究发现 SOTA 智能体在面对欺骗性 UI 时，被误导率高达 70% 以上。现有的研究要么只关注欺骗检测（脱离任务），要么只记录攻击手段（缺乏防御）。本文旨在赋予智能体“经验性韧性”，使其在完成任务的同时能够识别并避开界面陷阱。

## 2. 方法论 (DUDE 框架)
DUDE（Deceptive UI Detector & Evaluator）是一个两阶段的防御框架，其核心思想是在智能体动作执行前引入一个“评估器”作为安全闸门。
*   **Stage 1: 混合奖励学习 (Hybrid-Reward Learning)**：
    *   通过强化学习（采用 GRPO 算法）校准评估器。
    *   **非对称惩罚机制**：设计了特殊的严重性权重 $\omega$。对于“将欺骗性点击误判为安全”的灾难性错误（C4类），惩罚权重设为 10；而对于“将安全点击误判为不安全”的保守性错误，惩罚权重仅为 1。
    *   引入置信度调整（Confidence Adjustment）和注意力标量（Attention Scalar）来处理决策模糊性。
*   **Stage 2: 经验总结 (Experience Summarization)**：
    *   将 Stage 1 中出现的失败案例（截图、任务、错误动作）收集到失败池。
    *   利用多模态大模型迭代式地将这些失败模式提炼为紧凑的文本准则（Experience Context）。
    *   这些准则在推理时作为上下文注入，无需更新模型参数即可实现防御进化。

## 3. 实验设计
*   **RUC 基准 (Real UI Clickboxes)**：本文构建了一个包含 **1,407 个场景** 的基准测试，涵盖新闻、预订、购物、软件四大领域。
    *   **欺骗类别**：强制设计、认知操纵、上下文路径欺骗、情感操纵。
    *   **标注**：每个样本包含正确操作框（$B_c$）和欺骗性操作框（$B_d$）。
*   **对比方法**：
    *   **Vanilla**：原始模型（Qwen3-VL, UI-TARS, GLM-4.6V）。
    *   **Simple Refusal**：简单的拒绝策略。
    *   **DUDE 增强版**：应用了本文提出的两阶段防御。
*   **评估指标**：成功率 (SR)、欺骗失败率 (DFR)、平均交互步数 (Steps)。

## 4. 资源与算力
*   **硬件环境**：使用了 **4 张 NVIDIA A100 (80GB VRAM)** GPU。
*   **计算资源**：配备 AMD EPYC 7742 64核处理器和 512GB RAM。
*   **训练细节**：Stage 1 训练使用了 AdamW 优化器，学习率为 $1 \times 10^{-5}$，Batch Size 为 16，训练 3 个 Epoch。推理时固定温度为 0 以保证确定性。

## 5. 实验数量与充分性
*   **实验规模**：在 200 个独立任务的测试集上进行了跨模型验证。
*   **消融实验**：
    *   针对 Stage-1 和 Stage-2 的独立贡献进行了消融。
    *   针对奖励函数中的四个关键组件（严重性权重、注意力标量、置信度调整等）进行了详细对比。
    *   对比了不同 Prompt 策略（无、手动、自动总结）的效果。
*   **充分性**：实验覆盖了从 2B 到 7B 不同参数规模的模型，并验证了从开源模型学习到的经验向闭源模型（GLM-4.6V）的**零样本迁移能力**，实验设计较为全面且客观。

## 6. 主要结论与发现
*   **显著降低风险**：DUDE 将智能体的欺骗易感性平均降低了 **53.8%**，在多个场景下将 DFR 降至接近 0。
*   **意外的性能增益**：DUDE 不仅防欺骗，还显著提升了任务成功率（如 GLM 模型 SR 提升了 51%）。这是因为评估器的“拒绝并反思”机制能纠正智能体点击无效区域的普通错误。
*   **效率提升**：虽然单步增加了评估开销，但由于减少了无效的盲目尝试，总任务完成时间从 217.6s 缩短至 48.5s。

## 7. 优点
*   **非对称安全观**：敏锐地捕捉到安全防御中“漏报”代价远大于“误报”的特性，并将其量化为奖励函数。
*   **即插即用与迁移性**：Stage 2 生成的经验准则具有很强的通用性，可以直接提升其他未训练模型的安全性。
*   **闭环反馈**：通过“评估-拒绝-反思”循环，将原本被动的检测转化为主动的策略调整。

## 8. 不足与局限
*   **动态环境验证不足**：实验主要基于静态截图，对于需要多步动态交互（如异步加载的弹窗）的复杂环境覆盖有限。
*   **语言局限性**：RUC 基准目前仅限于英文界面，未测试跨语言（如中文网页）的泛化效果。
*   **防御深度**：主要针对视觉层面的欺骗，对于隐藏在代码逻辑中、无视觉表现的后端攻击（如隐蔽的脚本注入）无能为力。
*   **对抗强度**：RUC 侧重于真实世界的“暗黑模式”，在面对极高强度的恶意对抗攻击时，防御的鲁棒性仍有待进一步验证。
