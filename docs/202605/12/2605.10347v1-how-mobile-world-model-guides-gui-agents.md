## 摘要
视觉语言模型的最新进展使移动 GUI 智能体能够感知视觉界面并执行用户指令，但对于长时程和高风险交互而言，对动作后果的可靠预测仍然至关重要。现有的移动世界模型提供基于文本或基于图像的未来状态，但目前尚不清楚哪种表示更有用、生成的展开（rollouts）是否可以替代真实环境，以及测试时引导如何帮助不同能力的智能体。为了回答上述问题，我们过滤并标注了移动世界模型数据，随后训练了涵盖四种模态的世界模型：增量文本（delta text）、全文、基于扩散的图像以及可渲染代码。这些模型在 MobileWorldBench 和 Code2WorldBench 上均达到了 SOTA 性能。此外，通过在 AITZ、AndroidControl 和 AndroidWorld 上评估其下游效用，我们得到了三个发现。第一，可渲染代码重构实现了较高的分布内保真度，并为数据构建提供了有效的多模态监督，而基于文本的反馈在在线分布外（OOD）执行中表现得更为稳健。第二，世界模型生成的轨迹可以在训练过程中提供可迁移的交互经验，并提高智能体的端到端任务性能，尽管这些数据并未保持原始分布。最后，对于动作熵较低的过度自信移动智能体，事后自我反思带来的收益有限，这表明世界模型作为先验感知或训练监督比作为通用的事后验证器更为有效。

## Abstract
Recent advances in vision-language models have enabled mobile GUI agents to perceive visual interfaces and execute user instructions, but reliable prediction of action consequences remains critical for long-horizon and high-risk interactions. Existing mobile world models provide either text-based or image-based future states, yet it remains unclear which representation is useful, whether generated rollouts can replace real environments, and how test-time guidance helps agents of different strengths. To answer the above questions, we filter and annotate mobile world-model data, then train world models across four modalities: delta text, full text, diffusion-based images, and renderable code. These models achieve SoTA performance on both MobileWorldBench and Code2WorldBench. Furthermore, by evaluating their downstream utility on AITZ, AndroidControl, and AndroidWorld, we obtain three findings. First, renderable code reconstruction achieves high in-distribution fidelity and provides effective multimodal supervision for data construction, while text-based feedback is more robust for online out-of-distribution (OOD) execution. Second, world-model-generated trajectories can provide transferable interaction experience in the training process and improve agents' end-to-end task performance, although these data do not preserve the original distribution. Last, for overconfident mobile agents with low action entropy, posterior self-reflection provides limited gains, suggesting that world models are more effective as prior perception or training supervision than as universal post-hoc verifiers.

### 2. 方法论：核心思想与关键技术
*   **四种预测模态**：
    *   **增量文本（Delta Text）**：仅描述动作引起的界面变化，减少冗余。
    *   **全文（Full Text）**：描述动作后的完整界面状态。
    *   **代码转图像（Code2Image）**：预测可渲染的 HTML/代码，再渲染成图像，保证布局和文字的精确性。
    *   **扩散图像（Diffusion Image）**：利用生成式扩散模型直接合成下一帧截图。
*   **数据构建流水线（MWM Pipeline）**：
    *   **图级去重**：合并视觉和语义相似的状态转移。
    *   **保真度过滤**：剔除无法通过代码准确还原的样本。
    *   **动作逻辑过滤**：利用强模型（如 Gemini-3-Flash）验证动作与界面变化的一致性。
    *   **页面噪声去除**：移除高度同质化的空白页或搜索页。

### 3. 实验设计
*   **数据集**：整合了 AITW、AMEX、AITZ、AndroidControl 和 GUI-Odyssey 等多个主流移动数据集。
*   **基准测试（Benchmarks）**：
    *   **重建质量**：MobileWorldBench（文本）、Code2World（图像/代码）。
    *   **下游任务**：AITZ（离线导航）、AndroidControl（分布内测试）、AndroidWorld（在线端到端任务）。
*   **对比方法**：
    *   **基础智能体**：MAI-UI-8B、Qwen3-VL 系列（8B 到 238B）、Gemini-3-Flash、GPT-5.4。

### 4. 资源与算力
*   **模型规模**：使用了 Qwen3-4B/8B 作为基础模型进行微调，并对比了高达 238B 参数的大规模模型。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从重建质量评估到离线单步预测，再到在线长程任务执行的全流程实验。
*   **充分性**：实验设计非常全面。作者不仅对比了模态，还深入探讨了智能体的“动作熵”（置信度）如何影响世界模型的发挥。
*   **客观性**：通过引入“VLM-as-a-Judge”（如使用 Gemini-3.1-Pro 作为裁判）和多种自动化指标（Sad, Sid, Sele, Slay），保证了评估的客观性。

### 6. 主要结论与发现
*   **模态权衡**：**Code2Image** 在分布内（ID）重建中保真度最高，对强智能体引导效果好；而**文本反馈**在处理分布外（OOD）和在线环境时更具鲁棒性。扩散模型因文字渲染差且效率低，目前实用性较弱。
*   **智能体过度自信**：移动智能体通常具有极低的动作熵（非常自信但可能出错），这限制了“后验反思”的效果，因为智能体往往不愿尝试多样化的候选动作。
*   **先验优于后验**：世界模型作为**先验感知模块**（在动作前提供信息）比作为后验验证器更有效。
*   **想象力训练**：在世界模型生成的轨迹上进行微调（WM-SFT）虽然会改变动作分布（如点击坐标向中心偏移），但能显著提升智能体在未知环境（AndroidWorld）中的泛化成功率。

### 7. 优点
*   **系统性强**：首次对移动端世界模型的四种主流模态进行了公平、深入的消融对比。
*   **洞察深刻**：提出了“动作熵”是限制世界模型发挥的关键因素，解释了为什么简单的自我反思在 GUI 任务中往往无效。
*   **实用价值**：提出的数据过滤流水线显著提升了世界模型的训练质量，达到了 SOTA 水平。

### 8. 不足与局限
*   **分布偏移**：世界模型生成的页面往往比真实页面简单，导致在其上训练的智能体在点击精度上会出现系统性偏差（如倾向于点击屏幕中心）。
*   **推理开销**：高性能的引导（如 Code2Image 或先验感知）伴随着巨大的 Token 消耗和推理延迟，难以在移动端实时部署。
*   **依赖元素列表**：先验感知模式高度依赖于界面元素列表（Element List）的获取，在某些无法获取元数据的场景下受限。
