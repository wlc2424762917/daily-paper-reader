## 摘要
多模态大语言模型越来越多地通过调用外部工具（用于视觉检查、OCR、检索、计算和多步推理）来解决以视觉为中心的任务。目前的工具使用智能体通常会展示执行的工具轨迹和最终答案，但它们很少指明哪项工具观测结果支持了生成的每个断言。我们将这种缺失的断言级依赖结构称为“溯源鸿沟”（provenance gap）。这一鸿沟使得工具的使用难以验证和优化，因为有用的证据、冗余的探索和无根据的推理都混合在同一轨迹中。我们提出了 TRACER，这是一个用于多模态工具使用智能体的可验证生成式溯源框架。TRACER 并非在生成后添加引用，而是在生成每个答案句子的同时，生成一个结构化的溯源记录，该记录识别了支持该句子的工具轮次、证据单元和语义支持关系。其关系空间包含引用（Quotation）、压缩（Compression）和推理（Inference），涵盖了直接复用、忠实浓缩和有据推导。TRACER 通过模式检查、工具轮次对齐、来源真实性和关系合理性来验证每条记录，然后将验证后的溯源转化为可追溯性约束和基于溯源的局部信用（local credit），用于强化学习。我们进一步构建了 TRACE-Bench，这是一个用于从粗粒度多模态工具轨迹中进行句子级溯源重建的基准测试。在 TRACE-Bench 上，单纯增加工具往往会引入噪声。使用 Qwen3-VL-8B，TRACER 达到了 78.23% 的答案准确率和 95.72% 的摘要准确率，比最强的闭源工具增强基线高出 23.80 个百分点。与仅使用工具的有监督微调相比，它还将测试集总工具调用次数从 4949 次减少到 3486 次。这些结果表明，可靠的多模态工具推理取决于对观测结果的溯源感知使用，而不仅仅是增加工具调用次数。

## Abstract
Multimodal large language models increasingly solve vision-centric tasks by calling external tools for visual inspection, OCR, retrieval, calculation, and multi-step reasoning. Current tool-using agents usually expose the executed tool trajectory and the final answer, but they rarely specify which tool observation supports each generated claim. We call this missing claim-level dependency structure the provenance gap. The gap makes tool use hard to verify and hard to optimize, because useful evidence, redundant exploration, and unsupported reasoning are mixed in the same trajectory. We introduce TRACER, a framework for verifiable generative provenance in multimodal tool-using agents. Instead of adding citations after generation, TRACER generates each answer sentence together with a structured provenance record that identifies the supporting tool turn, evidence unit, and semantic support relation. Its relation space contains Quotation, Compression, and Inference, covering direct reuse, faithful condensation, and grounded derivation. TRACER verifies each record through schema checking, tool-turn alignment, source authenticity, and relation rationality, and then converts verified provenance into traceability constraints and provenance-derived local credit for reinforcement learning. We further construct TRACE-Bench, a benchmark for sentence-level provenance reconstruction from coarse multimodal tool trajectories. On TRACE-Bench, simply adding tools often introduces noise. With Qwen3-VL-8B, TRACER reaches 78.23% answer accuracy and 95.72% summary accuracy, outperforming the strongest closed-source tool-augmented baseline by 23.80 percentage points. Compared with tool-only supervised fine-tuning, it also reduces total test-set tool calls from 4949 to 3486. These results show that reliable multimodal tool reasoning depends on provenance-aware use of observations, not on more tool calls alone.

*   **核心问题：** 论文指出了多模态工具使用智能体（Agent）中存在的**“溯源鸿沟”（Provenance Gap）**。虽然目前的智能体能展示工具调用轨迹和最终答案，但无法明确说明答案中的每一个断言（Claim）具体是由哪一个工具的哪一部分观测结果支持的。
*   **研究动机：** 这种模糊性导致推理过程难以验证（无法区分是模型幻觉、参数记忆还是真实的工具使用），且难以优化（无法识别哪些工具调用是冗余的，哪些是关键证据）。
*   **整体含义：** 提出 TRACER 框架，旨在通过生成“可验证的生成式溯源”，建立断言级的依赖结构，从而提升多模态推理的可靠性和效率。

### 2. 方法论：核心思想与技术细节
*   **核心思想：** 将溯源视为生成过程的一部分，而非事后解释。模型在生成答案句子的同时，同步输出结构化的溯源记录。
*   **关键技术细节：**
    *   **结构化溯源记录：** 包含支持该句子的工具轮次（Tool Turn）、证据单元（Evidence Unit）以及语义支持关系。
    *   **关系空间（Relation Space）：** 定义了三种关系：**引用（Quotation）**（直接复用）、**压缩（Compression）**（忠实浓缩）和**推理（Inference）**（基于证据的推导）。
    *   **多维度验证：** 通过模式检查（Schema）、工具轮次对齐、来源真实性（Source Authenticity）和关系合理性（Relation Rationality）四项指标对生成的溯源进行自动校验。
    *   **溯源引导的强化学习（RL）：** 采用组相对策略优化（类似 GRPO），设计了综合奖励函数 $R_{total} = R_{vqa} \cdot R_{trace} \cdot (w_0 + w_{cite} \cdot R_{cite})$。其中 $R_{trace}$ 是硬性的可追溯性门控，$R_{cite}$ 奖励高效的工具引用。
    *   **局部信用分配：** 利用验证后的溯源信息，将奖励直接分配给产生关键证据的工具调用 Token，解决长期推理中的信用分配难题。

### 3. 实验设计
*   **数据集与基准：** 构建了 **TRACE-Bench**，基于 ToolVQA 轨迹重建而成。包含 14,183 条高质量溯源轨迹，涵盖 7 种工具（OCR、搜索、计算、视觉描述等）。
*   **对比方法：**
    *   **闭源模型：** ChatGPT-4o-latest, Claude-3-5-Sonnet, Gemini-2.5-Pro（均包含直接回答和工具增强模式）。
    *   **开源模型：** Qwen2-VL (2B/7B), InternVL3.5-7B, LLaVA-v1.5, MiMo-VL 等。
    *   **消融基线：** 仅工具 SFT（无溯源）、TRACER-SFT、以及移除不同奖励组件的 TRACER-RL。
*   **评测指标：** 答案准确率（Acc）、摘要准确率（SummAcc）、可追溯性准确率（Traceability Acc）、溯源 F1 值、工具调用次数及错误数。

### 4. 资源与算力
*   **硬件：** 使用了 **8 台 NVIDIA A100 GPU**。
*   **训练细节：** 
    *   **SFT 阶段：** 全参数微调，Batch Size 为 2，梯度累积 4 步，训练 2 个 Epoch，使用 DeepSpeed ZeRO-2。
    *   **RL 阶段：** 训练 200 步，每组采样 8 个候选输出进行拒绝采样优化。
*   **模型底座：** 统一基于 Qwen3-VL-8B-Instruct 进行实验。

### 5. 实验数量与充分性
*   **实验规模：** 进行了大规模的基准对比（涵盖 10 余种主流模型），并在 2,550 条测试集上验证。
*   **消融实验：** 针对 $R_{cite}$（引用效率）和 $R_{trace}$（可追溯性）奖励进行了专门的消融，验证了各组件对准确率和工具效率的贡献。
*   **充分性与公平性：** 实验设计较为客观，通过引入“摘要准确率”排除了工具环境差异的影响，且对比了同等参数规模下的 SFT 与 RL 效果，证明了提升源于溯源机制而非单纯的算力堆砌。

### 6. 主要结论与发现
*   **工具并非越多越好：** 简单的工具接入往往会引入噪声，导致闭源模型（如 GPT-4o）在某些任务上准确率反而下降。
*   **TRACER 表现卓越：** TRACER-RL 达到了 78.23% 的准确率，比最强的闭源工具增强基线（Gemini-2.5-Pro）高出 **23.80%**。
*   **效率显著提升：** 相比于标准 SFT，TRACER 在提升准确率的同时，将工具调用次数减少了约 **30%**（从 4949 降至 3486），证明模型学会了“按需调用”和“精准引用”。
*   **溯源质量：** 强化学习显著提升了溯源的忠实度，溯源 F1 值从 70.70% 提升至 90.52%。

### 7. 优点（亮点）
*   **首创性：** 首次系统性地定义并解决了多模态工具智能体中的“断言级溯源”问题。
*   **可验证性：** 将溯源信息结构化并引入自动验证流程，使 Agent 的推理过程从“黑盒”变为“白盒”。
*   **闭环优化：** 不仅将溯源用于解释，还将其作为强化学习的信号，实现了性能与效率的双重优化。

### 8. 不足与局限
*   **验证器依赖：** 溯源验证在很大程度上依赖于 LLM-as-a-judge（如 Gemini-2.5-Pro），可能存在评估偏见或提示词敏感性。
*   **工具覆盖面：** 目前仅涵盖 7 种常用工具，对于更复杂、长程的工业级工作流验证尚待观察。
*   **模型规模：** 实验主要集中在 8B 规模模型，尚未探讨该机制在超大规模模型（如 70B+）上的扩展效应（Scaling Laws）。
*   **因果性限制：** 溯源记录证明了证据与断言之间的逻辑支持关系，但不能完全等同于模型内部计算的因果解释。
