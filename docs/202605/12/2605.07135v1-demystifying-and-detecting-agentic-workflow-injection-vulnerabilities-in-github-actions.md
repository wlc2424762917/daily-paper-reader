## 摘要
GitHub Actions 越来越多地被用于部署基于大语言模型（LLM）的代理，以处理以仓库为中心的任务，如议题（issue）分类、拉取请求（pull-request）审查、代码修改和发布辅助。这些代理工作流通过代理能力扩展了传统的 CI/CD 自动化，但也创造了新的注入攻击面。在本文中，我们引入了代理工作流注入（Agentic Workflow Injection, AWI），这是一种工作流级别的注入缺陷，其中不可信的 GitHub 事件上下文（如议题正文、拉取请求描述或评论）被纳入代理提示词或代理消耗的输入中，并通过代理工具或下游工作流逻辑转化为受攻击者影响的行为。我们识别了两种核心 AWI 模式：提示词到代理（Prompt-to-Agent, P2A），即不可信内容到达代理提示词边界；以及提示词到脚本（Prompt-to-Script, P2S），即攻击者的影响通过模型或代理衍生的输出传播到后续脚本中。我们对 GitHub Actions 中的 AWI 进行了首次系统性研究。我们对 1,033 个真实世界的 AI 辅助 Action 进行了特征分析，并提取了 AWI 特有的污点规范，包括提示词边界、衍生输出、代理能力和访问控制接口。基于这些规范，我们设计了 TaintAWI，这是一种污点分析工具，用于追踪从不可信事件上下文到代理提示词输入以及安全敏感的工作流汇聚点（sinks）的流向。将 TaintAWI 应用于来自 10,792 个仓库的 13,392 个真实代理工作流，我们报告了 519 个潜在 AWI 漏洞，其中 496 个在我们的威胁模型下被确认可利用，准确率达到 95.6%。其中，343 个是此前未知的零日漏洞。我们优先披露了 187 个零日案例，收到了 26 份维护者回复，截至撰稿时，已有 24 个案例被接受或修复。

## Abstract
GitHub Actions is increasingly used to deploy LLM-based agents for repository-centric tasks such as issue triage, pull-request review, code modification, and release assistance. These agentic workflows extend traditional CI/CD automation with agentic capabilities but also create a new injection surface. In this paper, we introduce Agentic Workflow Injection (AWI), a workflow-level injection flaw where untrusted GitHub event context, such as issue bodies, pull-request descriptions, or comments, is incorporated into agent prompts or agent-consumed inputs and converted into attacker-influenced behavior through agent tools or downstream workflow logic. We identify two core AWI patterns: Prompt-to-Agent (P2A), where untrusted content reaches an agent prompt boundary, and Prompt-to-Script (P2S), where attacker influence propagates through model- or agent-derived outputs into later scripts. We present the first systematic study of AWI in GitHub Actions. We characterize 1,033 real-world AI-assisted actions and extract AWI-specific taint specifications, including prompt boundaries, derived outputs, agentic capabilities, and access-control interfaces. Based on these specifications, we design TaintAWI, a taint-analysis tool that tracks flows from untrusted event context to agent prompt inputs and security-sensitive workflow sinks. Applying TaintAWI to 13,392 real-world agentic workflows from 10,792 repositories, we report 519 potential AWI vulnerabilities, of which 496 are confirmed exploitable under our threat model, yielding a precision of 95.6%. Among them, 343 are previously unknown zero-day vulnerabilities. We prioritized disclosure for 187 zero-day cases, received 26 maintainer responses, and 24 cases have been accepted or fixed at the time of writing.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究背景**：GitHub Actions 正在从传统的确定性自动化转向集成大语言模型（LLM）的“代理工作流”（Agentic Workflows）。这些工作流允许代理根据自然语言指令执行文件编辑、命令运行和 API 调用等高权限操作。
*   **核心问题**：论文识别了一种新型漏洞 **AWI**。当不可信的 GitHub 事件上下文（如 Issue 内容、PR 描述、评论）被引入代理提示词（Prompt）时，攻击者可以通过语义诱导操纵代理的行为，或通过代理生成的输出影响下游脚本。
*   **研究意义**：这是首个系统性研究 CI/CD 环境中 LLM 代理注入风险的工作，揭示了传统安全工具无法覆盖的新型攻击面。

### 2. 方法论
论文提出了 **TAINTAWI**，一种专门用于检测 AWI 的静态污点分析工具，其核心流程包括：
*   **Action 级规范提取**：通过 LLM 辅助分析 1,033 个 AI Action，识别其代理能力、提示词边界、衍生输出接口和访问控制机制。
*   **工作流中间表示（IR）构建**：将 YAML 格式的工作流提升为结构化的 IR，捕捉触发器、作业（Job）间依赖、步骤（Step）间数据流及权限配置。

*   **AWI 污点传播规则**：
    *   **P2A（Prompt-to-Agent）**：追踪污点是否到达代理的提示词输入。
    *   **P2S（Prompt-to-Script）**：当污点到达提示词后，将代理的输出视为受污染，并继续追踪其是否流向敏感的下游汇聚点（如 `gh` 命令、`git push` 等）。
*   **可达性分析**：结合 Action 级（如用户白名单）和工作流级（如 `if` 条件检查）的防护机制，过滤不可利用的路径。

### 6. 主要结论与发现
*   **漏洞普遍性**：检测到 519 个潜在漏洞，确认 496 个（准确率 95.6%），其中 **343 个为零日漏洞**。

*   **首创性**：首次定义并系统化研究了 GitHub Actions 中的代理注入漏洞。
*   **高准确性**：通过结合 Action 语义和可达性分析，实现了极高的检测准确率（95.6%）。
*   **实战价值**：发现了大量真实世界的零日漏洞，并推动了相关厂商（如 Google）更新安全准则。
*   **深度建模**：不仅关注提示词注入，还通过“输出桥接”技术解决了 LLM 语义边界导致的污点断裂问题。

### 8. 不足与局限
*   **静态分析局限**：存在过度近似（Over-approximation），可能报告在特定运行时环境下难以触发的路径。
*   **Action 语义依赖**：依赖于预先定义的 Action 规范，对于快速更新或长尾的第三方 Action 可能存在漏报。
*   **脚本分析深度**：虽然支持 Shell 和 JavaScript 的轻量级分析，但对于复杂的跨文件脚本或动态生成的命令，分析能力有限。
*   **应用限制**：目前主要针对 GitHub Actions 环境，尚未扩展到其他 CI/CD 平台（如 GitLab CI）。
