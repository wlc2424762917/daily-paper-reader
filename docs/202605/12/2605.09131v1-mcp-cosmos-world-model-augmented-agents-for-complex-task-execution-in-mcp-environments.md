## 摘要
模型上下文协议 (MCP) 统一了大语言模型 (LLM) 与外部工具之间的接口，但在智能体如何概念化其运行环境方面仍存在根本性差距。当前的范式是分叉的：任务级规划通常忽略执行时的动态变化，而反应式执行则缺乏长远预见。我们提出了 MCP-Cosmos，这是一个将生成式世界模型 (WM) 注入 MCP 生态系统以实现预测性任务自动化的框架。通过统一 MCP、世界模型和智能体这三种不同的技术，我们证明了“自带世界模型” (BYOWM) 策略允许智能体在执行前在潜空间中模拟状态转换并优化计划。我们在 20 多个 MCP-Bench 任务上，使用 ReAct 和 SPIRAL 两种策略，结合 2 个规划模型和 3 个代表性世界模型进行了实验。我们观察到智能体环境交互 KPI 的提升，例如工具成功率和工具参数准确度。该框架还提供了诸如“执行质量”等新指标，以产生关于世界模型与基准模型相比有效性的新见解。

## Abstract
The Model Context Protocol (MCP) has unified the interface between Large Language Models (LLMs) and external tools, yet a fundamental gap remains in how agents conceptualize the environments within which they operate. Current paradigms are bifurcated: Task-level planning often ignores execution-time dynamics, while reactive execution lacks long-horizon foresight. We present MCP-Cosmos, a framework that infuses generative World Models (WM) into the MCP ecosystem to enable predictive task automation. By unifying three disparate technologies, namely MCP, World Model, and Agent, we demonstrate that a "Bring Your Own World Model" (BYOWM) strategy allows agents to simulate state transitions and refine plans in a latent space before execution. We conducted experiments using two strategies, namely ReAct and SPIRAL with 2 planning models and 3 representative world models over 20+ MCP-Bench tasks. We observed improvements in Agent's environment interaction KPI such as tool success rate and tool parameter accuracy. The framework also offers new metrics such as Execution Quality to generate new insights about the effectiveness of world models compared to baseline.

#### 1. 论文的核心问题与整体含义
*   **研究背景**：模型上下文协议（MCP）虽然统一了 LLM 与外部工具的接口，但现有的智能体架构存在缺陷。规划类框架往往忽略执行时的动态环境，而反应式框架（如 ReAct）则存在“视界近视”（Horizon Myopia），无法预见行动的长期后果，导致冗余的工具调用或不可逆的执行失败。
*   **核心问题**：如何在 MCP 环境中让智能体具备“预测性认知”能力，即在正式执行工具调用前，先在模拟环境中预判状态转移，从而优化复杂任务的执行路径。

#### 2. 方法论
*   **核心思想**：提出 **BYOWM（Bring Your Own World Model，自带世界模型）** 策略。将生成式世界模型（World Model, WM）集成到 MCP 交互循环中，使智能体能够在潜空间（Latent Space）模拟环境动态 $P(s_{t+1} | s_t, a_t)$。
*   **关键技术细节**：
    *   **两阶段流程**：
        1.  **模拟规划阶段**：智能体利用世界模型模拟工具调用的响应（Pseudo Observation），通过多次迭代生成、修正并选择最优的工具调用计划。
        2.  **真实执行阶段**：将选定的计划在真实 MCP 环境中运行，并记录执行轨迹。
    *   **智能体架构**：支持 **ReAct-Plan-Exec**（扩展的反应式规划）和 **SPIRAL-Exec**（基于蒙特卡洛树搜索 MCTS 的结构化规划）。

#### 6. 主要结论与发现
*   **性能提升**：世界模型的引入显著提升了**工具选择的恰当性**和**参数准确度**。
*   **效率与质量**：虽然 ReAct 基准在任务完成度上表现尚可，但其工具调用成功率低且冗余调用多。WM 增强型智能体在“执行质量”指标上远超基准。

*   **专用模型差距**：专门针对 MCP 训练的小型模型（Arctic-AWM-4B）在模拟精度上仍逊于超大规模通用 LLM（如 Claude）。

#### 7. 优点
*   **架构前瞻性**：BYOWM 架构具有高度的模块化和灵活性，适应了 MCP 生态的快速扩张。
*   **指标创新**：提出的“执行质量”指标更贴近实际生产环境（关注成本、延迟和 API 稳定性）。
*   **深度分析**：不仅展示了性能提升，还深入探讨了 Token 成本与性能之间的权衡（Trade-off）。

#### 8. 不足与局限
*   **计算成本**：模拟过程带来了巨大的 Token 消耗和推理延迟，对于实时性要求极高的场景可能不适用。
