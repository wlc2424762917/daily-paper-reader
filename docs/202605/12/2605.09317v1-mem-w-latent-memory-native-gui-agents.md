## 摘要
GUI 智能体正开始将 Web、移动端和桌面端作为交互式世界进行操作，成功的控制取决于在转瞬即逝的当前屏幕之外，延续视觉、程序和任务层面的证据。然而，大多数智能体仍将记忆视为外部的、人类可读的人造物：历史记录在被策略重新编码之前，被总结、分类、检索并作为文本或结构化记录重新插入。这导致了经验存储的表示形式与现代 GUI 策略实际作用的潜嵌入序列（latent embedding sequence）之间的不匹配。我们推出了 Mem-W，一系列潜空间记忆原生型（latent-memory-native）GUI 智能体，它们将记忆视为智能体连续上下文的一部分，而非辅助性的符号支架。Mem-W 通过共享的轨迹到潜空间压缩器，将历史轨迹（作为经验记忆）和会话内片段（作为工作记忆）编织成紧凑的记忆标记（memory tokens）。这些标记与当前的 GUI 观测和局部上下文编织成一个连续的嵌入序列，使智能体能够通过相同的机器原生接口读取成功、失败和未完成的进度。Mem-W 采用自蒸馏和结果感知监督进行训练，以保留与决策相关的状态，同时将记忆过滤为真正支持任务成功的证据。在四个 Web 和移动导航基准测试中，Mem-W 持续改进了多种骨干网络和记忆增强型基准模型，提升幅度高达 +30.0，这表明潜空间上下文原生记忆可以作为长程 GUI 代理的可扩展基础。

## Abstract
GUI agents are beginning to operate the web, mobile, and desktop as interactive worlds, where successful control depends on carrying forward visual, procedural, and task-level evidence beyond the fleeting present screen. Yet most agents still treat memory as an external, human-readable artifact: histories are summarized, categorized, retrieved, and reinserted as text or structured records before being encoded again by the policy. This creates a mismatch between the representational form in which experience is stored and the latent embedding sequence over which modern GUI policies actually act. We introduce Mem-W, a series of latent-memory-native GUI agents that treat memory as part of the agent's continuous context rather than as an auxiliary symbolic scaffold. Mem-W weaves both historical trajectories (as experiential memory) and in-session segments (as working memory) into compact memory tokens through a shared trajectory-to-latent compressor. These tokens are woven with the current GUI observation and local context into one continuous embedding sequence, allowing the agent to read successes, failures, and unfinished progress through the same machine-native interface. Mem-W is trained with self-distillation and outcome-aware supervision to preserve decision-relevant state while filtering memory toward evidence that truly supports task success. Across four web and mobile navigation benchmarks, Mem-W consistently improves diverse backbones and memory-enhanced baselines, with gains of up to $+30.0$, suggesting that latent-context-native memory can serve as a scalable foundation for long-horizon GUI agency.

### 1. 核心问题与整体含义（研究动机和背景）
现代 GUI 智能体（如操作网页或手机 App 的 AI）在处理长程任务时，面临如何有效利用历史信息的问题。
*   **现有瓶颈**：目前的记忆机制大多是“外部符号化”的，即将历史记录总结为文本、列表或图结构，再重新输入模型。这导致了**表征不匹配（Representation Mismatch）**：智能体内部是在连续的潜空间（Latent Space）中推理，而记忆却是离散的符号。
*   **研究动机**：能否让智能体直接在“机器原生”的潜空间中存储和读取经验？论文提出了 **Mem-W**，将记忆视为智能体连续上下文的一部分，而非外部挂载的支架，从而消除表征鸿沟，提升长程任务的控制能力。

### 2. 方法论：核心思想与关键技术
Mem-W 的核心是将“工作记忆”（当前会话进度）和“经验记忆”（检索到的历史成功/失败案例）统一编织进模型的潜空间。

*   **轨迹到潜空间压缩器（Trajectory-to-Latent Compressor, $C_\phi$）**：
    *   使用轻量级的 **Q-Former** 结构，将变长的 GUI 事件序列（截图+动作）映射为固定数量（$K$个）的潜空间 Token。
    *   **统一表征**：无论是当前会话的片段还是检索到的历史轨迹，都通过同一个压缩器处理，输出相同格式的嵌入向量。
*   **双尺度记忆编织（Dual-Scale Weaving）**：
    *   **经验记忆**：从外部库检索相似任务的成功或失败轨迹。
    *   **工作记忆**：将当前会话中较早的、已过期的片段进行压缩，释放上下文窗口。
    *   这些潜空间 Token 与当前屏幕的原始嵌入拼接，形成一个统一的输入序列。
*   **两阶段训练算法**：
    *   **阶段 1：自蒸馏（Self-Distillation）**。让学生模型（带压缩记忆）去模仿老师模型（带完整原始历史）的输出分布，确保压缩后的 Token 保留了决策关键信息。
    *   **阶段 2：结果感知优化（Outcome-Aware Optimization）**。利用环境反馈（成功/失败奖励），通过 **RLOO（REINFORCE Leave-One-Out）** 算法微调压缩器，使记忆能区分并利用“成功经验”和“失败教训”。

### 6. 主要结论与发现
*   **性能飞跃**：Mem-W 在所有基准上均显著提升了骨干网络的效果。在 MMINA 任务中，相比原始模型提升了 **+27.0 到 +30.0** 个百分点。
*   **超越大模型**：搭载 Mem-W 的 8B 模型在某些指标上超过了 GPT-5 和 32B 的开源模型，证明了**高效的记忆机制比单纯增加模型参数更重要**。
*   **可扩展性**：随着外部经验库的增大，Mem-W 的性能持续提升，显示出良好的 Scaling 潜力。
*   **效率平衡**：虽然增加了压缩步骤，但由于减少了原始 Token 的输入，整体推理延迟保持在可接受范围内，且任务完成更果断（减少了无效步骤）。

### 7. 优点与亮点
*   **架构优雅**：通过潜空间 Token 统一了不同来源、不同尺度的记忆，消除了手动设计复杂记忆分类（如“失败提醒层”、“任务进度层”）的必要。
*   **端到端学习**：记忆的保留和过滤是由梯度（任务成功率）驱动的，而非人工定义的规则。
*   **结果感知**：通过强化学习让模型学会从失败中吸取教训，这在 GUI 任务中至关重要。

### 8. 不足与局限
*   **固定预算限制**：压缩器使用固定的 Token 预算（$K=8$），对于极度复杂的长轨迹，可能会存在信息丢失。
*   **模态覆盖**：目前主要集中在 Web 和 Mobile，尚未验证在桌面端（Windows/Mac）复杂多窗口环境下的表现。
*   **模型依赖**：虽然理论上是通用的，但压缩器仍需针对特定的骨干网络进行微调，尚未实现完全跨模型的“通用记忆插件”。
