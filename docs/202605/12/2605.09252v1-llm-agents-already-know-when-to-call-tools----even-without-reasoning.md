## 摘要
工具增强的 LLM 智能体往往会不加区分地调用工具，即使模型可以直接回答。每次不必要的调用都会浪费 API 费用并增加延迟，然而目前尚无基准测试系统地研究何时真正需要调用工具。我们提出了 When2Tool，这是一个包含 18 个环境（15 个单跳，3 个多跳）的基准测试，涵盖了工具必要性的三个类别——计算规模、知识边界和执行可靠性——每个类别都具有受控的难度级别，从而在需要工具的任务和不需要工具的任务之间建立了清晰的决策边界。我们评估了两类无需训练的基准方法：仅提示（Prompt-only，通过改变提示词来抑制不必要的调用）和先推理后行动（Reason-then-Act，要求模型在行动前对工具必要性进行推理）。两者提供的控制都有限：仅提示方法在抑制不必要调用的同时也抑制了必要的调用，而先推理后行动方法在困难任务上仍会产生不成比例的准确率损失。为了理解这些基准方法失败的原因，我们探测了模型的隐藏状态，发现工具必要性可以从生成前的表示中线性解码，在六个模型中的 AUROC 达到 0.89--0.96，显著超过了模型自身的口头推理。这表明模型已经知道何时需要工具，但在生成过程中未能根据这一知识采取行动。基于这一发现，我们提出了 Probe&Prefill，它使用轻量级线性探测器读取隐藏状态信号，并用引导语句预填充模型的响应。在所有测试的模型中，Probe&Prefill 在仅损失 1.7% 准确率的情况下减少了 48% 的工具调用，而准确率相当的最佳基准方法仅减少了 6% 的工具调用，或者在实现相似的工具调用减少量时产生了 5 倍的准确率损失。我们的代码可在 https://github.com/Trustworthy-ML-Lab/when2tool 获取。

## Abstract
Tool-augmented LLM agents tend to call tools indiscriminately, even when the model can answer directly. Each unnecessary call wastes API fees and latency, yet no existing benchmark systematically studies when a tool call is actually needed. We propose When2Tool, a benchmark of 18 environments (15 single-hop, 3 multi-hop) spanning three categories of tool necessity -- computational scale, knowledge boundaries, and execution reliability -- each with controlled difficulty levels that create a clear decision boundary between tool-necessary and tool-unnecessary tasks. We evaluate two families of training-free baselines: Prompt-only (varying the prompt to discourage unnecessary calls) and Reason-then-Act (requiring the model to reason about tool necessity before acting). Both provide limited control: Prompt-only suppresses necessary calls alongside unnecessary ones, and Reason-then-Act still incurs a disproportionate accuracy cost on hard tasks. To understand why these baselines fail, we probe the models' hidden states and find that tool necessity is linearly decodable from the pre-generation representation with AUROC 0.89--0.96 across six models, substantially exceeding the model's own verbalized reasoning. This reveals that models already know when tools are needed, but fail to act on this knowledge during generation. Building on this finding, we propose Probe&Prefill, which uses a lightweight linear probe to read the hidden-state signal and prefills the model's response with a steering sentence. Across all models tested, Probe&Prefill reduces tool calls by 48% with only 1.7% accuracy loss, while the best baseline at comparable accuracy only reduces 6% of tool calls, or achieves a similar tool call reduction but incurs a 5$\times$ higher accuracy loss. Our code is available at https://github.com/Trustworthy-ML-Lab/when2tool

以下是对该论文的结构化总结：

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：当前的工具增强型 LLM 智能体存在“过度调用”工具的问题，即使模型本身具备直接回答的能力，也会盲目调用 API。这导致了不必要的 API 费用支出和响应延迟。
*   **研究动机**：现有的基准测试主要关注模型“如何正确使用工具”，而忽略了“何时应该使用工具”的决策能力。作者旨在探索模型是否具备判断自身能力边界的意识，以及如何高效地优化这一决策过程。

### 2. 论文提出的方法论
论文提出了一个名为 **Probe & Prefill** 的轻量级推理时干预方法：
*   **核心思想**：利用模型内部隐藏状态中已存在的“工具必要性”信号，通过外部引导语干预生成过程，而非依赖模型昂贵的显式推理。
*   **关键技术流程**：
    1.  **特征提取**：在模型处理完输入提示词（Prompt）的最后一个 Token 时，提取其所有层的隐藏状态（Hidden States）。
    2.  **线性探测（Linear Probe）**：训练一个简单的逻辑回归分类器（探针），输入为拼接后的隐藏状态，输出为“是否需要工具”的概率 $p$。
    3.  **响应预填（Prefill）**：根据探针预测结果，在模型生成响应前预填一句引导语。
        *   若不需要工具：预填“我可以不使用工具直接解决这个问题。”
        *   若需要工具：预填“我需要为这个问题使用一个工具。”
    4.  **阈值控制**：通过调整探针的决策阈值 $\tau$，可以精细化地平衡准确率与工具调用成本。

### 3. 实验设计
*   **基准测试（When2Tool）**：作者构建了首个专门针对工具调用决策的基准，包含 18 个环境（15 个单跳，3 个多跳），分为三大类：
    *   **计算规模**：测试模型对自身心算极限的认知（如：简单加法 vs. 万亿级乘法）。
    *   **知识边界**：测试模型对内部参数知识与外部实时/冷门知识的区分。
    *   **执行可靠性**：测试模型对复杂逻辑追踪能力的评估（如：简单代码执行 vs. 深度递归追踪）。
*   **对比方法**：
    *   **Prompt-only**：通过改变系统提示词（如“仅在必要时使用工具”）来控制。
    *   **Reason-then-Act**：要求模型在行动前先进行思考（类似 ReAct 模式）。
    *   **SFT（监督微调）**：作为强基准，对模型进行全参数微调以学习决策边界。
*   **测试模型**：涵盖 Qwen3 系列（1.7B 到 32B）和 Llama-3.1/3.3 系列（8B 和 70B）。

### 4. 资源与算力
*   **探针训练**：极其轻量，在 CPU 上仅需数秒即可完成训练。
*   **推理开销**：探针推理延迟小于 1ms，相对于 LLM 的生成时间几乎可以忽略不计。
*   **SFT 基准**：为了进行对比实验，作者使用了 **4 张 GPU**（具体型号未标明，但提到使用了 HuggingFace Trainer 和梯度检查点）进行全参数微调。

### 5. 实验数量与充分性
*   **实验规模**：总计 1,080 个训练任务和 2,700 个测试任务。
*   **充分性**：
    *   在 6 个不同规模和家族的模型上进行了验证。
    *   进行了**分布外（OOD）泛化实验**，证明探针在未见过的任务环境上依然有效。
    *   在真实世界的搜索智能体基准（Search-o1）上进行了扩展测试。
    *   包含了丰富的**消融实验**：如软/硬预填对比、温度缩放、层选择、数据效率（仅需 10% 数据即可达到较高 AUROC）等。
*   **公平性**：所有对比基准均在相同环境下运行，并报告了多次运行的均值和标准差。

### 6. 主要结论与发现
*   **模型“心知肚明”**：LLM 的隐藏状态中包含非常清晰的工具必要性信号（AUROC 高达 0.89--0.96），即使模型在口头推理中表现混乱，其内部表示依然准确。
*   **推理并非万能**：要求模型先推理再行动（Reason-then-Act）往往无法有效减少不必要的调用，甚至在某些模型（如 Llama）上会导致调用逻辑崩溃。
*   **Probe & Prefill 效果显著**：在仅损失 1.7% 准确率的情况下，减少了 48% 的工具调用。在 Search-o1 搜索任务中，减少了 20%--56% 的 API 调用且无准确率损失。

### 7. 优点
*   **机制性洞察**：揭示了模型内部表示与生成行为之间的脱节（即模型知道但做不到）。
*   **极低成本**：无需微调大模型，无需额外的推理 Token，仅靠一个线性层和预填语句即可实现高效控制。
*   **高可控性**：通过调整探针阈值，开发者可以像调节旋钮一样精确控制“省钱”与“准确”的平衡点。

### 8. 不足与局限
*   **依赖标注数据**：虽然所需数据量很小，但仍需要针对特定任务收集“是否需要工具”的二元标签来训练探针。
*   **指令遵循风险**：部分模型（如 Llama-3.1-8B）可能会忽略“软预填”的引导语，此时需要使用“硬预填”（强制输出格式）来补救。
*   **任务类型限制**：目前主要针对计算、知识和执行类任务，对于更复杂的开放式创造性任务，工具必要性的定义可能更加模糊。
