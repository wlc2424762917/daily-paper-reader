## 摘要
关系学习是一个具有挑战性的问题，它催生了广泛的研究方法，包括基于图的模型（如图神经网络、图 Transformer）、表格方法（如表格基础模型）以及基于序列的方法（如大语言模型），每种方法都有其自身的优势和局限性。我们提出了 RelAgent，一种基于 LLM 的关系学习自主数据科学家，它分为两个阶段运行。在搜索阶段，LLM 智能体利用数据库、验证和评估工作区工具来构建 SQL 特征程序并选择预测模型。在推理阶段，生成的程序在无需进一步调用 LLM 的情况下执行。最终的预测器由 SQL 查询和经典模型组成，能够实现快速、确定性且具有内在可解释性的预测：特征是人类可读的查询，且预测仅取决于生成的由查询定义的特征映射，从而能够使用标准数据库系统进行可扩展部署。

## Abstract
Relational learning is a challenging problem that has motivated a wide range of approaches, including graph-based models (e.g., graph neural networks, graph transformers), tabular methods (e.g., tabular foundation models), and sequence-based approaches (e.g., large language models), each with its own advantages and limitations. We propose RelAgent, an LLM-based autonomous data scientist for relational learning, which operates in two phases. In the search phase, an LLM agent uses database, validation, and evaluation workspace tools to construct SQL feature programs and select a predictive model. In the inference phase, the resulting program is executed without further LLM calls. The final predictor consists of SQL queries and a classical model, enabling fast, deterministic, and intrinsically interpretable predictions: features are human-readable queries, and predictions depend only on the resulting query-defined feature map, enabling scalable deployment using standard database systems.

### RelAgent: 作为关系学习数据科学家的 LLM 智能体 —— 论文总结

#### 1. 核心问题与整体含义
关系数据库（RDB）是存储结构化数据的标准范式，但在其上进行预测建模（关系学习）通常依赖于繁琐的手动特征工程（如编写复杂的 SQL 进行连接和聚合）。现有的自动化方法（如 GNN、表格基础模型或直接序列化表格的 LLM）在灵活性、可扩展性或可解释性方面存在权衡。
**研究动机：** 论文旨在开发一种能够像人类数据科学家一样，通过自主探索数据库模式、编写 SQL 特征程序并迭代优化预测模型的智能体框架，从而实现高效、可解释且无需在推理阶段调用 LLM 的关系学习。

#### 2. 方法论
RelAgent 框架将关系学习建模为一个在 SQL 特征程序和预测模型空间中的**智能体搜索过程**，分为两个阶段：
*   **搜索阶段（Search Phase）：**
    *   **核心思想：** LLM 智能体通过三个工具接口与环境交互：
        1.  **数据库探索：** 使用只读 SQL 检查模式、表关系和数据分布。
        2.  **程序验证：** 提交由 SQL 查询组成的“特征程序”和模型配置（如 XGBoost、CatBoost 及其超参数）。系统在训练集上拟合模型，在验证集上评估并返回性能指标及错误诊断。
        3.  **评估工作区（Evaluation Workspace）：** 一个持久化的数据库，记录所有历史尝试的预测结果和诊断信息。智能体可以查询该工作区，进行跨尝试的错误分析（如“哪些样本在修改特征后预测更准了”）。
    *   **迭代优化：** 智能体根据验证反馈和工作区分析，不断修正 SQL 查询和模型选择。
*   **推理阶段（Inference Phase）：**
    *   **脱离 LLM：** 搜索结束后，选定的最佳 SQL 程序和拟合好的经典机器学习模型被部署。推理时直接执行 SQL 提取特征并输入模型，无需 LLM 参与，保证了速度和确定性。

#### 3. 实验设计
*   **数据集与场景：** 涵盖了三个主流关系学习基准：**RelBenchV1**、**RelBenchV2** 和 **4DBInfer**。涉及电商、社交网络、医疗、学术和体育等多个领域，任务包括流失预测、参与度预测、LTV（终身价值）估计等分类和回归任务。
*   **对比方法：**
    *   **监督表格模型：** LightGBM, XGBoost, AutoGluon（基于原始表或人工特征）。
    *   **关系深度学习（RDL）：** GraphSAGE, GAT, RelGNN, RelGT（基于图的方法）。
    *   **关系基础模型：** Griffin, Relational Transformer, KumoRFM-v1/v2（闭源或开源的基础模型）。
    *   **LLM 特征工程方法：** FeatLLM, ReFuGe。

#### 4. 资源与算力
*   **LLM 后端：** 主要使用 **GPT-5.2** 作为智能体大脑。
*   **对比后端：** 实验中还测试了 **DeepSeek-V4-Pro** 和 **Qwen3-32B**。
*   **硬件支持：** 对于本地运行的 Qwen3-32B，使用了 **2 张 NVIDIA RTX 6000 Ada (48GB)** 显卡进行推理。
*   **搜索成本：** 每次任务运行约 40-50 个回合，GPT-5.2 的平均成本约为每任务 1.57 美元，而 DeepSeek-V4-Pro 仅需约 0.11 美元。

#### 5. 实验数量与充分性
*   **实验规模：** 在 32 个不同的预测任务上进行了评估，涵盖了分类和回归。
*   **消融实验：** 针对迭代反馈、评估工作区、语义信息（模式匿名化）、预测器选择和不同 LLM 后端进行了详尽的消融研究。
*   **充分性评价：** 实验设计非常充分且客观。通过 5 次独立搜索并取最佳验证分数的策略（Cross-rollout selection）降低了随机性影响，并提供了标准差分析。对比基线涵盖了从传统机器学习到最前沿的闭源基础模型，具有很强的说服力。

#### 6. 主要结论与发现
*   **性能卓越：** RelAgent 在多个基准上达到了 SOTA 或接近 SOTA 的水平，甚至在某些任务上超过了强大的闭源模型 KumoRFM-v2。
*   **搜索价值：** 消融实验证明，迭代反馈循环对性能至关重要（AUROC 从 66.86 提升至 78.38）。
*   **模型与特征协同：** 智能体能够根据任务特性（如长尾分布的回归任务）自动选择 L1 损失函数和对数变换，展现了超越单纯特征工程的综合数据科学能力。
*   **SQL 的表达力：** 智能体能发现复杂的关联模式（如三向自连接检测有向三角形），这是标准 GNN 难以捕捉的。

#### 7. 优点
*   **内在可解释性：** 预测依据是人类可读的 SQL 查询，每个特征的含义清晰透明。
*   **推理高效：** 推理阶段完全摆脱 LLM，可在标准数据库环境中低延迟运行。
*   **灵活性：** 不受固定图架构或序列长度限制，通过 SQL 表达复杂的业务逻辑。
*   **自我诊断：** 引入“评估工作区”让智能体具备了类似人类的“反思”和“对比实验”能力。

#### 8. 不足与局限
*   **搜索成本：** 虽然推理便宜，但搜索阶段需要多次调用 LLM，对于超大规模模式的数据库，Token 消耗和时间成本较高。
*   **搜索空间限制：** 在有限的回合内，智能体可能无法发现极深层次的连接（Join）或极其复杂的逻辑组合。
*   **非结构化数据处理：** 目前主要侧重于关系结构和数值/类别特征，对长文本字段或多模态属性的处理能力有限。
*   **验证集依赖：** 存在对验证集过拟合的潜在风险，尤其是在样本量极小的情况下。
