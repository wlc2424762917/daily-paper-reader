## 摘要
自动多智能体系统（MAS）旨在不依赖手动设计或固定编排的情况下实例化智能体工作流。然而，现有的自动 MAS 方法仅具有部分自适应性：它们要么执行无需训练的测试时搜索，要么在保持下游执行智能体冻结的情况下优化元级设计器，这造成了“冻结执行器”瓶颈，并使得自设计和自执行智能体模型的端到端训练仍未得到探索。为解决这一问题，我们引入了 MetaAgent-X，这是一个端到端强化学习框架，可联合优化自动 MAS 的设计与执行。MetaAgent-X 支持基于脚本的 MAS 生成、执行展开（rollout）收集，以及针对设计器和执行器轨迹的信用分配。为了支持稳定且可扩展的优化，我们提出了执行器-设计器分层展开（Executor Designer Hierarchical Rollout）和阶段性协同演化（Stagewise Co-evolution），以提高训练稳定性并揭示设计器-执行器协同演化的动态过程。MetaAgent-X 的表现始终优于现有的自动 MAS 基准，最高实现了 21.7% 的性能提升。全面的消融实验表明，设计器和执行器在整个训练过程中均有所提升，且有效的自动 MAS 学习遵循阶段性协同演化过程。这些结果确立了端到端可训练的自动 MAS 是构建自设计和自执行智能体模型的一种实用范式。

## Abstract
Automatic multi-agent systems aim to instantiate agent workflows without relying on manually designed or fixed orchestration. However, existing automatic MAS approaches remain only partially adaptive: they either perform training-free test-time search or optimize the meta-level designer while keeping downstream execution agents frozen, which creating a frozen-executor ceiling and leaving the end-to-end training of self-designing and self-executing agentic models unexplored. To address this, we introduce MetaAgent-X, an end-to-end reinforcement learning framework that jointly optimizes automatic MAS design and execution. MetaAgent-X enables script-based MAS generation, execution rollout collection, and credit assignment for both designer and executor trajectories. To support stable and scalable optimization, we propose Executor Designer Hierarchical Rollout and Stagewise Co-evolution to improve training stability and expose the dynamics of designer-executor co-evolution. MetaAgent-X consistently outperforms existing automatic MAS baselines, achieving up to 21.7% gains. Comprehensive ablations show that both designer and executor improve throughout training, and that effective automatic MAS learning follows a stagewise co-evolution process. These results establish end-to-end trainable automatic MAS as a practical paradigm for building self-designing and self-executing agentic models.

自动多智能体系统（Automatic MAS）旨在根据任务需求自动构建智能体工作流。然而，现有方法存在“部分自适应”的局限：
*   **冻结执行器瓶颈（Frozen-executor Ceiling）：** 现有方法要么在推理时进行搜索（无需训练），要么仅优化负责设计的“元智能体（Designer）”，而保持下游执行任务的“执行器（Executor）”处于冻结状态。
*   **参数级脱节：** 设计者与执行者仅通过提示词（Prompt）交互，缺乏参数层面的联合优化，导致系统无法诱导执行者产生特定的专业化行为。
*   **协同演化机制不明：** 业界尚不清楚设计者与执行者在共同训练下如何相互促进。
**MetaAgent-X** 的提出正是为了通过**端到端强化学习（End-to-End RL）**同时优化设计者和执行者，打破性能天花板。

#### 2. 方法论：核心思想与关键技术
MetaAgent-X 采用联合优化框架，将 MAS 的设计与执行视为一个耦合的在线强化学习问题。
*   **核心思想：** 设计者生成描述 MAS 架构的 Python 脚本，执行者根据脚本实例化并运行工作流，系统根据最终环境反馈（Reward）同时更新两者的策略参数。
*   **关键技术 1：执行者-设计者分层展开（Hierarchical Rollout）：**
    *   采用树状采样结构：针对每个问题采样 $M$ 个设计方案，每个方案进行 $N$ 次独立执行。
    *   **信用分配（Credit Assignment）：** 设计者的优势函数（Advantage）基于 $N$ 次执行的平均奖励，以消除执行过程中的随机噪声；执行者的优势函数则在问题级别进行归一化，对比不同设计下的执行表现。
*   **关键技术 2：阶段性协同演化（Stagewise Co-evolution）：**
    *   为了解决非平稳环境下的训练不稳定问题，将训练分为交替的阶段（如每 30 步切换一次）：在执行者阶段仅更新执行能力，在设计者阶段仅更新架构设计能力。
*   **优化算法：** 使用 **GRPO（Group Relative Policy Optimization）** 算法，通过组内相对优势进行策略更新，无需额外的价值函数网络。

*   **算法稳健：** 提出的分层展开和阶段性优化策略有效解决了多智能体强化学习中的信用分配难题和非平稳性问题。
