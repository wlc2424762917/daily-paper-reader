# Abstract 

Computer-use agents are increasingly capable of operating on real operating sys-tems, but this capability has also increased the risks posed by prompt injection, indirect instructions, and visual attacks. Existing defenses typically rely on analyz-ing the prompt or each potentially malicious input with a second large model at inference time, which can limit coverage or increase deployment cost. We propose ProjGuard, an alternative based on behavioral trajectory monitoring. At each step, we derive a lightweight scalar risk signal from the agent’s accumulated interaction history and evaluate, online, whether execution is beginning to drift toward an unsafe region. This enables early warnings before the trajectory reaches a poten-tially harmful action. When an alert is raised, we selectively activate an auxiliary vision-language model to propose a corrected next step and steer execution back toward task completion. Experiments on OS-Harm show that monitoring with on-demand correction reduces the unsafe rate from 16% to 3% while improving task completion from 59% to 65%. We further evaluate transfer to RiosWorld, where the method remains competitive, reaching 4% unsafe and 64% completion. Overall, these results support a hierarchical safety strategy in which always-on monitoring anticipates deviations and activates correction only when needed. 

# 1 Introduction 

Computer-use agents have evolved from conversational assistants into systems capable of processing screenshots, reading accessibility trees, interacting with browsers, editors, email clients, and terminals, and executing long action sequences on a real operating system [ 1 ]. This evolution creates immediate opportunities for automation, but it also introduces security issues: a malicious signal no longer merely alters generated text, but can instead induce concrete actions over files, credentials, web navigation, email, or system configuration, including the exfiltration of sensitive information such as passwords, Secure Shell (SSH) keys, or banking credentials [2]. Recent evidence confirms this fragility. The OS-Harm benchmark shows that both commercial and open computer-use agents remain vulnerable to deliberate misuse, prompt injection, and unsafe behavior in real desktop tasks [ 3]. In parallel, visual attacks such as Malicious Image Patches demonstrate that localized, nearly imperceptible perturbations can hijack the decision-making process of multimodal operating-system agents [4]. A different starting point is to recognize that existing defenses still face important limitations in an always-on monitoring setting. Another family of approaches constructs semantic safety spaces from previously observed benign and malicious examples and then uses that geometry to discriminate new inputs; the main issue is that such strategies depend heavily on prior knowledge of the attack form and become less robust when the surface realization changes or when the threat is indirect and multimodal. In addition, high-capacity semantic verification approaches such as LLM-as-a-Judge [ 6 ] can provide richer safety signals, but they typically incur higher operational cost when run continuously. Monitors that rely on a second LLM can provide stronger guardrails by reviewing instructions, tools, or execution steps, but they introduce cumulative latency, higher deployment complexity, and an inference cost that is difficult to sustain as a continuous defense [ 7]. Complementarily, learned-trajectory approaches such as TrajAD [ 8] show the potential of modeling behavioral anomalies, although they depend on learned representations and can increase inference cost in always-on deployments. Overall, the state-of-the-art still lacks a dynamic monitoring mechanism that is lightweight enough to run at each decision step in practice while remaining general enough to detect textual, indirect, and visual deviations. Such attacks often manifest in the agent’s behavioral trajectory. A compromised agent does not merely receive a malicious signal; it begins to generate anomalous responses, introduce justifications unrelated to the original goal, explore interfaces irrelevant to the task, or produce actions whose semantics drift away from the intended workflow. This suggests that safety violations are not only input-level phenomena but are reflected in the structure of the trajectory itself. We therefore hypothesize that agent trajectories can be partitioned into safe and compromised behaviors, and that this separation becomes evident in a low-dimensional representation space, enabling detection without explicitly analyzing the input prompt. In this work, we propose ProjGuard, a trajectory-based safety mechanism for computer-use agents that monitors behavior rather than inputs. At each step, we construct a lightweight representation of the accumulated trajectory and learn a projection into a low-dimensional space using Linear Discriminant Analysis, where unsafe behaviors become separable. This enables efficient, real-time detection of prompt injection by identifying deviations in the trajectory dynamics, without invoking an additional large model. When a potential risk is detected, we trigger a corrective escalation mechanism that activates a VLM-based agent only for flagged trajectories. Given the task, partial trajectory, and current observation, the corrective agent generates a revised next action that replaces the original one, steering execution back toward the intended objective. Otherwise, the agent proceeds unchanged. This conditional intervention ensures efficient monitoring while enabling targeted correction when unsafe drift is detected (Figure 1). The contributions of this work can be summarized as follows • We introduce ProjGuard, a lightweight trajectory-based monitoring framework that projects partial agent interactions into a low-dimensional representation space, enabling early detec-tion of unsafe behavioral drift. 2• We propose a hierarchical safety mechanism that combines continuous low-cost monitoring with selective escalation to a stronger VLM-based correction module only when risk is detected, substantially reducing unnecessary intervention costs. • We evaluate the proposed approach on the OS-Harm and RiosWorld benchmarks, showing that ProjGuard achieves strong safety-completion trade-offs while reducing monitoring overhead compared to other approaches. 

# 2 Related Work 

Trajectory-level and runtime safety monitors. TrajAD [ 8] addresses trajectory-level anomaly detection in LLM agents using learned behavioral representations and a trained verifier to identify unsafe drift. ShieldAgent [ 9] proposes an explicit policy-verification scheme in which candidate agent actions are checked against verifiable safety rules before execution. StepShield [ 10 ] focuses on intervention during execution, formulating when to intervene under potentially risky behavior in multi-step sequences. 

Agent benchmarks and attack surface. OSWorld introduced a standardized environment for evaluating multimodal agents on open-ended tasks over a real operating system, including multiple applications and observation modalities [ 1]. OS-Harm extended this line of work toward safety by formalizing scenarios of misuse, prompt injection, and unsafe agent behavior [ 3 ]. Beyond OS-level benchmarks, web-interaction environments have provided additional testbeds for long-horizon grounded decision making, including World of Bits [11], WebShop [12], WebArena [13], and large-scale web task collections such as Mind2Web [ 14 ]. Recent multimodal web agents and evaluation ecosystems further highlight the breadth of the surface, including WebVoyager [ 15 ], WorkArena [ 16 ], and BrowserGym [ 17 ]. These benchmarks are essential because they turn agent safety into an end-to-end measurable problem: what matters is not only whether the model generates problematic text, but whether it ultimately executes unsafe actions in the environment. In parallel, visual attacks have expanded the threat surface. Localized perturbations can alter the behavior of multimodal agents even when the signal is nearly imperceptible to humans [ 4 ], and this vulnerability is consistent with broader evidence on vision-language and visual models [ 18 , 19 ]. From a safety monitoring perspective, these results imply that a monitor that reasons only over visible user text may miss important causal signals that appear in rendered content. 

Prompt- and instruction-centered monitors. A dominant line of work in the literature attempts to reinforce model obedience to privileged instructions and reduce the influence of external content. Authors in [ 5 ] propose an instruction hierarchy to train models that prioritize trusted sources, while [ 2]documents the severity of indirect prompt injection in real applications. These approaches function as safety monitors or guardrails centered on the prompt, and are especially useful when the threat appears in explicit textual form. However, their weakness is also clear: they depend on the adversarial signal being observable in the instruction itself or in the available textual context. When the attack is embedded in a document, a webpage, a system notification, or an image, the textual monitor no longer sees the cause of the problem and can only react if the deviation appears later in the agent’s behavior. 

Tool-using agents and semantic verification. The rapid growth of tool-augmented agents has been enabled by prompting and training strategies that interleave reasoning with actions and external API calls [ 20 , 21 ]. More recent work shows that documentation and large-scale tool corpora can improve generalization to new tools [ 22 –24 ]. Complementary agent designs incorporate deliberation and self-improvement at test time via structured search and reflection [ 25 , 26 ], and extend these ideas to embodied settings [27]. These developments increase capability, but also amplify the importance of runtime checks that can detect when tool-using behavior begins to drift. 

Prompt attacks and robustness evaluations. Beyond indirect prompt injection, recent work has documented systematic jailbreak behaviors and transferable adversarial prompt constructions [28 , 29 ]. Correspondingly, proposed defenses and evaluation methodologies emphasize the difficulty of building broadly effective mitigations against optimized attacks [ 30 , 31 ]. At the training level, alignment schemes such as Constitutional AI aim to increase harmlessness and reduce unsafe generations, providing an additional layer of protection that can complement runtime monitoring [ 32 ]. 3Figure 2: Overview of the proposed method. # 3 Methodology 

To provide a high-level overview of the proposed method, Figure 2 summarizes a hierarchical safety architecture with continuous monitoring and on-demand correction. At each step t, the monitor constructs a low-dimensional representation from the partial trajectory, projects it to a scalar coordinate, and produces a risk score ρt that indicates whether execution is drifting toward an unsafe region. When the alert threshold is exceeded, a correction module is selectively activated to propose the next step and steer execution back toward the original task; otherwise, the agent proceeds without intervention. We formalize the project as follows.

# 4 Experiments 

Setup. The experimental evaluation begins with an agent performance baseline on the tasks under consideration. This measurement indicates how well the models perform across different observation settings. From that reference point, the evaluation focuses on prompt injection scenarios and on the trajectories they produce, and then studies whether safe and unsafe trajectories are separable in the proposed low-dimensional space. 5Benchmarks and tasks. The experiments build on OSWorld [ 1] and use the OS-Harm benchmark [3 ] to study prompt injection scenarios in desktop agents. We additionally evaluate transfer on the RiosWorld benchmark [ 33 ], using a matched subset of tasks to enable comparison across benchmarks. The tasks span web navigation, document editing, email, programming, and workflows involving multiple applications, which makes it possible to observe agent behavior in varied settings that are close to real use. 

Data. Each run produces a structured trace containing the instruction, the agent’s responses, the executed actions, the screenshots, and the final safety judgment. From these traces, an analysis set is built with fields for prompt , response , and action , together with a binary safe /unsafe label. In the current repository version, this set contains 196 trajectories associated with prompt injection scenarios, including both safe and unsafe executions. Although the complete trace includes responses and actions, the analysis developed in this section focuses on the response sequence. 

Baselines. We start by running unprotected agents on the evaluation tasks to establish the no-monitoring baseline. We report task completion and unsafe-trajectory rates for GPT-5.2 and Claude Opus 4.5, which serve as reference points for all subsequent comparisons (Table 4). For observation, we provide both screenshots and the accessibility tree; Appendix A reports an ablation by observation modality and motivates this choice for the main experiments. 

Metrics. We report trajectory-level safety (unsafe-trajectory rate) and utility (task-completion rate). When analyzing separability in the learned one-dimensional coordinate, we report the silhouette score and Davies-Bouldin index. When comparing monitoring strategies, we also report operational costs in terms of API calls, tokens, and latency.

Table 3 highlights three key observations. First, both alert-only and correction-only configurations significantly reduce the unsafe rate compared to the baseline (from 16% to 2%). This indicates that the trajectory-based monitoring signal alone is already effective at identifying unsafe behavior, even without intervention, and that corrective reasoning can mitigate unsafe outcomes when applied. Second, the correction-only setting achieves the highest completion rate (66%), but at a prohibitive computational cost. Since the VLM is invoked at every step, this configuration requires 708 API calls, 35,000 tokens, and 90 seconds of latency. This highlights a key limitation of step-by-step correction strategies: while effective, they scale poorly and are impractical for real-time deployment. Third, the full method (alert + correction) achieves a similar completion rate (65%, within 1 point of the correction-only method) while drastically reducing the computational cost to only 6 API calls, 300 tokens, and 75 seconds of latency. This reduction stems from the fact that the correction module is only activated when the monitoring stage detects risk, rather than at every step.

# 5 Conclusion 

This work studied the safety of multimodal desktop agents through their execution trajectories, rather than analyzing only the content of the prompt. The main result is that safe and unsafe trajectories occupy distinguishable regions in a lightweight one-dimensional space, and that this structure can be exploited for low-cost continuous monitoring. In particular, the Hashing (2048) + Linear Discriminant Analysis configuration achieved a silhouette score of 0.953 and a Davies–Bouldin index of 0.524, confirming that the observed separation is sufficiently clear to serve as the basis for the monitor. On that basis, the experiments showed that the main role of monitoring is to issue an alert when the trajectory enters an unsafe region and, from that signal, selectively activate a correction stage. The 9GPT-5.2 + Linear Discriminant Analysis + GPT-5.4 combination achieved 3% unsafe trajectories and 65% task completion with only 6 API calls, 300 tokens, and 75 s of latency. Although the StepJudge-Safety variant achieves 2% unsafe trajectories and 66% task completion, it requires 708 API calls, 35,000 tokens, and 90 s. Taken together, these results show that the proposed method preserves a level of safety and performance very close to that of the best baseline, while incurring a much lower computational cost. In addition, the transfer to RiosWorld was evaluated on an equivalent subset of tasks, enabling a consistent comparison with the prompt injection scenario in OS-Harm. Likewise, the trajectory-based formulation with selective correction is general enough to extend across both benchmarks considered in this work, OS-Harm and RiosWorld. Lightweight geometric representations can also be sensitive to domain shift, requiring recalibration when deployed to new environments.

# C Representation Separability Analysis 

This section provides additional exploratory analyses of geometric separability between benign and malicious instances under lightweight text representations. We visualize structure using low-dimensional projections. 

Projections. We use low-dimensional projections to visualize geometry. PCA provides a global linear view by maximizing variance but is not optimized for class separation. t-SNE and UMAP are nonlinear methods that emphasize neighborhood structure and can reveal clusters while distorting global geometry in different ways. LDA is a supervised projection that explicitly maximizes separation between benign and malicious instances, making it informative when labels are available. We compare lightweight representations that preserve different signals. TF-IDF downweights com-mon tokens and emphasizes rare informative terms; BoW encodes unordered token counts; SBERT produces dense embeddings intended to capture semantic similarity; and character n-grams capture local writing patterns that often reflect style or obfuscation cues.
