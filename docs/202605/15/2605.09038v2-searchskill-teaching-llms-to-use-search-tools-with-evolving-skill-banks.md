## 摘要
教大语言模型使用搜索工具不仅在于它们是否进行搜索，还在于它们是否能发出高质量的查询。这在开放域问答中尤为重要，因为宽泛或复制的查询往往会浪费检索预算，并干扰后续的推理。我们提出了 SearchSkill，这是一个通过可重用搜索技能使查询规划显式化的框架。在每一步中，模型首先选择一项技能，然后根据所选的技能卡生成搜索或回答动作。技能清单本身并非固定不变：SearchSkill 维护着一个不断演进的技能库（SkillBank），根据反复出现的失败模式对其进行扩展或优化，并在监督训练前重构受影响的轨迹。由此产生的两阶段监督微调（SFT）方案将训练与推理时的协议对齐，即先进行技能选择，再进行基于技能的执行。在开源和闭源模型上，SearchSkill 提高了知识密集型问答基准测试中的精确匹配率，并带来了更好的检索行为，包括减少了首个查询的复制、增加了更多针对原子跳跃（atomic hop）的查询，以及在较小的搜索预算内获得了更多正确答案。这些结果表明，显式的基于技能的查询规划是替代将搜索视为无差别动作的一种轻量化方案。

## Abstract
Teaching language models to use search tools is not only a question of whether they search, but also of whether they issue good queries. This is especially important in open-domain question answering, where broad or copied queries often waste retrieval budget and derail later reasoning. We propose \Ours, a framework that makes query planning explicit through reusable search skills. At each step, the model first selects a skill, then generates a search or answer action conditioned on the selected skill card. The skill inventory itself is not fixed: SearchSkill maintains an evolving SkillBank, expands or refines it from recurrent failure patterns, and reconstructs affected trajectories before supervised training. The resulting two-stage SFT recipe aligns training with the inference-time protocol of skill selection followed by skill-grounded execution. Across open-source and closed-source models, SearchSkill improves exact match on knowledge-intensive QA benchmarks and yields better retrieval behavior, including fewer copied first queries, more atomic hop-focused queries, and more correct answers within a small search budget. These results suggest that explicit skill-conditioned query planning is a lightweight alternative to treating search as an undifferentiated action.

### 1. 核心问题与整体含义（研究动机与背景）
*   **核心问题**：现有的 LLM 在使用搜索工具时，瓶颈往往不在于“是否搜索”，而在于“如何构建高质量查询”。在处理复杂的多跳问答（Multi-hop QA）时，模型常会生成过于宽泛或直接复制原始问题的查询，导致检索效率低下、噪声多且推理失败。
*   **研究背景**：虽然 RAG 和强化学习（RL）方法（如 Search-R1）提升了检索能力，但它们通常将搜索视为一种无差别的动作。本文认为，应该将搜索过程拆解为显式的“规划-执行”步骤，通过可重用的技能来规范模型的搜索行为。

### 2. 方法论：核心思想与关键技术
SearchSkill 的核心是将搜索过程转化为 **“选择-阅读-执行”（Select-Read-Act）** 协议：
*   **可演进的技能库（SkillBank）**：
    *   维护一个包含多种搜索策略的技能清单（如：桥接实体搜索、属性对比、时间范围提取、冲突检查等）。
    *   **演进机制**：从基础种子库（B0）开始，根据模型在不同难度数据集（单跳到多跳）上的失败模式，利用 GPT-5.4 迭代式地添加或优化技能卡片，最终形成 B4 版技能库。
*   **两阶段监督微调（SFT）**：
    *   **第一阶段（格式学习）**：训练模型掌握 `<skill>` 标签和搜索/回答动作的基本格式。
    *   **第二阶段（技能条件化执行）**：训练模型根据 SkillBank 索引选择技能 ID，并结合技能卡片的内容生成具体的查询动作。
*   **算法流程**：
    1.  模型输出 `<select_skill>` 标签选择技能。
    2.  系统将对应的技能卡片内容注入上下文。
    3.  模型基于技能指导输出具体的 `<search>` 查询或 `<answer>` 答案。
    4.  检索结果反馈给模型，循环直至得出答案。

### 6. 主要结论与发现
*   **性能领先**：SearchSkill 在 7B 和 3B 模型上均取得了最佳的平均精确匹配率（EM），优于大规模 RL 训练的搜索智能体。
*   **查询质量提升**：相比基准方法，SearchSkill 生成的查询更具“原子性”（针对特定子目标），减少了对原始问题的直接复制。
*   **效率更高**：在较小的搜索预算（如 3 次搜索以内）下，SearchSkill 的正确率显著更高。
*   **通用性**：即使不进行微调，将演进后的技能库作为 Prompt 提供给闭源模型（如 Claude），也能显著提升其搜索规划能力。

### 7. 优点与亮点
*   **显式规划**：将模糊的搜索动作转化为结构化的技能选择，增强了模型的可解释性和受控性。
*   **轻量高效**：证明了通过高质量数据和两阶段 SFT，可以达到甚至超过高成本强化学习（RL）的效果。
*   **动态演进**：技能库不是静态的，而是通过失败驱动（Failure-driven）不断优化，具有很强的适应性。

### 8. 不足与局限
*   **数据依赖**：技能库的演进高度依赖于强大的教师模型（如 GPT-5.4）进行失败分析和轨迹重构。
*   **RL 增益有限**：在 SFT 基础上引入 GRPO 强化学习后，准确率提升并不显著，说明目前的奖励函数或 RL 算法在技能空间内的优化仍有挑战。
*   **领域局限**：实验主要集中在百科类问答，对于需要实时性极强或极度专业化工具（如代码执行、数学计算）的搜索技能尚未充分验证。
