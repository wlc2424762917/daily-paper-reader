## 摘要
随着基于大语言模型（LLM）的智能体越来越多地代表用户浏览网页，一个自然而然的问题随之而来：网站能否被动地识别出驱动智能体的底层模型？这样做将构成重大的安全风险，使得针对已知模型漏洞的定向攻击成为可能。通过对 14 种前沿 LLM 和涵盖信息检索及购物任务的四个网络环境的研究，我们证明，通过被动 JavaScript 追踪器捕获的智能体动作和交互时序，足以识别底层模型，F1 分数高达 96%。我们通过证明在智能体动作上训练的分类器可以在不同模型规模和系列之间泛化，从而使这一攻击面正式化。我们进一步表明，仅需少量交互轨迹即可训练出强大的分类器，并且可以在一个回合（episode）的早期推断出智能体身份。在动作之间注入随机的时间延迟会显著降低分类器的性能，但并不能提供稳健的保护：在延迟轨迹上重新训练的分类器在很大程度上能够恢复性能。我们在此发布了我们的测试框架和带标签的智能体轨迹语料库。

## Abstract
As LLM-based agents increasingly browse the web on users' behalf, a natural question arises: can websites passively identify which underlying model powers an agent? Doing so would represent a significant security risk, enabling targeted attacks tailored to known model vulnerabilities. Across 14 frontier LLMs and four web environments spanning information retrieval and shopping tasks, we show that an agent's actions and interaction timings, captured via a passive JavaScript tracker, are sufficient to identify the underlying model with up to 96\% F1. We formalise this attack surface by demonstrating that classifiers trained on agent actions generalise across model sizes and families. We further show that strong classifiers can be trained from few interaction traces and that agent identity can be inferred early within an episode. Injecting randomised timing delays between actions substantially degrades classifier performance, but does not provide robust protection: a classifier retrained on delayed traces largely recovers performance. We release our harness and a labelled corpus of agent traces \href{https://github.com/KabakaWilliam/known_actions}{here}.

### 2. 方法论
*   **核心思想**：将智能体识别建模为一个基于行为轨迹的监督分类问题。研究假设不同的 LLM 在推理速度、规划逻辑和交互习惯上存在细微差异，这些差异会映射到 UI 操作的时序和结构中。
*   **关键技术细节**：
    *   **数据采集**：使用被动 JavaScript 追踪器记录事件流 $\tau = \{(s_t, u_t, \Delta t_t)\}$，包括动作类型 $u_t$（点击、滚动、按键）和事件间隔时间 $\Delta t_t$。
    *   **特征工程**：提取了 41 维标量特征，分为五类：
        1.  **时间动态**：事件间间隔（IEI）的均值、标准差、分位数等。
        2.  **滚动行为**：最大滚动深度、方向反转次数。
        3.  **点击分布**：点击坐标的离散度、特定区域（如导航栏）的点击比例。
        4.  **导航策略**：页面停留时间、动作/页面比例。
        5.  **结构化特征**：功能键（Enter/Tab）与普通字符按键的比例。
    *   **分类模型**：对比了 XGBoost（表现最好）、随机森林、逻辑回归和 LSTM。
    *   **识别模式**：分为“闭集识别”（从已知 14 种模型中选一）和“开集检测”（识别是否为未知模型）。

### 3. 实验设计
*   **实验场景**：涵盖了信息检索（2WikiMultiHop, FRAMES）和在线购物（Webshop, Deepshop）两大领域。
*   **Benchmark 与对比**：
    *   **模型库**：包含 14 种前沿 LLM，涵盖闭源（GPT-5.4, Claude 4.6, Gemini 3.1, Seed-2.0-Lite）和开源（Qwen3/3.5 系列, GLM-4.6V, Gemma-4, UI-TARS-1.5）。
    *   **基准线**：随机猜测（约 7% 的准确率）。
    *   **防御测试**：在动作间注入 0.5s 到 5s 的随机延迟，测试分类器的鲁棒性。

*   **时长**：全量数据采集耗时约 **3 天**。

### 5. 实验数量与充分性
*   **实验规模**：共收集了 **1050 条** 完整的交互轨迹。
*   **实验维度**：
    *   **闭集分类**：验证 14 种模型的区分度。
    *   **开集检测**：使用“留一法”（LOO）验证对未知模型的识别能力。
    *   **样本效率**：测试仅用 1/3 的训练数据或仅观察交互前 40% 的动作时的识别效果。
    *   **泛化性实验**：跨任务（QA 到购物）和跨网站的指纹迁移测试。
*   **评价**：实验设计非常全面，不仅验证了攻击的有效性，还深入探讨了攻击的效率、时机以及对抗防御（随机延迟）的手段，具有很高的客观性和说服力。

### 6 主要结论与发现
*   **高识别率**：在闭集设置下，识别底层模型的 Macro F1 分数最高可达 **96%**。
*   **时间是关键但非唯一**：时间特征（IEI）是识别的最强信号，但即使注入随机延迟，分类器通过重新训练，利用点击坐标和动作结构特征仍能恢复大部分性能。
*   **早期识别**：仅需观察一个会话中前 **15 个左右的动作**（或前 40% 的轨迹），即可准确锁定模型身份。
*   **防御失效**：简单的随机延迟防御无法阻止“自适应攻击者”，因为智能体的固有行为模式（做什么）比节奏（何时做）更难掩盖。

### 7. 优点（亮点）
*   **新颖的视角**：首次证明了 LLM 智能体的底层模型身份会通过 UI 行为“泄露”，将指纹识别从“人机辨长”提升到了“模型辨识”的高度。
*   **实战化威胁模型**：提出的“被动、同位置攻击者”模型非常符合现实中网站运营者的角色，且攻击成本极低。
*   **开源贡献**：发布了首个带标签的智能体交互轨迹数据集和测试框架，填补了该领域的空白。

### 8. 不足与局限
*   **框架依赖性**：实验主要基于 Midscene.js 这一种智能体框架，尚未验证不同框架（如 Playwright 原生调用 vs. 其他封装库）是否会引入干扰指纹。
*   **开集检测尚不完美**：虽然开集检测优于随机猜测，但对于某些模型（如 Seed-2-lite），其在未知环境下的识别表现仍有待提高。
*   **防御手段单一**：论文主要讨论了随机延迟这一种防御，未深入探讨更复杂的混淆策略（如注入诱导性冗余动作）。
