## 摘要
GUI 定位是使 GUI 智能体能够执行点击和拖动等任务的关键能力。然而，在 ScreenSpot-Pro 基准测试等复杂场景中，现有模型的性能往往不尽如人意。利用所提出的掩码预测分布（MPD）归因方法，我们发现误差的主要来源有两个方面：高图像分辨率（导致精度偏差）和复杂的界面元素（导致歧义偏差）。为了应对这些挑战，我们引入了偏差感知操作推理（BAMI），它结合了由粗到精的聚焦和候选选择这两项关键操作，以有效缓解这些偏差。我们广泛的实验结果表明，在无需训练的情况下，BAMI 显著提高了各种 GUI 定位模型的准确性。例如，将我们的方法应用于 TianXi-Action-7B 模型，使其在 ScreenSpot-Pro 基准测试中的准确率从 51.9% 提升至 57.8%。此外，消融研究证实了 BAMI 方法在不同参数配置下的鲁棒性，突显了其稳定性和有效性。代码可在 https://github.com/Neur-IO/BAMI 获取。

## Abstract
GUI grounding is a critical capability for enabling GUI agents to execute tasks such as clicking and dragging. However, in complex scenarios like the ScreenSpot-Pro benchmark, existing models often suffer from suboptimal performance. Utilizing the proposed \textbf{Masked Prediction Distribution (MPD)} attribution method, we identify that the primary sources of errors are twofold: high image resolution (leading to precision bias) and intricate interface elements (resulting in ambiguity bias). To address these challenges, we introduce \textbf{Bias-Aware Manipulation Inference (BAMI)}, which incorporates two key manipulations, coarse-to-fine focus and candidate selection, to effectively mitigate these biases. Our extensive experimental results demonstrate that BAMI significantly enhances the accuracy of various GUI grounding models in a training-free setting. For instance, applying our method to the TianXi-Action-7B model boosts its accuracy on the ScreenSpot-Pro benchmark from 51.9\% to 57.8\%. Furthermore, ablation studies confirm the robustness of the BAMI approach across diverse parameter configurations, highlighting its stability and effectiveness. Code is available at https://github.com/Neur-IO/BAMI.

### 论文总结：BAMI —— GUI 定位中无需训练的偏差缓解

#### 1. 核心问题与整体含义（研究动机和背景）
GUI 定位（GUI Grounding）是 GUI 智能体执行任务（如点击、拖拽）的基础。尽管多模态大模型（MLLM）在通用任务上表现出色，但在处理高分辨率、元素密集的复杂界面（如专业软件）时，定位准确率往往低于 50%。
*   **研究动机**：作者通过新提出的 **MPD（掩码预测分布）** 归因方法分析发现，定位失败的主要原因并非完全是知识匮乏（仅占 14%），更多源于**归纳偏差（Inductive Bias）**，包括：
    *   **精度偏差（Precision Bias）**：由于坐标离散化标记化，模型在高分辨率下难以达到像素级精确。
    *   **歧义偏差（Ambiguity Bias）**：模型训练目标是最小化 Token 的编辑距离，而非欧氏空间距离，导致模型在相似元素间产生选择错误。
*   **核心目标**：开发一种无需重新训练、即插即用的推理增强方法来缓解这些偏差。

#### 2. 方法论：BAMI 推理框架
BAMI（Bias-Aware Manipulation Inference）将单步定位转变为多步结构化推理过程，包含两个核心操作：
*   **由粗到精的聚焦（Coarse-to-Fine Focus）**：
    *   **流程**：先进行初步预测，根据预测坐标对原始图像进行局部裁剪（Crop），再将放大后的局部图输入模型进行二次精细定位。
    *   **作用**：通过减小搜索空间和提高局部分辨率，有效缓解精度偏差。
*   **候选选择（Candidate Selection）**：
    *   **流程**：利用**掩码预测（Masking）**机制生成多个互斥的候选框（预测一个框后遮盖该区域再预测下一个），随后引入一个外部“修正模型”（如 GPT-5 或本地微调的 Qwen3-VL）。
    *   **关键技术**：在 Prompt 中注入 GUI 先验规则（如：优先考虑功能性元素、交互式组件优于静态文本、遵循 UI 典型模式等），引导修正模型从候选框中选出最符合指令的目标。
    *   **作用**：纠正模型在 Token 空间与几何空间之间的不一致性，缓解歧义偏差。

#### 3. 实验设计
*   **数据集**：
    *   **ScreenSpot-Pro**：包含专业软件（CAD、科学计算、创意设计等）的高分辨率复杂场景，是主要的挑战基准。
    *   **ScreenSpot-V2**：涵盖移动端、桌面端和 Web 端的常规场景。
*   **对比方法**：
    *   **闭源模型**：GPT-4o, Claude 3.5 Computer Use。
    *   **开源 SFT 模型**：OS-Atlas, UI-TARS, UGround, CogAgent 等。
    *   **强化学习模型**：UI-R1, GUI-G2, InfiGUI-R1 等。
    *   **推理增强方法**：GUI-RC, DiMo-GUI。

#### 4. 资源与算力
*   **推理算力**：所有推理实验均在单张 **NVIDIA RTX 4090 GPU** 上完成。
*   **训练算力（本地修正模型）**：为了实现脱离 API 的独立部署，作者使用 **8 张 A100 (80GB) GPU** 对 Qwen3-VL-8B 进行了 LoRA 微调，训练时长约 **12 小时**，使用了约 12.8 万条双框选择样本。
*   **效率**：MPD 归因分析单样本约需 20 分钟；BAMI 推理由于涉及多次模型调用，耗时高于单次推理。

#### 5. 实验数量与充分性
*   **实验规模**：对 50 个错误样本进行了深入的 MPD 归因统计；在两个主流基准上测试了 4 种不同的开源底座模型（7B 规模）。
*   **消融实验**：非常充分。涵盖了裁剪比例（Crop Ratio）、迭代次数、Prompt 设计（CoT 与先验规则的影响）、以及不同修正模型（GPT-5, Gemini, Qwen 等）的对比。
*   **客观性**：通过对比多种底座模型证明了方法的通用性，且本地修正模型的实验证明了提升并非完全依赖于更强的外部 API（如 GPT-5）。

#### 6. 主要结论与发现
*   **性能提升**：BAMI 在所有测试底座上均实现显著提升。例如，将 TianXi-Action-7B 在 ScreenSpot-Pro 上的准确率从 **51.9% 提升至 57.8%**。
*   **偏差来源**：证实了“歧义偏差”是复杂 GUI 定位中的最大杀手（占比 54%）。
*   **推理侧潜力**：证明了无需昂贵的强化学习或大规模微调，仅通过优化推理路径（Test-time Reasoning）即可挖掘现有模型的定位潜力。

#### 7. 优点
*   **无需训练（Training-Free）**：对现有模型友好，易于集成到各种 GUI 智能体中。
*   **诊断工具创新**：提出的 MPD 方法为理解多模态模型的定位错误提供了直观的归因手段。
*   **鲁棒性强**：实验显示该方法对参数设置（如裁剪比例）不敏感，且在不同类型的 UI 元素（图标 vs 文本）上表现均衡。

#### 8. 不足与局限
*   **推理延迟**：由于采用了多轮预测和裁剪操作，推理时间成倍增加，可能影响实时交互体验。
*   **依赖修正模型**：虽然本地模型有效，但 BAMI 的上限仍受限于修正模型对 GUI 布局和指令的理解能力。
*   **知识缺失无法解决**：如归因分析所示，对于模型完全不认识的罕见 UI 元素（知识鸿沟），BAMI 无法通过推理优化来弥补。
