## 摘要
单步检索增强生成 (RAG) 为简单问答任务提供了一种整合外部信息的有效方式，但在处理复杂问题时表现不佳。智能体 RAG (Agentic RAG) 扩展了这一范式，将单步检索替换为多步过程，其中大语言模型 (LLM) 充当搜索智能体，生成中间思维和子查询，以与检索系统进行迭代交互。由于冗长思维和子查询的自回归生成，这种迭代过程会产生巨大的延迟。为了解决这一局限性，我们提出了 LatentRAG，这是一个将推理和检索从离散语言空间转移到连续潜空间的新型框架。与现有的逐标记生成自然语言思维或子查询的显式方法不同，LatentRAG 在单次前向传播中直接从隐藏状态生成思维和子查询的潜标记 (latent tokens)。我们在潜空间中将 LLM 与稠密检索模型对齐，实现了基于潜子查询标记的检索，并支持端到端的联合优化。为了提高透明度并促进具有语义意义的潜表示，我们引入了一种并行潜解码机制，将潜标记翻译回自然语言。在七个基准数据集上的广泛实验表明，LatentRAG 达到了与显式智能体 RAG 方法相当的性能，同时将推理延迟降低了约 90%，大幅缩小了与传统单步 RAG 的延迟差距。

## Abstract
Single-step retrieval-augmented generation (RAG) provides an efficient way to incorporate external information for simple question answering tasks but struggles with complex questions. Agentic RAG extends this paradigm by replacing single-step retrieval with a multi-step process, in which the large language model (LLM) acts as a search agent that generates intermediate thoughts and subqueries to iteratively interact with the retrieval system. This iterative process incurs substantial latency due to the autoregressive generation of lengthy thoughts and subqueries. To address this limitation, we propose LatentRAG, a novel framework that shifts both reasoning and retrieval from discrete language space to continuous latent space. Unlike existing explicit methods that generate natural language thoughts or subqueries token-by-token, LatentRAG produces latent tokens for thoughts and subqueries directly from the hidden states in a single forward pass. We align LLMs with dense retrieval models in the latent space, enabling retrieval over latent subquery tokens and supporting end-to-end joint optimization. To improve transparency and encourage semantically meaningful latent representations, we incorporate a parallel latent decoding mechanism that translates latent tokens back into natural language. Extensive experiments on seven benchmark datasets show that LatentRAG achieves performance comparable to explicit agentic RAG methods while reducing inference latency by approximately 90%, substantially narrowing the latency gap with traditional single-step RAG.

*   **核心问题**：传统的“智能体 RAG”（Agentic RAG）虽然能通过多步推理和检索解决复杂问题，但其**推理延迟极高**。
*   **研究背景**：Agentic RAG 需要 LLM 像搜索智能体一样，通过自回归方式逐个标记（token-by-token）生成冗长的中间思维（Thoughts）和搜索子查询（Subqueries）。实验发现，这类显式生成过程占用了总推理时间的约 90%，导致其延迟通常是传统单步 RAG 的 15-20 倍。
*   **整体含义**：本文旨在通过将推理和检索过程从“离散语言空间”转移到“连续潜空间”（Latent Space），在不损失复杂推理能力的前提下，大幅提升 Agentic RAG 的运行效率。

### 2. 方法论：核心思想与关键技术
*   **核心思想**：利用 LLM 的隐藏状态（Hidden States）作为“潜标记”（Latent Tokens）来承载推理和检索信息，从而通过单次前向传播代替耗时的自回归生成。
*   **关键技术细节**：
    *   **潜标记生成**：在输入中插入特殊的思维标记（`<think>`）和查询标记（`<query>`），LLM 在一次前向传播中产出这些标记对应的最后层隐藏状态，作为潜推理和潜子查询。
    *   **潜空间检索对齐**：引入轻量级投影层（Projector），将 LLM 的潜子查询标记映射到检索模型的向量空间。使用 **KL 散度损失** 将潜子查询的检索分布与自然语言子查询的检索分布进行对齐，实现端到端优化。
    *   **并行潜解码机制**：为了解决潜空间不可解释的问题，设计了一个可选的解码分支，将潜标记重新翻译为自然语言。由于各步骤的潜标记已生成，所有步骤的思维和查询可以**并行解码**，远快于传统的串行生成。
    *   **联合训练目标**：损失函数由生成损失（$L_{gen}$）、检索对齐损失（$L_{ret}$）和潜解码损失（$L_{dec}$）加权组成。

### 6. 主要结论与发现
*   **极高的效率提升**：LatentRAG 在保持与 Search-R1 等显式方法相当性能（差异小于 5%）的情况下，**推理延迟降低了约 90%**，其速度接近于最简单的单步 RAG。
*   **并行解码的优势**：即使开启旨在提高透明度的潜解码功能，LatentRAG 仍比显式方法快 40%-60%。
*   **可扩展性**：随着 LLM 和检索模型规模的扩大，LatentRAG 的性能持续提升，且延迟增长远低于显式方法。
*   **检索模型敏感性**：发现检索模型的“各向异性”（Anisotropy）会影响对齐效果，分布越均匀的模型（如 Qwen3-Embedding）效果越好。

### 7. 优点：亮点总结
*   **首创性**：率先将潜推理（Latent Reasoning）引入到 Agentic RAG 的多步检索场景中。
*   **端到端优化**：通过潜空间对齐，解决了离散 Token 生成无法传递梯度的问题，使 LLM 和检索器能更好地协同。
*   **平衡透明度与效率**：通过并行解码机制，既保留了潜空间的高效，又提供了人类可读的中间步骤，增强了系统的可解释性。

### 8. 不足与局限
*   **对训练数据的依赖**：LatentRAG 依赖于现有显式 Agentic RAG 生成的轨迹进行监督微调（SFT），其性能上限受限于“教师模型”生成的轨迹质量。
*   **精确度微降**：在某些需要极高词汇精确度的任务中，潜表示可能导致微小的性能损失（如 Failure Case 中出现的拼写细微偏差）。
*   **应用限制**：目前主要在问答任务上验证，对于需要极长链条推理或极其复杂工具调用的场景，潜空间的表达能力是否足够仍需进一步探索。
