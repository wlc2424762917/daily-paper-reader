## 摘要
代理式安全系统越来越多地将大语言模型（LLM）规划器与能够发现、验证和报告漏洞的工具相结合。这产生了一个非对称控制问题：系统在授权参与范围内应保持强大的攻击能力，而在范围之外则必须禁止这些能力。现有的护栏（guardrails）提供了有用的策略控制，但它们并未将这一边界转化为针对可观测效应（observable effects）的一等形式合约。我们引入了对齐合约（alignment contracts），这是一个用于在可观测效应轨迹上指定和执行行为约束的框架。合约定义了范围、允许和禁止的效应、资源预算以及披露策略。我们为该语言赋予了有限轨迹语义，将满足性表征为具有有限违规见证（violation witnesses）的安全属性，开发了用于模块化合约工程的精化（refinement）和单向组合规则，并证明了可接纳性检查（admissibility checking）是可判定的。我们针对以 Web 为中心的代理式安全工作流实例化了该框架，并展示了相同的结构如何扩展到其他效应剖面（effect profiles）。在显式的效应可观测性假设（Effect Observability Assumption）下，即所有 $\SigmaEff$-效应均受调解，可靠性定理对代理模型进行了量

## Abstract
Agentic security systems increasingly combine LLM planners with tools that can discover, validate, and report vulnerabilities. This creates an asymmetric control problem: the system should retain strong offensive capability inside an authorized engagement, while the same capabilities must be denied outside scope. Existing guardrails provide useful policy controls, but they do not make this boundary a first-class formal contract over observable effects.   We introduce alignment contracts, a framework for specifying and enforcing behavioral constraints over observable effect traces. A contract defines scope, allowed and forbidden effects, resource budgets, and disclosure policies. We give the language finite-trace semantics, characterize satisfaction as a safety property with finite violation witnesses, develop refinement and one-way composition rules for modular contract engineering, and show that admissibility checking is decidable. We instantiate the framework for web-focused agentic security workflows and show how the same structure extends to other effect profiles.   Under an explicit Effect Observability Assumption, where all $\SigmaEff$-effects are mediated, the soundness theorem quantifies over the agent model and gives guarantees for mediated $\SigmaEff$-effects, including enforcement soundness for monitor-realized traces. We also state an assumption-lifted adaptation result and formalize limits through undecidability transfer and observability-boundary theorems. A Lean 4 artifact checks the formal core theorems used by the paper.

### 2. 方法论：核心思想与关键技术
*   **核心思想**：将安全约束定义为对“效应轨迹”（Effect Traces）的形式化合约。只要 Agent 的所有外部效应都经过受信任的监视器（Monitor），即使 LLM 本身被劫持或产生恶意意图，其最终实现的效应仍能保持在合约定义的范围内。
*   **关键技术细节**：
    *   **合约元组 ($C$)**：定义为 $C = \langle S, E_{allow}, E_{forbid}, B, D, Res, cost, flows \rangle$。
        *   $S$ (Scope)：授权的目标范围（如特定的 IP 或域名）。
        *   $E_{allow}/E_{forbid}$：允许或禁止的效应类别（如仅限 GET 请求，禁止文件写入）。
        *   $B$ (Budget)：资源预算（如请求总数上限）。
        *   $D$ (Disclosure)：披露政策（敏感信息只能流向授权的 Sink）。
    *   **效应模型**：将 Agent 的行为抽象为事件序列 $e = \langle \sigma, p, t, i \rangle$（类型、参数、时间戳、Agent ID）。
    *   **准入检查 (Admissibility)**：监视器根据当前轨迹前缀判断下一个事件是否合规。
    *   **形式化验证**：使用 **Lean 4** 证明了框架的可靠性定理（Soundness），即在“效应可观测性假设”（EOA）下，监视器实现的轨迹必然满足合约。

### 6. 主要结论与发现
*   **强制执行可靠性**：在完全调解（Mediation）的前提下，无论 LLM 内部状态如何（即使被注入恶意指令），其外部效应都能被严格限制在合约内。
*   **可判定性**：证明了合约合规性检查是可判定的，为实时监控提供了理论基础。
*   **模块化能力**：通过合约精化和组合，可以实现复杂安全策略的解耦和复用。
*   **不可判定性转移**：发现对于图灵完备的工具语言，静态预判 Agent 是否会尝试禁止效应是不可判定的，这进一步强调了**运行时监控**的必要性。

### 7. 优点
*   **形式化严谨性**：使用 Lean 4 进行机械化证明，显著提升了安全保证的可信度。
*   **信任转移**：将安全信任从不可解释的“黑盒”LLM 转移到了小巧、可审计的监视器和合约上。
*   **灵活性**：合约代数允许安全团队像编写代码一样组合和精化安全策略。

### 8. 不足与局限
*   **效应可观测性依赖 (EOA)**：如果 Agent 能够通过未被监控的通道（如绕过代理的底层系统调用）产生效应，则所有保证失效。
*   **内容盲区**：目前的框架主要关注效应的“元数据”（目标、方法、预算），对合法通道内隐藏的恶意载荷（如在允许的 HTTP 请求体中嵌入复杂的隐写信息）防御能力有限。
*   **静态合约限制**：核心模型假设合约在执行期间是静态的，虽然讨论了动态更新，但尚未在形式化核心中完全实现。
*   **应用门槛**：编写和验证形式化合约需要较高的专业知识，可能限制其在普通开发者中的普及。
