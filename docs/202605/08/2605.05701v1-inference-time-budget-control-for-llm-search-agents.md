## 摘要
大语言模型（LLM）搜索智能体在推理时日益依赖工具，但其轨迹通常受到工具调用次数和生成 Token 数量的双重硬性限制。在这种双重预算下，获得更好的答案不仅需要更强大的模型，还需要明确控制哪个搜索动作应获得下一个预算单位，以及何时积累的证据足以提交最终答案。我们在多跳问答（QA）中研究了这一问题，并将其表述为两阶段推理时预算控制。在搜索阶段，我们的控制器为每个可行操作分配一个任务级信息价值（VOI）分数，该分数定义为在当前搜索状态和剩余双重预算下，每单位预算的边际任务价值的操作性估计，并利用该分数在检索、分解和答案提交之间进行选择。搜索结束后，一个基于证据的选择性定稿器将轨迹答案与精炼后的候选答案进行比较，且仅在残差误差表现为低风险的答案形式错误时才进行重写。在四个多跳问答基准测试、三个 LLM 主干模型和四个预算级别上，该方法在相同的硬性双重预算协议下，相比四个经过审计的基准模型实现了正向的综合收益。消融实验表明，搜索时预算控制（尤其是与预算相关的惩罚）提供了主要的性能提升，而答案时控制主要在检索路径已经充分时发挥作用。这些结果表明，LLM 搜索智能体的推理时预算控制应当同时管理搜索过程中的预算支出方式以及最终答案的提交方式。

## Abstract
LLM search agents increasingly rely on tools at inference time, but their trajectories are often constrained by hard limits on both tool calls and generated tokens. Under such dual budgets, better answers require not only stronger models, but also explicit control over which search action should receive the next budget unit and when the accumulated evidence is sufficient to commit a final answer. We study this problem in multi-hop question answering (QA) and formulate it as two-stage inference-time budget control. At search time, our controller assigns each feasible action a task-level Value-of-Information (VOI) score, defined as an operational estimate of marginal task value per unit budget under the current search state and remaining dual budget, and uses this score to choose among retrieval, decomposition, and answer commitment. After search, a selective evidence-grounded finalizer compares the trajectory answer with a refined candidate and rewrites only when the residual error appears to be a low-risk answer-form error. Across four multi-hop QA benchmarks, three LLM backbones, and four budget levels, the method yields positive aggregate gains over four audited baselines under the same hard dual-budget protocol. Ablations show that search-time budget control, especially budget-dependent penalty, provides the main performance gain, while answer-time control helps mainly when the retrieval path is already adequate. These results suggest that inference-time budget control for LLM search agents should govern both how budget is spent during search and how the final answer is committed.

### 论文总结：大语言模型搜索智能体的推理时预算控制（Inference-Time Budget Control for LLM Search Agents）

#### 1. 核心问题与研究动机
随着 LLM 搜索智能体（如 Search-o1, ReAct 等）的普及，推理过程不再是简单的文本生成，而是一个涉及外部工具调用的动态决策过程。然而，实际部署中智能体面临**工具调用次数**和**生成 Token 数量**的双重硬性预算限制。
*   **研究动机**：现有的方法往往在预算分配上缺乏精细化控制，导致在有限资源下出现冗余搜索、过早回答或 Token 浪费。
*   **核心问题**：在双重预算约束下，如何决定“下一步该做什么”（检索、分解问题还是直接回答）以及“何时提交最终答案”以最大化任务准确率。

#### 2. 方法论
论文提出了一个**两阶段推理时预算控制框架**，旨在不进行额外模型训练的情况下优化智能体的决策路径。

*   **第一阶段：搜索时预算分配（Search-Time Allocation）**
    *   **核心思想**：引入**任务级信息价值（VOI）评分**。该分数衡量在当前状态和剩余预算下，每单位预算投入所能带来的边际任务价值。
    *   **关键技术**：
        1.  **效用计算**：结合评论模型（Critic）提供的进度信号、结构化信号（如是否存在未解决的桥接实体）以及**预算相关惩罚项**（预算越少，搜索动作的代价感越高）。
        2.  **归一化与保护机制**：将效用除以动作成本进行归一化，并应用硬性保护规则（如防止在证据不足时过早回答，或在简单事实问题上过度分解）。
*   **第二阶段：回答时定稿控制（Answer-Time Finalization）**
    *   **核心思想**：在搜索结束后，不直接输出结果，而是通过一个**证据驱动的定稿器**进行二次校验。
    *   **关键技术**：定稿器会对比原始答案和基于证据精炼后的候选答案。它采用**风险控制策略**：仅当错误属于“答案形式错误”（如 Yes/No 极性反转、槽位填充不全）且证据充分时才进行重写；若涉及复杂的推理结构风险，则保持原样以避免误改。

#### 3. 实验设计
*   **数据集**：涵盖四个多跳问答（Multi-hop QA）基准：**HotpotQA, 2WikiMultihopQA, MuSiQue, Bamboogle**。
*   **实验场景**：设置了四个预算等级（从低到高），严格限制工具调用次数（1-3次）和 Token 数量（100-500个）。
*   **对比方法（Baselines）**：
    *   **BAVT**：基于价值树搜索的基准。
    *   **BATS**：预算感知型工具使用方法。
    *   **AFlow**：自动化工作流生成方法。
    *   **Search-o1**：增强型搜索推理模型。
*   **主干模型**：使用了 Qwen3-32B、Qwen3.5-122B 和 GPT-5.4-Mini 三种不同规模和系列的 LLM。

#### 4. 资源与算力
*   **硬件环境**：实验在一台配备 **4 张 NVIDIA RTX 5880 Ada GPU** 的工作站上运行。
*   **算力消耗**：由于该方法是 **Training-free（免训练）** 的，主要的算力消耗在于推理时的 LLM 调用和检索过程。论文未详细列出总推理时长，但提到 VOI 控制器相比 BAVT 基准能减少约 27.2% 的平均推理时间，因为它通过优化决策减少了无效的搜索步骤。

#### 5. 实验数量与充分性
*   **实验规模**：论文在 3 个模型、4 个数据集、4 个预算等级上进行了交叉验证（共 48 组核心实验单元），并提供了详细的 EM（精确匹配）和 F1 分数。
*   **消融实验**：针对 VOI 的各个组件（惩罚项、归一化、结构信号、保护机制）以及两阶段的贡献度分别进行了消融分析。
*   **充分性评价**：实验设计非常充分且客观。它采用了**硬性双重预算审计协议**，确保所有对比方法在完全相同的资源限制下竞争，避免了因 Token 长度不一导致的评估偏差。

#### 6. 主要结论与发现
*   **预算控制的有效性**：VOI 控制器在大多数设置下优于基准，尤其在**中低预算**环境下提升最显著，因为此时每一单位预算的分配都至关重要。
*   **关键组件**：消融实验显示，**预算相关惩罚项**是性能提升的核心，它能有效抑制智能体在预算枯竭边缘进行低效搜索。
*   **分工明确**：第一阶段（搜索控制）贡献了大部分性能增益；第二阶段（定稿控制）则在检索路径已正确但表达不精准时起到“画龙点睛”的作用。
*   **非单调性**：增加预算并不总是能提升性能，有时更多的搜索会引入噪声，而 VOI 控制器能缓解这种负面影响。

#### 7. 优点
*   **免训练（Training-free）**：无需微调模型，易于集成到现有的智能体框架中。
*   **双重预算感知**：首次明确在同一框架下同时处理工具调用和 Token 消耗的硬约束。
*   **可解释性强**：VOI 评分基于明确的启发式信号和逻辑规则，决策过程透明。
*   **效率提升**：在提升准确率的同时，通过减少冗余动作降低了推理延迟。

#### 8. 不足与局限
*   **模型依赖性**：实验显示在极强大的主干模型（如 Qwen3.5-122B）上，控制器的边际收益会有所收窄。
*   **检索路径修复能力有限**：如果初始检索路径完全错误，第二阶段的定稿器无法通过重写来挽回损失。
*   **超参数敏感性**：VOI 公式中的系数（如惩罚权重）目前是静态设置的，可能需要针对不同领域进行微调。
*   **任务局限**：目前主要针对多跳问答任务，在更开放、长程的网页浏览或软件操作任务中的表现尚待验证。
