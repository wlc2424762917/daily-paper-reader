## 摘要
长期运行的自主 AI 智能体面临着一个已被广泛记录的记忆一致性问题：由于现有扁平文件记忆系统中的四种复合故障模式，在 72 小时的运行窗口内，工具执行成功率下降了 14 个百分点。我们提出了 MEMTIER，这是一种用于 OpenClaw 智能体运行时的三部分记忆架构，它引入了结构化情境 JSONL 存储、五信号加权检索引擎、注意力归因认知权重更新循环、将情境事实提升至语义层的异步整合守护进程，以及用于自适应检索权重的基于 PPO 的策略框架（基础设施已验证；性能提升待定稿）。在包含 500 个问题的完整 LongMemEval-S 基准测试（Wu 等人，2025）中，MEMTIER 使用 Qwen2.5-7B 在消费级 6GB GPU 上实现了 Acc=0.382 和 F1=0.412，相比全上下文基线（0.050 -> 0.382，即 5% -> 38%）提升了 33 个百分点。通过 DeepSeek-V4-Flash 事实预填充，单会话召回率达到 0.686-0.714，在这些类别上超过了论文中 RAG BM25 GPT-4o 的基线（0.560）。时间推理能力提升至 0.323，多会话综合能力提升至 0.173，证明了结构化语义预填充从定性上改变了轻量级检索所能达到的效果。所有阶段均在配备 6GB GPU 的消费级笔记本电脑上本地运行。

## Abstract
Long-running autonomous AI agents suffer from a well-documented memory coherence problem: tool-execution success rates degrade 14 percentage points over 72-hour operation windows due to four compounding failure modes in existing flat-file memory systems. We present MEMTIER, a tripartite memory architecture for the OpenClaw agent runtime that introduces a structured episodic JSONL store, a five-signal weighted retrieval engine, an attention-attributed cognitive weight update loop, an asynchronous consolidation daemon promoting episodic facts to a semantic tier, and a PPO-based policy framework for adapting retrieval weights (infrastructure validated; performance gains pending camera-ready). On the full 500-question LongMemEval-S benchmark (Wu et al., 2025), MEMTIER achieves Acc=0.382, F1=0.412 with Qwen2.5-7B on a consumer 6GB GPU - a +33 percentage point improvement over the full-context baseline (0.050 -> 0.382, i.e., 5% -> 38%). With DeepSeek-V4-Flash fact pre-population, single-session recall reaches 0.686-0.714, exceeding the paper's RAG BM25 GPT-4o baseline (0.560) on those categories. Temporal reasoning rises to 0.323 and multi-session synthesis to 0.173, demonstrating that structured semantic pre-population qualitatively changes what lightweight retrieval can achieve. All phases run locally on a consumer laptop with a 6GB GPU.

#### 1. 核心问题与研究动机
长期运行的自主 AI 智能体（如 OpenClaw 框架下的智能体）在持续运行（如 72 小时窗口）中面临严重的**记忆一致性退化**问题，工具执行成功率通常会下降约 14%。
研究发现现有扁平化记忆系统存在四大失效模式：
*   **上下文崩溃**：固定容量限制导致信息非优雅截断。
*   **压缩不连续性**：上下文压缩事件常导致智能体行为中断。
*   **结构盲区**：扁平文本无法区分实体间的深层关系。
*   **缺乏归因反馈**：工具执行的结果无法反馈至记忆系统以优化后续检索。

#### 2. 方法论：MEMTIER 三层架构
MEMTIER 提出了一种三部分组成的记忆管理系统，旨在解决上述瓶颈：
*   **结构化情境存储（Episodic JSONL Store）**：采用 JSONL 格式记录会话，包含时间戳、项目隔离标签及**认知权重（Cognitive Weight, CW）**。CW 衡量记忆条目对工具执行成功的贡献度。
*   **五信号加权检索引擎**：检索评分函数 $S(q, m_i) = w^\top \phi(q, m_i)$ 综合了五个信号：BM25 相关性、指数时间衰减（半衰期约 14 天）、认知权重、层级提升（Tier Boost）以及语义相关性。
*   **两阶段检索策略**：第一阶段在语义层检索相关会话 ID，第二阶段在选定会话的情境记忆中进行精细评分，有效防止上下文溢出。
*   **异步整合守护进程（Consolidation Daemon）**：利用 LLM（如 DeepSeek-V4-Flash）将零散的情境记忆提炼为精简的语义事实，实现 164 倍的信息压缩。
*   **PPO 策略优化**：引入强化学习（PPO）框架，根据任务成功与否动态调整检索信号的权重向量 $w$。

#### 6. 主要结论与发现
*   **性能飞跃**：在 LongMemEval-S 上，MEMTIER 将 7B 模型的准确率从 5% 提升至 **38.2%**，在单会话召回上甚至超过了 GPT-4o 的 RAG 基线。

*   **闭环学习**：通过认知权重和 PPO 实现了记忆系统的自我进化能力。
