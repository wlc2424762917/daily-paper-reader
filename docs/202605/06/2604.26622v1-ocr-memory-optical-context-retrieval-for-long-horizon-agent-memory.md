## 摘要
自主大语言模型（LLM）智能体越来越多地在长程、交互式环境中运行，其成功取决于对长期积累经验的复用。然而，现有的智能体记忆系统从根本上受到文本上下文预算的限制：存储或回顾原始轨迹的 Token 开销极高，而摘要和纯文本检索虽然节省了 Token，却以信息丢失和证据碎片化为代价。为了解决这一局限性，我们提出了光学上下文检索记忆（OCR-Memory），这是一种利用视觉模态作为智能体经验高密度表示的记忆框架，能够在检索时以极低的提示词（prompt）开销保留任意长度的历史记录。具体而言，OCR-Memory 将历史轨迹渲染为带有唯一视觉标识符注释的图像。OCR-Memory 通过“定位并转录”（locate-and-transcribe）范式检索存储的经验，该范式通过视觉锚点选择相关区域并检索相应的逐字文本，从而避免了自由文本生成并减少了幻觉。在长程智能体基准测试上的实验表明，在严格的上下文限制下，该方法取得了持续的性能提升，证明了光学编码在增加有效记忆容量的同时，能够保持忠实的证据恢复。

## Abstract
Autonomous LLM agents increasingly operate in long-horizon, interactive settings where success depends on reusing experience accumulated over extended histories. However, existing agent memory systems are fundamentally constrained by text-context budgets: storing or revisiting raw trajectories is prohibitively token-expensive, while summarization and text-only retrieval trade token savings for information loss and fragmented evidence. To address this limitation, we propose Optical Context Retrieval Memory (OCR-Memory), a memory framework that leverages the visual modality as a high-density representation of agent experience, enabling retention of arbitrarily long histories with minimal prompt overhead at retrieval time. Specifically, OCR-Memory renders historical trajectories into images annotated with unique visual identifiers. OCR-Memory retrieves stored experience via a \emph{locate-and-transcribe} paradigm that selects relevant regions through visual anchors and retrieves the corresponding verbatim text, avoiding free-form generation and reducing hallucination. Experiments on long-horizon agent benchmarks show consistent gains under strict context limits, demonstrating that optical encoding increases effective memory capacity while preserving faithful evidence recovery.

### 2. 方法论：核心思想与关键技术
OCR-Memory 的核心思想是将历史轨迹转化为带标注的图像流，并采用“定位后转录”的非生成式检索机制。

*   **视觉存储与 SoM 标注**：将智能体的交互日志（推理链、工具调用、环境反馈）渲染成图像。使用 **Set-of-Mark (SoM)** 技术，在图像中的每个文本段落周围绘制红色边界框并标注唯一数字 ID。
*   **定位并转录（Locate-and-Transcribe）范式**：
    *   **定位**：给定查询 $q$，光学检索模型（基于 DeepSeek-OCR）扫描视觉记忆，输出相关片段的 ID 索引，而非直接生成文本内容。
    *   **转录**：根据预测的索引，从外部数据库中**确定性地（Deterministic）**提取原始逐字文本。这种设计彻底消除了检索阶段的文本生成幻觉。
*   **多分辨率与主动召回（Active Recall）**：
    *   **记忆衰减**：模拟人类记忆，较旧的轨迹被存储为低分辨率缩略图（如 $512 \times 512$），大幅降低视觉 Token 开销。
    *   **动态刷新**：一旦低分辨率记忆被检索命中，系统会立即将其“上采样”恢复为高分辨率（如 $1024 \times 1024$），确保后续推理获得完整细节。
*   **检索评分机制**：利用 Logits 计算校准后的相关性概率，采用“阈值触发 + Top-K 保底”的混合策略，确保检索的高召回率。

*   **创新的模态转换**：首次将智能体记忆问题转化为光学字符识别与定位问题，避开了文本压缩的信息损耗。
*   **抗幻觉设计**：Locate-and-Transcribe 机制极具实用价值，解决了 RAG 系统中常见的检索内容畸变问题。
*   **动态记忆模型**：多分辨率主动召回机制巧妙地平衡了长期存储成本与短期推理精度。

### 8. 不足与局限
*   **训练成本**：与无需训练的 RAG 相比，该方法需要专门微调光学模型，增加了前期开销。
*   **存储负担**：图像存储占用的磁盘空间远大于纯文本，对于存储受限的设备可能不友好。
*   **渲染延迟**：将日志实时渲染为图像并进行 OCR 处理会增加每一步交互的端到端延迟。
*   **视觉依赖**：如果原始文本包含极其密集的微小字符，低分辨率模式下可能存在识别风险（尽管有主动召回机制补偿）。
