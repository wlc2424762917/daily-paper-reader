## 摘要
随着基于大语言模型（LLM）的智能体越来越多地被部署用于执行复杂的现实任务，它们面临着一类日益增多的攻击。这些攻击利用用户-智能体-环境之间长期的交互，来实现单轮对话设置中难以达成的恶意目标。这种长程威胁对 LLM 智能体在关键领域的安全部署构成了重大风险。在本文中，我们提出了 MAGE（Memory As Guardrail Enforcement），这是一种旨在应对各种长程威胁的新型防御框架。受系统安全中“影子栈”（shadow stack）抽象概念的启发，MAGE 维护了一个专门的、以安全为中心的智能体内存，用于提取并保留智能体整个执行轨迹中与安全相关的关键上下文，并利用该影子内存，在待执行操作实施前主动评估其风险。广泛的评估表明，在多种长程威胁下，MAGE 在检测准确率方面显著优于现有防御措施，实现了对大多数攻击的早期检测，且对智能体效用的开销微乎其微。据我们所知，MAGE 是首个利用智能体内存方法来检测和缓解长程威胁的框架，为这一关键挑战建立了新的范式，并为未来的研究开辟了充满前景的方向。

## Abstract
As large language model (LLM)-powered agents are increasingly deployed to perform complex, real-world tasks, they face a growing class of attacks that exploit extended user-agent-environment interactions to pursue malicious objectives improbable in single-turn settings. Such long-horizon threats pose significant risks to the safe deployment of LLM agents in critical domains. In this paper, we present MAGE (Memory As Guardrail Enforcement), a novel defensive framework designed to counter a wide range of long-horizon threats. Inspired by the "shadow stack" abstraction in systems security, MAGE maintains a dedicated, safety-focused agentic memory that distills and retains safety-critical context across the agent's full execution trajectory, leveraging this shadow memory to proactively assess the risk of pending actions prior to their execution. Extensive evaluation demonstrates that MAGE substantially outperforms existing defenses across diverse long-horizon threats in detection accuracy, achieves early-stage detection for the majority of attacks, and introduces only negligible overhead to agent utility. To our best knowledge, MAGE represents the first framework to detect and mitigate long-horizon threats using an agentic memory approach, establishing a new paradigm for this critical challenge and opening promising directions for future research.

#### 1. 核心问题与整体含义（研究动机和背景）
随着 LLM 智能体被部署于处理复杂、多轮交互的任务，它们面临着一种新型安全威胁——**长程威胁（Long-Horizon Threats）**。这类攻击（如多轮中毒、工具链攻击、持续提示词注入）的特点是：**恶意意图在单轮交互中是隐蔽的，仅在整个执行轨迹（Trajectory）层面显现**。
现有的防御手段（如关键词过滤、单轮分类器）由于缺乏对长上下文的理解能力，在面对这种跨越多轮交互的攻击时往往失效。论文的研究动机在于：如何提炼并保留执行轨迹中的安全关键信息，从而在恶意行为最终发生前进行拦截。

#### 2. 方法论：核心思想与关键技术
MAGE 借鉴了系统安全中的“**影子栈（Shadow Stack）**”概念，提出了一种名为 **MAGE (Memory As Guardrail Enforcement)** 的防御框架。

*   **核心思想**：在智能体的工作内存之外，维护一个独立的、以安全为中心的“**影子内存（Shadow Memory）**”。它负责蒸馏（Distill）并保留整个交互过程中的安全关键上下文。
*   **关键组件**：
    *   **内存管理器 ($M$)**：在每一轮交互后，将当前动作、环境观察和用户指令提炼成紧凑的安全摘要，更新影子内存。
    *   **裁判 ($J$)**：在智能体执行下一个动作前，结合影子内存评估该动作的风险。
*   **算法流程**：
    1.  智能体提出待执行动作 $a_t$。
    2.  $M$ 更新影子内存 $m_t$。
    3.  $J$ 基于 $m_t$ 做出“批准”或“拒绝”的决策。
    4.  若拒绝，则拦截动作并向用户报告理由；若批准，则执行动作并进入下一轮。
*   **模型优化**：采用**逐轮强化学习（Turn-wise RL）**，基于 GRPO 算法优化。设计了特殊的奖励函数：
    *   **裁判奖励**：基于决策的准确性。
    *   **内存奖励**：采用时间折扣因子，将未来的风险评估结果回传给当前的内存更新，以解决信用分配问题。
    *   **格式与长度惩罚**：确保输出结构化且内存紧凑。

#### 6. 主要结论与发现
*   **高效防御**：MAGE 将工具链攻击的成功率从 100% 降低至 **8.3%**，将长程注入攻击（PI2）的成功率降至 **0%**。

#### 7. 优点
*   **范式创新**：首次将智能体内存机制转化为安全原语，解决了长程威胁中上下文稀释的问题。
*   **即插即用**：无需修改智能体核心模型或推理流程，具有很强的通用性。
*   **可解释性**：影子内存和裁判决策均生成人类可读的理由，方便审计。
*   **端到端优化**：通过逐轮 RL 联合训练内存管理和风险评估，使两者高度协同。

#### 8. 不足与局限
*   **模式依赖**：目前的影子内存依赖于预定义的模式（如用户意图、已知攻击模式），对于完全未知的攻击类型可能存在泛化风险。
*   **单智能体限制**：研究目前集中在单智能体场景，尚未探讨在多智能体协作系统中的防御效果。
*   **静态 Schema**：影子内存的结构是固定的，未来可能需要动态调整 Schema 以适应更复杂的任务。
