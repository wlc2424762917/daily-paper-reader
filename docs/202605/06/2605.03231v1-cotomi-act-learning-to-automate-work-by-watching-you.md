## 摘要
如果浏览器智能体只需通过观察你的操作就能学习你的工作，会怎样？我们提出了 cotomi Act，这是一款基于浏览器的计算机使用智能体，它将可靠的多步任务执行与从用户行为中学习到的持久性组织知识相结合。在执行方面，该智能体架构采用了自适应延迟观察（adaptive lazy observation）、基于文本差异（verbal-diff）的历史压缩、粗粒度动作以及通过 Best-of-N 动作选择实现的测试时缩放（test-time scaling），在包含 179 个任务的 WebArena 人类评估子集上达到了 80.4% 的成功率，超过了报告的 78.2% 的人类基准。在组织知识方面，一种“行为到知识”流水线被动地观察用户的浏览行为，并将其逐步抽象为工件（如任务板、维基），这些工件通过用户和智能体均可编辑的共享工作区展示。一项受控代理评估证实，随着源自行为的知识不断积累，任务成功率也会随之提高。在我们的现场演示中，参与者在真实浏览器中与系统交互，发布任务并观察端到端的自主执行和共享知识管理。

## Abstract
What if a browser agent could learn your work simply by watching you do it? We present cotomi Act, a browser-based computer-using agent that combines reliable multi-step task execution with persistent organizational knowledge learned from user behavior. For execution, an agent scaffold with adaptive lazy observation, verbal-diff-based history compression, coarse-grained actions, and test-time scaling via best-of-N action selection achieves 80.4% on the 179-task WebArena human-evaluation subset, exceeding the reported 78.2% human baseline. For organizational knowledge, a behavior-to-knowledge pipeline passively observes the user's browsing and progressively abstracts it into artifacts (task boards, wiki) exposed through a shared workspace editable by both user and agent. A controlled proxy evaluation confirms that task success improves as behavior-derived knowledge accumulates. In our live demonstration, attendees interact with the system in a real browser, issuing tasks and observing end-to-end autonomous execution and shared knowledge management.

### 1. 核心问题与整体含义（研究动机和背景）
目前的计算机使用智能体（CUA）主要面临两大挑战：
*   **缺乏组织知识（Tacit Knowledge）：** 现有的智能体通常是“无状态”的执行者，不了解特定组织的内部术语、审批流程或用户偏好。这些知识往往未被记录，仅存在于员工的日常操作中。
*   **长程任务的推理可靠性：** 在复杂的 Web UI 中，长路径操作会导致上下文窗口迅速膨胀（Context Bloat），充满冗余信息，从而降低智能体的决策质量。

**cotomi Act** 的核心目标是创建一个既能高效执行多步 Web 任务，又能通过“观察”用户日常操作来被动学习并积累组织知识的浏览器智能体。

### 2. 方法论：核心思想与关键技术
该系统由三个核心组件构成：

*   **智能体架构（Agent Scaffold）：**
    *   **自适应延迟观察（Adaptive Lazy Observation）：** 默认仅观察视口内元素，仅在需要时请求完整树，以平衡精度与延迟。
    *   **基于文本差异的历史压缩（Verbal-diff History Compression）：** 不存储原始页面状态，而是记录自然语言描述的 DOM 变化（如“新增了提交按钮”），大幅降低 Token 消耗。
    *   **粗粒度动作（Coarse-grained Actions）：** 使用更高层的抽象动作（如 `scrollInto`）替代像素级操作，缩短任务步数。
    *   **测试时缩放（Test-time Scaling）：** 引入 **Best-of-N 动作选择**（通过多数投票减少单次推理误差）和 **任务分解**（将复杂指令拆分为并行子任务）。

*   **用户行为记录器（User Behavior Logger）：**
    *   后台被动记录用户的站点操作、标签切换和截图。
    *   采用 **Agentic ETL 流水线**：利用 LLM 自动对原始日志进行分段、摘要，并将其抽象为可重用的知识。

*   **共享知识工作区（Shared Knowledge Workspace）：**
    *   将学习到的知识转化为人类可读、智能体可查的工件（任务板、维基页面、活动时间线）。
    *   **双向协作：** 智能体根据观察提议更新知识，用户可以进行纠正和编辑，确保人机对齐。

### 6. 主要结论与发现
*   **超越人类水平：** cotomi Act 在 WebArena 人类评估子集上达到 **80.4%** 的成功率，超过了 **78.2%** 的人类基准。
*   **上下文优化的重要性：** 使用文本差异（Verbal-diff）代替原始观察，在提升精度的同时，将每步 Token 成本降低了约 **3.8 倍**。
*   **知识积累的价值：** 在 WorkArena-L1 实验中，随着行为知识覆盖率的增加，任务成功率最高提升了约 **10 个百分点**。
*   **抽象的必要性：** 实验发现，将原始轨迹抽象为“脚本”或“洞察”比直接存储原始轨迹在知识检索中更稳定、更有效。

### 7. 优点
*   **被动学习机制：** 无需用户显式编写文档，通过观察即可捕获“隐性知识”，降低了用户负担。
*   **透明的人机协作：** 共享工作区将智能体的“记忆”外显化为日常办公工件，增强了系统的可解释性和可控性。
*   **高效的上下文管理：** 通过 Verbal-diff 和延迟观察，巧妙解决了长程任务中的上下文膨胀问题。

### 8. 不足与局限
*   **隐私风险：** 尽管提到了 PII 脱敏和用户授权，但后台持续记录行为仍具有较高的隐私敏感性。
*   **代理评估的局限：** 行为知识的评估使用了智能体生成的成功轨迹作为“用户行为”的代理，尚未在包含大量错误、中断和探索性操作的真实人类杂乱日志中进行全面验证。
*   **应用限制：** 目前主要集中在浏览器环境，对于跨应用程序（如桌面软件）的自动化支持尚未讨论。
