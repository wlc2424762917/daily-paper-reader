## 摘要
现代语言智能体必须在长程、多轮历史记录上运行，但在小语言模型（SLM）上部署此类智能体仍然存在根本性困难。全上下文提示会导致上下文溢出，扁平化检索会使模型暴露在噪声证据中，而开放式智能体循环在推理能力有限的情况下并不可靠。我们认为，SLM 内存失效的很大一部分原因在于内存操作不匹配：不同的查询类型需要截然不同的检索策略、证据转换和上下文预算，而 SLM 无法通过开放式推理可靠地自我编排这些操作。我们引入了 MemFlow，这是一种无需训练的内存编排框架，它将内存规划从 SLM 中外置。路由智能体（Router Agent）根据意图对每个查询进行分类，并将其分发给内存智能体（Memory Agent），后者执行三个专门层级（配置文件查找、针对性检索或深度推理）之一，并在动态的、感知层级的 Token 预算下组装生成的证据。随后，回答智能体（Answer Agent）根据这一紧凑的上下文生成响应，当响应未得到所提供证据的支持时，验证智能体（Validator Agent）可选择使用更重的内存层级进行重试。这种“先路由后编译”的设计避免了工具选择幻觉和推理循环，同时保持了回答上下文的紧凑性。在冻结的 Qwen3-1.7B 骨干模型上，通过 LongMemEval、LoCoMo 和 LongBench 等长程内存基准测试进行评估，MemFlow 的准确率比全上下文 SLM 基准提高了近 2 倍。这些结果表明，结构化的意图路由和确定性的证据准备可以使能力有限的模型在资源受限的长程智能体中发挥更大的效用。

## Abstract
Modern language agents must operate over long-horizon, multi-turn histories, yet deploying such agents with Small Language Models (SLMs) remains fundamentally difficult. Full-context prompting causes context overflow, flat retrieval exposes the model to noisy evidence, and open-ended agentic loops are unreliable under limited reasoning capacity. We argue that a substantial portion of SLM memory failure arises from mismatched memory operations: different query types demand categorically different retrieval strategies, evidence transformations, and context budgets that SLMs cannot reliably self-orchestrate through open-ended reasoning. We introduce MemFlow, a training-free memory orchestration framework that externalizes memory planning from the SLM. A Router Agent classifies each query by intent and dispatches it to the Memory Agent, which executes one of three specialized tiers (Profile Lookup, Targeted Retrieval, or Deep Reasoning) and assembles the resulting evidence under a dynamic, tier-aware token budget. An Answer Agent then generates a response from this compact context, and a Validator Agent optionally retries with a heavier memory tier when the response is not supported by the provided evidence. This route-then-compile design avoids tool-selection hallucination and reasoning loops while keeping the answer context compact. Evaluated on a frozen Qwen3-1.7B backbone across long-horizon memory benchmarks - LongMemEval, LoCoMo, and LongBench - MemFlow improves accuracy by nearly 2x over full-context SLM baselines. These results suggest that structured intent routing and deterministic evidence preparation can make limited-capacity models substantially more effective in resource-constrained long-horizon agents.

#### 1. 核心问题与整体含义（研究动机和背景）
在长程对话和多轮交互中，智能体需要处理海量的历史信息。**小语言模型（SLM，通常指参数量 <3B 的模型）** 在此场景下面临三大挑战：
*   **上下文溢出：** 全量历史输入会导致超出模型处理窗口或推理性能下降。
*   **检索噪声：** 传统的 RAG（检索增强生成）采用统一检索策略，无法区分不同查询对证据的不同需求（如偏好查询 vs. 时间线推理）。
*   **推理不可靠：** 类似于 ReAct 的开放式推理循环在 SLM 上极易产生幻觉或逻辑断裂。
**核心假设：** SLM 的内存失效本质上是“意图路由不匹配”。论文提出将内存规划从模型内部外置，通过结构化的编排框架来提升 SLM 的长程记忆能力。

#### 2. 方法论：核心思想与技术细节
MemFlow 采用一种**“先路由后编译”（Route-then-Compile）**的无需训练框架，包含四个核心代理：
*   **路由智能体 (Router Agent)：** 使用三层级联（规则层 -> SLM 分类层 -> 关键词启发层）将查询归类为 7 种意图标签（如个人画像注入、时间推理、冲突解决等）。
*   **内存智能体 (Memory Agent)：** 根据意图执行三个专门的执行路径：
    *   **第一层 (Profile Lookup)：** 直接加载预编译的用户画像，无需检索。
    *   **第二层 (Targeted Retrieval)：** 执行多路实体感知的混合检索（BM25 + 向量）。
    *   **第三层 (Deep Reasoning)：** 引入**确定性预处理**（如时间排序、日期计算、规则过滤），在 SLM 看到证据前完成复杂的逻辑运算。
*   **上下文打包器 (Context Packer)：** 根据意图标签动态分配 Token 预算，按优先级（固定事实 > 预计算摘要 > 原始片段）组装紧凑的上下文。
*   **回答智能体 (Answer Agent)：** 使用特定标签的系统提示词生成最终响应。
*   **验证智能体 (Validator Agent)：** 检查回答是否得到证据支持。若失败，则触发**升级机制 (Escalation)**，切换到更重的内存层级重新检索和生成。

*   **实验规模：** 涵盖了 3 个主流基准测试，共计 **4,236 个测试项**，样本量充足。

#### 6. 主要结论与发现
*   **性能翻倍：** 在 Qwen3-1.7B 上，MemFlow 的准确率比全上下文基准提高了近 **2 倍**。
*   **优于复杂方案：** 相比于 ReAct 和 RAG，MemFlow 在保持更小回答上下文的同时，准确率提升了 **6.2 个百分点**。
*   **结构化胜过开放式：** 实验证明，对于 SLM 而言，确定性的路径选择比让模型自主进行工具调用（ReAct）更有效。
*   **低端模型救星：** 即使是表现极差的 1B 级模型，在 MemFlow 框架下也能恢复出连贯且准确的回答。

#### 7. 优点
*   **无需训练：** 易于集成到现有的冻结模型中，部署成本低。
*   **确定性处理：** 通过外部脚本处理 SLM 难以胜任的日期计算和排序，规避了模型本身的推理短板。
*   **动态预算：** 智能分配 Token，确保关键证据不被截断，提高了推理效率。
*   **鲁棒性：** 验证与升级机制有效减少了幻觉，尤其在处理“不可回答”的问题时表现优异。

#### 8. 不足与局限
*   **编排开销：** 虽然回答时的上下文很紧凑，但整个流水线（路由、验证等）的总 Token 消耗较高（平均约 10k tokens）。
*   **单意图限制：** 目前的路由只能选择单一标签，可能无法处理需要多种内存操作的复合查询。
*   **动态事实更新：** 对于跨会话频繁变动的事实，冲突解决的精度仍有提升空间。
*   **评测偏差：** 依赖 LLM 作为法官可能存在语义理解上的偏差，且目前仅在英文数据集上进行了验证。
