## 摘要
大语言模型（LLM）智能体日益依赖可重用的技能，即结合了指令、控制流、约束和工具调用的能力包。然而，在大多数当前的智能体系统中，技能仍由重文本的产物表示，包括 SKILL.md 风格的文档和结构化记录，其机器可用的证据在很大程度上仍嵌入在自然语言描述中。这给以技能为中心的智能体系统带来了挑战：管理技能集合以及利用技能支持智能体，都需要对调用接口、执行结构和具体的副作用进行推理，而这些内容往往纠缠在单一的文本表层中。因此，技能知识的显式表示可能有助于使这些产物更易于机器获取和利用。借鉴 Schank 和 Abelson 在语言知识表示经典著作中的记忆组织包（Memory Organization Packets）、脚本理论（Script Theory）和概念依赖（Conceptual Dependency），我们引入了据我们所知首个针对智能体技能产物的结构化表示，它解耦了技能级调度信号、场景级执行结构以及逻辑级动作和资源使用证据：即调度-结构-逻辑（SSL）表示。我们使用基于 LLM 的归一化器实例化了 SSL，并在技能发现和风险评估两个任务的技能语料库上对其进行了评估，其表现显著优于纯文本基准：在技能发现中，SSL 将 MRR 从 0.573 提高到 0.707；在风险评估中，它将宏观 F1 从 0.744 提高到 0.787。这些发现表明，显式的、基于源的结构使智能体技能更易于搜索和审查。它们还表明，SSL 最好被理解为迈向更具可检查性、可重用性和操作可行性的智能体系统技能表示的务实一步，而非一个成熟的标准或管理和使用技能的端到端机制。

## Abstract
LLM agents increasingly rely on reusable skills, capability packages that combine instructions, control flow, constraints, and tool calls. In most current agent systems, however, skills are still represented by text-heavy artifacts, including SKILL{.}md-style documents and structured records whose machine-usable evidence remains embedded largely in natural-language descriptions. This poses a challenge for skill-centered agent systems: managing skill collections and using skills to support agent both require reasoning over invocation interfaces, execution structure, and concrete side effects that are often entangled in a single textual surface. An explicit representation of skill knowledge may therefore help make these artifacts easier for machines to acquire and leverage. Drawing on Memory Organization Packets, Script Theory, and Conceptual Dependency from Schank and Abelson's classical work on linguistic knowledge representation, we introduce what is, to our knowledge, the first structured representation for agent skill artifacts that disentangles skill-level scheduling signals, scene-level execution structure, and logic-level action and resource-use evidence: the Scheduling-Structural-Logical (SSL) representation. We instantiate SSL with an LLM-based normalizer and evaluate it on a corpus of skills in two tasks, Skill Discovery and Risk Assessment, and superiorly outperform the text-only baselines: in Skill Discovery, SSL improves MRR from 0.573 to 0.707; in Risk Assessment, it improves macro F1 from 0.744 to 0.787. These findings reveal that explicit, source-grounded structure makes agent skills easier to search and review. They also suggest that SSL is best understood as a practical step toward more inspectable, reusable, and operationally actionable skill representations for agent systems, rather than as a finished standard or an end-to-end mechanism for managing and using skills.

### 1. 核心问题与研究动机
*   **研究背景**：随着智能体系统的发展，复杂的任务被封装为“技能”（包含指令、控制流、约束和工具调用）。
*   **核心问题**：目前的技能主要以 `SKILL.md` 等重文本形式存在。这种自然语言描述虽然易于人类阅读，但对机器而言，其**调用接口（何时用）**、**执行结构（怎么做）**和**逻辑副作用（风险点）**高度纠缠在一起，导致技能的自动发现、管理和风险评估变得困难且低效。
*   **研究目标**：建立一种显式的、结构化的技能表示方法，将纠缠的文本信号解构为机器可直接利用的证据。

### 2. 方法论：SSL 表示法
论文借鉴了 Schank 和 Abelson 的经典语言知识表示理论，提出了 **SSL（Scheduling-Structural-Logical）** 三层结构化表示：
*   **调度层 (Scheduling Layer)**：关注技能级接口。定义技能的目标、输入/输出契约、意图签名和依赖关系。其设计灵感来自“记忆组织包（MOPs）”。
*   **结构层 (Structural Layer)**：关注场景级执行阶段。将技能分解为有序的“场景”（如准备、获取、动作、验证、恢复等），描述阶段间的转换逻辑。其设计灵感来自“脚本理论（Script Theory）”。
*   **逻辑层 (Logical Layer)**：关注原子动作与资源证据。记录最细粒度的操作（如 READ, WRITE, CALL_TOOL）及其涉及的资源范围（如内存、网络、凭据）。其设计灵感来自“概念依赖（CD）”。
*   **归一化流水线**：利用 LLM 作为归一化器（Normalizer），通过四阶段提示词工程将非结构化的 `SKILL.md` 转换为符合 SSL 模式的 JSON 图结构，并进行严格的结构验证和重试机制。

### 6. 主要结论与发现
*   **性能提升**：在技能发现任务中，SSL-Rich 将 MRR 从 0.573 提升至 **0.707**；在风险评估中，文档+SSL 的组合将 F1 从 0.744 提升至 **0.787**。
*   **结构化的价值**：显式的结构化表示比单纯增加文本长度更有效。SSL 能够过滤掉冗余的自然语言噪声，使关键的接口和操作信号更加突出。
*   **互补性**：SSL 并非要取代原始文档，而是作为一种“证据接口”。结构化数据便于检索和初步审查，而原始文本则提供更深层的上下文解释。

### 7. 优点
*   **理论扎实**：将 20 世纪 70 年代的经典认知科学/语言学理论（Schank 等）成功应用于现代 LLM 智能体领域。
*   **实用性强**：直接针对当前智能体系统中的“黑盒技能”问题，提供了可落地的 JSON Schema 和归一化工具。
*   **开源贡献**：提供了包含 6,184 个技能的标注语料库和评估数据集，对社区具有重要价值。

### 8. 不足与局限
*   **静态分析限制**：SSL 是从静态文档中提取的，无法捕捉运行时的动态行为（如动态下载的负载或复杂的条件分支）。
*   **归一化保真度**：SSL 的质量高度依赖于底层 LLM 的提取能力，对于描述模糊或存在欺骗性的技能，可能会产生错误的结构化信息。
*   **应用范围**：目前仅在管理（发现与风险评估）层面验证，尚未直接测试 SSL 如何在智能体规划（Planning）和执行（Execution）过程中实时指导智能体。
