## 摘要
尽管大型视觉语言模型（LVLMs）取得了快速进展，但细粒度的、基于状态条件的 GUI 交互仍然具有挑战性。当前的评估覆盖范围有限，目标状态定义不精确，且过度依赖最终任务的成功，从而掩盖了智能体失败的位置和原因。为了填补这一空白，我们推出了 FineState-Bench，这是一个评估智能体能否将指令正确定位到预期的 UI 控件并达到精确目标状态的基准测试。FineState-Bench 包含跨桌面、Web 和移动平台的 2,209 个实例，涵盖 4 个交互系列和 23 种 UI 组件类型，每个实例都明确指定了用于细粒度状态设置的精确目标状态。我们进一步提出了 FineState-Metrics，这是一个具有阶段性成功率的四阶段诊断流水线：定位成功率（SR@Loc）、交互成功率（SR@Int）、定位时的精确状态成功率（ES-SR@Loc）以及交互时的精确状态成功率（ES-SR@Int），以及一个即插即用的视觉诊断助手（VDA），它通过生成描述和边界框定位提示，利用有无提示的对照实验来诊断视觉定位的原因。在 FineState-Bench 上，精确目标状态的成功率仍然较低：ES-SR@Int 在 Web 上达到 32.8% 的峰值，跨平台的平均值为 22.8%。在 VDA 定位提示的帮助下，Gemini-2.5-Flash 的 ES-SR@Int 提升了 14.9 个百分点，这表明改进视觉定位具有巨大的提升空间，但整体准确率对于可靠的细粒度状态条件交互而言仍然不足。

## Abstract
Despite the rapid progress of large vision-language models (LVLMs), fine-grained, state-conditioned GUI interaction remains challenging. Current evaluations offer limited coverage, imprecise target-state definitions, and an overreliance on final-task success, obscuring where and why agents fail. To address this gap, we introduce \textbf{FineState-Bench}, a benchmark that evaluates whether an agent can correctly ground an instruction to the intended UI control and reach the exact target state. FineState-Bench comprises 2,209 instances across desktop, web, and mobile platforms, spanning four interaction families and 23 UI component types, with each instance explicitly specifying an exact target state for fine-grained state setting. We further propose \textit{FineState-Metrics}, a four-stage diagnostic pipeline with stage-wise success rates: Localization Success Rate (SR@Loc), Interaction Success Rate (SR@Int), Exact State Success Rate at Locate (ES-SR@Loc), and Exact State Success Rate at Interact (ES-SR@Int), and a plug-and-play \textit{Visual Diagnostic Assistant} (VDA) that generates a Description and a bounding-box Localization Hint to diagnose visual grounding reason via controlled w/ vs.\ w/o comparisons. On FineState-Bench, exact goal-state success remains low: ES-SR@Int peaks at 32.8\% on Web and 22.8\% on average across platforms. With VDA localization hints, Gemini-2.5-Flash gains +14.9 ES-SR@Int points, suggesting substantial headroom from improved visual grounding, yet overall accuracy is still insufficient for reliable fine-grained state-conditioned interaction \href{https://github.com/FengxianJi/FineState-Bench}{Github.}

*   **研究动机**：
    *   **评估不精确**：现有基准测试（如 ScreenSpot, AITW）无法区分智能体是由于“没找到控件”、“点错了位置”还是“无法理解目标状态”而失败。
    *   **状态条件缺失**：大多数任务只要求点击，而不要求将 UI 控件设置为特定的、可验证的精确状态。
    *   **诊断困难**：缺乏阶段性的诊断指标来定位感知、定位与交互之间的性能瓶颈。

### 2. 论文提出的方法论
论文构建了一个名为 **FineState** 的评估框架，包含以下核心组件：
*   **FineState-Bench**：一个包含 2,209 个实例的跨平台基准测试，涵盖桌面、Web 和移动端。其核心特色是**双区域标注**：
    *   **定位框 ($B_{loc}$)**：覆盖控件的可见范围。
    *   **交互核心框 ($B_{int}$)**：触发状态改变的精确核心区域（如滑块的游标）。
*   **FineState-Metrics（四阶段诊断指标）**：
    1.  **SR@Loc**：定位成功率（点击点是否在当前控件范围内）。
    2.  **SR@Int**：交互成功率（点击点是否在当前交互核心范围内）。
    3.  **ES-SR@Loc**：定位时的精确状态成功率（在目标配置下定位正确且状态达成）。
    4.  **ES-SR@Int**：交互时的精确状态成功率（在目标配置下交互核心定位正确且状态达成）。
*   **视觉诊断助手 (VDA)**：一个即插即用的诊断工具（基于 GPT-4o），通过“描述-定位”两步走策略生成结构化描述和定位提示（Bounding Box Hint）。通过对比“有无 VDA 提示”下智能体的表现，量化视觉定位对失败的影响。

### 6. 论文的主要结论与发现
*   **性能低迷**：当前最强模型在精确状态达成率（ES-SR@Int）上平均仅为 22.8%，远低于其在粗粒度定位任务中的表现。
*   **瓶颈定位**：主要的性能损失发生在从“粗略控件定位”到“精确交互核心定位”的转换过程中。一旦智能体能点中交互核心，状态达成的成功率通常较高。
*   **视觉定位是关键**：引入 VDA 定位提示后，Gemini-2.5-Flash 的性能提升了 14.9%，说明当前的失败很大程度上归因于视觉定位精度不足，而非逻辑推理失败。
*   **组件差异**：连续值调节组件（如滑块、进度条）的失败率远高于离散选择组件（如开关、复选框）。

### 7. 优点（亮点）
*   **细粒度诊断**：通过双框标注和四阶段指标，首次实现了对 GUI 交互失败原因的量化拆解。
*   **跨平台覆盖**：同时覆盖了桌面、移动和 Web，是目前较为全面的 GUI 状态设置基准。
*   **即插即用的诊断工具**：VDA 的设计为未来分析模型视觉瓶颈提供了一个标准化的方法。

### 8. 不足与局限
*   **静态评估限制**：该基准测试基于静态截图和单步交互，无法评估智能体在真实动态环境中的多步纠错能力。
*   **VDA 依赖性**：诊断工具 VDA 本身依赖于高性能模型（如 GPT-4o），在处理极度复杂的专业软件界面时，VDA 自身的定位准确性可能成为瓶颈。
*   **未涉及长程推理**：侧重于单步状态设置，不涉及需要复杂逻辑链条的多步 GUI 任务。
