## 摘要
将多模态基础模型集成到企业生态系统中面临着根本性的软件架构挑战。架构师必须平衡相互竞争的质量属性：视觉语言动作（VLA）模型的高延迟和非确定性，与企业控制回路所要求的严格确定性和实时性能。在本研究中，我们为视觉智能体提出了一种架构模式语言，将快速、确定性的反射与缓慢、概率性的监督相分离。该语言包含四种架构设计模式：(1) 混合示能性集成 (Hybrid Affordance Integration)，(2) 自适应视觉锚定 (Adaptive Visual Anchoring)，(3) 视觉层次合成 (Visual Hierarchy Synthesis)，以及 (4) 语义场景图 (Semantic Scene Graph)。

## Abstract
Integrating multimodal foundation models into enterprise ecosystems presents a fundamental software architecture challenge. Architects must balance competing quality attributes: the high latency and non-determinism of vision language action (VLA) models versus the strict determinism and real-time performance required by enterprise control loops. In this study, we propose an architectural pattern language for visual agents that separates fast, deterministic reflexes from slow, probabilistic supervision. It consists of four architectural design patterns: (1) Hybrid Affordance Integration, (2) Adaptive Visual Anchoring, (3) Visual Hierarchy Synthesis, and (4) Semantic Scene Graph.

### 1. 核心问题与研究动机
*   **研究背景**：企业级自动化正在从基于 API 的集成转向直接与图形用户界面（GUI）交互的“具身智能体”。这在处理缺乏 API 的遗留系统（如 ERP、远程桌面）时尤为重要。
*   **核心矛盾**：
    *   **传统 RPA（机器人流程自动化）**：速度快、确定性强，但极其脆弱，UI 的微小变动（如布局偏移、分辨率变化）就会导致任务失败。
    *   **端到端 VLA（视觉-语言-动作）模型**：语义理解能力强、鲁棒性高，但存在高延迟（单次推理 2-10 秒）、高成本、非确定性（可能产生幻觉）以及架构耦合（CACE 反模式）等问题。
*   **研究目的**：提出一种架构模式语言，通过分离“快速确定性反射”与“慢速概率性监督”，解决实时控制与语义自适应之间的冲突。

### 2. 方法论：核心思想与关键技术
论文借鉴了机器人学中的**包容架构（Subsumption Architecture）**和心理学中的**双系统理论（System 1 & System 2）**，提出了包含四个模式的架构语言：

*   **核心思想：摊销推理（Amortized Inference）**。即大部分时间使用低成本的本地反射（System 1）执行，仅在遇到不确定性或 UI 漂移时才调用高成本的 VLA 监督者（System 2）进行修复。
*   **四种架构模式**：
    1.  **混合示能性集成（Hybrid Affordance Integration）**：多模态感知融合层。并行运行 OCR 和对象检测，通过几何融合和冲突解决机制过滤噪声，防止“示能性幻觉”（如误把文本当按钮）。
    2.  **自适应视觉锚定（Adaptive Visual Anchoring）**：建立快速缓存组件与慢速 VLA 监督者之间的回退路由。当本地匹配置信度低于阈值（如 $\tau < 0.9$）时，触发 VLA 重新定位元素并更新本地缓存。
    3.  **视觉层级合成（Visual Hierarchy Synthesis）**：利用格式塔原理（如邻近性、对齐性）将扁平的检测结果转化为层级化的 UI 树，实现相对寻址（如“登录表单中的提交按钮”），增强对布局平移的鲁棒性。
    4.  **语义场景图（Semantic Scene Graph）**：构建可查询的静态世界模型。在执行高风险操作前，通过遍历图结构验证前提条件，确保决策的透明度和可解释性。
