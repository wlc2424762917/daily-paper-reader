## 摘要
计算机使用智能体（CUA）依赖于图形用户界面的视觉观测，其中每张截图都被编码为大量的视觉 token。随着交互轨迹的增长，token 开销迅速增加，限制了在固定上下文和计算预算下可以纳入的历史信息量。与其他领域不同，这导致在使用历史信息时，性能几乎没有提升或提升非常有限。我们通过引入 ReVision 来解决这一低效问题。ReVision 用于在轨迹上训练多模态语言模型，其中冗余的视觉 patch 会通过一个学习到的 patch 选择器被移除；该选择器在对比连续截图之间的 patch 表示的同时，保留了模型所需的空间结构。在 OSWorld、WebTailBench 和 AgentNetBench 三个基准测试中，当使用 Qwen2.5-VL-7B 处理包含 5 张历史截图的轨迹时，ReVision 平均减少了约 46% 的 token 使用量，同时相比于不丢弃 patch 的基准线，成功率提高了 3%。这确立了明显的效率提升，使智能体能够以更少的 token 处理更长的轨迹。凭借这种效率的提升，我们重新审视了历史信息在 CUA 中的作用，并发现当冗余被移除后，随着更多过去观测值的加入，性能会持续提升。这表明，通常观察到的视觉历史信息饱和并非由于过去信息的效用有限，而是 token 表示效率低下的结果。

## Abstract
Computer-use agents (CUAs) rely on visual observations of graphical user interfaces, where each screenshot is encoded into a large number of visual tokens. As interaction trajectories grow, the token cost increases rapidly, limiting the amount of history that can be incorporated under fixed context and compute budgets. This has resulted in no or very limited improvement in the performance when using history unlike other domains. We address this inefficiency by introducing ReVision, which is used to train multimodal language models on trajectories where redundant visual patches are removed using a learned patch selector that compares patch representations across consecutive screenshots while preserving spatial structure required by the model. Across three benchmarks, OSWorld, WebTailBench, and AgentNetBench, when processing trajectories with 5 history screenshots using Qwen2.5-VL-7B, ReVision reduces token usage by approximately 46% on average while improving success rate by 3% over the no drop baseline. This establishes a clear efficiency gain, enabling agents to process longer trajectories with fewer tokens. With this improved efficiency, we revisit the role of history in CUAs and find that performance continues to improve as more past observations are incorporated when redundancy is removed. This suggests that the commonly observed saturation in visual history is not due to limited usefulness of past information, but rather a consequence of inefficient token representations.

### 1. 核心问题与整体含义
*   **研究动机**：计算机使用智能体（CUA）在执行多步任务时，通常需要参考历史截图。然而，现有的多模态大模型（MLLM）将每张截图编码为数千个视觉 Token，导致上下文窗口迅速被填满，计算成本激增。
*   **核心问题**：连续的 GUI 截图之间存在极高的视觉冗余（平均 45.4% 的区域保持不变）。这种冗余不仅浪费算力，还导致模型在处理长历史信息时出现性能“饱和”甚至下降。
*   **整体含义**：论文旨在通过剔除时间维度上的视觉冗余，使智能体能够以更低的成本处理更长的历史轨迹，从而提升长程任务的成功率。

### 2. 方法论
*   **核心思想**：引入一个轻量级的“补丁选择器”（RTS），通过对比连续两帧之间对应位置的补丁（Patch）表示，识别并移除未发生变化的视觉 Token，同时保留空间位置信息。
*   **关键技术细节**：
    *   **RTS 模块**：一个三层 MLP 分类器。它接收当前帧和前一帧对应补丁的嵌入向量，预测该补丁是否冗余。
    *   **监督学习**：利用 OmniParserV2 对截图进行语义区域分割，通过计算区域重叠（IoU）来自动生成冗余标签，用于训练 RTS。
    *   **空间结构保留**：被保留的 Token 维持其原始的空间位置索引（Position IDs），确保模型（如使用 m-ROPE 编码的模型）能够理解残缺图像的空间布局。
    *   **训练策略**：在过滤后的轨迹上对 MLLM 进行微调，使模型学会从之前的帧中“找回”被剔除的信息，实现时序上的证据融合。

### 3. 实验设计
*   **数据集/场景**：
    *   **OSWorld**：真实的桌面操作系统环境基准。
    *   **WebTailBench**：复杂的长程 Web 任务基准。
    *   **AgentNetBench**：离线交互轨迹基准。
*   **对比方法**：
    *   **通用模型**：Qwen2.5-VL (7B/32B/72B), Qwen3-VL (8B/32B), Kimi-VL-A3B。
    *   **专用智能体**：UI-TARS, OpenCUA。
    *   **消融策略**：随机丢弃、螺旋丢弃、基于像素相似度丢弃、基于嵌入余弦相似度丢弃。

*   **算力设备**：使用了 **8 × NVIDIA H200 GPU** 组成的集群。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 3 个主流基准测试，对比了超过 10 种不同的模型配置。
*   **消融实验**：非常充分。作者测试了不同历史窗口长度（从 1 到 15 张图像）、不同的 Token 剔除策略、跨模型家族（Qwen2.5 vs Qwen3）的泛化性，以及在单图任务（Grounding）中是否会退化。

### 6. 主要结论与发现
*   **效率提升**：在 5 张历史截图的设置下，ReVision 平均减少了 **46%** 的 Token 使用量。
*   **性能增强**：在大幅减小 Token 负担的同时，成功率反而提升了约 **3%**（在 WebTailBench 上提升更显著）。
*   **打破饱和点**：传统模型在历史图像超过 7 张时性能开始下降，而 ReVision 在 11 张以上仍能持续提升。这证明了**历史信息的效用饱和是由 Token 表示效率低下引起的，而非历史信息本身无用**。
*   **决策效率**：ReVision 能够以更少的交互步数完成任务，显示出更强的推理效率。

### 7. 优点
*   **即插即用**：作为一种 Token 过滤机制，无需修改 MLLM 的底层架构。
*   **低延迟**：相比于基于区域分割（如 OmniParser）的过滤，RTS 模块速度极快，适合实时交互。
*   **空间感知**：通过保留位置 ID，解决了视觉 Token 剔除后可能导致的空间错位问题。
*   **揭示本质**：通过实验有力地反驳了“长历史对 GUI 智能体无用”的既有偏见。

### 8. 不足与局限
*   **风险点**：如果 RTS 错误地剔除了微小但关键的视觉变化（如极小的状态指示灯改变），可能会导致智能体决策失败。
*   **训练依赖**：模型需要针对“过滤后的分布”进行专门训练，直接在现成模型上推理可能效果不佳。
*   **环境偏差**：虽然在三大基准上表现良好，但在极端动态或视觉噪声极大的 UI 环境下（如视频播放界面），冗余检测的准确性可能受限。
