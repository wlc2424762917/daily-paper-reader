Abstract 

LLM-based conversational AI agents struggle to maintain coherent behavior over long horizons due to limited context. While RAG-based approaches are increasingly adopted to overcome this limitation by storing interactions in external memory modules and performing retrieval from them, their effectiveness in answering challenging questions (e.g., multi-hop, commonsense, etc.) ultimately depends on the agent’s ability to reason over the retrieved information. However, existing methods typically retrieve memory by semantic similarity to the raw user utterance, which lacks explicit reasoning over missing intermediate facts and often returns evidence that is irrelevant or insufficient for grounded reasoning. In this work, we introduce GOAL -M EM , a goal-oriented reasoning framework for RAG-based agentic memory that performs explicit backward chaining from the user’s utterance as a goal. Rather than progressively expanding from retrieved context, G OAL -M EM 

decomposes each goal into atomic subgoals, performs targeted memory retrieval to satisfy each subgoal, and iteratively identifies what information from memory should be retrieved when intermediate goals cannot be resolved. We formalize this process in Natural Language Logic, a logical system that combines the verifiability of reasoning provided by FOL with the expressivity of natural language. Through extensive experiments on two datasets, comparing to eight strong memory baselines, we show that G OAL -M EM consistently improves performance, particularly on tasks requiring multi-hop reasoning and implicit inference. 

1 Introduction 

Large Language Models (LLMs) are increasingly deployed as conversational and agentic systems for complex, long-horizon tasks [ 21 , 36 , 9 , 19 , 15 ]. Yet their ability to provide coherent responses across long interaction horizons remains fundamentally constrained by finite context windows and the difficulty of reliably using information across long contexts [ 3 , 16 ]. Recent work augments the conversational agentic LLM systems with explicit external long-term memory : a persistent store that encodes information from prior interactions and makes it available for later retrieval [ 37 , 32 ]. Within this line of work, retrieval-augmented generation (RAG) -based memory stores detailed records of

These approaches support responses that remain coherent with past interactions while keeping the input within a feasible context window. Several works have focused on more effective encoding, organizing, and retrieving episodic traces, such as Mem0 [ 4 ], MAGMA [ 10 ], A-MEM [ 33 ], and MemTree [ 24 ]. However, a key question remains underexplored: which queries should the agent form to retrieve the right memory trace? . Existing methods merely focus on the surface form of the utterance, relying on semantic similarity through global dense retrieval over stored memory traces [ 27 ] or traversal heuristics over the predefined structures [ 33 , 10 ]. However, to satisfy the goal of generating appropriate responses, conversational agentic systems require substantial reasoning over the memory [ 1 ], and the information needed to support such reasoning is not always semantically adjacent to the utterance. Thus, existing approaches often fail on multi-hop questions that require chaining intermediate queries across memory entries. Reasoning over external long-term memory poses several challenges. The agent must correctly identify intermediate queries to recover the required memory traces across multi-hop steps [ 30 , 18 ]and ground the final response in retrieved memory facts rather than relying on the model’s parametric knowledge [ 22 ]. These challenges make many popular forward reasoning strategies, such as self-reflection and query reformulation, brittle: early irrelevant retrievals or unsupported inferences can propagate across steps, causing the agent to accumulate facts without converging on the correct set of memory traces, as illustrated in Figure 1 (middle). Such approaches also lack an explicit verification mechanism to ensure that the final answer is grounded in retrieved memory facts. To address this, we propose G OAL -M EM : Goal -Oriented Reasoning for RAG-based Mem ory, a framework rooted in the classical principle of backward chaining [ 7], which can be applied to arbitrary memory backbones. Instead of expanding outward from the utterance and the memory retrieved so far, G OAL -M EM treats the user’s question as the goal and decomposes a set of intermediate subgoals whose satisfaction is necessary to establish the goal (cf. right of Figure 1), and retrieves memory traces targeted at each subgoal. If the retrieved memory is insufficient to resolve any subgoals, the agent then recursively derives their antecedents as queries, until the original goal can be answered. To make this process verifiable, we represent both goals and memory traces in typed Natural Language Logic (NL-Logic). NL-Logic retains a language-like expressiveness since predicates and types remain NL phrases, while inheriting the structural guarantees of First-Order Logic (FOL) through typed variables and explicit substitutions. As a key component of NL-Logic, we introduce a verifiable unification for memory where the subgoal is grounded only when the retrieved memory facts are 2type-consistent and logically entailed via LLM-judged entailment over NL predicates, ruling out the spurious pure semantic matches. In summary, our contributions are as follows: • We propose GOAL -M EM , a novel goal-oriented reasoning framework for RAG-based memory with NL-Logic. G OAL -M EM performs backward chaining from the user utterance and retrieves only the goal-relevant facts with a verifiable response generation. • We show that G OAL -M EM is structure-agnostic and can be seamlessly applied to popular RAG-based memory backbones, yielding substantial empirical improvements. • We compare G OAL -M EM against popular forward reasoning approaches using the same memory backbones and show that G OAL -M EM consistently yields larger gains. 

2 Related Work 

Retrieval-Augmented Generation (RAG) Based Memory RAG-based memory retrieves a relevant subset of stored conversational traces or structured records at inference time. This paradigm has been widely studied in recent LLM-agent memory systems, where prior interactions are stored externally and selectively recalled to support long-term or task-grounded responses [ 37 ]. We therefore focus on representative retrieval-augmented memory backbones that cover the major design choices in this line of work. Early work like MemoryBank introduced updatable long-term memory with forgetting mechanisms [ 39 ]. More recent systems focus on advanced structural organization and routing heuristics. For relational structures, Mem0 [ 4] and A-MEM [ 33 ] utilize dynamic graphs and linked notes, concepts further advanced by agentic graph routing in AdaMem [ 34 ] and causal tool retrieval in AMA-Agent [ 38 ]. To manage abstraction, memory is increasingly structured hierarchically; MemTree [ 24 ] and RAPTOR [ 27 ] construct multi-level trees. Furthermore, methods like MAGMA perform query-adaptive traversal across multi-view graphs [ 10 ]. However, G OAL -M EM is orthogonal to all these works. Whereas prior methods primarily focus on memory structure and retrieval, the problem is addressing what information should be queried for the utterance. G OAL -M EM serves as a structure-agnostic reasoning layer that can be flexibly integrated with these memory backbones to enforce goal-oriented verifiable reasoning. 

LLM Reasoning via Backward Chaining Backward chaining is an established paradigm in logical reasoning. Unlike forward reasoning approaches such as query reformulation [ 20 ], self-reflection [ 35 , 28 ], iterative retrieval with LLM reranking [ 13 , 17 ], and memory-specific methods such as MemGuide [ 5] that aim to derive the goal from observed facts progressively, backward chaining works backward from the goal, avoiding exhaustive exploration of facts and thereby yielding substantial efficiency gains [ 26 , 2 ]. This goal-directed approach has recently been used in LLM-based reasoning. LAMBADA [ 12 ] pioneered this integration by prompting the LLM to emulate the backward-chaining process by decomposing complex goals into sub-goals. Symba [ 14 ] leverages a symbolic solver that undergoes backward chaining and calls the LLM to provide additional facts. LLM-TRes [ 29 ]employs theory resolution to unify LLM-derived commonsense axioms into backward chaining. Despite the collective success of these works in enhancing LLM reasoning efficiency, they remain constrained by a reliance on static knowledge, and to our knowledge, no research has extended the potential of backward LLM reasoning to the dynamic domain of memory-augmented agents, where backward chaining must be tightly coupled with adaptive memory and continual context updates. 

3 Preliminaries 

Retrieval-Augmented Generation (RAG) Based Memory We consider a conversation as a sequence of turns. At turn t, the user provides an utterance ut, and the agent generates a response at.Let [( u1, a 1), . . . , (ut−1, a t−1)] denote the dialogue history prior to turn t. A RAG-based memory-augmented LLM is additionally equipped with an external long-term memory M that stores episodic traces distilled from dialogue history. A RAG-based memory backbone B consists of three components: (1) a persistent external long-term memory store M, instantiated as a flat vector database, graph-structured memory, or tree-structured memory; (2) a retrieval function Bretrieve (qt, M) that returns memory facts from M given a query 

qt; and (3) an update mechanism to encode information in ut into M. Given the utterance ut,the agent first constructs one or more queries qt that encapsulate the information needs in ut.3Each query is passed to the memory backbone B, which retrieves relevant memory facts Mt:

Mt = Bretrieve (qt, M). The agent then generates a response conditioned on both the current utterance and the retrieved fact, at = f LLM (ut, Mt). The backbone updates M with the new turn. As noted in section 2, choices on what information in dialogue is encoded, how it is organized and updated, and how queries are matched to traces in M are made by B and fall outside the scope of our work. We focus on constructing a sequence of queries qt via goal-oriented reasoning. 

Natural Language Logic (NL-Logic) While conversation and memories are inherently expressed in NL, using LLMs for reasoning over pure NL induces the risk of generating ungrounded or hallucinated content. Moreover, NL-based reasoning is often difficult to verify, as the intermediate steps leading to a conclusion are not explicitly structured. Formal logical systems such as first-order logic (FOL) [ 26 ] provide a structured framework with explicit variables, substitutions, and entailment relations, enabling transparent and verifiable reasoning. However, its reliance on predefined symbols limits its expressiveness for memory traces, motivating the need for a hybrid logical NL representation. To combine the strengths of both paradigms, we extend FOL into NL-Logic, where predicates, objects, and relations are expressed directly in NL, preserving logical structure while maintaining expressiveness. Let E denote the constants appearing in the current utterance or retrieved memory facts, V a set of variables, and T the set of NL type descriptions. A typed term is written as t : τ ,where t ∈ E ∪ V and τ ∈ T . Unlike conventional typed logical systems, we do not assume that T is a fixed, predefined type vocabulary. Instead, types are open-ended, on-demand NL phrases and type checking is performed by the LLM through NL entailment. This allows more flexible definitions. In NL-Logic, an atomic formula is R(t1, . . . , t n), where R is an NL relational phrase and each argument ti is explicitly typed (e.g., “(Alice : user ) likes (x : flavor ).” ). Type constraints are enforced on variables: x : τ ⇒ x ∈ E τ meaning that x belongs to Eτ , the subset of constants in E

with type τ . Type specification for constants (e.g., “Alice” , “Matcha” ) is optional, as their names implicitly encode their semantic type. Thus, an atomic formula in NL-Logic is represented as a structured NL statement with typed arguments, rather than as a purely symbolic predicate application. In this logical system, a knowledge base (e.g. Memory) can be represented as a set of ground atoms: 

M = {R (e1, . . . , e n) | (e1, . . . , e n) ∈ E n} (1) 

4 Methodology 

Given a user utterance ut at turn t, the objective is to generate a grounded response. G OAL -M EM first maps ut to a formal NL-Logic goal ϕut (x1, . . . , x m), which captures both the semantic intent of the utterance and the set of variables {x1, . . . , x m} that must be instantiated to satisfy the user’s request.

> User utterance: ut=“Alice: Which drink did I order at cafe Momoco? ”NL-Logic goal: ϕut=“( x:drink ) ordered by ( Alice:human ) at ( Momoco:cafe ).”

Answering the user requires resolving variable x, whose type is drink . The response-generation problem thus reduces to a logical inference: finding a substitution θ = {x1/e 1, . . . , x m/e m}, where each variable xi is replaced by a type-consistent constant ei drawn from the retrieved memory Mt.

Memory Backbones and Baselines. We instantiate G OAL -M EM with eight retrieval-augmented memory backbones: BM25-RAG [ 25 ], Dense-RAG [ 11 ], Mem0 and Mem0-Graph [ 4], MemTree [ 24 ], RAPTOR [ 27 ], GraphRAG [ 6], and A-MEM [ 33 ]. These cover unstructured, tree-based, and graph-based memory designs. Since public implementations differ in availability and interfaces, we implement all methods in a unified evaluation pipeline following their original algorithmic descriptions, while controlling for retrieval budgets, prompts, and backbone. We compare against forward reasoning baselines: query reformulation [ 20 ], LLM-based self-reflection, MemGuide [ 5], and ReAct [35] under the same memory backbone B. Please refer to Appendix F for details. 

Datasets and Setup. We evaluate G OAL -M EM on two long-term conversational memory bench-marks: LoCoMo [ 21 ] and LongMemEval-Small [ 31 ]. LoCoMo emphasizes long-range, multi-session memory with many multi-hop questions over dispersed evidence, while LongMemEval evaluates core assistant memory abilities, including extraction, multi-session and temporal reasoning, knowl-edge updates, and abstention (cf. Appendix C for details). We use two LLM backbones: GPT-5.4 mini [ 23 ] and Gemma-4-26B-A4B [ 8]. We use identical hyperparameters in the standalone and GOAL -M EM -augmented settings. The performance delta ( ∆) isolates the contribution of G OAL -MEM from backbone-specific tuning. We maintain a consistent retrieval budget across methods: the memory backbone alone retrieves up to k = 60 memories in a single call, while reasoning-based approaches distribute the same retrieval cap across iterations. The maximum breadth is consistently set to 3, and the maximum depth is set to 5. We evaluate the generated responses with reference answer via LLM-as-judge using prompts provided by LoCoMo and LongMemEval, and token F1. 

5.1 Experimental Results RQ1. Table 1 reports performance with and without G OAL -M EM across two datasets and a range of memory backbones. G OAL -M EM delivers consistent gains in both LLM-as-judge accuracy and token-level F1, regardless of how the underlying memory is organized. The largest absolute improvements appear on the simplest flat backbones, BM25-RAG and Dense-RAG, suggesting that goal-oriented backward chaining can compensate for limited memory structure. Mem0 and Mem0-Graph show smaller gains because their abstract memory facts omit fine-grained cues such as temporal details and raw dialogue context, weakening intermediate subgoal retrieval, which can be mitigated through a modification in structures, but altering the original backbone designs. To localize the source of these gains, Figure 3 breaks down accuracy by question type on LoCoMo. GOAL -M EM shows a substantial lift on multi-hop questions, even for backbones whose overall improvements are smaller. This pattern supports the claim that semantic similarity alone is insufficient for multi-hop questions because the intermediate queries needed for chaining are often missing. GOAL -M EM ’s goal-directed backward chaining directly addresses this problem.

RQ3. We next ablate the two budgets that govern G OAL -M EM ’s search: the maximum depth (subgoal refinement) and the maximum breadth (alternative goal decompositions). Figure 4 shows cumulative LLM-as-judge accuracy as each budget is increased, for two backbone LLMs. Increasing depth yields accuracy improvements until max depth ≥ 3, and the gains are largest for backbones with weaker baseline performance. When the first retrieved memory is incomplete or noisy, antecedent-guided refinement lets agents navigate the memory space more systematically, recovering missing context that weaker backbones are especially likely to miss. Increasing breadth produces little additional gain. We attribute this to the strength of SOTA LLMs; at a single goal, decomposition is typically sufficient, and the depth loop also refines potential failures. Allocating the search budget to depth rather than breadth is therefore the more practical strategy. 

6 Conclusion 

We introduced G OAL -M EM , a framework that enhances the reasoning capabilities of memory-augmented LLM agents through goal-oriented backward chaining. By converting user utterances into NL-Logic goals and iteratively grounding subgoals via targeted retrieval and verifiable unification, GOAL -M EM enables more effective use of external memory. In addition, its compatibility with diverse memory backbones makes it broadly applicable across existing RAG-based systems. Experimental results show consistent improvements across benchmarks, with particularly strong gains on tasks requiring multi-step inference. Overall, G OAL -M EM provides an effective approach for improving memory-grounded reasoning in agentic LLM systems. 90 1 2 3 4 5

The workflow of the G OAL -M EM reasoning framework is summarized in Algorithm 1. The reasoning process begins by mapping the utterance ut into a formal goal ϕut , which specifies the variables that must be instantiated to answer the query. An initially empty substitution θ is maintained to store variable bindings as they are discovered. The algorithm proceeds through the breadth loop . At each breadth step b, the goal ϕut is decomposed into a set of subgoals ψ(b) using an LLM-based decomposition function, conditioned on previously explored decompositions {ψ(0) , . . . , ψ (b−1) } (with ψ(0) = ∅). Each subgoal represents a necessary condition for satisfying the original goal. For each subgoal ψi ∈ ψ(b), the algorithm performs targeted retrieval by querying the memory backbone. The retrieved facts are aggregated into a memory set M(b, 0)  

> t

. The algorithm then attempts to ground the subgoals via verifiable unification , which matches subgoals against retrieved memory while enforcing three constraints: (i) type consistency between variables and candidate values, (ii) consistency with previously established substitutions, and (iii) logical entailment of the grounded subgoal by the retrieved memory (as judged by the LLM). This step returns an updated substitution θ(b, 0) , a set of unresolved subgoals U(b, 0) , and a set of supporting memory facts F that successfully grounded subgoals. We also maintain the accumulated subgoal set Ψ(b, ≤0) = ψ(b), and define the set of resolved subgoals S(b, 0) = Ψ (b, ≤0) \ U (b, 0) .If all subgoals are successfully grounded (i.e., U(b, 0) = ∅), the algorithm terminates and generates the final response conditioned on the utterance, the substitution, and only the supporting memory F.Otherwise, the algorithm enters a depth loop , which performs recursive refinement of unresolved subgoals. At each depth step d, only the subgoals from the previous refinement frontier ψ(b,d −1) 

(initialized as U(b, 0) ) are refined using an LLM-based refinement function, producing a new set ψ(b,d )

of intermediate subgoals. These refined subgoals are used for additional targeted retrieval, and the newly retrieved facts are accumulated into the memory M(b,d ) 

> t

.The accumulated subgoal set is updated as Ψ(b, ≤d) = Ψ (b, ≤d−1) ∪ ψ(b,d ). The unification procedure is then re-applied over Ψ(b, ≤d) and M(b,d ) 

> t

to attempt further grounding. To avoid redundant computation while preserving correctness, subgoals that have already been successfully grounded (tracked in S(b,d −1) ) are skipped during unification, although they remain part of the accumulated subgoal set for consistency. This step produces updated substitution θ(b,d ), unresolved subgoals 

U(b,d ), and supporting facts F, and updates the resolved set as S(b,d ) = Ψ (b, ≤d) \ U (b,d ).This refinement process continues until either (i) all subgoals are grounded, (ii) no new memory can be retrieved, or (iii) the maximum depth is reached. If grounding succeeds at any depth, the algorithm generates the final response using the grounded substitution and the supporting memory F.If the current decomposition fails to yield a complete grounding after exhausting the depth budget, the algorithm returns to the breadth loop and considers an alternative decomposition of the original goal, enabling exploration of multiple reasoning strategies.

C Additional Information on the Evaluation Datasets 

We evaluate G OAL -M EM on two long-term conversational memory benchmarks: LoCoMo [ 21 ] and LongMemEval-Small [ 31 ]. These datasets are designed to assess the ability of LLM-based agents to store, retrieve, and reason over information accumulated across extended interactions. 

LoCoMo. LoCoMo is a benchmark for evaluating long-range conversational memory in multi-session settings. It consists of dialogues that span multiple sessions, where relevant information required to answer a query may be distributed across distant parts of the interaction history. The dataset emphasizes questions that require multi-hop reasoning , where the model must retrieve and combine multiple pieces of evidence to arrive at the correct answer. In addition to multi-hop questions, LOCOMO includes single-hop, temporal, and open-domain queries, enabling a fine-grained analysis of model performance across different reasoning types. A key challenge in L OCOMO is that the information necessary to answer a question is often not semantically similar to the query itself, requiring the model to identify and retrieve intermediate facts. This makes it particularly suitable for evaluating the effectiveness of goal-directed retrieval and backward chaining. 

LONG MEM EVAL -S MALL . LONG MEM EVAL -S MALL is a benchmark focused on core memory capabilities of conversational agents. It evaluates a range of skills, including information extraction from dialogue, reasoning across multiple sessions, handling temporal dependencies, updating knowl-edge over time, and abstaining when sufficient evidence is not available. Compared to L OCOMO,this dataset places greater emphasis on the correctness and consistency of memory usage, as well as the ability to track evolving user information. The dataset includes diverse query types that test whether the model can accurately retrieve and ground responses in previously stored memory, rather than relying on parametric knowledge. 

Evaluation Protocol. For both datasets, we evaluate model outputs using an LLM-as-a-judge framework, which assesses the correctness and grounding of generated responses with respect to the reference answers. In addition, we report token-level F1 scores to provide a complementary quantitative measure of answer overlap. These evaluation metrics allow us to capture both semantic correctness and lexical fidelity, offering a comprehensive view of model performance on long-term memory reasoning tasks.

F Forward Reasoning Baseline Implementations 

This appendix describes the forward reasoning baselines used in RQ2: Query Reformulation, Self-Reflection, ReAct, and MemGuide. All four methods are implemented as wrappers around the same retrieval-augmented memory backbone B used by G OAL -M EM . Thus, these baselines do not change the memory storage, indexing, or update procedure of the underlying backbone; they only change how queries are generated, how retrieved memory units are accumulated or filtered, and how the final answer is produced. 19 Shared retrieval protocol. For a controlled comparison, all baselines use the same inference LLM, decoding configuration, memory backbone B, and answer-generation prompt as the corresponding GOAL -M EM run. We use a shared retrieved-memory cap of 

kret max = 60 , (7) where kret max denotes the maximum number of unique retrieved memory units that can be exposed to the final answer stage for one question. This retrieval cap is distinct from Kmax in the complexity analysis, where Kmax denotes the maximum number of subgoals produced by a goal decomposition. For iterative baselines, the same cap is distributed across retrieval actions: each retrieval action calls 

Bretrieve with a bounded per-query fan-out, retrieved units are deduplicated, and the merged pool is clipped to at most kret max units before final answer generation or filtering. 

Query Reformulation. The Query Reformulation baseline follows the query-rewriting view of retrieval-augmented generation, where an LLM adapts the user input before retrieval to reduce the gap between the surface form of the question and the information needed by the reader [ 20 ]. Given the user utterance ut, the LLM generates nqr = 5 diverse retrieval queries. The original question is also included as a retrieval query, giving 

Qqr (ut) = {ut} ∪ Rewrite LLM (ut; nqr = 3) . (8) The prompt asks the LLM to target different entities, events, temporal expressions, synonyms, and background aspects that may appear in memory, while preserving important entities and events from the original question. The retrieved results from all queries are merged, deduplicated, and clipped to 

kret max = 60 . The final answer is then generated using the standard answer prompt over the resulting memory pool. This baseline is intentionally simple: it performs no symbolic decomposition, no unification, no missing-slot refinement, and no post-retrieval reranking. 

Self-Reflection. The Self-Reflection baseline adapts the reflective-agent principle of Reflexion to the single-question memory retrieval setting. Reflexion uses verbal feedback and an episodic memory buffer to improve later decisions; in our evaluation, we use the same forward-style reflection idea without carrying reflections across evaluation questions, so that the baseline does not receive extra persistent feedback unavailable to the other methods. The method first retrieves memories using the original question. At each reflection turn, the LLM is shown the question, all previously issued queries, and the accumulated retrieved memory pool. It outputs a JSON decision with three fields: whether the pool is sufficient, one concise next query if more evidence is needed, and a short rationale. If the pool is judged sufficient, the loop terminates; otherwise, the follow-up query is sent to Bretrieve ,and the newly retrieved units are merged and deduplicated with the existing pool. We set the maximum number of reflection turns to T sr max = 5 . The method can therefore terminate because the LLM judges the memory pool sufficient, because no new retrieval query is proposed, because retrieval returns no new memory units, or because the turn limit is reached. The final answer is generated from the accumulated memory pool after clipping to kret max = 60 . Unlike G OAL -M EM ,Self-Reflection does not maintain typed variables or substitutions and does not verify that a retrieved memory entails a particular subgoal; each additional query is generated forward from the current question and evidence pool. 

ReAct. The ReAct baseline follows the reasoning-and-acting paradigm in which the LLM alternates between natural-language reasoning traces and actions that interact with an external environment [ 35 ]. In our memory-QA adaptation, the external environment is the shared memory backbone B. At each step, the model emits exactly one Thought and one Action . The action space is restricted to 

Retrieve[query] or Finish[answer] . (9) When the model selects Retrieve[query] , the query is sent to Bretrieve , and the returned memories are appended to the trajectory as an Observation . When the model selects Finish[answer] , the answer is returned directly. This preserves the ReAct design choice of interleaving reasoning traces with environment actions, while making the environment identical to the memory backend used by GOAL -M EM .We set the maximum number of ReAct steps to Sreact max = 5 . If the model output does not match the required Thought /Action format, the implementation retries once with a stricter format reminder. 20 If the step limit is reached before a Finish action, the model is forced to produce a final Finish 

answer using only the observations collected so far. Retrieved observations are deduplicated, and the accumulated memory pool is clipped to kret max = 60 . As with Self-Reflection, ReAct is a forward reasoning baseline: it decides the next retrieval action from the current trajectory, rather than decomposing the target answer into subgoals and verifying substitutions. 

MemGuide. MemGuide is a memory-selection method for goal-oriented multi-session LLM agents. The original method introduces a two-stage intent-driven framework: Intent-Aligned Retrieval retrieves memory units that share the current task intent, and Missing-Slot Guided Filtering reranks candidate memory units according to their marginal gain in filling missing task slots [ 5]. Our implementation follows this two-stage structure while adapting it to the unified memory-backbone interface used in our experiments. First, an intent-capture prompt maps the question to a compact intent description and generates 

nintent = 5 intent-aligned retrieval queries. The original question is included as an additional seed query. These queries retrieve an initial candidate pool from B. Second, a missing-slot reasoner inspects the current candidate pool and decides whether the pool is sufficient. If not, it enumerates concrete missing fields, such as a date, location, counterpart person, event outcome, or relation, and proposes at most F mg max = 5 follow-up queries for the next refinement round. We allow at most 

Rmg max = 3 missing-slot refinement rounds. After retrieval, MemGuide applies slot-guided filtering. Candidate memory units are scored by an LLM judger according to how directly they answer the question or fill the identified missing slots. The implementation combines the LLM score with the retrieval prior, 

s(f ) = λs LLM (f ) + (1 − λ)sret (f ), λ = 0 .7, (10) where sret (f ) favors earlier retrieved candidates. The final selected memory pool is sorted by this score and clipped to kret max = 60 before answer generation. The candidate pool is also clipped to the same cap after retrieval and deduplication. This adaptation is faithful to the algorithmic role of MemGuide as an intent- and slot-aware forward memory selector, but it does not assume MemGuide’s original task-oriented-dialogue memory format or its task-specific fine-tuned filtering model. Instead, all candidate units are retrieved from the same memory backbone B used by the other baselines, and the slot filter is implemented with the shared inference LLM. This keeps the comparison controlled: MemGuide receives the benefit of intent-aligned retrieval and missing-slot filtering, but it does not receive a method-specific memory representation or extra supervised training signal unavailable to the other methods. 

G Broader Impacts 

Improving the coherence and factual grounding of long-horizon conversational agents has clear posi-tive societal value in applications such as healthcare assistance, education, and personal productivity. By anchoring responses in retrieved memory rather than parametric knowledge, G OAL -M EM reduces the risk of hallucinated but plausible-sounding responses. However, as with all existing RAG-based memory works and LLM-based systems, there may be concerns about the possibility of mistakes and the risk of over-reliance on the system’s outputs. 

H Limitations 

As we observed in Section 5.1, G OAL -M EM consistently improves the performance of existing RAG-based agentic memory systems. At the same time, since it builds upon an underlying memory and retrieval backbone and leverages an LLM for components such as goal decomposition and answer generation, its effectiveness naturally depends on the quality of these components.
