## 摘要
随着基于大语言模型（LLM）的智能体越来越多地依赖外部工具，评估其在熟悉的工作流和短程交互之外，维持工具驱动推理的能力变得至关重要。我们推出了 AgentEscapeBench，这是一个密室逃脱风格的基准测试，旨在测试智能体在显式长程依赖约束下，能否推断、执行并修正新颖的工具使用流程。每个任务在工具和物品上定义了一个有向无环依赖图，要求智能体调用真实的外部函数，追踪逐步揭示的隐藏状态，传递中间结果，并提交一个可确定性验证的最终答案。AgentEscapeBench 包含分布在五个难度层级的 270 个实例，并支持全自动评估。对 16 个 LLM 智能体和人类参与者的实验表明，随着依赖深度的增加，性能急剧下降：人类的成功率从难度 5 的 98.3% 下降到难度 25 的 80.0%，而表现最好的模型则从 90.0% 下降到 60.0%。轨迹分析将模型的失败主要归因于长程状态追踪、线索遵循以及中间结果传递的崩溃。这些发现表明，当前的智能体通常能够处理局部工具使用，但在深层上下文依赖方面仍面临挑战。我们希望 AgentEscapeBench 能够作为一个诊断性测试平台，用于衡量当前智能体的能力，并为未来旨在实现更稳健的通用推理、行动和适应的训练工作提供参考。

## Abstract
As LLM-based agents increasingly rely on external tools, it is important to evaluate their ability to sustain tool-grounded reasoning beyond familiar workflows and short-range interactions. We introduce AgentEscapeBench, an escape-room-style benchmark that tests whether agents can infer, execute, and revise novel tool-use procedures under explicit long-range dependency constraints. Each task defines a directed acyclic dependency graph over tools and items, requiring agents to invoke real external functions, track hidden state revealed incrementally, propagate intermediate results, and submit a deterministically verifiable final answer. AgentEscapeBench includes 270 instances across five difficulty tiers and supports fully automated evaluation. Experiments with sixteen LLM agents and human participants show that performance drops sharply as dependency depth increases: humans decline from 98.3% success at difficulty-5 to 80.0% at difficulty-25, while the best model drops from 90.0% to 60.0%. Trajectory analysis attributes model failures mainly to breakdowns in long-range state tracking, clue adherence, and intermediate-result propagation. These findings suggest that current agents can often handle local tool use but still struggle with deep contextual dependencies. We hope AgentEscapeBench can serve as a diagnostic testbed for measuring current agent capabilities and informing future training efforts toward more robust general-purpose reasoning, action, and adaptation.

AgentEscapeBench 的核心是一个**基于 DAG（有向无环图）的自动化任务生成流水线**，包含六个阶段：
1.  **模板库构建**：包含 32 种工具（涉及大数运算、加密、编码、文件操作等）、16 种物品和 4 种容器模板。
2.  **DAG 骨架生成**：采用**逆向生成算法**。从目标节点（胜利节点）开始，递归地为输入端口寻找匹配的输出端口，直到达到预设的节点数量（难度等级）。
3.  **源节点标注**：识别所有不需要前置输入的叶子节点作为初始线索来源。
4.  **数值实例化**：利用 LLM 为源节点生成符合类型约束和逻辑一致性的具体数值（如 AES 密钥长度、模幂运算的底数等）。
5.  **确定性前向执行**：在沙盒中执行整个 DAG，计算出唯一的、可验证的最终答案（Flag）。
6.  **叙事生成与验证**：LLM 根据节点关系生成不同主题（如赛博朋克、侦探、科幻）的背景故事，并隐藏部分线索。
*   **关键机制**：**增量披露（Incremental Disclosure）**。某些工具或物品最初是不可见的，只有当智能体解开前置容器或触发特定掉落事件后才会显现，这测试了智能体的动态探索能力。

*   **实验规模**：270 个任务实例，每个模型在不同难度下进行数十次测试，总计分析了数千条交互轨迹。
*   **充分性**：实验设计较为充分。不仅提供了最终成功率，还进行了**细粒度的轨迹分析**，包括反馈驱动的收敛速度、早熟调用率（顺序错误）和线索遵循率。

### 6. 主要结论与发现
*   **性能随深度剧降**：所有模型性能随 DAG 深度增加而显著下降。最强模型 Claude-Opus-4.6 在难度 5 时 SR 为 90%，但在难度 25 时降至 60%；而人类在难度 25 下仍保持 80% 的成功率。
*   **长程依赖是瓶颈**：模型失败的主要原因不是单个工具调用错误，而是**中间结果在长链条中的传递失效**（线索遵循率低）以及**状态追踪混乱**（过早调用尚未解锁的工具）。
*   **推理模型未必占优**：DeepSeek-Reasoner 等专门的推理模型在某些任务上表现不如其对话版本，说明长程工具交互需要的是“环境感知下的动态调整”，而非单纯的内部思维链。
*   **人类-模型差距**：人类在面对陌生规则时表现出极强的泛化和适应能力，而 LLM 在脱离熟悉模板后表现出明显的“迁移赤字”。

### 7. 优点与亮点
*   **诊断性强**：通过 DAG 结构将复杂的智能体行为分解为可追踪的步骤，能够精准定位模型是在哪一步“掉链子”。
*   **消除先验干扰**：密室逃脱的设定有效地排除了模型通过记忆训练数据获胜的可能性，真正测试了 OOD（域外）推理。
*   **全自动流水线**：任务生成、执行验证和轨迹分析均实现了自动化，具有良好的可扩展性。

### 8. 不足与局限
*   **语言局限**：目前所有任务描述和工具接口均为英文，未测试多语言环境下的推理能力。
*   **单次尝试偏差**：由于 API 成本限制，每个实例仅运行一次，未能完全消除模型输出的随机性波动。
*   **固定步数预算**：为所有模型设置了统一的步数上限，可能限制了某些“慢思考”模型的潜力。
*   **应用场景单一**：虽然测试了通用推理，但密室逃脱的形式与真实的工业/科研工作流仍有一定距离。
