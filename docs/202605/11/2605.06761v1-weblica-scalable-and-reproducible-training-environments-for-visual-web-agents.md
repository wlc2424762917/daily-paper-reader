## 摘要
Web 环境复杂、开放且不断变化，这使得扩展视觉 Web 智能体的训练数据具有挑战性。现有的数据收集尝试仍局限于用于监督微调的离线轨迹，或少数用于强化学习（RL）训练的模拟环境，因此未能捕捉到 Web 的多样性。我们提出了 Weblica (Web Replica)，这是一个用于构建可复现且可扩展的 Web 环境的框架。我们的框架利用了：1) HTTP 级缓存，在保留交互行为的同时捕获并重放稳定的视觉状态；2) 基于大语言模型（LLM）的环境合成，该合成基于真实世界的网站和核心 Web 导航技能。利用该框架，我们将强化学习训练扩展到了数千个多样的环境和任务中。我们表现最好的模型 Weblica-8B 在多个 Web 导航基准测试中优于同等规模的开源权重基准模型，同时使用的推理步骤更少，在增加测试时计算量时表现出良好的扩展性，并且与 API 模型相比具有竞争力。

## Abstract
The web is complex, open-ended, and constantly changing, making it challenging to scale training data for visual web agents. Existing data collection attempts remain limited to offline trajectories for supervised fine-tuning or a handful of simulated environments for RL training, thus failing to capture web diversity. We propose Weblica (Web Replica), a framework for constructing reproducible and scalable web environments. Our framework leverages 1) HTTP-level caching to capture and replay stable visual states while preserving interactive behavior and 2) LLM-based environment synthesis grounded in real-world websites and core web navigation skills. Using this framework, we scale RL training to thousands of diverse environments and tasks. Our best model, Weblica-8B, outperforms open-weight baselines of similar size across multiple web navigation benchmarks while using fewer inference steps, scales favorably with additional test-time compute, and is competitive with API models.

### 1. 核心问题与整体含义（研究动机和背景）
构建能够自主导航网页的视觉智能体（Visual Web Agents）面临核心瓶颈：**训练数据的可扩展性与环境的可复现性之间的矛盾**。
*   **现有方法的缺陷**：
    *   **离线轨迹（SFT）**：缺乏交互性，智能体无法从错误中学习或进行探索。
    *   **模拟环境（如 WebArena）**：多样性有限，难以覆盖真实 Web 的复杂性。
    *   **实时网页训练**：受限于网络延迟、机器人检测（Bot Detection）和网页内容的频繁变动，导致训练极不稳定且不可复现。
*   **研究目标**：创建一个既能模拟真实网页多样性，又能保证高吞吐量、稳定且完全可复现的交互式训练框架。

### 2. 方法论：Weblica 框架
Weblica 引入了两种互补的机制来构建离线交互环境：

*   **Weblica-Cache（HTTP 级缓存）**：
    *   **记录与重放**：利用 Playwright 捕获真实网站的 HTTP 流量。
    *   **自动规则生成**：通过对比多次访问的差异，识别并过滤掉导致缓存失效的动态参数（如时间戳、Session Token），确保在完全断网的情况下也能实现确定性的网页重放。
    *   **任务来源**：结合 InstaV3 数据集，在缓存环境中验证任务的可解性。
*   **Weblica-Synth（LLM 基于能力的合成）**：
    *   **能力驱动**：从真实轨迹中提取 144 种核心 Web 导航技能（如日期选择、表单提交、地图交互）。
    *   **自动化构建**：使用 Claude Code (Opus 4.5) 编写无框架的 HTML/JS/CSS 代码，生成具有特定视觉风格（如极简主义、复古风）和行业类别（如航空、银行）的静态网站。
    *   **自验证循环**：编码智能体通过截图和 Playwright 自动测试，迭代修复 CSS 和功能问题，确保环境可用。
*   **训练流程**：
    *   **模型基础**：基于 Qwen3-VL 系列（2B, 4B, 8B），仅使用原始截图和像素坐标，不依赖 DOM 或辅助标记。
    *   **两阶段训练**：先进行经过 LLM 判别器筛选的 **SFT（监督微调）**，再使用 **Dr. GRPO（一种强化学习算法）** 在 Weblica 环境中进行大规模 RL 训练。
    *   **奖励机制**：采用 **LLM-as-Judge**，由 GPT-4o 根据任务标准和轨迹截图判定成功与否。

### 4. 资源与算力
*   **硬件**：所有 SFT 和 RL 实验均在 **8 台 NVIDIA B200 GPU** 上完成。
*   **效率**：由于环境本地化部署并跳过了动画，单次动作响应时间从 ~1.5s 降至 50-150ms。
*   **RL 训练规模**：每个 RL 步骤生成 1024 条轨迹（256 个提示词 × 4 个采样），每步耗时约 28 分钟。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 2B 到 8B 的三种模型参数规模，验证了 Scaling Law。

### 6. 主要结论与发现
*   **性能领先**：Weblica-8B 在 OM2W 上达到 39.2% 的 Pass@1，显著优于同规模的开源模型（如 MolmoWeb-8B 的 35.3%），且推理步数更少。
*   **扩展性强**：性能随模型规模和测试时采样次数（k 值）显著提升。在 Pass@8 时，OM2W 准确率可飙升至 68.8%。
*   **能力保持**：在提升导航能力的同时，模型在 MMBench-GUI 等视觉定位（Grounding）基准上的表现并未退化，反而略有提升。
*   **RL 的关键作用**：RL 训练使模型能够更有效地利用长序列（更多步骤）来解决复杂任务，而基础模型在增加步数时提升有限。

### 7. 优点
*   **解决了环境痛点**：通过缓存和合成，摆脱了对实时网络和人工标注轨迹的依赖，实现了训练的“工业化”扩展。
*   **高效率**：本地环境服务器极大提升了 RL 训练的吞吐量。
*   **纯视觉方案**：不依赖不稳定的 DOM 结构，增强了模型在不同前端框架下的鲁棒性。

### 8. 不足与局限
*   **Sim-to-Real 差距**：合成环境虽然多样，但与真实复杂网页（如重度依赖后端逻辑的应用）之间仍存在差距。
*   **静态快照限制**：缓存环境是静态的，无法反映网页随时间的动态更新。
*   **任务类型**：目前主要集中在单次任务，尚未深入探索多轮对话、长期记忆或需要人类实时反馈的复杂交互场景。
*   **判别器依赖**：RL 训练高度依赖 GPT-4o 作为判别器，这可能引入判别器的偏见或成本问题。
