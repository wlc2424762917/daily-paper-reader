## 摘要
我们提出了一种动态策略学习方法，该方法结合了广义规划和分层任务分解，适用于基于大语言模型（LLM）的智能体。我们的方法，即分层广义策略组件学习（HCL-GP），学习可在不同任务实例间泛化的参数化策略，并从成功的执行中自动提取可重用组件，将其组织成组件库以进行组合式策略生成。我们解决了三个挑战：（1）通过自动分解学习组件；（2）泛化组件以最大化重用；（3）通过语义搜索进行高效检索。在 AppWorld 基准测试上的评估结果显示，我们的方法在常规任务上达到了 98.2% 的准确率，在包含未见过的应用程序的挑战性任务上达到了 97.8%，在挑战性场景中比静态合成方法提高了 15.8 个百分点。对于开源模型，动态重用实现了 62.5% 的成功率，而没有重用时成功率接近于零。这表明经典规划概念可以有效地与 LLM 智能体集成，从而提高准确性和效率。

## Abstract
We present a dynamic policy-learning approach that combines generalized planning and hierarchical task decomposition for LLM-based agents. Our method, Hierarchical Component Learning for Generalized Policies (HCL-GP ), learns parameterized policies that generalize across task instances and automatically extracts reusable components from successful executions, organizing them into a component library for compositional policy generation. We address three challenges: (1) learning components through automated decomposition, (2) generalizing components to maximize reuse, and (3) efficient retrieval via semantic search. Evaluated on the AppWorld benchmark, our approach achieves 98.2% accuracy on normal tasks and 97.8% on challenge tasks with unseen applications, improving 15.8 points over static synthesis on challenging scenarios. For open-source models, dynamic reuse enables 62.5% success versus near-zero without reuse. This demonstrates that classical planning concepts can be effectively integrated with LLM agents for improved accuracy and efficiency.

*   **研究动机**：当前的 LLM 智能体在解决复杂、多步骤的任务时，通常将每个任务视为孤立的实例。这导致智能体在面对相似任务时会重复“重新发明轮子”，无法积累和重用过程性知识。
*   **核心问题**：如何在不依赖符号化领域模型（如 PDDL）或手动编码任务网络的情况下，让 LLM 智能体能够动态地学习、泛化并重用可执行的策略组件，从而实现高效的分层广义规划（Hierarchical Generalized Planning）。

#### 2. 方法论：HCL-GP 框架
HCL-GP（Hierarchical Component Learning for Generalized Policies）结合了广义规划（参数化策略）和分层任务分解的思想。其核心流程包含三个动态循环：
*   **策略生成（Policy Generation）**：
    *   **任务抽象**：智能体首先分析一组相关任务，提取共享的参数化签名（Signature）和高层步骤。
    *   **组件检索**：利用语义搜索（BAAI 嵌入模型）从已有的“组件库”中检索最相关的可重用代码片段。
    *   **迭代调试**：生成 Python 策略代码，在模拟器中执行，并根据反馈（错误信息、执行轨迹）进行自我修复。
*   **组件学习（Learning Components）**：
    *   **自动分解**：从成功的策略代码中，由“分解智能体”识别并提取出具有独立逻辑功能的代码块（如身份验证、特定数据过滤）。
    *   **验证**：确保分解出的组件在原任务中依然能正确运行。
*   **组件泛化（Generalizing Components）**：
    *   **聚类与去重**：对新学习的组件进行语义聚类，合并功能相似的组件。
    *   **抽象化**：将特定领域的组件（如 `login_to_spotify`）泛化为通用组件（如 `login_to_app`），通过参数化提高重用率。
