## 摘要
单智能体系统（SAS）已成为大语言模型（LLM）驱动的科学工作流的默认模式，但在单个上下文窗口中进行路由规划、工具使用和综合处理会带来众所周知的代价：随着工具规范和观察轨迹的累积，每个决策可用的有效上下文会缩小，端到端的可靠性也会受损。我们提出了一个用于流体力学的多智能体系统（MAS）原型，其中专门的智能体通过层级执行图（LEG）进行协调。规划智能体（planner agent）根据自然语言路由启发式方法构建特定于查询的执行拓扑，这些启发式方法捕获了领域知识，而无需将其硬编码为僵化的控制逻辑；专家智能体（specialist agents）在严格的工具白名单下运行，并承担互补的数据类别角色。在各层之间，整合智能体（consolidator agents）将并行输出融合为简洁的摘要，报告智能体（reporter agent）合成最终响应，同时运行时环境记录每次工具调用的溯源信息以支持可审计性。所有基准测试、消融实验和压力测试均使用 Claude Sonnet 4.6 作为专家智能体和通用智能体的骨干模型。在涵盖六个复杂度类别的 37 个查询评估中，该原型实现了 93.6% 的事实准确率和 100% 的通过率。从单线程到五个独立并行轨道的运行中，准确率保持在 90% 以上；在模拟单个数据源丢失的情况下，系统表现出优雅降级，仍能返回实质性的部分答案。总之，这些结果表明，由规划器引导的图结构多智能体编排可以有效缓解限制单体单智能体架构的上下文饱和瓶颈。

## Abstract
Single-agent systems (SAS) have become the default pattern for LLM-driven scientific workflows, but routing planning, tool use, and synthesis through a single context window comes with a well-known cost: as tool specifications and observational traces accumulate, the effective context available for each decision shrinks, and end-to-end reliability suffers. We present a multi-agent system (MAS) prototype for hydrodynamics in which specialized agents are coordinated through a Layer Execution Graph (LEG). A planner agent constructs query-specific execution topologies from natural-language routing heuristics that capture domain knowledge without hard-coding it as rigid control logic; specialist agents operate under strict tool allowlists and occupy complementary data-class roles. Between layers, consolidator agents fuse parallel outputs into concise briefs, and a reporter agent synthesizes the final response, while the runtime logs provenance for every tool invocation to support auditability. All benchmarks, ablations, and stress tests use Claude Sonnet~4.6 as the backbone model for both specialist and general-purpose agents. Evaluated on 37 queries spanning six complexity categories, the prototype achieves 93.6% factual precision with a 100% pass rate. Accuracy remains above 90% across runs from single-threaded to five independent parallel tracks, and under simulated loss of individual data sources the system degrades gracefully, still returning substantive partial answers. Together, these results suggest that planner-guided, graph-structured multi-agent orchestration can meaningfully alleviate the context-saturation bottlenecks that constrain monolithic single-agent architectures.

### 2. 核心方法论
*   **核心思想**：将复杂任务分解为由多个专门智能体协作的图结构流程，通过“规划-执行-整合-报告”的四阶段流水线实现。
*   **关键技术细节**：
    *   **层级执行图（LEG）**：一种同步的图结构，将工作分为多个层级。每一层可以包含多个并行运行的专家智能体，层与层之间通过“屏障”同步。
    *   **智能体角色分类**：
        *   **规划者（Graph Architect）**：利用自然语言启发式规则动态构建执行拓扑（单轨道或多轨道并行）。
        *   **专家智能体（Specialists）**：包括 NHC（飓风中心）、NOAA（水位）、USGS（高水位标记）、FEMA（洪水图）、STOFS（预报模型）和 OSM（地图）。
        *   **通用智能体**：整合者（Consolidator）负责压缩上下文；图像智能体（Image）负责解释等值线图；报告者（Reporter）合成最终答案。
    *   **路由启发式（Routing Heuristics）**：将领域知识（如“查询观测数据前先获取风暴路径”）以自然语言形式存入规划者，而非硬编码逻辑。
    *   **溯源审计**：运行时记录所有工具调用的参数、哈希值和 URL，确保结果可追溯。

### 3. 实验设计
*   **数据集/场景**：涵盖了历史飓风（如 Katrina, Ike, Sandy, Ian, Helene 等）的真实观测数据、FEMA 监管数据和 STOFS 预报输出。

### 4. 资源与算力
*   **模型支持**：所有实验均使用 **Claude Sonnet 4.6** 作为骨干 LLM。
*   **算力说明**：论文未提及具体的 GPU 训练算力，因为该系统是基于 API 调用的推理框架。

### 5. 实验数量与充分性
*   **实验规模**：进行了 37 组端到端查询、7 组代表性消融实验、12 组改写测试、5 组对抗测试和 3 组故障测试。
*   **充分性评价**：实验设计较为全面，不仅关注准确率，还深入探讨了拓扑选择、智能体 F1 分数和来源引用（Source Attribution）等维度。压力测试部分客观地评估了系统在极端情况下的表现，具有较高的说服力。

### 6. 主要结论与发现
*   **高准确性**：系统实现了 93.6% 的综合得分，事实准确率达 93.2%，且 100% 完成了所有任务。
*   **动态规划的必要性**：消融实验显示，移除动态规划（固定图结构）会导致准确率下降 28.7%，证明了灵活编排是核心优势。
*   **优雅降级**：在模拟 API 故障时，系统不会崩溃，而是利用剩余数据源提供部分答案。
*   **上下文管理**：通过整合者（Consolidator）进行上下文压缩，虽然会略微损失技术细节（如站号），但显著提升了最终回答的连贯性和准确性。

### 7. 优点
*   **架构灵活性**：LEG 结构既保证了执行的确定性，又保留了 LLM 处理自然语言的灵活性。
*   **多模态集成**：通过图像智能体将高分辨率数值模型（STOFS）的输出转化为文本理解，解决了 LLM 难以直接处理大规模数值网格的问题。
*   **可解释性**：详细的溯源日志为科学应用提供了必要的审计追踪。

### 8. 不足与局限
*   **成本与延迟**：多智能体并行和多次整合导致 Token 消耗较高，且复杂查询的平均延迟接近 100 秒。
*   **缺乏自我修正机制**：目前系统缺乏“监督者”角色，如果上游专家智能体提供错误数据，下游智能体难以实时纠错（需等待最终人工审核）。
*   **应用范围限制**：目前主要侧重于数据检索和初步合成，尚未实现直接在智能体内部运行重型流体力学数值模拟（CFD）代码。
