## 摘要
大语言模型（LLMs）在长期个性化对话中缺乏持久性记忆。现有的基于图的记忆系统面临信息稀释、缺乏溯源追踪以及忽略查询上下文的统一检索等问题。我们提出了 MemORAI（基于自适应图智能的记忆组织与检索），该框架集成了三项创新：采用双层压缩的选择性记忆过滤以保留与用户画像相关的内容；通过富含溯源信息的多关系图在轮次级别追踪事实来源；以及利用动态加权 PageRank（Dynamic Weighted PageRank）进行查询自适应子图检索，该方法应用了基于查询条件的边权重计算。在 LOCOMO 和 LongMemEval 基准测试上的评估结果显示，MemORAI 在记忆检索和个性化响应生成方面达到了最先进的性能，证明了选择性存储、丰富的表示和自适应检索对于构建连贯、个性化的 LLM 智能体至关重要。

## Abstract
Large Language Models (LLMs) lack persistent memory for long-term personalized conversations. Existing graph-based memory systems suffer from information dilution, absent provenance tracking, and uniform retrieval that ignores query context. We introduce MemORAI (Memory Organization and Retrieval via Adaptive Graph Intelligence), a framework that integrates three innovations: selective memory filtering with dual-layer compression to retain user-persona-relevant content, a provenance-enriched multi-relational graph tracking factual origins at the turn level, and query-adaptive subgraph retrieval with Dynamic Weighted PageRank that applies query-conditioned edge weighting. Evaluated on LOCOMO and LongMemEval benchmarks, MemORAI achieves state-of-the-art performance in memory retrieval and personalized response generation, demonstrating that selective storage, enriched representation, and adaptive retrieval are essential for coherent, personalized LLM agents.

论文针对大语言模型（LLM）在长期个性化对话中缺乏持久记忆的问题展开研究。现有的记忆增强系统（如 RAG 或基于图的系统）存在三大缺陷：

*   **缺乏溯源**：无法准确追踪事实起源于哪一轮对话，导致回复的透明度和可审计性差。
*   **检索机制僵化**：如 HippoRAG 等系统在图上进行评分传播时采用统一权重，忽略了查询（Query）的具体上下文语义。

**MemORAI** 旨在通过自适应图智能技术，实现高效、可溯源且具备上下文感知能力的长期记忆管理。

### 2. 方法论：核心思想与关键技术
MemORAI 提出了一个三阶段的流水线：

*   **会话分割与选择性压缩（Selective Memory Filtering）**：
    *   利用 LLM 将对话按话题分割成段。
    *   **记忆门（Memory Gate）**：仅保留与用户画像（个人事实、偏好、承诺等）相关的细粒度内容。
    *   **双层压缩**：保留关键原始轮次的同时，生成段落级摘要作为全局上下文锚点，大幅减少存储开销并过滤噪声。
*   **溯源增强图构建（Provenance-Enriched Graph）**：
    *   构建异构图，包含三种节点：**实体（Entity）**、**轮次（Turn）**、**段落（Segment）**。
    *   显式记录**轮次级溯源**：每个实体和关系三元组都链接到具体的 `turn_id`，支持透明审计。
*   **查询自适应子图检索与动态加权 PageRank（DW-PR）**：
    *   **子图提取**：检索时先通过语义相似度识别种子节点，仅在局部子图上操作，降低延迟。
    *   **动态加权算法**：改进了传统的 PageRank。边的权重 $w(u \to v)$ 不再是固定的，而是根据查询 $q$ 与节点/关系描述之间的语义相似度动态计算。
    *   **公式逻辑**：$PR_{t+1}(v) = (1-d) \cdot seed(v) + d \cdot \sum \frac{w(u \to v)}{\sum w(u \to *)} PR_t(u)$。这确保了语义相关但连接数较少的节点也能获得高排名。

*   **精细的溯源机制**：通过 Turn 节点连接实体与原始文本，解决了 RAG 系统中常见的“事实幻觉”和“来源不明”问题。
*   **高效的子图策略**：通过局部子图检索平衡了图检索的深度与实时性需求。
