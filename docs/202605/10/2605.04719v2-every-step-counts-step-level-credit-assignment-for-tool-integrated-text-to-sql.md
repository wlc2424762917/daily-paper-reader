## 摘要
工具集成的 Text-to-SQL 解析已成为一种极具前景的范式，它将 SQL 生成构建为与工具执行交织的序列决策过程。然而，现有的强化学习方法主要依赖于粗粒度的结果监督，导致了根本性的信用分配问题：模型对于任何产生正确答案的轨迹都会获得相同的奖励，即使中间步骤是冗余、低效或错误的。因此，模型被鼓励去探索次优的推理空间，限制了效率和泛化能力。为了解决这一问题，我们提出了 FineStep，这是一个用于工具增强型 Text-to-SQL 中步骤级信用分配的新型框架。首先，我们引入了一种带有独立过程奖励的奖励设计，以缓解结果监督的信号稀疏性。其次，我们提出了一种步骤级信用分配机制，以精确量化每个推理步骤的价值。最后，我们开发了一种基于步骤级优势的策略优化方法，以实现高效更新。在 BIRD 基准测试上的广泛实验表明，FineStep 达到了最先进的性能并减少了冗余的工具交互，在 4B 规模下相比 GRPO 实现了 3.25% 的平均 EX 增益。

## Abstract
Tool-integrated Text-to-SQL parsing has emerged as a promising paradigm, framing SQL generation as a sequential decision-making process interleaved with tool execution. However, existing reinforcement learning approaches mainly rely on coarse-grained outcome supervision, resulting in a fundamental credit assignment problem: models receive the same reward for any trajectory that yields the correct answer, even when intermediate steps are redundant, inefficient, or erroneous. Consequently, models are encouraged to explore suboptimal reasoning spaces, limiting both efficiency and generalization. To address this problem, we propose FineStep, a novel framework for step-level credit assignment in tool-augmented Text-to-SQL. First, we introduce a reward design with independent process rewards to alleviate the signal sparsity of outcome supervision. Next, we present a step-level credit assignment mechanism to precisely quantify the value of each reasoning step. Finally, we develop a policy optimization method based on step-level advantages for efficient updates. Extensive experiments on BIRD benchmarks show that FineStep achieves state-of-the-art performance and reduces redundant tool interactions, with a 3.25% average EX gain over GRPO at the 4B scale.

### 1. 核心问题与整体含义
*   **研究背景**：当前的 Text-to-SQL 范式正从单步生成转向“模型-工具-反馈”的多轮交互推理（Agentic Reasoning）。
*   **核心问题**：现有的强化学习方法（如 GRPO）主要依赖**结果监督（Outcome Supervision）**。这意味着只要最终 SQL 正确，整个推理轨迹（包括其中的冗余步骤、错误尝试或无效工具调用）都会获得正向奖励。
*   **后果**：这种“黑盒”式的奖励机制导致模型学习到低效的“盲目试错”策略，而非真正的逻辑推理，限制了模型的泛化能力和执行效率。

### 2. 方法论：FineStep 框架
FineStep 的核心思想是将信用分配从“轨迹级”细化到“步骤级”，包含以下三个关键部分：

*   **多粒度奖励设计**：
    *   **结果奖励 ($R_{out}$)**：评估最终 SQL 的格式和执行正确性。
    *   **过程奖励 ($r_{proc}$)**：由**硬约束**（检查工具调用是否合法、是否重复）和**基于召回率的软奖励**（计算中间查询结果与最终正确答案的单元格级召回率）组成，用以量化每一步的信息增益。
*   **步骤级信用分配机制**：
    *   **结果奖励折扣（Discounting）**：引入折扣因子 $\gamma$，使越靠近结果的步骤获得更高的结果权重，体现时间异质性。
    *   **过程奖励平滑（Smoothing）**：通过反向平滑机制，将未来的潜力回溯到当前步骤，肯定那些具有“铺垫价值”的探索性操作。
    *   **步骤级优势估计**：在采样组内对所有有效步骤进行标准化处理，识别出“失败轨迹中的正确步骤”和“成功轨迹中的冗余步骤”。
*   **策略优化**：扩展 GRPO 算法，使用步骤级优势（Advantage）进行参数更新，并在反向传播时掩码（Mask）掉环境反馈 Token，确保模型只优化自身的推理逻辑。

### 7. 优点与亮点
*   **细粒度归因**：成功打破了 RL 训练中的“轨迹黑盒”，能够精确惩罚成功轨迹中的冗余步。
*   **创新的软奖励**：利用 SQL 执行结果的“子集/超集”特性设计召回率奖励，非常符合数据库查询的逻辑。
*   **即插即用**：该框架不依赖昂贵的人工过程标注（PRM），而是通过规则和执行反馈自动生成过程信号。

### 8. 不足与局限
*   **领域局限性**：目前的方法高度依赖 SQL 执行器的反馈，对于没有明确中间执行结果的通用推理任务（如常识推理），软奖励的设计需要重新构思。
*   **规则依赖**：过程奖励仍基于预设规则（如召回率），虽然解释性强，但可能无法捕捉到某些非常规但有效的推理路径。
*   **未来方向**：论文提到未来可引入学习型过程奖励模型（PRM）来进一步提升灵活性。
