## 摘要
大语言模型（LLMs）的最新进展使得在基于微服务的系统（MSS）中实现自动化根因分析（RCA）的初步尝试成为可能。然而，以往的工作通常依赖于沿单一诊断路径进行的线性推理过程。在本文中，我们提出了 LATS-RCA，这是一个基于 LLM 的用于 MSS 根因分析的多智能体框架。LATS-RCA 使用语言智能体树搜索（Language Agent Tree Search）算法，将 RCA 建模为一种由反思引导的树状结构搜索。在 LATS-RCA 中，多个由 LLM 驱动的智能体通过对每个微服务的执行日志和性能指标进行推理，迭代地执行 RCA，以收集用于根因探索的运行证据。源自中间诊断状态的反思评分被用于根据累积的证据引导搜索走向最可能的根因。我们在开源工业级 MSS Light-OAuth2 (LO2) 上使用公开数据集，并在一家运营复杂度显著更高的案例公司的生产微服务环境 (Prod) 中对 LATS-RCA 进行了评估。LO2 是一个采用同构技术栈的小型团队 Java 系统。在 LO2 上的结果表明，LATS-RCA 实现了较高的诊断准确率，我们还进一步基准测试了其相关的计算成本。与 LO2 相比，Prod 的诊断准确率较低且计算成本更高。Prod 的部署证明了 LATS-RCA 在现实世界 MSS 中的实际适用性，并反映了生产规模 MSS 中多语言技术栈、源组件不同的日志记录实践以及多因素根因所带来的挑战。

## Abstract
Recent advances in large language models (LLMs) have enabled early attempts to automate root cause analysis (RCA) in microservice-based systems (MSS). Yet, prior works typically rely on a linear reasoning process that proceeds along a single diagnostic path. In this paper, we propose LATS-RCA, an LLM-based multi-agent framework for RCA in MSS. LATS-RCA formulates RCA as a reflection-guided tree-structured search using a Language Agent Tree Search algorithm. In LATS-RCA, multiple LLM-driven agents iteratively perform RCA for each microservice by reasoning over its execution logs and performance metrics to collect operational evidence for root cause exploration. Reflection scores derived from intermediate diagnostic states are used to guide the search toward the most likely root cause based on accumulated evidence. We evaluate LATS-RCA on the open-source industrial MSS, Light-OAuth2 (LO2), using a publicly available dataset and in a production microservice environment (Prod) in a case company with substantially higher operational complexity. LO2 is a small-team Java system with a homogeneous technology stack. The results on LO2 show that LATS-RCA achieves high diagnostic accuracy, and we further benchmark its associated computational costs. Compared to LO2, Prod attains lower diagnostic accuracy and incurs higher computational cost. The Prod deployment demonstrates the practical applicability of LATS-RCA in real-world MSS and reflects the challenges introduced by polyglot tech stack, varied logging practices of source components, and multi-factor root-causes by production-scale MSS.

*   **研究动机**：在实际生产中，一个异常可能指向多个潜在根因。线性推理难以处理这种不确定性，容易陷入局部错误结论。因此，需要一种能够同时探索多个假设、并根据证据进行自我修正和反思的系统。

### 2. 方法论：LATS-RCA 框架
*   **核心思想**：将根因分析建模为一个**由反思引导的树状结构搜索过程**，采用语言智能体树搜索（Language Agent Tree Search, LATS）算法。
*   **关键技术细节**：
    *   **多智能体架构**：包含**日志智能体（Log Agent）**、**指标智能体（Metric Agent）**和**主管智能体（Supervisor）**。主管负责跨模态调度和最终结论汇总。
    *   **搜索流程（MCTS 风格）**：
        1.  **选择（Selection）**：使用 **UCT（置信区间上界）公式**平衡“开发”（选择高分路径）和“探索”（尝试未充分研究的假设）。
        2.  **扩展（Expansion）**：在当前节点生成 5 个候选动作（如查询特定日志或指标）。
        3.  **反思与奖励（Reflection & Reward）**：LLM 对动作进行评分，标准包括证据质量、诊断完整性和内部一致性。
        4.  **回传（Backpropagation）**：将奖励值沿路径回传，更新节点价值估计。
    *   **跨模态移交**：当某一智能体进展不足时，主管智能体通过摘要形式将当前发现移交给另一模态的智能体，避免循环推理。

### 6. 主要结论与发现
*   **性能优异**：在 LO2 基准测试中，LATS-RCA 准确率达到 **91.3%**，远超单智能体 ReAct (39.8%) 和多智能体 ReAct (57.4%)。
*   **生产环境挑战**：在真实生产环境 (Prod) 中，准确率下降至 **60-70%**。这反映了多语言栈、非标准日志和多因素根因带来的巨大挑战。
*   **成本与收益**：高准确率是以更高的计算成本（Token 消耗和时间）为代价的。LATS-RCA 的优势在于更系统地探索了假设空间，而非仅仅收集了更多数据。

### 7. 优点
*   **非线性推理**：引入树搜索机制，允许系统在发现路径不通时进行回溯和重新评估。
*   **多智能体协同**：有效地结合了日志和指标两种不同维度的观测数据。
*   **实战验证**：提供了 LLM 智能体在真实、大规模、多语言微服务生产环境中部署的宝贵经验和数据。

### 8. 不足与局限
*   **计算成本高昂**：相比线性方法，Token 消耗和诊断延迟显著增加，可能限制其在极高实时性要求的场景下的应用。
*   **环境依赖性**：在生产环境中，准确率受限于日志的规范化程度。如果系统日志极度非结构化，性能会受到影响。
*   **评估局限**：目前主要采用“精确匹配”作为准确率标准，但在生产环境中，故障往往是多因素的，二元对错的评估可能无法完全体现系统的辅助价值。
