## Abstract 

Automated red-teaming methods for large language models typically optimize attack prompts within a fixed, human-designed strategy, leaving the attack strategy itself unchanged. We instead optimize the strategy. We propose A UTO RISE, a method that searches over executable attack programs rather than individual prompts. At each iteration, a coding agent edits a strategy and a fixed evaluation harness scores the resulting attacks, returning both a scalar objective and per-example diagnostics that guide subsequent edits. This allows structural changes, including new attack components and altered control flow, that prompt-level meth-ods do not directly express. We also release two benchmark suites developed on disjoint target sets and evaluate on 11 models from five families against seven established jailbreak datasets. Across held-out models, A UTO RISE improves average attack success rate by 17.0 points over the strongest baseline, and improves attack success by up to 16 points on frontier targets with low baseline success rates. Ablations against parametric and strategy-library baselines suggest that these gains arise from unrestricted program search, particularly compositional techniques and control-flow edits. A UTO RISE operates in a black-box, inference-only setting, requiring no fine-tuning, human annotation, or GPU compute. 

## 1 Introduction 

Large language models (LLMs) are now widely deployed in high-stakes applications [OpenAI, 2023, Anthropic, 2024, Google DeepMind, 2024], yet despite extensive safety training they remain susceptible to jailbreak attacks : carefully crafted prompts that bypass safety guardrails and elicit harmful content [Zou et al., 2023, Wei et al., 2023]. Jailbreak techniques exploit the tension between helpfulness and safety through diverse surface forms including persona manipulation, encoding tricks, hypothetical framing, and prompt injection [Shen et al., 2023, Liu et al., 2024d]. 

Red-teaming [Ganguli et al., 2022, Perez et al., 2022] has become a core safety practice, but manual red-teaming is expensive and cannot keep pace with modern deployment cycles. Automated red-teaming addresses this by using LLMs to generate and iterate on attacks [Perez et al., 2022, Mehrotra et al., 2024, Chao et al., 2024b], yet existing methods operate at the prompt level : they refine individual attack prompts within a fixed strategy [Chao et al., 2024b, Mehrotra et al., 2024] or fine-tune attacker models [Ge et al., 2024, Author, 2025], without modifying the attack methodology itself. This leaves an open question: can the research loop of forming hypotheses about why certain attacks succeed, designing new strategies, and validating them be automated end-to-end? In this work, we answer this question in the affirmative. We present A UTO RISE ( Auto nomous 

Red-teaming via Iterative Strategy Evolution), an automated red-teaming method that elevates the search space from individual prompts to attack strategy code . An AI coding agent edits a single

## 2 Related Work 

2.1 Jailbreak Attacks and Automated Red-Teaming 

Jailbreak attacks exploit the tension between helpfulness and safety refusal in LLM training [Wei et al., 2023, Ganguli et al., 2022, Perez et al., 2022]. Reported attack families include persona overrides [Shen et al., 2023], gradient-based suffix optimization [Zou et al., 2023], genetic-algorithm prompt evolution [Liu et al., 2024c], encoding tricks [Jiang et al., 2024a], persuasion framing [Zeng et al., 2024], iterative dialogue refinement [Chao et al., 2024b, Mehrotra et al., 2024], and multi-turn escalation [Russinovich et al., 2024, Yu et al., 2024]. Defenses span training-time approaches (RLHF [Ouyang et al., 2022], Constitutional AI [Bai et al., 2022], adversarial training [Mazeika et al.,

22024]) and test-time filtering [Robey et al., 2023, Jain et al., 2023, Inan et al., 2023], but no defense has proven robust to adaptive adversaries [Wei et al., 2023]. Static safety benchmarks [Zou et al., 2023, Mazeika et al., 2024, Chao et al., 2024a] face saturation and contamination concerns [Beyer et al., 2025, Eiras et al., 2025, Hollinsworth et al., 2025]; JBDISTILL [Zhang et al., 2025b] addresses this by selecting high-quality subsets from existing attacks. Automated red-teaming methods can be organized by the search space they optimize over. Token-level 

methods such as GCG [Zou et al., 2023] and AutoDAN [Liu et al., 2024c] search over adversarial suf-fixes using gradients or genetic algorithms. Prompt-level methods such as PAIR [Chao et al., 2024b] and TAP [Mehrotra et al., 2024] use an attacker LLM to iteratively refine natural-language prompts; WildTeaming [Jiang et al., 2024b] mines in-the-wild jailbreak tactics; Rainbow Teaming [Samvelyan et al., 2024] casts the problem as quality-diversity search via MAP-Elites. Strategy-description 

methods such as AutoDAN-Turbo [Liu et al., 2024b] and AutoRedTeamer [Zhou et al., 2025] ac-cumulate a library of natural-language attack strategies that future attempts draw from. In all cases, the generation pipeline (how strategies are instantiated into prompts, how targets are selected, how techniques are composed) is fixed code authored once by the researchers. AUTO RISE moves one level higher: the search space is executable strategy code . An AI coding agent edits the attack pipeline program, enabling parametric, template, and structural changes (new generation stages, target-adaptive routing, novel composition pipelines) that no fixed-dimensional parameterization can express. A UTO RISE additionally optimizes a multi-objective composite score (ASR, diversity, novelty, coverage) rather than per-prompt ASR on a single target. 

2.2 Agent-Driven Autonomous Research 

AUTO RISE adapts the AutoResearch paradigm [Karpathy, 2026], in which a coding agent edits code, runs experiments, and accepts or reverts changes based on a measured score. Concurrent work applies similar loops to general scientific discovery [Lu et al., 2024, 2025, Baek et al., 2025]. These systems share the hypothesis-experiment-revise loop but operate in general ML domains rather than adversarial red-teaming. 

## 3 Methodology 

We frame A UTO RISE as an optimization problem over attack strategy programs. An AI coding agent edits a single Python file ( strategy.py ) encoding the entire jailbreak generation pipeline, and a fixed evaluation harness executes the strategy against target LLMs and a three-judge ensemble, returning a composite score together with rich per-sample diagnostics. A cycle is a single edit-and-evaluate step: the agent rewrites part of the strategy, the harness runs it for a fixed wall-clock budget, and the edit is kept or reverted based on the score. The search space is the set of valid programs expressible in strategy.py , which is unbounded and admits parametric, template, and structural changes; the feedback channel is rich (per-technique rates, per-target breakdowns, judge rationales) rather than a bare scalar. 

3.1 Architecture and Search Space 

The system comprises three read-only components that fix the scoring function across cycles: the 

evaluation harness (prepare.py ), which owns the target LLM pool, judge ensemble, embedding client (used to compute the diversity and novelty components of the composite score; Section 3.4), and scoring function; the agent specification (program.md ), a natural-language description of the agent’s autonomy boundaries and per-cycle workflow (full text in Section A.1); and the target and judge endpoints , accessed indirectly through the harness. The mutable component is a single file, the 

strategy program (strategy.py ), containing prompt templates, generation parameters, technique mixing ratios, category targeting logic, post-filtering, and the experiment loop. This is the only file the agent edits and constitutes the entire search space. The agent’s edit surface is layered from parametric tweaks (mixing weights, temperatures, batch sizes) through template rewrites (prompt and system strings) to structural changes (new techniques, multi-stage pipelines, target-adaptive routing, programmatic prompt builders); Section B.2 expands these three levels and relates them to prior prompt-level methods. 33.2 Target Pool and Jailbreak Generation 

AUTO RISE attacks a configurable pool of target LLM deployments simultaneously via chat comple-tion APIs. During each cycle the strategy selects a target uniformly at random for each attempt, unless the agent has installed a custom routing rule. The seed strategy generates prompts with a generator model (a capable LLM distinct from the targets) using a structured template system: a system prompt establishing a red-team research context, plus a technique-specific user prompt parameterized by the target harm category. The seed portfolio contains nine attack techniques with configurable mixing weights (Section B.1), targeting eight harm categories. Generation proceeds in micro-batches with concurrent API calls, and a post-filter discards degenerate outputs (too short, too long, or beginning with refusal prefixes). 

3.3 Three-Judge Evaluation Ensemble 

The evaluation is anchored by an ensemble of three independent LLM-based judges from distinct model families: GPT-5.2 (OpenAI), Grok-3 (xAI), and DeepSeek-V3.2 (DeepSeek). Each judge scores a (jailbreak_prompt, target_response) pair with the same structured rubric, out-putting a binary harmful-compliance verdict, a severity score s ∈ { 0, . . . , 5}, a confidence estimate, and a brief rationale. A jailbreak is counted as successful only when a majority of the three judges agree ( ≥ 2/3). The rubric is inspired by StrongREJECT [Souly et al., 2024] but redesigned to focus on response content rather than proxy dimensions such as convincingness and specificity (Section 4.1 validates it against human annotations; Section C reproduces the full text). Cross-family judging reduces correlated errors from shared training data and mitigates the risk of embedding any one provider’s content policy into the scoring signal. 

3.4 Composite Score 

The scalar score returned to the agent at the end of each cycle is 

S = 0 .60 · rsuccess + 0 .10 · ddiversity + 0 .10 · nnovelty + 0 .10 · ccategory + 0 .10 · ctarget , (1) where rsuccess is the fraction of attempts judged successful by majority vote; ddiversity is the mean pairwise cosine dissimilarity of jailbreak-prompt embeddings; nnovelty is the fraction of current-cycle prompts whose maximum cosine similarity to any prompt in the most recent three cycles is below 

τ = 0 .85 ; ccategory is the fraction of harm categories with at least three successful jailbreaks (requiring depth, not just breadth); and ctarget is the fraction of target deployments with at least one success. The multi-objective formulation incentivizes diverse, novel attacks across categories and targets rather than overfitting to a single technique. S enters the accept/revert decision, but the harness also exposes a structured diagnostic channel (per-technique success rates, per-target breakdowns, per-judge verdicts, and sample responses) that the agent uses for hypothesis formation before committing the next action. 

3.5 Hypothesis-Driven Strategy Evolution 

Algorithm 1 formalizes the loop, adapted from the AutoResearch paradigm [Karpathy, 2026]. Each cycle runs for a fixed wall-clock budget B to ensure comparability regardless of strategy complexity. The agent (Claude Code) inspects the previous cycle’s diagnostics, retrieves relevant prior observa-tions from a persistent scratchpad, conjectures a causal explanation for the score movement, and proposes a targeted edit that tests its hypothesis.

4.1 Evaluator Validation 

Strategy evolution is driven entirely by the composite reward, so the credibility of every downstream result depends on the judge ensemble. We validate it against 1,100 human-labeled prompt–response pairs from three established benchmarks: JailbreakBench ( N =100 ) [Chao et al., 2024a], WildGuard-Mix ( N =500 ) [Han et al., 2024], and JailJudge ( N =500 ) [Liu et al., 2024a]. Our initial implementation applies the StrongREJECT rubric [Souly et al., 2024] verbatim. It achieves high recall but excessive false positives because it conflates topical engagement with harmful compliance (Table 1). We replace it with a response-harm rubric that asks whether the response provides actionable wrongdoing instructions, materially enables harm, or otherwise facilitates unsafe behavior (full text in Section C). The response-harm rubric raises mean F1 from 74 .5 to 88 .2 with a large gain in precision ( 66 .5% →

91 .3% ), while recall stays high ( 82 .8–94 .5% ). As we want to ensure high confidence in our judge’s jailbreak prediction, all subsequent experiments use the proposed response-harm rubric. 

4.2 Search-Space Ablation 

We test the central claim, namely that searching the program space discovers attacks prompt- or parameter-level search cannot, by running A UTO RISE in three modes that differ only in the agent’s action space. All three arms run for 30 episodes against GPT-4.1 from the same 9-technique seed strategy (DAN-style, roleplay persona, hypothetical framing, gradual escalation, context camouflage, prompt injection, encoding tricks, few-shot poisoning, token smuggling), with a DeepSeek-V3.2/Grok-4 generator pool. 5AR-Parametric. The agent may only edit numeric parameters of the seed strategy: per-technique mixing weights, temperature, batch size, per-category sampling weights, and judge thresholds. Prompt templates, helper functions, and control flow are frozen. The action space is a fixed-dimensional vector of seed-technique knobs. The standing instructions given to the agent, including the exhaustive list of allowed edits and a post-hoc diff verifier that checks the constraint after every cycle, appear verbatim in Section A.4. 

AR-Library. The agent may add new independent strategy entries to the technique library and tune all parameters (weights, temperature, batch size, system prompt, category descriptions), but may not modify the generation pipeline, control flow, or routing logic. Each new technique must be a natural-language template that the generator LLM interprets, not a programmatic prompt builder. This mirrors strategy-library methods such as AutoDAN-Turbo that accumulate independent strategy descriptions but leave the dispatch and generation pipeline fixed. The instructions appear in Section A.3. 

AR-Full (unrestricted). The agent has unrestricted edit access to strategy.py : parametric edits plus prompt-template rewrites, new helper functions, new programmatic prompt builders, target- or category-adaptive routing, multi-stage pipelines, and any other structural change that compiles. This is the default A UTO RISE mode and exposes the full program space rather than a fixed parameterization or a prompt-template slot. The full instructions for this arm appear in Section A.2. 

What the agents did. The AR-Library agent added seven new template-string techniques and rewrote the system prompt with target-specific guidance (its single largest improvement), but each new template gave only 1–2 cycles of novelty recovery before embedding saturation recurred; diversity plateaued at ∼0.40 regardless of library size. The AR-Full agent invented eight pro-grammatic prompt builders that fuse two or more seed techniques into structurally novel sur-face forms (Table 5): academic_peer_review , incident_debrief , regulatory_inquiry ,

classified_cable , hybrid_compound (a cross-fusion meta-technique), sworn_testimony ,

procurement_rfp , and training_manual . It also installed three structural control-flow changes neither narrower arm can express: category-adaptive routing, weighted category sampling, and per-category framing pools with 42 institutional cover framings. Section D.2 presents the highest-scoring jailbreak example for each of the eight builders, illustrating the structural novelty of the agent’s inventions.

4.4 Cost Analysis 

AUTO RISE runs entirely on inference-only API calls, with no GPU rental, no fine-tuning, and no human annotation. The two runs together cost $115.85 in API spend ($41.07 for Standard, $74.78 for Frontier) over ∼ 15 .4 hours, with judge inference dominating both runs ( 54 –72% of spend) since every sample is scored by three judges on the full prompt+response context; generator and embedding costs are negligible. Full per-component cost breakdowns appear in Table 6. Figure 2 shows that AR-Standard yields near-linear jailbreak accumulation from the start, while AR-Frontier exhibits an extended exploration phase before cipher and many-shot attacks come online. The marginal cost of evaluating an additional held-out target is $1 –$3 per model at frontier-model rates. 

Comparison to prior cost regimes. Training-based methods [Ge et al., 2023, Liu et al., 2024b] report hundreds to thousands of GPU-hours; per-target optimization (PAIR, AutoDan-Turbo) avoids 8training but pays fresh inference per model. A UTO RISE pays discovery once ($116) and amortizes across arbitrary held-out targets, two orders of magnitude cheaper end-to-end. 

## 5 Discussion 

Implications for LLM safety. Our results demonstrate that an autonomous agent with unrestricted access to the program space discovers attack strategies that transfer across five model families at a cost under 120 . The monotonic ordering in our ablation (parametric < library < program-space) shows that each expansion of the attacker’s action space unlocks qualitatively new attacks. This implies that defenders should model adversaries who can modify their attack methodology, not just their prompts. More broadly, we see agentic red-teaming as the natural next step for adversarial evaluation: an agent that forms hypotheses, writes code, and iterates on failures captures a threat model closer to a real-world adversary than any fixed prompt optimizer can. 

Limitations. Our judge-ensemble, while validated against existing jailbreak baselines, uses closed-source models whose behavior may shift across API versions. The composite score weights are set a priori; adaptive or preference-based objectives may yield different exploration dynamics. Our ablation runs on a single dev target (GPT-4.1) and extending it to frontier targets would clarify whether the program-space advantage grows or shrinks with defender strength. 

Ethical considerations and responsible disclosure. AUTO RISE produces attack content for defensive use. We release benchmarks under a safety-research license, share frontier-target findings with affected providers prior to publication following established disclosure practices [Ganguli et al., 2022, Mazeika et al., 2024], and release the evaluation harness and scrapbooks for reproducibility. 

## 6 Conclusion 

We presented A UTO RISE, an automated red-teaming method that lifts the search space from individ-ual prompts to executable attack-strategy code. By casting an AI coding agent as the policy and a fixed evaluation harness as the environment, A UTO RISE enables hypothesis-driven optimization over attack methodologies. The two released benchmarks improve on the strongest baseline by 17 .0 points of held-out ASR on standard targets and 16 .2/10 .8 points on frontier targets, for $116 in API spend with no model training. Our three-arm ablation establishes that this gain is causal to the action space: the monotonic ordering parametric < library < program-space shows that each expansion of the action space unlocks qualitatively new moves that the narrower spaces cannot express. These results suggest that the next unit of automation in adversarial evaluation is not a better prompt optimizer but a better research agent . Natural extensions include giving the agent access to GPU compute so it can explore attack methods beyond API-based inference, such as reinforcement learning or gradient-based optimization, and symmetric defense-side evolution for co-evolutionary red-teaming.

B.2 Action-Space Layers 

The agent’s edit surface decomposes into three levels of abstraction, in increasing structural depth: • Parametric actions : adjusting technique mixing weights, temperature, batch sizes, or thresholds. These are the analogue of hyperparameter tuning and correspond to the AR-Parametric ablation arm. • Strategy-library actions : adding new natural-language technique entries to the strategy portfolio, rewriting prompt templates, and tuning system prompts. Structural actions modify the execution graph of the attack pipeline. Empirically, almost all of the gains in our runs come from structural actions discovered late in training (e.g., the bijection-cipher and many-shot context-stuffing attacks introduced during A UTO RISE-Frontier evolution), not from parametric or strategy-library tweaks. The matched ablation in Section 4.2 isolates this distinction by comparing three action-space regimes: one arm restricted to parametric actions only (AR-Parametric), one arm that additionally permits adding new strategy-library entries (AR-Library), and an unrestricted arm with access to all three levels (AR-Full). 

B.3 Judge Ensemble: Design Rationale 

Using multiple independent judges from different model families provides robustness against system-atic biases in any one model’s safety training. A false positive would require at least two models from different families to independently misclassify a safe response as harmful, which is unlikely when the models do not share training or alignment procedures. Conversely, a false negative would require at least two judges to miss genuinely harmful content. The three-judge majority vote thus provides a conservative, high-precision jailbreak signal suitable for driving autonomous strategy evolution. Cross-family judging also mitigates a subtler problem: individual models encode the safety policies 20 of their respective providers, and these policies differ in where they draw the line between harmful and acceptable content. A single-model judge would bias the evaluation toward one provider’s policy, producing verdicts that reflect that company’s content standards rather than a generalizable notion of harmful compliance. By averaging over judges from different providers via majority vote, the ensemble produces a verdict that is robust to idiosyncratic policy differences: a response is flagged only when multiple independent policy perspectives agree it constitutes harmful compliance.

ASR stabilized at 10 –17% overall (combining both targets), and the novelty score 34 rose from ∼0.30 to 0.50 –0.70 . Sexual content was finally cracked at C136 via cipher, completing all eight harm categories. The 200 -prompt benchmark slice was reached at C138. 

Summary observations. The agent’s notes record three meta-lessons that the structural inventions in A UTO RISE-Frontier required, none of which any seed-technique tuning would have surfaced: (i) input-side safety training does not generalize to in-context-learned ciphers, so attacks that move the harmful content out of the model’s input distribution succeed where in-distribution attacks fail; (ii) o4-mini’s reasoning engine identifies cipher prompts as attacks but is fooled by many-shot context-stuffing that mimics a benign continuation task, indicating that the two frontier targets have qualitatively different defensive mechanisms; (iii) target-specific routing is itself an action the agent only takes once it has accumulated enough per-target diagnostic evidence, illustrating the value of the rich diagnostic channel beyond a scalar score. 

## E Benchmark Implementation Details 

Two of the benchmarks in Table 2 required custom implementation rather than direct usage: TeleAI-Safety and AutoDAN-Turbo-R. This section documents the configuration choices and construction details for each. 

E.1 TeleAI-Safety 

TeleAI-Safety [Chen et al., 2025a] is a comprehensive jailbreaking benchmark that provides attack prompts generated by multiple methods. We construct our evaluation subset using two of its attack methods: • ReNeLLM [Ding et al., 2024] (250 prompts): An automatic framework that uses LLMs themselves to generate jailbreak prompts through prompt rewriting and scenario nesting. We use the pre-generated attack prompts from the TeleAI-Safety repository, produced with Grok 3 as the attack model. • TAP [Mehrotra et al., 2024] (104 prompts): Tree of Attacks with Pruning, an iterative black-box method that uses a tree-search procedure to refine jailbreak prompts. These prompts were generated with Grok 3 as the attacker against GPT-5.2, with tree depth = 3 .All 250 unique harmful queries from TeleAI-Safety are covered: ReNeLLM provides one attack prompt per query, while TAP covers a 104 -query subset. The resulting 354 pre-transformed prompts are sent directly to each target model AR-is, with no further optimization, consistent with how all static benchmarks are evaluated in Table 2. 

E.2 AutoDAN-Turbo-R 

AutoDAN-Turbo-R [Liu et al., 2024c,b] is a per-target prompt optimizer that iteratively refines jailbreak prompts using an attacker LLM, a scorer, and a retrieval-augmented strategy library. Unlike the static benchmarks above, AutoDAN optimizes prompts individually for each target. 

Setup. We run AutoDAN-Turbo-R with a pre-trained strategy library (from the official reposi-tory) against the JailbreakBench prompt set ( 100 prompts). The attacker, scorer, and summarizer all use DeepSeek-V3.2; embeddings use text-embedding-3-small . Each prompt undergoes 2

optimization epochs: epoch 0 generates an initial jailbreak prompt, sends it to the target, and scores the response; epoch 1 retrieves relevant strategies from the library, generates a refined prompt, and scores again. In total, each prompt incurs 6 attacker/scorer calls (DeepSeek-V3.2), 3 target calls, and 

1 embedding call. 

Epoch budget. The original AutoDAN-Turbo paper uses 150 epochs per prompt with early stopping at a scorer threshold of 8.5. Running 150 epochs is prohibitively expensive for a multi-target evaluation: at ∼5 h per 100 -prompt target run with 2 epochs, the full 150 -epoch setting would require 

∼75 × more wall-clock time per target. We set epochs = 2 as a practical low-budget configuration that still exercises the full pipeline (retrieval, strategy-guided generation, scoring) while keeping total cost on the same order as the other benchmarks. We evaluate the resulting prompts with the 35 same three-judge majority-vote ensemble used throughout this paper, so ASR numbers in Table 2 are directly comparable across all columns.
