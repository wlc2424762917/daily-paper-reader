## 摘要
大语言模型（LLM）智能体可以自动化数据科学工作流，但由于 LLM 在统计知识和工具检索方面存在困难，许多在 R 语言中实现的严谨统计方法仍未得到充分利用。现有的检索增强方法侧重于函数级语义而忽略了数据分布，导致匹配效果不佳。我们提出了 DARE（分布感知检索嵌入），这是一种轻量级、即插即用的检索模型，它将数据分布信息整合到函数表示中，用于 R 包检索。我们的主要贡献包括：(i) RPKB，一个源自 8,191 个高质量 CRAN 软件包的精选 R 包知识库；(ii) DARE，一种融合了分布特征与函数元数据的嵌入模型，旨在提高检索相关性；(iii) RCodingAgent，一个面向 R 的 LLM 智能体，用于可靠的 R 代码生成，以及一套用于在现实分析场景中系统评估 LLM 智能体的统计分析任务。实验表明，DARE 的 NDCG@10 达到了 93.47%，在 R 包检索方面优于最先进的开源嵌入模型高达 17%，且使用的参数量显著更少。将 DARE 集成到 RCodingAgent 中，在下游分析任务上取得了显著提升。这项工作有助于缩小 LLM 自动化与成熟的 R 统计生态系统之间的差距。

## Abstract
Large Language Model (LLM) agents can automate data-science workflows, but many rigorous statistical methods implemented in R remain underused because LLMs struggle with statistical knowledge and tool retrieval. Existing retrieval-augmented approaches focus on function-level semantics and ignore data distribution, producing suboptimal matches. We propose DARE (Distribution-Aware Retrieval Embedding), a lightweight, plug-and-play retrieval model that incorporates data distribution information into function representations for R package retrieval. Our main contributions are: (i) RPKB, a curated R Package Knowledge Base derived from 8,191 high-quality CRAN packages; (ii) DARE, an embedding model that fuses distributional features with function metadata to improve retrieval relevance; and (iii) RCodingAgent, an R-oriented LLM agent for reliable R code generation and a suite of statistical analysis tasks for systematically evaluating LLM agents in realistic analytical scenarios. Empirically, DARE achieves an NDCG at 10 of 93.47%, outperforming state-of-the-art open-source embedding models by up to 17% on package retrieval while using substantially fewer parameters. Integrating DARE into RCodingAgent yields significant gains on downstream analysis tasks. This work helps narrow the gap between LLM automation and the mature R statistical ecosystem.

尽管大语言模型（LLM）智能体在自动化数据科学领域取得了进展，但它们主要集中在 Python 生态。相比之下，拥有数十年统计学积淀的 **R 语言生态（如 CRAN）** 未被充分利用。
*   **研究动机**：LLM 在处理 R 语言时存在系统性缺陷：(1) 倾向于使用 Python 替代方案，即使 R 的统计方法更严谨；(2) 容易产生幻觉（虚构函数名或参数）。
*   **核心挑战**：现有的检索增强生成（RAG）主要依赖语义相似度，而统计方法的适用性高度依赖于**数据分布特征**（如稀疏性、维度、分布假设）。通用嵌入模型往往无法捕捉这些细微但关键的统计约束，导致工具检索错误。

*   **RPKB (R Package Knowledge Base)**：从 8,191 个高质量 CRAN 包中提取并清洗出的结构化知识库，包含函数描述、用法、参数和返回值。
*   **DARE (Distribution-Aware Retrieval Embedding)**：
    *   **核心思想**：在检索阶段显式引入“数据概况（Data Profile）”。

    *   **训练目标**：使用 InfoNCE 损失函数进行对比学习，使模型学习在语义相似但统计约束不同的函数间进行区分（例如区分处理正态分布和泊松分布的函数）。
*   **RCodingAgent**：一个专门设计的 R 语言智能体，通过 DARE 检索工具，并进行迭代推理、代码生成和执行验证。

#### 6. 主要结论与发现
*   **检索性能突破**：DARE 在 NDCG@10 上达到 **93.47%**，比参数量大得多的最强基准模型（Snowflake Arctic）高出 **17.8%**。

#### 7. 优点
*   **轻量且高效**：用极小的参数量击败了巨型通用模型，体现了领域对齐（Domain Alignment）的威力。
*   **即插即用**：作为检索模块，可以无缝集成到任何现有的 LLM 智能体框架中。
*   **填补空白**：首次系统性地解决了 LLM 在 R 统计生态中的工具调用难题，并贡献了高质量的 RPKB 知识库。

#### 8. 不足与局限
*   **合成数据依赖**：训练查询主要由 LLM 合成，可能与真实用户的自然语言分布存在一定偏差。
*   **任务覆盖面**：虽然 16 个下游任务具有代表性，但相对于 R 语言数万个包的规模，评估范围仍有扩展空间。
*   **R 语言原生能力限制**：如果 LLM 本身对 R 语法极度陌生，即使检索到了正确的工具，仍可能在复杂逻辑编写上失败（尽管 DARE 缓解了这一问题）。
