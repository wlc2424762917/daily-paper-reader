## 摘要
大语言模型（LLMs）正成为能够利用工具解决复杂任务的自主智能体的基础。强化学习（RL）已成为注入此类智能体能力的常用方法，但通常处于严格受控的训练设置下。它往往依赖于精心构建的任务-解决方案对和大量的人工监督，这为通往超智能系统的开放式自进化制造了根本性障碍。在本文中，我们提出了 Tool-R0 框架，在零数据假设下，通过自我博弈强化学习（self-play RL）从零开始训练通用工具调用智能体。Tool-R0 从同一个基础 LLM 初始化，通过互补奖励共同进化生成器（Generator）和求解器（Solver）：前者在后者的能力边界提出针对性的挑战性任务，而后者则学习通过真实的工具调用来解决这些任务。这创造了一个无需预先存在任务或数据集的自进化循环。在不同工具使用基准测试上的评估表明，Tool-R0 相比基础模型实现了 92.5% 的相对提升，并在相同设置下超越了全监督工具调用基准。我们的工作通过分析协同进化、课程动态和扩展行为，进一步为自我博弈 LLM 智能体提供了实证见解。

## Abstract
Large language models (LLMs) are becoming the foundation for autonomous agents that can use tools to solve complex tasks. Reinforcement learning (RL) has emerged as a common approach for injecting such agentic capabilities, but typically under tightly controlled training setups. It often depends on carefully constructed task-solution pairs and substantial human supervision, which creates a fundamental obstacle to open-ended self-evolution toward superintelligent systems. In this paper, we propose Tool-R0 framework for training general purpose tool-calling agents from scratch with self-play RL, under a zero-data assumption. Initialized from the same base LLM, Tool-R0 co-evolves a Generator and a Solver with complementary rewards: one proposes targeted challenging tasks at the other's competence frontier and the other learns to solve them with real-world tool calls. This creates a self-evolving cycle that requires no pre-existing tasks or datasets. Evaluation on different tool-use benchmarks show that Tool-R0 yields 92.5 relative improvement over the base model and surpasses fully supervised tool-calling baselines under the same setting. Our work further provides empirical insights into self-play LLM agents by analyzing co-evolution, curriculum dynamics, and scaling behavior.

### 2. 方法论：Tool-R0 框架
Tool-R0 的核心思想是让同一个基础模型的两个副本承担不同角色：**生成器（Generator）**和**求解器（Solver）**，通过互补的奖励机制实现协同进化。

*   **协同进化循环**：
    1.  **生成器训练**：生成器学习合成可验证的工具使用任务（包括用户请求、工具菜单和标准答案）。
    2.  **数据集构建**：利用训练好的生成器合成大量任务，通过去重、交叉验证和难度排序构建课程数据集。
    3.  **求解器训练**：求解器在合成数据集上进行强化学习，提升工具调用准确性。
    4.  **迭代**：求解器的进步会反馈给生成器，促使其生成更具挑战性的任务。

*   **关键技术细节**：
    *   **GRPO 算法**：采用群体相对策略优化（Group Relative Policy Optimization）进行训练。
    *   **难度引导奖励（$r_{curr}$）**：这是核心创新。生成器不仅要生成正确的任务，还要生成处于求解器“能力边界”的任务。通过蒙特卡洛采样估计求解器的成功率 $\hat{p}_{succ}$，若成功率在 $[0.25, 0.75]$ 之间则给予高奖励，过难或过易则奖励衰减（高斯衰减）。
    *   **接地任务规范（Grounded Task Specification）**：为了防止生成器模式崩溃，通过动态采样领域（如金融、医疗等）和约束条件（工具数量、调用步数）来引导生成。

### 6. 主要结论与发现
*   **零数据胜过人工数据**：Tool-R0 在 1.5B 模型上实现了 92.5% 的相对提升，且性能超越了使用 21 万条人工数据的监督模型。
*   **角色分离至关重要**：生成器和求解器必须使用独立参数，否则会因为奖励目标冲突（探索 vs. 执行）导致灾难性遗忘。
*   **自适应课程优势**：模型生成的课程比静态的人工数据集更均衡、覆盖面更广，能精准针对模型自身的能力短板。
*   **作为“中段训练”的潜力**：将 Tool-R0 作为预训练后的中间阶段，能显著增强后续监督微调的效果。

### 7. 优点与亮点
*   **完全脱离人工数据**：实现了从零开始的工具学习，验证了 LLM 智能体自我进化的可行性。
*   **难度感知机制**：通过带通滤波器（Band-pass）形式的奖励函数，巧妙地实现了自动课程学习。
*   **通用性强**：不局限于特定工具（如 Python 代码），而是面向通用的 JSON API 调用。

### 8. 不足与局限
*   **小模型饱和问题**：在 0.5B 等极小模型上，自博弈在 3 次迭代后会出现性能饱和甚至轻微下降，显示出模型容量对自我进化的限制。
*   **计算开销**：为了估计任务难度，生成器训练期间需要对求解器进行多次采样，这增加了训练时的计算成本。
*   **奖励黑客风险**：虽然引入了语义一致性检查，但在极少数情况下，模型仍可能通过生成看似正确但逻辑有误的任务来“骗取”奖励。
