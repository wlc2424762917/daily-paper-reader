## 摘要
具有先进认知能力的大语言模型（LLMs）正逐渐成为处理各种推理和规划任务的智能体。传统的评估通常侧重于受控环境下的特定推理或规划问题。最近的研究探索了将旅行规划作为一种媒介，将各种语言推理任务整合到现实世界的情境中。然而，推理任务不仅限于语言推理，对 LLMs 的全面评估需要一个包含多个认知领域任务的测试平台。为了填补这一空白，我们推出了 ItinBench，这是一个在保留传统语言推理任务的同时，将空间推理任务（即路线优化）引入行程规划的基准。ItinBench 同时评估了多种 LLMs 在不同任务中的表现，包括 Llama 3.1 8B、Mistral Large、Gemini 1.5 Pro 以及 GPT 系列。我们的研究结果表明，当同时处理多个认知维度时，LLMs 难以保持高水平且一致的性能。通过整合来自不同人类水平认知领域的任务，ItinBench 为构建更全面、更能反映现实挑战的推理测试平台提供了新的见解。代码和数据集：https://ethanwtl.github.io/IBweb/

## Abstract
Large language models (LLMs) with advanced cognitive capabilities are emerging as agents for various reasoning and planning tasks. Traditional evaluations often focus on specific reasoning or planning questions within controlled environments. Recent studies have explored travel planning as a medium to integrate various verbal reasoning tasks into real-world contexts. However, reasoning tasks extend beyond verbal reasoning alone, and a comprehensive evaluation of LLMs requires a testbed that incorporates tasks from multiple cognitive domains. To address this gap, we introduce ItinBench, a benchmark that features one task of spatial reasoning, i.e., route optimization, into trip itinerary planning while keeping the traditional verbal reasoning tasks. ItinBench evaluates various LLMs across diverse tasks simultaneously, including Llama 3.1 8B, Mistral Large, Gemini 1.5 Pro, and GPT family. Our findings reveal that LLMs struggle to maintain high and consistent performance when concurrently handling multiple cognitive dimensions. By incorporating tasks from distinct human-level cognitive domains, ItinBench provides new insights into building more comprehensive reasoning testbeds that better reflect real-world challenges. The code and dataset: https://ethanwtl.github.io/IBweb/

*   **研究动机**：尽管 LLM 在语言推理（逻辑、数学、语义理解）方面表现出色，但人类的规划能力是多维度的，包含空间推理（Spatial Reasoning）。现有的旅行规划基准主要关注文本约束的满足，而忽略了现实世界规划中至关重要的空间布局和路径优化。
*   **核心问题**：当 LLM 需要同时处理复杂的语言约束（如用户偏好、预算）和空间优化（如减少行驶距离）时，其性能是否稳健？是否存在不同认知维度之间的性能权衡（Trade-off）？
*   **整体含义**：论文提出了 **ItinBench**，这是一个将空间推理（路径优化）整合进传统语言推理任务的行程规划基准，旨在提供一个更接近人类真实认知挑战的测试平台。

#### 2. 论文提出的方法论
*   **核心思想**：通过构建一个包含真实地理信息（经纬度、地址）和用户评论数据的环境，要求 LLM 生成既符合用户偏好又在空间上最优的旅行行程。
*   **关键技术细节**：
    *   **数据管道**：从 Yelp 提取费城（Philadelphia）的商家数据（餐厅、酒店、景点），并利用 LLM 从海量用户评论中提取细粒度属性（如口味、服务、氛围等）。
    *   **空间聚类**：使用 K-means 算法对 POI（兴趣点）进行地理聚类，并将聚类信息作为可选的文本提示提供给模型，以测试模型是依赖“空间想象”还是“语义关联”。
    *   **任务设计**：设计了四种递进任务：
        1.  全数据无优化要求（纯语言推理）。
        2.  全数据有优化要求（双维度挑战）。
        3.  过滤数据有优化要求（降低语言难度，聚焦空间推理）。
        4.  工具使用（ReAct 模式，模拟真实智能体行为）。
*   **算法流程**：引入了适配后的**旅行商问题（TSP）算法**作为金标准（Ground Truth），用于计算 LLM 生成路径与理论最短路径之间的距离差距（Distance Gap）。

#### 3. 实验设计
*   **数据集/场景**：以费城为主要场景，包含 500 个类人查询（Query），每个查询包含 6-10 个偏好约束。此外，还使用圣巴巴拉（Santa Barbara）数据进行了泛化性验证。
*   **对比方法（模型）**：
    *   **开源模型**：Llama 3.1 8B, Mistral Large (123B)。
    *   **闭源模型**：Gemini 1.5 Pro, GPT-4o, OpenAI o1。
*   **评估指标**：
    *   **语言维度**：池外选择率（OOP）、信息缺失率（MI）、微观/宏观偏好匹配率、最终验证率（VR）。
    *   **空间维度**：推荐数量偏差（ARG）、日均距离差距（DG）、总距离差距（Total-DG）、额外聚类跳跃率（ECJ）。

#### 4. 资源与算力
*   **算力说明**：论文明确提到所有实验均通过 **API 调用方式**完成，因此**不需要本地 GPU 显存**进行推理。
*   **训练时长**：该研究属于基准测试评估，不涉及模型的重新训练或微调，因此未提供训练算力数据。

#### 5. 实验数量与充分性
*   **实验规模**：针对 5 种主流模型，在 4 种不同任务设置下进行了大规模测试，总计生成并评估了数千个行程方案。
*   **充分性与公平性**：
    *   **多维度验证**：除了自动评估，还进行了 240 组人工评估（由博士生和资深旅行者参与），验证了自动指标与人类偏好的一致性。
    *   **消融/对比**：通过提供/不提供聚类信息，成功区分了模型的语义处理能力与空间推理能力。
    *   **泛化测试**：在不同规模的城市（圣巴巴拉）重复实验，证明了结论的普适性。实验设计客观、公平，基准线（Greedy/TSP）设置合理。

#### 6. 论文的主要结论与发现
*   **多维度处理能力不足**：LLM 在同时处理语言和空间任务时表现挣扎。即使在信息完全透明的情况下，验证通过率（VR）也仅在 60% 左右。
*   **空间推理的“幻觉”**：LLM 的空间优化能力高度依赖于文本形式的“空间线索”（如聚类标签）。如果没有这些标签，即使是 GPT-4o 也会产生 15%-38% 的额外无效行驶距离。
*   **性能权衡**：强推理模型（如 o1）虽然在空间优化上表现较好（距离差距降至 7%-9%），但其语言偏好匹配的准确度反而有所下降，显示出模型在分配注意力资源时的局限性。
*   **工具使用的局限**：在工具使用任务中，模型在识别弥散在文本中的复杂偏好（如餐厅口味）时容易出错。

#### 7. 优点
*   **维度创新**：首次在旅行规划基准中深度整合了空间推理与路径优化，填补了 LLM 认知评估的空白。
*   **评估严谨**：利用经典的 TSP 算法为空间规划提供了量化的、非语义的硬性评价标准。
*   **真实性强**：基于真实世界的 Yelp 数据和复杂的类人查询，比传统的网格世界或棋盘游戏更具实际应用参考价值。

#### 8. 不足与局限
*   **空间表示单一**：目前主要依赖经纬度和地址，尚未涉及视觉地图或更复杂的拓扑结构推理。
*   **地理范围限制**：实验主要集中在单城市规划，未扩展到多城市、跨国等更复杂的长途规划场景。
*   **动态性缺失**：未考虑现实中的动态因素，如交通拥堵、营业时间变动等实时约束。
*   **模型依赖**：空间能力的提升目前看来更多是语义关联的产物，而非模型真正理解了几何空间，这暗示了当前架构在处理非符号推理时的根本性缺陷。
