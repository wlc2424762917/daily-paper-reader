## 摘要
使用工具的大语言模型（LLM）智能体通常在回答质量与执行成本之间面临根本性的权衡。固定工作流虽然稳定但缺乏灵活性，而像 ReAct 这样的自由形式多步推理方法虽然可能提高任务性能，但代价是过多的工具调用、更长的执行轨迹、更高的 Token 消耗以及增加的延迟。在本文中，我们将智能体编排视为一个显式的决策问题，而不是完全将其留给提示词层面的行为。我们提出了一种效用引导的编排策略，通过平衡预估收益、单步成本、不确定性和冗余，在响应、检索、工具调用、验证和停止等动作中进行选择。我们的目标并非追求普适的最佳任务性能，而是提供一个可控且可分析的策略框架，用于研究使用工具的 LLM 智能体在质量与成本之间的权衡。在直接回答、阈值控制、固定工作流、ReAct 以及几种策略变体上的实验表明，显式的编排信号会显著影响智能体的行为。对成本定义、工作流公平性和冗余控制的额外分析进一步证明，轻量级的效用设计可以为智能体控制提供一种合理且实用的机制。

## Abstract
Tool-using large language model (LLM) agents often face a fundamental tension between answer quality and execution cost. Fixed workflows are stable but inflexible, while free-form multi-step reasoning methods such as ReAct may improve task performance at the expense of excessive tool calls, longer trajectories, higher token consumption, and increased latency. In this paper, we study agent orchestration as an explicit decision problem rather than leaving it entirely to prompt-level behavior. We propose a utility-guided orchestration policy that selects among actions such as respond, retrieve, tool call, verify, and stop by balancing estimated gain, step cost, uncertainty, and redundancy. Our goal is not to claim universally best task performance, but to provide a controllable and analyzable policy framework for studying quality-cost trade-offs in tool-using LLM agents. Experiments across direct answering, threshold control, fixed workflows, ReAct, and several policy variants show that explicit orchestration signals substantially affect agent behavior. Additional analyses on cost definitions, workflow fairness, and redundancy control further demonstrate that lightweight utility design can provide a defensible and practical mechanism for agent control.

### 2. 方法论：效用引导的编排框架
*   **核心思想**：在每个步骤中，智能体不只是生成下一步内容，而是从预定义的动作空间中选择效用最大化的动作。
*   **动作空间 $\mathcal{A}$**：包含 `respond`（响应）、`retrieve`（检索）、`tool_call`（工具调用）、`verify`（验证）和 `stop`（停止）。

    *   **Step Cost (单步成本)**：内部控制信号，用于抑制轨迹无限制扩张。
    *   **Uncertainty (不确定性)**：评估当前证据是否充足，高不确定性倾向于继续探索。
    *   **Redundancy (冗余度)**：惩罚重复或语义相似的动作，以压缩执行轨迹。
*   **执行流程**：构建状态 $s_t$（含查询、历史、观测、预算状态） $\rightarrow$ 计算各动作效用 $\rightarrow$ 选择最高效用动作执行 $\rightarrow$ 更新状态并循环，直到选择 `stop` 或耗尽预算。

### 6. 主要结论与发现
*   **显式编排有效性**：效用引导策略能提供比固定工作流更优的灵活性，且比 ReAct 更具成本效益。
*   **收益递减效应**：增加推理步数初期能显著提升 F1，但随后边际收益迅速下降，证明了主动“停止”决策的重要性。
*   **启发式信号的价值**：虽然 `Gain` 和 `Uncertainty` 是 LLM 自评估的启发式信号（非精确概率），但它们与智能体的持续/停止行为具有显著的相关性，足以支撑有效的决策。
*   **冗余控制**：引入语义冗余控制能显著减少 Token 消耗和工具调用次数，而对答案质量影响极小。

### 7. 优点与亮点
*   **解耦控制与推理**：将“如何做”与“是否值得做”分离，使智能体行为更易于解释和调试。
*   **系统级视角**：不单纯追求最高准确率，而是将 LLM 智能体视为一个受资源约束的系统进行优化。
*   **轻量化**：无需复杂的强化学习训练，通过简单的效用公式即可实现对智能体行为的精细调节。

### 8. 不足与局限
*   **信号校准问题**：效用组件（如收益和不确定性）依赖于 LLM 的自评估，可能存在过度自信或评估不准的情况，缺乏严格的概率校准。
*   **绝对性能上限**：在某些配置下，其绝对 F1 分数可能略低于不受限的 ReAct，说明在追求效率时可能牺牲了极小部分的性能上限。
*   **延迟开销**：引入语义冗余检查等复杂效用计算时，虽然减少了 Token 消耗，但可能因额外的计算步骤增加墙钟延迟。
*   **场景单一**：实验主要集中在 HotpotQA 问答任务，在更长程、更复杂的交互环境（如 Web 导航或多工具协作）中的表现尚待验证。
