# Formal Semantics for Agentic Tool Protocols: A Process Calculus Approach

Abstract 

The emergence of large language model agents capable of invoking external tools has created urgent need for formal verification of agent protocols. Two paradigms dominate this space: Schema-Guided Dialogue (SGD), a research framework for zero-shot API generalization, and the Model Context Protocol (MCP), an industry standard for agent–tool integration. While both enable dynamic service discovery through schema descriptions, their formal relationship remains unexplored. Building on prior work establishing the conceptual convergence of these paradigms [Schlapbach, 2026], we present the first process calculus formalization of SGD and MCP, proving they are structurally bisimilar under a well-defined mapping Φ. However, we demonstrate that the reverse mapping Φ−1 is partial and lossy, revealing critical gaps in MCP’s expressivity. Through bidirectional analysis, we identify five principles—semantic completeness, explicit action boundaries, failure mode documentation, progressive disclosure compatibility, and inter-tool relationship declaration—as necessary and sufficient conditions for full behavioral equivalence. We formalize these principles as type-system extensions MCP +, proving 

MCP + ∼= SGD . Our work provides the first formal foundation for verified agent systems and establishes schema quality as a provable safety property. 

Keywords: Process calculus, bisimulation, schema-guided dialogue, model context protocol, agent verification, type safety. 

# 1 Introduction 

The rapid deployment of large language model (LLM) agents [Vaswani et al., 2017, Brown et al., 2020] in production systems has created a critical gap: while these agents—capable of multi-step reasoning [Wei et al., 2022] and reactive tool use [Yao et al., 2023]—orchestrate complex workflows across multiple services, we lack formal methods to verify their correctness, safety, and behavioral properties. Two paradigms have emerged to address agent–tool integration: 1. Schema-Guided Dialogue (SGD) [Rastogi et al., 2020]: A research framework demonstrating that dialogue models can generalize zero-shot to new APIs when provided natural-language schema descriptions at runtime. 2. Model Context Protocol (MCP) [Anthropic, 2025]: An industry standard introduced by Anthropic to solve the N-to-M integration problem , enabling any LLM host to connect with any compliant server through standardized primitives. Both paradigms share a core principle: agents discover and reason about services through machine-readable schemas without retraining. Schlapbach [2026] established that SGD and MCP converge conceptually around five schema design principles, demonstrating empirical improvements in agent routing accuracy. However, the formal relationship between the two protocols remains unexplored. Can we prove they are equivalent? What properties are preserved under transformation? What gaps exist in either framework? This paper provides rigorous answers by grounding the analysis in process calculus.

## 1.2 Motivation: Why Formal Semantics Matter 

Consider a banking agent orchestrating account transfers: 1. verify _balance (account _id ) → balance 

2. validate _transfer (amount , balance ) → authorized 

3. execute _transfer (from , to , amount ) → confirmation 

Questions requiring formal answers include: Does the agent always check balance before transfer? Can step 3 execute without step 2 approval? What happens if step 3 fails after debiting the account? Can we prove properties of agent 1 | agent 2?Current approaches rely on testing (incomplete coverage), prompt engineering [Wei et al., 2022] (no guarantees, brittle), or human review (not scalable). Agent frameworks grounded in BDI theory [Rao and Georgeff, 1995] offer structured reasoning, but stop short of protocol-level verification. Formal semantics enable verification (prove safety for all executions), composition (build complex systems from verified components), optimization (preserve equivalence under transformation), and debugging (trace violations to specific protocol violations). 

# 2 Background and Related Work 

## 2.1 Process Calculi for Concurrency 

Process calculi provide mathematical frameworks for reasoning about concurrent, communicating systems. The π-calculus [Milner, 1999] extends CCS [Milner, 1989] with mobile channels, enabling dynamic network topologies—essential for agent systems where tool availability changes at runtime. 

Key concepts. Processes are computational entities that evolve through transitions; channels are communication primitives for message passing; operators include parallel composition (|), restriction (ν),and replication (! ·); bisimulation is a behavioral equivalence relation preserving observations. Related work includes session types [Honda et al., 1998] for protocol verification, choreography cal-culi [Carbone et al., 2012] for multi-party workflows, and probabilistic process algebras [D’Argenio et al., 2022] for stochastic systems. Foundational correctness arguments for concurrent programs trace back to Lamport [Lamport, 1977]. However, all these frameworks assume static protocols. Agent systems require runtime schema interpretation. 22.2 Schema-Guided Dialogue Systems 

The SGD dataset [Rastogi et al., 2020] introduced a paradigm shift in task-oriented dialogue: rather than training on fixed ontologies, SGD models learn to interpret arbitrary API schemas at runtime. Key components are intents (high-level user goals), slots (parameters to collect), schemas (JSON definitions with natural-language descriptions), and state tracking (frame-based representation across turns). Dialogue state tracking has a longer history in shared tasks [Williams et al., 2016], and BERT-based encoders [Devlin et al., 2019] enabled strong cross-service generalisation. Fast and robust SGD trackers [Gupta et al., 2020] demonstrated practical viability at scale. More recently, in-context learning has shown that few demonstration examples suffice to adapt to new schemas—mirroring the zero-shot generalisation that SGD pioneered. 

## 2.3 Model Context Protocol 

MCP [Anthropic, 2025] standardizes agent–tool communication through three primitives: Tools (exe-cutable functions), Resources (read-only context), and Prompts (reusable workflow templates). The architecture comprises a host (LLM environment), a client (protocol handler), and a server (service exposing primitives via JSON-RPC 2.0). Schemas are expressed in OpenAPI/JSON Schema [Foun-dation, 2021], distinguishing MCP from alternative API paradigms such as gRPC [Google, 2015] and GraphQL [Facebook, 2015]. Active tool discovery at runtime—where agents autonomously locate and load new servers—is explored by MCP-Zero [Gao et al., 2025]. 

## 2.4 Formal Methods for Agent Systems 

Recent work applies formal methods to LLM agents: LMQL [Beurer-Kellner et al., 2023] constrains outputs via formal grammars; ToolEmu [Patil et al., 2024] tests tool-use agents with emulated environments; MCPBench [Chen et al., 2025] benchmarks MCP tool selection; PARSE [Shrimal et al., 2025] applies LLM-driven schema optimisation for reliable entity extraction; and PromptBench [Zhu et al., 2024] evaluates LLM robustness under adversarial prompts. Formal verification of neural components has been approached via SMT solving [Katz et al., 2017] and certified defences [Wong and Kolter, 2018]. Safe reinforcement learning [García and Fernández, 2015] provides constrained optimisation guarantees orthogonal to our protocol-level approach. Weighted majority voting [Xie et al., 2026] has been applied to improve LLM decision reliability under uncertainty. None of these works provide process calculus semantics for agent–tool protocols.

Definition 8.1 (Capability-Confined Process) . A process P is capability-confined iff in every state reachable from P no unauthorized channel access occurs.

# 9 Conclusion 

We have presented the first formal semantics for agentic tool protocols, proving that SGD and MCP are structurally bisimilar under the mapping Φ (Theorem 4.4). However, Φ−1 is partial and lossy (Theorem 5.2), revealing four critical gaps in MCP’s expressivity. Through bidirectional analysis we identified five principles as necessary and sufficient conditions for full behavioral equivalence, and formalized them as type-system extensions defining MCP + with MCP + ∼= SGD 

(Theorem 7.1). Our framework enables: 

• Compositional reasoning about multi-agent orchestration via parallel composition in π-calculus .

• Security properties as provable process invariants (confinement, no unapproved side effects, dependency ordering). 

• Behavioral equivalence preserved under protocol transformation. 

• Schema quality as a type-checkable property. As LLM agents govern financial transactions, healthcare decisions, and infrastructure management, formal verification becomes essential. This paper provides the mathematical foundations for that verification, transforming agent protocols from ad-hoc conventions into provably safe systems. The convergence of SGD and MCP signals the emergence of Software 3.0 , where autonomous agents dynamically discover and orchestrate capabilities through standardized, machine-readable interfaces—and our formalization ensures this future is not only powerful but verifiable. 

# 10 Future Work 

Several directions follow naturally from the foundations established here. 

Mechanized proofs. The bisimulation proofs (Theorems 4.4 and 7.1) are pen-and-paper arguments by structural induction. Formalizing them in a proof assistant such as Isabelle/HOL or Coq—using existing 

π-calculus libraries such as Nominal Isabelle [Milner, 1999]—would yield machine-checked guarantees and expose any residual gaps. This is the most direct path to a fully verified MCP + type checker. 

Probabilistic bisimulation. LLM tool selection is inherently stochastic: the same prompt yields different tool calls with varying probability. Extending the labeled transition systems to continuous-time Markov chains and adopting Larsen–Skou probabilistic bisimulation would allow reasoning about expected 

behavioral equivalence and provide a formal measure of divergence between SGD and MCP agents under sampling. 

Multi-agent choreographies. The current calculus models a single agent against a single server. A natural extension is a choreography layer: given agents A1 | A2 | · · · | An each holding a disjoint subset of MCP + tools, prove global safety properties—deadlock freedom, resource exclusivity, progress—from local type-checking alone. This connects directly to multiparty session types [Honda et al., 1998] and choreography calculi [Carbone et al., 2012]. 15 Temporal safety via model checking. The security theorems of Section 8 are proved structurally. An orthogonal approach expresses them in CTL or LTL and applies off-the-shelf model checkers (SPIN, nuXmv) to finite MCP + instances. This would enable property verification for new protocol variants without manual proof, and could integrate with existing MCP server validation pipelines. 

Resource-aware semantics. Principle 4 (progressive disclosure) is motivated by token cost but formalized only loosely. A sub-structural or resource-sensitive type system would make token consumption a type-level property: a tool call exceeding the context budget becomes a type error rather than a runtime failure. This connects to work on bounded process calculi and quantitative information flow. 

Dynamic schema evolution. The current model treats schemas as static. Production MCP servers update tool definitions at runtime through versioning and deprecation. Extending the calculus with schema migration operations—and proving bisimulation is preserved under compatible migrations or characterizing when it breaks—is directly relevant to industrial deployment of the framework. 

Empirical calibration of Φ. Schlapbach [2026] demonstrated that the five principles improve routing accuracy empirically. The information-loss characterization of Φ−1 (Theorem 5.2) provides a theoretical explanation. Future work could derive a formal lower bound on routing error as a function of the information dropped by Φ, then validate the bound against the empirical routing curves of that companion study. 

Formal prompt-injection model. The tool-poisoning prevention result (Section 8) assumes descrip-tions are fully controlled by the server. A richer threat model in which description fields are partially adversary-controlled (e.g., via retrieval-augmented schema injection) requires an information-flow type system supporting non-interference and taint analysis, connecting process-calculus security to classical declassification frameworks.
