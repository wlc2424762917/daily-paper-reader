## 摘要
尽管大语言模型（LLMs）推动了通用智能体的发展，但在未见任务上实现鲁棒的泛化仍是一项重大挑战。目前的方法通常依赖于微调或利用检索经验的免训练记忆增强生成，但两者均存在局限性：微调往往难以推导至新任务，而经验检索的表现通常逊于有监督基准。在本研究中，我们提出将这两种方法结合，并系统地研究如何训练检索增强型 LLM 智能体，以在上下文中有效利用检索到的轨迹。首先，我们利用 LoRA 建立了一套鲁棒的有监督微调（SFT）方案，其性能优于多种先进的智能体训练流水线。其次，我们对经验检索的关键设计选择进行了详细分析，确定了存储、查询和轨迹选择的最优策略。最后，我们提出了一种将经验检索集成到微调过程中的流水线。实验结果表明，这种结合方法显著提升了对未见任务的泛化能力，为构建能够从经验中学习的智能体提供了一个可扩展且有效的框架。

## Abstract
While large language models (LLMs) have advanced the development of general-purpose agents, achieving robust generalization to unseen tasks remains a significant challenge. Current approaches typically rely on either fine-tuning or training-free memory-augmented generation using retrieved experience; yet both have limitations: fine-tuning often fails to extrapolate to new tasks, while experience retrieval often underperforms compared to supervised baselines. In this work, we propose to combine these approaches and systematically study how to train retrieval-augmented LLM agents to effectively leverage retrieved trajectories in-context. First, we establish a robust supervised fine-tuning (SFT) recipe using LoRA that outperforms several state-of-the-art agent training pipelines. Second, we provide a detailed analysis of key design choices for experience retrieval, identifying optimal strategies for storage, querying, and trajectory selection. Finally, we propose a pipeline that integrates experience retrieval into the fine-tuning process. Our results demonstrate that this combined approach significantly improves generalization to unseen tasks, providing a scalable and effective framework for building agents that learn to learn from experience.

### 2. 方法论：ExpRAG 与 ExpRAG-LoRA
论文提出了 **ExpRAG (Experience Retrieval-Augmented Generation)** 框架，核心思想是将智能体的历史交互轨迹作为“经验”进行检索和利用。

*   **经验银行（Experience Bank）**：离线收集智能体的交互轨迹 $\tau$，并将其编码为键向量 $\phi(\tau)$ 建立索引。
*   **检索机制**：
    *   **静态检索**：在任务开始时（$t=0$）根据任务描述检索一次。
    *   **动态检索**：在每个决策步骤根据当前交互历史刷新检索结果。
*   **核心技术细节**：
    *   **多轮对话编码**：将轨迹序列化为 LLM 原生的多轮对话格式（Chat Template），相比逐步骤编码，这种方式能更好地利用 KV 缓存，提升训练和推理效率。
    *   **ExpRAG-LoRA（关键创新）**：在 LoRA 微调过程中，将检索到的经验轨迹直接注入到系统提示词中。这迫使模型在训练阶段就学会“如何参考上下文中的经验”来预测动作，而不是仅仅死记硬背训练参数。
    *   **内存块格式**：在提示词中区分成功和失败的轨迹，引导模型模仿成功案例并规避失败教训。

### 3. 实验设计
*   **数据集/场景**：
    *   **ALFWorld**：文本交互式家庭环境任务。
    *   **ScienceWorld**：复杂的科学实验任务（小学科学课程水平）。
*   **Benchmark 设置**：将任务分为 **Easy（训练集）** 和 **Hard（测试集/未见任务）**，以严格评估跨任务的泛化能力。
*   **对比方法**：
    *   **免训练方法**：Zero-shot, ReAct, Mem0, Reflexion, A-MEM 等。
    *   **微调方法**：NAT, IWM, ETO, 以及作者建立的高强度 LoRA 基准。
*   **消融实验**：对比了不同的检索数量 $K$（1, 2, 4）、检索模式（静态 vs 动态）、索引组成（全量、仅简单、仅困难）以及轨迹序列化格式（JSON vs 文本）。

### 4. 资源与算力
*   **硬件**：所有模型均在单张 **NVIDIA A100 (80GB)** GPU 上训练。

*   **训练时长**：论文提到进行了长达 **50 个 Epoch** 的训练，以观察泛化动力学。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 4 种不同架构和规模的基座模型，在两个主流智能体基准上进行了详尽测试。
*   **充分性**：作者不仅对比了最终成功率，还深入分析了检索参数（如 $K$ 值）对性能的影响，并进行了 3 个随机种子的稳定性测试。
*   **客观性**：作者首先建立了一个极强的 LoRA 基准（其表现甚至超过了许多已发表的复杂智能体框架），确保了后续 ExpRAG 提升的含金量。

### 6. 主要结论与发现
*   **ExpRAG-LoRA 的优越性**：在未见过的 Hard 任务上，ExpRAG-LoRA 显著优于标准 LoRA。例如在 ALFWorld 的 Hard 任务中，Ministral 3-8B 的成功率从 34.4% 提升至 **88.5%**。
*   **延迟泛化（Delayed Generalization）**：发现智能体在微调时存在类似“Grokking”的现象——即使验证集损失（Loss）开始上升（看似过拟合），模型在未见任务上的实际成功率往往仍会继续提高。
*   **简单即有效**：简单的、只读的经验轨迹检索，在性能上可以媲美甚至超越复杂的、具有读写逻辑的记忆管理系统。
*   **动态检索的局限**：动态检索虽然能提供更实时的参考，但在索引不匹配时可能引入不稳定性（Context Churn）。

### 7. 优点
*   **基准扎实**：通过优化 LoRA 训练细节，提供了一个非常诚实的 Baseline。
*   **实用性强**：方法不依赖昂贵的闭源模型，在 3B-8B 规模的开源模型上效果显著。
*   **深入的动力学分析**：对训练 Epoch 与泛化能力关系的探讨，为智能体微调提供了新的实践指导。

### 8. 不足与局限
*   **依赖专家轨迹**：实验中的经验银行主要由专家脚本生成的轨迹组成，尚未充分探讨当经验库包含大量 LLM 自身生成的噪声/错误轨迹时的鲁棒性。
*   **索引缺失风险**：当环境中完全没有相关任务的经验可供检索时，ExpRAG-LoRA 的性能会大幅下降（甚至低于标准 LoRA），表现出对检索上下文的强依赖。
*   **静态经验库**：目前采用的是固定的、只读的经验库，缺乏在线更新和知识压缩（如摘要化）的机制。
