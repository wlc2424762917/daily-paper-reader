## 摘要
记忆嵌入对于 OpenClaw 等记忆增强系统至关重要，但在目前的文本嵌入基准测试中，对其评估尚不充分。现有的基准测试主要集中在传统的段落检索，未能评估模型处理涉及碎片化、上下文相关且时间跨度较大的信息的长跨度记忆检索任务的能力。为了解决这一问题，我们推出了长跨度记忆嵌入基准（LMEB），这是一个评估嵌入模型处理复杂长跨度记忆检索任务能力的综合框架。LMEB 涵盖了 22 个数据集和 193 个零样本检索任务，涉及四种记忆类型：情景记忆、对话记忆、语义记忆和程序记忆，包含 AI 生成和人工标注的数据。这些记忆类型在抽象程度和时间依赖性方面各不相同，捕捉了记忆检索的不同维度，反映了现实世界的各种挑战。我们评估了 15 个广泛使用的嵌入模型，参数量从数亿到百亿不等。结果显示：(1) LMEB 提供了合理的难度水平；(2) 更大的模型并不总是表现更好；(3) LMEB 与 MTEB 表现出正交性。这表明该领域尚未收敛出一个能够胜任所有记忆检索任务的通用模型，且传统段落检索的表现可能无法推广到长跨度记忆检索。总之，通过提供标准化且可重复的评估框架，LMEB 填补了记忆嵌入评估的关键空白，推动了文本嵌入在处理长期、上下文相关的记忆检索方面的进一步发展。LMEB 已在 https://github.com/KaLM-Embedding/LMEB 开源。

## Abstract
Memory embeddings are crucial for memory-augmented systems, such as OpenClaw, but their evaluation is underexplored in current text embedding benchmarks, which narrowly focus on traditional passage retrieval and fail to assess models' ability to handle long-horizon memory retrieval tasks involving fragmented, context-dependent, and temporally distant information. To address this, we introduce the Long-horizon Memory Embedding Benchmark (LMEB), a comprehensive framework that evaluates embedding models' capabilities in handling complex, long-horizon memory retrieval tasks. LMEB spans 22 datasets and 193 zero-shot retrieval tasks across 4 memory types: episodic, dialogue, semantic, and procedural, with both AI-generated and human-annotated data. These memory types differ in terms of level of abstraction and temporal dependency, capturing distinct aspects of memory retrieval that reflect the diverse challenges of the real world. We evaluate 15 widely used embedding models, ranging from hundreds of millions to ten billion parameters. The results reveal that (1) LMEB provides a reasonable level of difficulty; (2) Larger models do not always perform better; (3) LMEB and MTEB exhibit orthogonality. This suggests that the field has yet to converge on a universal model capable of excelling across all memory retrieval tasks, and that performance in traditional passage retrieval may not generalize to long-horizon memory retrieval. In summary, by providing a standardized and reproducible evaluation framework, LMEB fills a crucial gap in memory embedding evaluation, driving further advancements in text embedding for handling long-term, context-dependent memory retrieval. LMEB is available at https://github.com/KaLM-Embedding/LMEB.

## 1. 核心问题与整体含义（研究动机和背景）
论文的核心问题在于**现有文本嵌入基准（如 MTEB、BEIR）无法有效评估模型在“长跨度记忆（Long-horizon Memory）”检索任务中的表现**。
*   **背景**：随着 Agent 系统（如 OpenClaw）和长文本应用的发展，系统需要从海量、碎片化、上下文相关且时间跨度大的记忆库中检索信息。
*   **痛点**：传统基准侧重于结构良好的段落检索，而真实的记忆检索往往涉及：
    1.  **碎片化信息**：信息散落在不同时间点的对话或事件中。
    2.  **上下文依赖**：检索结果必须依赖特定的背景或时间线。
    3.  **长时依赖**：需要跨越很长的时间跨度定位特定记忆。
*   **含义**：LMEB 的提出旨在填补这一空白，为记忆增强型系统提供一个标准化的评估框架。

## 2. 论文提出的方法论
论文提出了 **LMEB (Long-horizon Memory Embedding Benchmark)**，其核心思想是将记忆检索按**抽象程度（Abstraction）**和**时间依赖性（Temporal Dependency）**两个维度划分为四类：

*   **核心分类体系**：
    1.  **情景记忆 (Episodic Memory)**：低抽象、高时间依赖。关注特定时间、地点发生的具体事件。
    2.  **对话记忆 (Dialogue Memory)**：高抽象、高时间依赖。关注多轮交互中的上下文、用户偏好和历史对话。
    3.  **语义记忆 (Semantic Memory)**：低抽象、低时间依赖。关注稳定的通用知识和事实（如学术论文检索）。
    4.  **程序记忆 (Procedural Memory)**：高抽象、低时间依赖。关注技能、操作序列和任务解决路径（如 API 调用、代码逻辑）。
*   **技术细节**：
    *   **统一格式**：将所有数据集转换为标准的 IR（信息检索）格式，包含 `queries`（查询）、`corpus`（语料库）、`qrels`（相关性标注）和可选的 `candidates`（候选池，用于限制检索范围）。
    *   **时间锚点**：针对带有相对时间描述的查询（如“两天前”），在查询文本中附加显式的当前时间锚点以消除歧义。
    *   **评估协议**：基于 MTEB v2 框架，支持零样本（Zero-shot）评估。

## 6. 论文的主要结论与发现
1.  **难度适中且具有区分度**：Top 模型（bge-multilingual-gemma2）的平均 NDCG@10 为 61.41，说明基准既有挑战性，又未达到饱和。
2.  **规模效应不绝对**：**更大的模型并不总是表现更好**。例如，300M 的 EmbeddingGemma 在某些任务上优于 12B 的模型，说明架构优化和训练数据质量比参数量更重要。
3.  **与 MTEB 的正交性**：LMEB 与 MTEB 的相关系数（Pearson/Spearman）接近 0。这意味着在传统段落检索上表现优异的模型，在长程记忆检索上可能表现平平。
4.  **指令敏感性差异**：不同模型对任务指令（Instruction）的偏好不同，部分模型在无指令下表现更好。

## 7. 优点：亮点与创新
*   **填补空白**：首次系统性地定义并构建了针对“长跨度记忆”的嵌入评估基准。
*   **分类科学**：基于心理学记忆分类（情景、语义等）并结合 AI 场景（对话、程序）构建的四象限分类法非常严谨。
*   **关注时间维度**：在检索中引入了时间依赖性和相对时间解析，这是此前基准普遍缺失的。
*   **高度兼容**：完全兼容 MTEB 接口，易于社区集成新模型。

## 8. 不足与局限
*   **语言局限性**：目前 LMEB 主要集中在**英语**数据集，缺乏对多语言记忆检索的评估。
*   **数据偏差风险**：部分数据集（如 EPBench）由 AI 生成，可能存在合成数据的分布偏差，与真实人类记忆特征可能存在细微差异。
*   **多模态缺失**：目前的记忆仅限于文本，尚未涵盖视觉或音频等多模态记忆检索。
*   **应用限制**：主要关注检索阶段，未评估检索后的推理或生成质量（尽管这是嵌入基准的通用限制）。
