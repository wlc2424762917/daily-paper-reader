## 摘要
多轮大语言模型（LLM）智能体在解决复杂的交互式任务中变得越来越重要，而强化学习（RL）是改善其长程行为的关键要素。然而，强化学习训练需要生成大量的沙箱化 rollout 轨迹，且现有的基础设施通常将 rollout 编排与训练循环耦合在一起，导致系统难以迁移和维护。秉持“rollout 即服务”的理念，我们提出了 ProRL Agent，这是一个可扩展的基础设施，通过 API 服务为完整的智能体 rollout 生命周期提供支持。ProRL Agent 还提供了标准化且可扩展的沙箱环境，支持在无根权限（rootless）的高性能计算（HPC）环境下执行多种智能体任务。我们通过在软件工程、数学、STEM 和编程任务上的强化学习训练验证了 ProRL Agent。ProRL Agent 已开源，并作为 NVIDIA NeMo Gym 的一部分进行了集成。

## Abstract
Multi-turn LLM agents are increasingly important for solving complex, interactive tasks, and reinforcement learning (RL) is a key ingredient for improving their long-horizon behavior. However, RL training requires generating large numbers of sandboxed rollout trajectories, and existing infrastructures often couple rollout orchestration with the training loop, making systems hard to migrate and maintain. Under the rollout-as-a-service philosophy, we present ProRL Agent , a scalable infrastructure that serves the full agentic rollout lifecycle through an API service. ProRL Agent also provides standardized and extensible sandbox environments that support diverse agentic tasks in rootless HPC settings. We validate ProRL Agent through RL training on software engineering, math, STEM, and coding tasks. ProRL Agent is open-sourced and integrated as part of NVIDIA NeMo Gym.

#### 1. 论文的核心问题与整体含义
随着大语言模型（LLM）智能体从单轮对话转向复杂的多轮交互任务（如软件工程、网页浏览、操作系统操作），强化学习（RL）成为提升其长程规划能力的关键。然而，现有的 RL 训练框架通常将 **Rollout（轨迹生成）** 与 **训练循环（策略优化）** 紧密耦合。
*   **研究动机：** 这种耦合导致了资源冲突（Rollout 是 I/O 密集型，训练是 GPU 密集型）、系统迁移困难（更换训练后端需重写执行管线）以及维护成本高。
*   **核心目标：** 提出“Rollout-as-a-Service”（RaaS，Rollout 即服务）理念，通过解耦两者，提供一个可扩展、高性能且兼容 HPC（高性能计算）环境的基础设施。

#### 2. 核心方法论
ProRL Agent 的核心思想是将智能体与环境交互的完整生命周期封装为一个独立的 HTTP 服务。
*   **三阶段异步流水线：** 服务器内部将任务分为 `INIT`（环境初始化）、`RUN`（智能体执行）和 `EVAL`（结果评估）三个阶段，每个阶段拥有独立的线程池和队列，实现任务重叠执行，最大化吞吐量。
*   **标准化沙盒环境（AgentHandler）：** 提供统一接口，支持不同任务（如代码、数学）的插件式扩展。
*   **HPC 兼容的容器技术：** 采用 **Singularity** 运行时而非 Docker，支持在无 Root 权限的集群（如 Slurm 管理的 HPC）中运行，并利用虚拟 IP 和 Unix 域套接字（UDS）优化通信延迟。
*   **LLM 后端管理：** 维护一个 LLM 推理服务器池，使用**最小堆（Min-Heap）**进行负载均衡，并支持在训练过程中动态注册新服务器或更换模型权重。
*   **Token-in/Token-out：** 全程使用 Token ID 通信，避免了文本重分词（Re-tokenization）带来的分布偏移问题。
*   **高效 DAPO 调度：** 针对动态采样策略优化（DAPO）算法，实现了异步补充机制和早期停止，减少了 GPU 空闲时间。
