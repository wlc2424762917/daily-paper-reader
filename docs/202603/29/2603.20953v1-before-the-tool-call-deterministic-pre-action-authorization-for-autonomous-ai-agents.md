## 摘要
如今的 AI 智能体拥有密码，但没有授权凭证。它们执行工具调用（如资金转账、数据库查询、Shell 命令、子智能体委派），但在动作执行前缺乏强制授权的标准机制。当前的安全架构依赖于模型对齐（概率性的、训练时的）和事后评估（回顾性的、批量的），两者都无法在单个工具调用级别提供基于策略的确定性强制执行。我们将这一差距定义为动作前授权问题，并提出了开放智能体护照（Open Agent Passport, OAP）。这是一种开放规范和参考实现，可在执行前同步拦截工具调用，根据声明式策略对其进行评估，并生成经加密签名的审计记录。实验表明，OAP 执行授权决策的中位数耗时为 53 毫秒（N=1,000）。在一个实战对抗测试平台中（涵盖 1,151 个会话中的 4,437 次授权决策，设有 5,000 美元悬赏），在宽松策略下，针对模型的社会工程学攻击成功率为 74.6%；而在严格的 OAP 策略下，同等规模的攻击者在 879 次尝试中实现了 0% 的成功率。我们将动作前授权与沙箱执行（限制影响范围但不能防止未经授权的动作）以及基于模型的筛选（概率性的）区分开来，并证明它们是互补的。强制执行安全约束（如支出限制、能力范围）的同一基础设施，也同样强制执行质量门禁、操作合约和合规控制。该规范以 Apache 2.0 协议发布（DOI: 10.5281/zenodo.18901596）。

## Abstract
AI agents today have passwords but no permission slips. They execute tool calls (fund transfers, database queries, shell commands, sub-agent delegation) with no standard mechanism to enforce authorization before the action executes. Current safety architectures rely on model alignment (probabilistic, training-time) and post-hoc evaluation (retrospective, batch). Neither provides deterministic, policy-based enforcement at the individual tool call level.   We characterize this gap as the pre-action authorization problem and present the Open Agent Passport (OAP), an open specification and reference implementation that intercepts tool calls synchronously before execution, evaluates them against a declarative policy, and produces a cryptographically signed audit record. OAP enforces authorization decisions in a measured median of 53 ms (N=1,000). In a live adversarial testbed (4,437 authorization decisions across 1,151 sessions, $5,000 bounty), social engineering succeeded against the model 74.6% of the time under a permissive policy; under a restrictive OAP policy, a comparable population of attackers achieved a 0% success rate across 879 attempts.   We distinguish pre-action authorization from sandboxed execution (contains blast radius but does not prevent unauthorized actions) and model-based screening (probabilistic), and show they are complementary. The same infrastructure that enforces security constraints (spending limits, capability scoping) also enforces quality gates, operational contracts, and compliance controls. The specification is released under Apache 2.0 (DOI: 10.5281/zenodo.18901596).

#### 2. 方法论：核心思想与技术细节
论文提出了 **Open Agent Passport (OAP)** 规范，其核心思想是在智能体框架层植入一个同步拦截钩子。
*   **核心组件**：
    *   **智能体护照 (Agent Passport)**：一种经 Ed25519 加密签名的凭证，将智能体身份与其授权的能力范围（Capabilities）和限制（Limits）绑定。
    *   **策略包 (Policy Packs)**：使用声明式 JSON/YAML 定义的规则集，规定了特定动作（如转账）必须满足的约束（如单笔限额、收款人白名单）。
    *   **拦截钩子 (before_tool_call)**：在工具执行前触发，是强制执行点。
*   **算法流程**：
    1.  **拦截**：框架捕获工具调用请求（名称、参数、上下文）。
    2.  **验证**：检查护照签名是否有效、是否过期或被吊销。
    3.  **匹配**：确认请求的工具是否在护照授权的能力列表中。
    4.  **评估**：将工具参数与策略包中的约束进行确定性比对（如 `amount > max_per_tx`）。
    5.  **决策**：返回 `ALLOW`（允许）、`DENY`（拒绝）或 `ESCALATE`（人工介入）。
    6.  **审计**：生成并签署不可篡改的审计记录。

#### 6. 主要结论与发现
*   **确定性防御的必要性**：在宽松策略下，社会工程学攻击对模型的成功率高达 **74.6%**；而在严格的 OAP 策略下，攻击成功率降至 **0%**。
*   **性能可行性**：53ms 的延迟对于通常耗时数百毫秒甚至数秒的工具调用（I/O 密集型）来说几乎可以忽略不计。
*   **互补性**：PAA（动作前授权）、沙箱执行和模型对齐是互补的。PAA 解决了语义层面的业务规则强制执行问题，这是沙箱无法做到的。

#### 7. 优点：亮点与创新
*   **确定性 (Deterministic)**：不依赖概率性的模型推理，消除了随机性带来的安全隐患。
*   **框架无关性**：作为一种开放规范，可以适配多种 AI 框架，具有良好的通用性。
*   **可审计性**：通过 Ed25519 签名提供不可篡改的审计追踪，满足合规性需求。
*   **故障闭合 (Fail-closed)**：设计上遵循安全首要原则，任何验证失败均默认拒绝。

#### 8. 不足与局限
*   **委派链缺失**：OAP v1.0 尚未形式化多智能体协作中的委派模型（计划在 v1.1 解决）。
*   **策略表达力限制**：采用声明式 JSON，虽然易于审计，但难以处理极其复杂的逻辑（如跨 30 天的滚动平均值计算）。
*   **静态性**：策略是静态定义的，无法像某些研究（如 AgentGuardian）那样根据执行轨迹自适应学习。
*   **平台信任依赖**：系统假设智能体运行框架本身未被篡改，如果框架层被攻破，拦截钩子可能被绕过。
