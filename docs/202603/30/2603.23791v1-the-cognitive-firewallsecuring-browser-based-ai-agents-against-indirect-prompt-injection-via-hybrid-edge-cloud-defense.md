## 摘要
将大语言模型（LLMs）部署为自主浏览器智能体，会以间接提示注入（IPI）的形式暴露巨大的攻击面。基于云的防御虽然能提供强大的语义分析，但会引入延迟并引发隐私担忧。我们提出了“认知防火墙”（Cognitive Firewall），这是一种三阶段分步计算架构，将安全检查分布在客户端和云端。该系统由本地视觉哨兵（Sentinel）、基于云的深度规划器（Deep Planner）以及在执行时强制执行策略的确定性守卫（Guard）组成。在 1,000 个对抗样本的测试中，仅靠边缘端的防御无法检测到 86.9% 的语义攻击。相比之下，完整的混合架构将总攻击成功率（ASR）降低至 1% 以下（静态评估下为 0.88%，自适应评估下为 0.67%），同时保持了对具有副作用操作的确定性约束。通过在本地过滤表现层攻击，该系统避免了不必要的云端推理，并比纯云端基准实现了约 17,000 倍的延迟优势。这些结果表明，在执行边界进行确定性强制执行可以补充概率性语言模型，且分步计算为保护交互式 LLM 智能体提供了实用的基础。

## Abstract
Deploying large language models (LLMs) as autonomous browser agents exposes a significant attack surface in the form of Indirect Prompt Injection (IPI). Cloud-based defenses can provide strong semantic analysis, but they introduce latency and raise privacy concerns. We present the Cognitive Firewall, a three-stage split-compute architecture that distributes security checks across the client and the cloud. The system consists of a local visual Sentinel, a cloud-based Deep Planner, and a deterministic Guard that enforces execution-time policies. Across 1,000 adversarial samples, edge-only defenses fail to detect 86.9% of semantic attacks. In contrast, the full hybrid architecture reduces the overall attack success rate (ASR) to below 1% (0.88% under static evaluation and 0.67% under adaptive evaluation), while maintaining deterministic constraints on side-effecting actions. By filtering presentation-layer attacks locally, the system avoids unnecessary cloud inference and achieves an approximately 17,000x latency advantage over cloud-only baselines. These results indicate that deterministic enforcement at the execution boundary can complement probabilistic language models, and that split-compute provides a practical foundation for securing interactive LLM agents.

#### 1. 核心问题与整体含义（研究动机和背景）
随着基于大语言模型（LLM）的自主浏览器智能体（如自动订票、管理 Jira 等）的兴起，**间接提示注入（Indirect Prompt Injection, IPI）** 成为重大安全威胁。攻击者通过在网页中嵌入不可见或伪装的恶意指令，诱导智能体执行未授权操作（如泄露隐私、删除数据）。
*   **研究动机**：现有的防御方案面临“延迟-深度-隐私”的三角难题。纯云端防御语义分析强但延迟高（>500ms）且有隐私风险；纯边缘防御速度快但难以识别复杂的语义攻击。
*   **核心目标**：构建一种既能保持低延迟响应，又能深度拦截语义攻击的端云协同防御架构。

#### 2. 方法论：核心思想与关键技术
论文提出了**“防御漏斗”（Defense Funnel）**模型，将安全检查分为三个阶段：
*   **L1：边缘哨兵（Edge Sentinel - 视觉感知层）**
    *   **核心思想**：在本地过滤表现层攻击。利用浏览器内置的轻量级模型（如 Gemini Nano）和 DOM/CSSOM 启发式算法。
    *   **技术细节**：检查元素的透明度（opacity < 0.1）、字体大小（≈0）或位置（是否在视口外）。如果发现“用户不可见但智能体可见”的文本，直接在本地拦截。
*   **L2：云端深度规划器（Deep Planner - 语义推理层）**
    *   **核心思想**：处理通过 L1 过滤的文本，进行深度意图分析。
    *   **技术细节**：在云端安全隔离区运行大模型（如 Llama 3 8B），使用专门的“安全分析师”系统提示词，识别越狱、角色扮演或指令覆盖等复杂语义陷阱。
*   **L3：执行守卫（Origin Guard - 确定性执行层）**
    *   **核心思想**：最后的确定性防线，不依赖概率模型。
    *   **技术细节**：通过同步 JavaScript 拦截器监控 API 调用（如 `fetch`）。强制执行**域名白名单**和**谓词策略**（例如：如果用户目标是“总结页面”，则拦截任何 `POST` 或 `DELETE` 请求）。

*   **实验规模**：进行了 1,000 组静态样本实验和 300 组针对性的自适应对抗实验。

*   **客观性**：考虑了自适应攻击者（White-box adversary）的情景，证明了即使 L1/L2 被绕过，L3 仍能提供确定性保护，实验结果具有较强的说服力。

#### 6. 主要结论与发现
*   **防御效能**：完整架构将攻击成功率（ASR）从 100% 降低至 **0.88%**（静态）和 **0.67%**（自适应）。
*   **延迟优势**：边缘哨兵（L1）的平均延迟仅为 **0.06ms**，比云端推理快约 **17,000 倍**。通过本地拦截 13.1% 的高频视觉攻击，显著节省了云端算力和 Token 消耗。
*   **协同效应**：边缘端无法识别 86.9% 的语义攻击，证明了云端深度分析的必要性；而云端模型可能被“良性包装”欺骗，证明了 L3 确定性守卫的不可或缺性。

#### 7. 优点
*   **分层防御思想**：将昂贵的语义检查与廉价的语法/视觉检查分离，平衡了性能与安全。
*   **确定性与概率性结合**：用确定性的代码拦截（L3）弥补了 LLM 概率性输出（L2）的不确定性，这是安全工程的优秀实践。
*   **隐私保护**：L1 在本地处理原始 DOM，减少了敏感数据上传的需求。

#### 8. 不足与局限
*   **多模态覆盖不足**：目前的哨兵主要基于文本和 DOM 属性，无法检测嵌入在图像中的视觉提示注入（如 QR 码或图片隐写指令）。
*   **可用性权衡**：L3 的严格策略导致了 **1.7% 的误报率**（False Positive），可能会拦截一些复杂的合法操作。
*   **语义残留风险**：在自适应攻击中，仍有 2% 的“良性包装”攻击成功，说明意图分类在面对极具说服力的伪装时仍存在瓶颈。
*   **人机交互缺失**：目前系统倾向于直接拦截，缺乏“引入人类确认”的机制来处理高熵或模糊的指令。
