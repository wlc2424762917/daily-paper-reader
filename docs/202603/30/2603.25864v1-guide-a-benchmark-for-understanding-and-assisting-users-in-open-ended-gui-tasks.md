## 摘要
图形用户界面（GUI）智能体具有辅助用户与复杂软件（如 PowerPoint、Photoshop）进行交互的潜力。虽然先前的研究主要集中在通过点击和按键实现用户操作的自动化，但这种范式忽略了人类意图，即用户在保持自主性的同时，非常看重探索、迭代和完善其想法的能力。为了超越自动化并迈向协作，GUI 智能体必须理解用户正在做什么以及为什么要这样做。我们推出了 GUIDE（GUI 用户意图检测评估），这是一个评估 AI 模型在开放式 GUI 任务中感知用户行为、推断意图并提供辅助能力的基准测试。GUIDE 包含来自 120 次新手用户演示的 67.5 小时屏幕录像，涵盖 10 种软件，并配有出声思维（think-aloud）叙述。GUIDE 定义了三个任务：(i) 行为状态检测，(ii) 意图预测，以及 (iii) 辅助预测，旨在测试模型识别行为状态、推理目标以及决定何时及如何提供帮助的能力。对八个最先进的多模态模型的评估显示，所有模型都表现不佳，在行为状态和辅助预测方面的准确率仅为 44.6% 和 55.0%。然而，提供用户上下文显著提高了性能，将辅助预测的准确率提升了高达 50.2 个百分点，突显了结构化用户理解在有效辅助中的关键作用。我们的数据集可在 https://guide-bench.github.io 获取。

## Abstract
Graphical User Interface (GUI) agents have the potential to assist users in interacting with complex software (e.g., PowerPoint, Photoshop). While prior research has primarily focused on automating user actions through clicks and keystrokes, this paradigm overlooks human intention, where users value the ability to explore, iterate, and refine their ideas while maintaining agency. To move beyond automation and toward collaboration, GUI agents must understand what users are doing and why. We introduce GUIDE (GUI User Intent Detection Evaluation), a benchmark that evaluates AI models on their ability to perceive user behavior, infer intent, and provide assistance in open-ended GUI tasks. GUIDE consists of 67.5 hours of screen recordings from 120 novice user demonstrations with think-aloud narrations, across 10 software. GUIDE defines three tasks - (i) Behavior State Detection, (ii) Intent Prediction, and (iii) Help Prediction that test a model's ability to recognize behavior state, reason about goals, and decide when and how to help. Evaluations across eight state-of-the-art multimodal models reveal that all models struggled, achieving only 44.6% and 55.0% accuracy on behavior state and help prediction. However, providing user context significantly improved the performance, raising help prediction by up to 50.2pp, highlighting the critical role of structured user understanding in effective assistance. Our dataset is available at https://guide-bench.github.io.

*   **核心问题**：当前的 AI 模型缺乏对用户**认知上下文**和**动态意图**的理解，无法在用户遇到困难时提供适时的、协作式的辅助。
*   **研究目标**：从“自动化”转向“人机协作”，开发能够感知用户行为状态、推断潜在意图并决定何时/如何提供帮助的智能系统。

#### 2. 论文提出的方法论
论文提出了 **GUIDE (GUI User Intent Detection Evaluation)** 基准，其核心思想是将用户理解建模为一个三阶段的递进框架：**理解 → 推理 → 辅助**。
*   **行为状态检测 (Behavior State Detection)**：基于视觉信号识别用户当前处于哪种行为模式。论文构建了一个包含 **9 种状态的分类体系**，涵盖四个阶段：
    *   *规划 (Planning)*：任务理解、构思。
    *   *执行 (Execution)*：探索决策、执行动作。
    *   *问题解决 (Problem-Solving)*：挫败感、调试、寻求外部帮助。
    *   *评估 (Evaluation)*：等待监控、成果评估。
*   **意图预测 (Intent Prediction)**：推断用户的短期、即时目标（例如“对齐对象”或“尝试另一种颜色”）。
*   **帮助预测 (Help Prediction)**：
    *   *需求检测*：判断用户是否需要帮助（二分类）。

#### 6. 论文的主要结论与发现
*   **模型表现欠佳**：所有模型在行为状态检测（最高 44.6%）和帮助预测（最高 55.0%）上均表现吃力。
*   **难以识别“挣扎”信号**：模型经常将用户的“挫败感”或“调试”行为误判为正常的“执行动作”，说明模型对重复点击、犹豫等细微的负面行为特征缺乏敏感度。
*   **上下文的重要性**：当显式提供用户的行为状态和意图作为输入时，帮助预测的准确率显著提升（最高提升 50.2 个百分点）。这证明了**结构化理解**是实现精准辅助的前提。
*   **时间上下文**：在在线设置中，随着视频片段的增加，模型表现稳步提升，说明时间序列信息的积累对预测至关重要。

#### 7. 优点
*   **视角独特**：不同于以往关注“专家演示”和“任务自动化”的基准，GUIDE 关注“新手用户”和“开放式协作”，更贴近真实的人机交互需求。
*   **任务链条完整**：将感知、推理、决策三个环节解耦并串联，为开发下一代主动式智能体提供了清晰的评估路径。
*   **高质量标注**：结合了出声思维叙述和人工校验，确保了意图和心理状态标注的真实性。

#### 8. 不足与局限
*   **模态限制**：虽然标注参考了音频，但评估时仅给模型提供了视觉帧，忽略了语音语调等可能包含情感和意图的重要信号。
*   **样本规模**：虽然在 GUI 领域已属较大规模，但相比通用视频理解数据集，样本量仍有提升空间。
*   **应用局限**：目前仅评估了模型的“判断”能力，尚未在实时交互系统中验证这些判断如何转化为实际的 UI 干预动作。
