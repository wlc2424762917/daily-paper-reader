## 摘要
Text-to-SQL 解析在全模式假设（Full Schema Assumption）下取得了显著进展。然而，这一前提在现实世界的企业环境中往往失效，因为那里的数据库包含数百张表以及海量的噪声元数据。智能体必须主动识别并验证相关的子集，而不是预先注入完整的模式，这便产生了本研究探讨的“未知模式”（Unknown Schema）场景。为此，我们提出了 TRUST-SQL（Truthful Reasoning with Unknown Schema via Tools）。我们将该任务建模为一个部分可观测马尔可夫决策过程（POMDP），其中自主智能体采用结构化的四阶段协议，将推理建立在经过验证的元数据之上。至关重要的是，该协议为我们新颖的双轨 GRPO（Dual-Track GRPO）策略提供了结构边界。通过应用 Token 级掩码优势，该策略将探索奖励与执行结果隔离开来，以解决信用分配问题，相比标准 GRPO 实现了 9.9% 的相对提升。在五个基准测试上的广泛实验表明，TRUST-SQL 的 4B 和 8B 变体相较于其基础模型，分别实现了 30.6% 和 16.6% 的平均绝对提升。值得注意的是，尽管在完全没有预加载元数据的情况下运行，我们的框架始终能达到或超过依赖模式预填充（schema prefilling）的强基准模型。

## Abstract
Text-to-SQL parsing has achieved remarkable progress under the Full Schema Assumption. However, this premise fails in real-world enterprise environments where databases contain hundreds of tables with massive noisy metadata. Rather than injecting the full schema upfront, an agent must actively identify and verify only the relevant subset, giving rise to the Unknown Schema scenario we study in this work. To address this, we propose TRUST-SQL (Truthful Reasoning with Unknown Schema via Tools). We formulate the task as a Partially Observable Markov Decision Process where our autonomous agent employs a structured four-phase protocol to ground reasoning in verified metadata. Crucially, this protocol provides a structural boundary for our novel Dual-Track GRPO strategy. By applying token-level masked advantages, this strategy isolates exploration rewards from execution outcomes to resolve credit assignment, yielding a 9.9% relative improvement over standard GRPO. Extensive experiments across five benchmarks demonstrate that TRUST-SQL achieves an average absolute improvement of 30.6% and 16.6% for the 4B and 8B variants respectively over their base models. Remarkably, despite operating entirely without pre-loaded metadata, our framework consistently matches or surpasses strong baselines that rely on schema prefilling.

#### 1. 核心问题与整体含义（研究动机和背景）
传统的 Text-to-SQL 研究大多基于**全模式假设（Full Schema Assumption）**，即假设数据库的所有表结构和元数据都能预先加载到模型的上下文窗口中。然而，在真实的企业级场景中，数据库往往包含数百张表和海量的噪声元数据，超出了上下文限制，且模式（Schema）会动态演变。
本文的核心问题是：**如何在数据库模式未知且不可观测的情况下，让智能体通过自主探索和验证，生成准确的 SQL？** 论文提出了 TRUST-SQL 框架，将该任务从“静态翻译”转变为“多轮交互式决策”，旨在解决未知模式下的幻觉问题和长路径交互中的信用分配（Credit Assignment）难题。

#### 2. 方法论：核心思想与关键技术
TRUST-SQL 将任务建模为**部分可观测马尔可夫决策过程（POMDP）**，核心包含两个部分：

*   **四阶段交互协议（EPGC）**：
    1.  **探索（Explore）**：调用工具查询数据库元数据（如表名、列名、外键）。
    2.  **提议（Propose）**：强制性的认知检查点，要求智能体提交已验证的模式知识，以抑制基于先验知识的幻觉。
    3.  **生成（Generate）**：基于提议的模式生成候选 SQL 并观察执行结果。
    4.  **确认（Confirm）**：提交最终 SQL。
*   **双轨 GRPO（Dual-Track GRPO）策略**：
    *   基于 DeepSeek 的组相对策略优化（GRPO）进行改进。
    *   **双轨设计**：将轨迹分为“模式轨道（Schema Track）”和“全轨迹轨道（Full Track）”。
    *   **独立奖励**：模式轨道根据提议模式与真实模式的匹配度给予奖励；全轨迹轨道根据 SQL 执行结果和格式规范给予奖励。
    *   **令牌级掩码优势（Token-level Masked Advantages）**：通过掩码技术，确保探索奖励仅影响探索阶段的 Token，执行奖励影响全局，从而精准解决长路径下的信用分配问题。

*   **超越特权基准**：尽管 TRUST-SQL 不预加载任何模式，其表现依然媲美或超过了那些预先知道完整模式的强基准模型。

#### 7. 优点
*   **场景真实**：直面企业级数据库中“模式不可观测”的痛点，具有极强的实用价值。
*   **算法创新**：双轨 GRPO 巧妙地利用协议边界解决了多轮交互中的奖励混淆问题。

#### 8. 不足与局限
*   **推理开销**：多轮交互自然比单轮翻译带来更高的推理延迟和 API 调用成本。
