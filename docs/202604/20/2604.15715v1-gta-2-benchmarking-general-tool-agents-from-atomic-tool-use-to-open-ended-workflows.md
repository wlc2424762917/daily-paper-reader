## 摘要
通用智能体的发展需要从执行简单指令转向完成复杂的现实世界生产力工作流。然而，目前的工具使用评测基准仍与现实需求脱节，依赖于 AI 生成的查询、虚拟工具以及有限的系统级协作。为此，我们提出了 GTA-2，这是一个针对通用工具智能体（GTA）的分层评测基准，涵盖了从原子级工具使用到开放式工作流的范围。该基准基于现实世界的真实性，利用了真实的用户查询、已部署的工具和多模态上下文：(i) GTA-Atomic 继承自我们之前的 GTA 基准，用于评估短程、闭口式工具使用的精准度；(ii) GTA-Workflow 引入了长程、开放式任务，用于评估现实的端到端完成情况。为了评估开放式交付成果，我们提出了一种基于递归检查点的评估机制，将目标分解为可验证的子目标，从而实现对模型能力和智能体执行框架（即执行容器）的统一评估。实验揭示了显著的能力断层：虽然前沿模型在原子任务上已经表现吃力（低于 50%），但它们在工作流任务上基本宣告失败，顶尖模型的成功率仅为 14.39%。进一步分析表明，基于检查点的引导反馈可以提升性能，而 Manus 和 OpenClaw 等先进框架显著增强了工作流的完成度，这凸显了在底层模型能力之外，执行框架设计的重要性。这些发现为开发可靠的个人和专业助手提供了指导。数据集和代码将在 https://github.com/open-compass/GTA 发布。

## Abstract
The development of general-purpose agents requires a shift from executing simple instructions to completing complex, real-world productivity workflows. However, current tool-use benchmarks remain misaligned with real-world requirements, relying on AI-generated queries, dummy tools, and limited system-level coordination. To address this, we propose GTA-2, a hierarchical benchmark for General Tool Agents (GTA) spanning atomic tool use and open-ended workflows. Built on real-world authenticity, it leverages real user queries, deployed tools, and multimodal contexts. (i) GTA-Atomic, inherited from our prior GTA benchmark, evaluates short-horizon, closed-ended tool-use precision. (ii) GTA-Workflow introduces long-horizon, open-ended tasks for realistic end-to-end completion. To evaluate open-ended deliverables, we propose a recursive checkpoint-based evaluation mechanism that decomposes objectives into verifiable sub-goals, enabling unified evaluation of both model capabilities and agent execution frameworks (i.e., execution harnesses). Experiments reveal a pronounced capability cliff: while frontier models already struggle on atomic tasks (below 50%), they largely fail on workflows, with top models achieving only 14.39% success. Further analysis shows that checkpoint-guided feedback improves performance, while advanced frameworks such as Manus and OpenClaw substantially enhance workflow completion, highlighting the importance of execution harness design beyond the underlying model capacity. These findings provide guidance for developing reliable personal and professional assistants. Dataset and code will be available at https://github.com/open-compass/GTA.

#### 1. 核心问题与整体含义（研究动机和背景）
随着大语言模型（LLM）向自主智能体（Agents）演进，其核心能力已从简单的指令遵循转向处理复杂的现实世界生产力工作流。然而，现有的工具使用基准测试（如 ToolBench, APIBank 等）存在三大缺陷：
*   **脱离现实**：多依赖 AI 生成的查询，而非真实用户需求。
*   **环境简化**：使用虚拟（Dummy）工具或纯文本环境，缺乏真实部署的工具和多模态上下文。
*   **维度单一**：侧重于短程（Short-horizon）的原子级工具调用，无法评估长程（Long-horizon）、开放式的工作流完成能力。
**GTA-2** 的提出旨在填补这一空白，通过构建一个分层的、基于真实世界场景的基准，系统性地评估智能体从基础工具精度到复杂跨域工作流掌控的全方位能力。

#### 2. 论文提出的方法论
GTA-2 采用分层架构，由两个独立但互补的部分组成：
*   **GTA-Atomic（原子层）**：继承自前作 GTA，专注于评估短程、闭口任务的工具调用精度（如：选择哪个工具、参数是否正确）。
*   **GTA-Workflow（工作流层）**：核心创新点，针对长程、开放式生产力任务。
    *   **核心思想**：不再评估执行路径（Trajectory），而是评估最终交付物（Deliverables）。
    *   **递归检查点评估机制（Recursive Checkpoint-based Evaluation）**：
        1.  将复杂的总目标分解为树状结构的**可验证子目标（Checkpoints）**。
        2.  每个叶子节点代表一个具体的、可客观评估的要求（如“生成一个 2.5-3.5 分钟的音频”）。
        3.  使用强模型（如 GPT-5.2）作为裁判（LLM Judge），对叶子节点按 0-10 分评分。
        4.  通过递归加权聚合，计算父节点及根节点的最终得分。
    *   **执行容器（Execution Harness）评估**：不仅评估底层 LLM，还评估智能体框架（如 Manus, OpenClaw）在规划、内存管理和错误恢复方面的系统级表现。

#### 6. 论文的主要结论与发现
*   **能力断崖（Capability Cliff）**：前沿模型在原子任务上表现尚可（<50%），但在工作流任务中集体“折戟”，顶尖模型（Gemini-2.5-Pro）的成功率仅为 **14.39%**。
*   **框架胜过模型**：先进的执行框架（如 Manus, OpenClaw）能显著提升任务完成度（成功率从 0% 提升至 50% 以上），证明了系统级设计（规划、内存、重试机制）在长程任务中比底层模型能力更关键。
*   **失败主因**：长程任务的失败主要源于**执行不稳定性**和**交付物格式/整合错误**，而非单纯的逻辑推理错误。
*   **反馈有效性**：细粒度的检查点反馈比模糊的错误提示能更有效地帮助智能体自我修正。

#### 7. 优点：方法或实验设计亮点
*   **真实性极高**：坚持使用真实工具和多模态输入，避免了“实验室环境”下的虚高表现。
*   **评估范式创新**：提出的递归检查点机制解决了开放式任务难以自动量化评估的难题，实现了“过程灵活、结果严格”的平衡。
*   **系统级视角**：首次将“执行框架（Harness）”作为核心变量进行评估，为未来 Agent 的工程化落地指明了方向。

#### 8. 不足与局限
*   **构建偏差**：虽然源自真实查询，但任务的重构和检查点生成仍依赖 LLM，可能引入模型偏好或特定的任务模式。
*   **安全评估缺失**：目前的基准侧重于“能力”，未充分考虑智能体在调用真实工具时的安全性、隐私保护和权限控制。
*   **失败分类的启发式属性**：失败原因的分类（如 PLAN, EXECUTE 等）带有一定的人为定义色彩，可能无法完全覆盖所有复杂的因果链条。
*   **成本问题**：使用强模型作为 Judge 进行大规模评测的成本较高，限制了其在小规模开发中的频繁使用。
