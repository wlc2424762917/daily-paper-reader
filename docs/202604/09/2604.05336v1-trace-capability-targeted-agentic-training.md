## 摘要
部署在智能体环境中的大语言模型（LLMs）必须在不同的任务实例中行使多种能力，其中能力是指在轨迹中执行一个或多个动作，这些动作对于成功解决环境中的子集任务是必需的。许多现有方法要么依赖于并非针对模型在目标环境中实际能力缺陷的合成训练数据，要么直接在目标环境上进行训练，使得模型需要跨任务隐式地学习这些能力。我们提出了 TRACE（将循环智能体失败转化为针对能力的训练环境），这是一个用于特定环境智能体自我改进的端到端系统。TRACE 通过对比成功和失败的轨迹来自动识别缺失的能力，并为每种能力合成一个针对性的训练环境，该环境会根据是否行使了该能力提供奖励。随后，TRACE 通过强化学习在每个合成环境上训练 LoRA 适配器，并在推理时路由至相关的适配器。实验结果表明，TRACE 在不同环境中具有良好的泛化性，在 $τ^2$-bench（客户服务）上比基础智能体提升了 14.1 分，在 ToolSandbox（工具使用）上增加了 7 个满分，分别比最强基线高出 7.4 分和 4 个满分。在给定相同数量的展开（rollouts）时，TRACE 的扩展效率优于基线，在 $τ^2$-bench 上分别超过 GRPO 和 GEPA 9.2 分和 7.4 分。

## Abstract
Large Language Models (LLMs) deployed in agentic environments must exercise multiple capabilities across different task instances, where a capability is performing one or more actions in a trajectory that are necessary for successfully solving a subset of tasks in the environment. Many existing approaches either rely on synthetic training data that is not targeted to the model's actual capability deficits in the target environment or train directly on the target environment, where the model needs to implicitly learn the capabilities across tasks. We introduce TRACE (Turning Recurrent Agent failures into Capability-targeted training Environments), an end-to-end system for environment-specific agent self-improvement. TRACE contrasts successful and failed trajectories to automatically identify lacking capabilities, synthesizes a targeted training environment for each that rewards whether the capability was exercised, and trains a LoRA adapter via RL on each synthetic environment, routing to the relevant adapter at inference. Empirically, TRACE generalizes across different environments, improving over the base agent by +14.1 points on $τ^2$-bench (customer service) and +7 perfect scores on ToolSandbox (tool use), outperforming the strongest baseline by +7.4 points and +4 perfect scores, respectively. Given the same number of rollouts, TRACE scales more efficiently than baselines, outperforming GRPO and GEPA by +9.2 and +7.4 points on $τ^2$-bench.

### 1. 核心问题与整体含义（研究动机和背景）
论文关注的核心问题是：**如何高效地提升大语言模型（LLM）智能体在特定复杂环境中的表现？**
*   **研究动机**：现有的智能体训练方法（如直接在目标环境进行强化学习或使用通用合成数据）存在效率低下的问题。这是因为目标环境的奖励信号往往是稀疏且模糊的，无法告诉模型具体是哪种底层能力（如数据检索、逻辑推理、工具调用精度等）导致了失败。
*   **核心理念**：TRACE 认为，与其让模型在复杂任务中隐式学习，不如先诊断出模型最缺什么能力，然后为这些能力“量身定制”简单的模拟训练场，让模型在这些专项训练场中快速进化。

### 2. 方法论：核心思想与技术流程
TRACE（Turning Recurrent Agent failures into Capability-targeted training Environments）是一个端到端的自我改进系统，包含四个关键阶段：
1.  **对比式能力识别（Contrastive Identification）**：
    *   让基础模型在目标环境中运行，收集成功和失败的轨迹。
    *   使用一个“分析智能体”对比两类轨迹，识别出导致失败的共性能力缺陷（如“结构化数据推理”）。
    *   通过计算“对比差距”（Contrastive Gap）和“失败覆盖率”来筛选高影响力的能力。
2.  **针对性环境合成（Environment Synthesis）**：
    *   为每种识别出的能力缺陷自动生成一个合成的、可验证的训练环境。
    *   这些环境保留了目标环境的工具 Schema 和交互协议，但任务被简化为专门考察该特定能力。
    *   环境能够根据随机种子程序化生成无限的训练实例，并提供密集的、自动化的奖励信号。
3.  **强化学习训练（RL Training）**：
    *   针对每个合成环境，冻结基础模型，仅训练一个专门的 **LoRA 适配器**。
    *   使用 **GRPO**（组相对策略优化）算法，通过组内轨迹的奖励方差来提供学习信号，无需显式的价值函数。
4.  **能力组合与路由（Composition & Routing）**：
    *   在推理阶段，系统不合并权重，而是使用基础模型作为“路由器”。
    *   根据当前任务的自然语言描述，动态选择并激活最相关的 LoRA 适配器。

### 6. 主要结论与发现
*   **性能显著提升**：TRACE 在 $\tau^2$-Bench 上比基础模型提升了 **14.1%** 的成功率，在 ToolSandBox 上增加了 **7 个** 满分。
*   **扩展效率极高**：在相同的训练数据量（Rollouts）下，TRACE 的表现远超直接 RL（GRPO）和提示词优化（GEPA）。
*   **路由优于合并**：实验证明，动态路由单个能力适配器的效果优于将多个能力强行合并到一个模型中，避免了参数干扰。
*   **训练胜过提示**：即使是经过优化的提示词，其效果也无法达到针对性 RL 训练的高度。

### 7. 优点：亮点总结
*   **精准诊断**：通过对比分析自动定位“病灶”，避免了盲目的大规模合成数据训练。
*   **信度分配清晰**：合成环境将复杂任务解耦，使强化学习的奖励信号直接作用于特定能力，极大提高了采样效率。
*   **模块化扩展**：采用 LoRA 适配器 + 路由的架构，使得系统可以像插件一样轻松添加新能力，而不会导致旧能力的灾难性遗忘。

### 8. 不足与局限
*   **级联依赖**：系统的有效性高度依赖于“分析智能体”识别能力的准确性以及“生成智能体”合成环境的质量。如果分析出错，后续训练将偏离目标。
*   **路由开销**：虽然单次路由开销小，但如果任务极其微小且对延迟极度敏感，路由过程仍是一个额外负担。
*   **能力重叠**：目前的路由策略是“单选”，对于某些需要同时深度运用多种能力的极复杂任务，可能需要更复杂的混合专家（MoE）路由机制。
*   **环境闭环**：目前主要针对可程序化验证的环境，对于缺乏明确成功准则（如开放式创意写作）的智能体任务，合成环境的构建会非常困难。
