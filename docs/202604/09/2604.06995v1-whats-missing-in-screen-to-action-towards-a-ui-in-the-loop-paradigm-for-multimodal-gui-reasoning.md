## 摘要
现有的图形用户界面（GUI）推理任务仍然具有挑战性，特别是在 UI 理解方面。当前的方法通常依赖于直接基于屏幕的决策，这缺乏可解释性，并忽视了对 UI 元素的全面理解，最终导致任务失败。为了增强对 UI 的理解和交互，我们提出了一种创新的 GUI 推理范式，称为 UI 在环（UILoop）。我们的方法将 GUI 推理任务视为一个循环的“屏幕-UI 元素-动作”过程。通过使多模态大语言模型（MLLM）显式地学习关键 UI 元素的定位、语义功能和实际用法，UILoop 实现了精确的元素发现并执行可解释的推理。此外，我们引入了一个更具挑战性的、以 UI 元素为中心的 UI 理解任务，并提出了三个评估指标。相应地，我们贡献了一个包含 2.6 万个样本的基准测试（UI Comprehension-Bench），以全面评估现有方法对 UI 元素的掌握程度。大量实验表明，UILoop 在实现最先进的 UI 理解性能的同时，在 GUI 推理任务中也取得了优异的结果。

## Abstract
Existing Graphical User Interface (GUI) reasoning tasks remain challenging, particularly in UI understanding. Current methods typically rely on direct screen-based decision-making, which lacks interpretability and overlooks a comprehensive understanding of UI elements, ultimately leading to task failure. To enhance the understanding and interaction with UIs, we propose an innovative GUI reasoning paradigm called UI-in-the-Loop (UILoop). Our approach treats the GUI reasoning task as a cyclic Screen-UI elements-Action process. By enabling Multimodal Large Language Models (MLLMs) to explicitly learn the localization, semantic functions, and practical usage of key UI elements, UILoop achieves precise element discovery and performs interpretable reasoning. Furthermore, we introduce a more challenging UI Comprehension task centered on UI elements with three evaluation metrics. Correspondingly, we contribute a benchmark of 26K samples (UI Comprehension-Bench) to comprehensively evaluate existing methods' mastery of UI elements. Extensive experiments demonstrate that UILoop achieves state-of-the-art UI understanding performance while yielding superior results in GUI reasoning tasks.

### 2. 方法论：UI-in-the-Loop (UILoop)
*   **核心思想**：将 GUI 推理重构为循环的“**屏幕-UI 元素-动作**”（Screen-UI Elements-Action）范式。模型在输出动作前，必须先显式地识别关键 UI 元素、描述其语义功能并推断其用法。
*   **关键技术细节**：

    *   **UI 元素驱动的强化学习微调 (RFT)**：采用 **GRPO**（群组相对策略优化）算法进行训练。
    *   **多维奖励函数设计**：
        1.  **格式奖励 (Format Reward)**：确保模型输出符合特定的 XML 标签格式。
        2.  **定位奖励 (Location Reward)**：计算预测坐标与真实坐标的欧氏距离。
        3.  **语义化奖励 (Lingualization Reward)**：评估模型对 UI 元素功能描述的语义相似度。
        4.  **利用奖励 (Leverage Reward)**：评估模型基于该元素生成的动作类型和参数是否准确。
    *   **训练逻辑**：设计了启发式阈值，只有当模型准确识别和理解了 UI 元素时，才会给予利用奖励，从而强制模型先学好“看”再学“做”。

### 6. 主要结论与发现
*   **性能领先**：UILoop 在 Android Control-High 和 ScreenSpot-Pro 上均取得了 SOTA 性能，显著优于传统的“屏幕到动作”模型。

*   **可解释性增强**：通过显式的推理链（Thought），UILoop 能够清晰地展示其识别了哪个元素以及为何点击该元素。

### 7. 优点
*   **范式创新**：提出了更符合人类逻辑的“UI 在环”推理模式，打破了端到端黑盒推理的局限。
*   **高质量资源**：贡献了包含 26K 细粒度标注的 UI Comprehension-Bench，填补了社区在 UI 语义理解数据集上的空白。
*   **强化学习策略**：巧妙地将 UI 理解的中间步骤转化为 RL 的奖励项，有效引导了模型的学习方向。

### 8. 不足与局限
*   **布局粒度**：目前主要关注单个 UI 元素，对于由多个元素组成的“粗粒度布局”或区域级理解尚有不足。
*   **模型覆盖**：实验主要基于 Qwen2.5-VL 架构，尚未在 Llama-V 或其他闭源模型架构上进行广泛验证。
*   **推理开销**：由于引入了显式的 UI 描述和推理步骤，推理时的 Token 消耗和延迟可能会有所增加。
