## 摘要
Web 智能体可自动执行浏览器任务，涵盖从简单的表单填写到如订购杂货等复杂的流程。虽然目前的基准测试评估了通用性能（如 WebArena）或针对恶意行为的安全性（如 SafeArena），但尚无现有框架评估智能体成功执行面向用户的网站安全与隐私任务的能力，例如管理 Cookie 偏好、配置隐私敏感的账户设置或撤销非活跃会话。为了填补这一空白，我们推出了 WebSP-Eval，这是一个用于衡量 Web 智能体在网站安全与隐私任务上表现的评估框架。WebSP-Eval 包含：1) 一个包含 28 个网站、200 个任务实例的人工构建任务数据集；2) 一个强大的智能体系统，利用自定义 Google Chrome 扩展程序支持跨运行的账户和初始状态管理；以及 3) 一个自动评估器。我们使用最先进的多模态大语言模型评估了总共 8 种 Web 智能体实例化方案，并针对网站、任务类别和 UI 元素进行了细粒度分析。我们的评估表明，当前模型在可靠解决网站安全与隐私任务方面的自主探索能力有限，且在特定任务类别和网站上表现欠佳。至关重要的是，我们发现有状态的 UI 元素（如开关和复选框）是智能体失败的主要原因，在包含这些元素的任务中，许多模型的失败率超过 45%。

## Abstract
Web agents automate browser tasks, ranging from simple form completion to complex workflows like ordering groceries. While current benchmarks evaluate general-purpose performance~(e.g., WebArena) or safety against malicious actions~(e.g., SafeArena), no existing framework assesses an agent's ability to successfully execute user-facing website security and privacy tasks, such as managing cookie preferences, configuring privacy-sensitive account settings, or revoking inactive sessions. To address this gap, we introduce WebSP-Eval, an evaluation framework for measuring web agent performance on website security and privacy tasks. WebSP-Eval comprises 1) a manually crafted task dataset of 200 task instances across 28 websites; 2) a robust agentic system supporting account and initial state management across runs using a custom Google Chrome extension; and 3) an automated evaluator. We evaluate a total of 8 web agent instantiations using state-of-the-art multimodal large language models, conducting a fine-grained analysis across websites, task categories, and UI elements. Our evaluation reveals that current models suffer from limited autonomous exploration capabilities to reliably solve website security and privacy tasks, and struggle with specific task categories and websites. Crucially, we identify stateful UI elements such as toggles and checkboxes are a primary reason for agent failure, failing at a rate of more than 45\% in tasks containing these elements across many models.

随着多模态大模型（MLLM）驱动的 Web 智能体（如 OpenAI 的 Operator、Perplexity 的 Comet）日益普及，用户开始授权智能体处理复杂的浏览器任务。然而，现有的基准测试（如 WebArena, WebVoyager）主要关注信息检索或通用任务，或者关注智能体是否会执行恶意操作（如 SafeArena）。
**核心问题：** 目前缺乏评估 Web 智能体执行“面向用户的安全与隐私任务”能力的框架。这类任务包括管理 Cookie 偏好、配置隐私设置、撤销非活跃会话等。如果智能体在这些任务中出错，可能会直接导致用户账户安全削弱或隐私数据泄露。

*   **任务策划（Task Curation）：** 手工构建了包含 138 个独特任务、200 个任务实例的数据集，涵盖 28 个流行网站。任务被分为 9 个类别（如账户安全、Cookie 管理、社交安全等）。
*   **智能体实例化与状态管理（Agent Instantiation）：** 
    *   基于 WebVoyager 进行了扩展，增强了动作空间（如“滚动到末尾”、“在弹窗内滚动”等）。

*   **自动化验证（Automated Verification）：** 采用 **MLLM-as-a-Judge** 方案。使用 Gemini 3 Pro、Gemini 3.1 Pro 和 Claude 4.6 Opus 组成集成评估器，通过分析智能体的完整操作轨迹和截图来判定任务是否成功。

*   **数据集/场景：** 28 个真实网站（如 Amazon, GitHub, Reddit, Steam 等），200 个任务实例。

#### 5. 实验数量与充分性
*   **实验规模：** 对 8 个模型在 200 个实例上进行了双变体（有/无导航）测试，总计生成了数千条操作轨迹。

#### 6. 主要结论与发现
*   **自主探索能力不足：** 所有模型在失去显式导航指令时性能均显著下降。Gemini 3 Pro 表现最佳（成功率 76.5%），而小型模型（如 Gemini 2.5 Flash）下降幅度达 11.5%。
*   **有状态 UI 元素是“重灾区”：** 智能体在处理“开关（Toggle）”和“复选框（Checkbox）”时失败率极高（部分模型超过 45%）。
*   **状态感知偏差：** 模型存在“行动偏见”，即即使初始状态已经符合任务要求（例如要求关闭已关闭的开关），智能体仍倾向于去点击它，导致结果反转。
*   **开源模型差距：** 开源模型 Gemma-3-27B 在复杂安全任务上的表现远逊于闭源推理模型。

#### 7. 优点
*   **填补空白：** 首次针对 Web 智能体在安全与隐私配置这一关键领域的表现进行了系统性评估。
*   **创新的基础设施：** 开发的 Chrome 扩展程序解决了实时网站评估中“状态一致性”的痛点，具有很高的实用价值。
*   **细粒度的失败分析：** 深入探讨了哪些具体的 UI 元素导致了模型失败，为未来改进智能体提供了明确方向。

#### 8. 不足与局限
*   **实时网站的波动性：** 实验依赖实时网站，网站 UI 的更新可能导致录制的脚本失效，维护成本高。
*   **地理位置偏差：** 网站的隐私选项（如 Cookie 弹窗）常因地区（GDPR/CCPA）而异，实验结果可能受地理位置影响。
*   **复现难度：** 由于涉及人工创建的账户和复杂的登录流程，其他研究者完全复现实验结果存在较高的门槛。
*   **覆盖范围：** 避开了需要生物识别或强制多因素认证（MFA）的极高安全性网站（如银行）。
