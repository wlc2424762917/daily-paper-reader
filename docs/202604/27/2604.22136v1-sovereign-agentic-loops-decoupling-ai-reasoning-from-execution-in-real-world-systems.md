## 摘要
大语言模型 (LLM) 智能体越来越多地发出改变现实系统的 API 调用，然而许多当前的架构直接将随机的模型输出传递给执行层。我们认为这种耦合产生了安全风险，因为在执行时无法假定模型的正确性、上下文感知能力和对齐性。我们引入了主权智能体循环 (SAL)，这是一种控制平面架构，其中模型发出带有理由的结构化意图，控制平面在执行前根据真实的系统状态和策略验证这些意图。SAL 结合了混淆膜（限制模型访问身份敏感状态）和加密链接的证据链（用于审计和重放）。我们对 SAL 进行了形式化，并证明在所述假设下，它提供了受策略约束的执行、身份隔离和确定性重放。在针对云基础设施的 OpenKedge 原型中，SAL 在策略层拦截了 93% 的不安全意图，通过一致性检查拒绝了剩余的 7%，在我们的基准测试中防止了不安全执行，并增加了 12.4 毫秒的中值延迟。

## Abstract
Large language model (LLM) agents increasingly issue API calls that mutate real systems, yet many current architectures pass stochastic model outputs directly to execution layers. We argue that this coupling creates a safety risk because model correctness, context awareness, and alignment cannot be assumed at execution time. We introduce Sovereign Agentic Loops (SAL), a control-plane architecture in which models emit structured intents with justifications, and the control plane validates those intents against true system state and policy before execution. SAL combines an obfuscation membrane, which limits model access to identity-sensitive state, with a cryptographically linked Evidence Chain for auditability and replay. We formalize SAL and show that, under the stated assumptions, it provides policy-bounded execution, identity isolation, and deterministic replay. In an OpenKedge prototype for cloud infrastructure, SAL blocks 93% of unsafe intents at the policy layer, rejects the remaining 7% via consistency checks, prevents unsafe executions in our benchmark, and adds 12.4 ms median latency.

### 1. 核心问题与整体含义
*   **研究动机**：当前的 AI 智能体架构通常将 LLM 的输出直接视为可执行命令（API 调用）。由于 LLM 具有随机性（Stochastic）、可能产生幻觉或缺乏实时上下文感知，这种“推理与执行紧密耦合”的模式在操作关键基础设施（如数据库、电力网）时存在巨大风险。
*   **核心问题**：如何防止 AI 智能体因错误的推理而发出破坏性的指令？
*   **整体含义**：论文提出了“解耦原则”（Decoupling Principle），主张将 AI 的**语义推理**与**执行权限**分离。AI 只负责提出“意图（Intent）”，而由一个受控的“主权控制平面”负责验证并最终执行，从而确保系统安全不受 AI 随机性的影响。

### 2. 方法论
SAL 架构通过以下关键技术实现推理与执行的解耦：
*   **混淆膜（Obfuscation Membrane, $\Pi$）**：在将系统状态发送给 AI 前，剥离身份敏感信息（如 IP 地址、具体 ID），仅保留结构化拓扑。这确保了 AI 在不接触敏感数据的情况下进行推理，维护了数据主权。
*   **意图协议（Intent Protocol）**：AI 不直接调用 API，而是输出结构化的“意图”和“理由（Justification）”。
*   **确定性主权评估（Sovereign Evaluation）**：
    *   **策略对齐（$E_{policy}$）**：使用 Open Policy Agent (OPA) 等引擎，根据预设规则（如“禁止删除生产环境数据库”）检查意图。
    *   **一致性检查（$E_{consistency}$）**：对比 AI 提供的理由与真实的系统遥测数据。如果 AI 声称“因 CPU 过高而重启”但实际 CPU 仅 10%，则拒绝执行。
*   **加密证据链（Evidence Chain）**：将状态、意图、理由、评估结果和执行动作通过哈希函数链接，形成不可篡改的账本，支持审计和确定性重放。

### 6. 主要结论与发现
*   **安全性**：SAL 成功拦截了 **100%** 的不安全意图。其中 93% 被策略层拦截，剩余 7%（语义矛盾类）被一致性检查层拦截。

*   **可审计性**：通过证据链实现了 100% 的确定性重放验证，证明了审计机制的有效性。

### 7. 优点
*   **架构级安全**：不依赖于 LLM 本身的“对齐”或“提示词工程”，而是通过外部硬性约束确保安全。
*   **主权保护**：混淆膜设计巧妙地解决了“推理需要上下文”与“主权要求隐私”之间的矛盾。
*   **实用性强**：极低的延迟开销使其具备在生产环境中大规模部署的潜力。

### 8. 不足与局限
*   **策略依赖性**：系统的安全性高度依赖于人类编写的 OPA 策略和一致性检查逻辑的完备性。
*   **封闭执行假设**：SAL 要求所有系统变更必须通过控制平面。在存在遗留系统或多路径操作的环境中，难以强制执行“封闭执行”。
*   **应用领域局限**：目前仅在云基础设施领域验证，对于物理机器人或毫秒级实时控制系统（如自动驾驶），12.4ms 的延迟和复杂的意图解析可能仍显过重。
