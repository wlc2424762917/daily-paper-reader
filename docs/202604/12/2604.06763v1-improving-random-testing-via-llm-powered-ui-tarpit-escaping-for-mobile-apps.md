## 摘要
随机 GUI 测试是一种广泛用于移动应用测试的技术。然而，其有效性受限于一个臭名昭著的问题——UI 探索陷阱（UI exploration tarpits），即探索过程被困在局部 UI 区域，从而阻碍了测试覆盖率和缺陷发现。在本篇经验论文中，我们介绍了 LLM 驱动的随机 GUI 测试，这是一种旨在缓解随机测试过程中 UI 陷阱的新型混合测试方法。我们的方法通过监测 UI 相似性来识别陷阱，并查询大语言模型（LLM）以建议有助于逃离所遇陷阱的潜在事件。我们在两种不同的移动应用自动化输入生成（AIG）工具之上实现了该方法：(1) 基于业界常用工具 Monkey 的 HybridMonkey；(2) 基于最先进工具 Droidbot 的 HybridDroidbot。我们在 12 个流行的真实应用上对其进行了评估。结果显示，HybridMonkey 和 HybridDroidbot 优于所有基准方法，平均覆盖率分别提升了 54.8% 和 44.8%，并检测到了数量最多的唯一崩溃。我们总共发现了 75 个唯一缺陷，其中包括 34 个此前未知的缺陷。截至目前，已有 26 个缺陷得到确认并修复。我们还将 HybridMonkey 应用于拥有数十亿月活跃用户的流行工业级应用微信（WeChat）。HybridMonkey 实现了比随机测试更高的 Activity 覆盖率，并发现了更多缺陷。

## Abstract
Random GUI testing is a widely-used technique for testing mobile apps. However, its effectiveness is limited by the notorious issue -- UI exploration tarpits, where the exploration is trapped in local UI regions, thus impeding test coverage and bug discovery. In this experience paper, we introduce LLM-powered random GUI Testing, a novel hybrid testing approach to mitigating UI tarpits during random testing. Our approach monitors UI similarity to identify tarpits and query LLMs to suggest promising events for escaping the encountered tarpits. We implement our approach on top of two different automated input generation (AIG) tools for mobile apps: (1) HybridMonkey upon Monkey, a state-of-the-practice tool; and (2) HybridDroidbot upon Droidbot, a state-of-the-art tool. We evaluated them on 12 popular, real-world apps. The results show that HybridMonkey and HybridDroidbot outperform all baselines, achieving average coverage improvements of 54.8% and 44.8%, respectively, and detecting the highest number of unique crashes. In total, we found 75 unique bugs, including 34 previously unknown bugs. To date, 26 bugs have been confirmed and fixed. We also applied HybridMonkey on WeChat, a popular industrial app with billions of monthly active users. HybridMonkey achieved higher activity coverage and found more bugs than random testing.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：随机 GUI 测试（如 Android Monkey）虽然高效且能触达深层状态，但由于缺乏语义理解，极易陷入 **“UI 探索陷阱”（UI Tarpits）**。在这种状态下，测试工具在视觉相似的局部区域内循环往复（如不断点击无效按钮或在两个页面间切换），导致约 50% 的测试时间被浪费，严重阻碍了代码覆盖率的提升和深层漏洞的发现。
*   **研究背景**：现有的解决方法（如 Vet 或 Aurora）要么过于保守（直接禁用某些动作），要么依赖硬编码的启发式规则，难以应对复杂多变的现代 App 界面。

### 2. 方法论
该论文提出了一种名为 **LLM-powered Random Testing** 的混合框架，核心思想是：**“随机测试负责深度探索，LLM 负责跳出陷阱”**。
*   **动态 UI 陷阱检测**：通过监控执行过程中的 UI 页面序列，利用**感知哈希（Perceptual Hash）**计算连续 $k$ 帧（默认 $k=8$）界面的视觉相似度。如果相似度超过阈值，则判定为陷入陷阱。
*   **LLM 驱动的逃逸机制**：
    *   **提示词构造（Prompting）**：将 UI 布局树线性化为文本，并引入**空间遮挡过滤算法**，剔除被浮窗或弹窗遮挡的无效组件，确保 LLM 看到的上下文是准确的。
    *   **局部历史感知**：向 LLM 提供当前陷阱内的失败尝试记录，避免重复无效操作。
*   **效率优化技术**：
    *   **陷阱记忆库（Tarpit Memorizer）**：记录成功逃离特定陷阱的动作。
    *   **概率性重用（Probabilistic Reuse）**：再次遇到相同陷阱时，以 80% 的概率重用历史成功动作，20% 的概率请求 LLM 生成新动作，平衡了成本与探索多样性。

### 6. 主要结论与发现
*   **覆盖率提升**：HybridMonkey 在行覆盖率和 Activity 覆盖率上分别比基准工具平均提升了 54.8% 和 44.8%。
*   **漏洞发现**：在 12 个 App 中共发现 **75 个唯一漏洞**，其中 **34 个为此前未知的 Bug**（26 个已被开发者确认或修复）。
*   **逃逸有效性**：LLM 逃逸成功率达到 72.85%，且 82.55% 的陷阱能在第一次尝试时即被 LLM 成功破解。
*   **工业价值**：在微信上的实验证明，该方法能有效提升超大型 App 的 Activity 遍历深度。

### 7. 优点与亮点
*   **混合架构的平衡性**：巧妙结合了随机测试的低成本高频率和 LLM 的高语义理解力，避免了全量使用 LLM 带来的高昂成本和响应延迟。
*   **视觉驱动的鲁棒性**：使用图像相似度而非布局树对比来检测陷阱，能更好地处理动态加载、ID 变化等布局树不稳定的情况。
*   **即插即用**：该方法作为增强模块，可以轻松集成到现有的任何随机测试工具（如 Monkey 或 Droidbot）中。

### 8. 不足与局限
*   **元数据依赖**：约 65% 的逃逸失败归因于 Android 布局树中缺乏文本标签或描述，导致 LLM 无法理解组件功能。
*   **逻辑循环限制**：目前的检测机制仅针对“视觉相似”的陷阱，对于跨越多个不同页面的“逻辑死循环”（如 A->B->C->A）尚无法有效识别。
*   **复杂输入难题**：在需要特定格式数据（如注册表单、多字段关联输入）的场景下，LLM 的引导能力仍受限。
