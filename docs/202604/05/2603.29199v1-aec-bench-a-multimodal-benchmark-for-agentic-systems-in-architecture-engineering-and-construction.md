## 摘要
AEC-Bench 是一个多模态基准测试，用于评估建筑、工程与施工 (AEC) 领域真实任务中的智能体系统。该基准涵盖了需要图纸理解、跨图纸推理以及建筑项目级协调的任务。本报告介绍了该基准的动机、数据集分类、评估协议，以及在多个领域特定基础模型框架下的基准测试结果。我们利用 AEC-Bench 识别出了一系列一致的工具和框架设计技术，这些技术能够统一提升基础模型在其自身基础框架（如 Claude Code 和 Codex）中的性能。我们在 Apache 2 许可证下公开了基准数据集、智能体框架和评估代码，以实现完全的可重复性，项目地址为 https://github.com/nomic-ai/aec-bench。

## Abstract
The AEC-Bench is a multimodal benchmark for evaluating agentic systems on real-world tasks in the Architecture, Engineering, and Construction (AEC) domain. The benchmark covers tasks requiring drawing understanding, cross-sheet reasoning, and construction project-level coordination. This report describes the benchmark motivation, dataset taxonomy, evaluation protocol, and baseline results across several domain-specific foundation model harnesses. We use AEC-Bench to identify consistent tools and harness design techniques that uniformly improve performance across foundation models in their own base harnesses, such as Claude Code and Codex. We openly release our benchmark dataset, agent harness, and evaluation code for full replicability at https://github.com/nomic-ai/aec-bench under an Apache 2 license.

#### 6. 论文的主要结论与发现
*   **检索是主要瓶颈**：智能体往往在找到正确图纸之前就失败了。一旦通过结构化解析（H+）改善了检索，性能在检索敏感型任务上平均提升了 18%-32%。
*   **视觉接地（Visual Grounding）依然薄弱**：即使提供了图像，智能体在需要追踪引线或理解几何关系的视觉任务上表现依然很差（如注释准确性任务得分下降）。
*   **路径依赖**：智能体倾向于使用“代码思维”，过度依赖 `pdftotext` 等文本工具，导致空间信息的丢失。
*   **复杂判断失效**：在报审文件审查（Submittal Review）等需要高层级工程判断的任务中，所有模型表现均不佳（最高分仅 23.1）。

#### 7. 优点
*   **真实性强**：任务设计紧贴 AEC 行业专家的实际工作流，而非简单的视觉问答。
*   **开源贡献**：公开了数据集、智能体框架和评估代码，具有极高的可重复性。
*   **多模态深度**：不仅测试文本，还深入探讨了图纸几何结构与文本之间的对齐问题。

#### 8. 不足与局限
*   **样本量限制**：实例数量（196个）相对较少，可能无法完全覆盖 AEC 领域的所有学科（如机电、暖通等）的复杂性。

*   **模型偏差**：实验中观察到智能体存在严重的“假阳性”倾向，即在复杂任务中过度生成错误发现。
