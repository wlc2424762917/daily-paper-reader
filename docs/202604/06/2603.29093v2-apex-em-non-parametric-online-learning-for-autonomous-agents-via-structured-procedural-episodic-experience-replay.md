## 摘要
基于大语言模型（LLM）的自主智能体缺乏持久的程序性记忆：即使之前已经解决过结构相同的任务，它们仍会从头开始推导解决方案。我们提出了 APEX-EM，这是一种非参数化在线学习框架，能够在不修改模型权重的情况下积累、检索和重用结构化程序计划。APEX-EM 引入了：(1) 一种结构化经验表示，编码了每次执行的完整程序-情境轨迹——包括规划步骤、中间产物、带有错误分析的迭代历史以及质量评分；(2) 一种包含任务验证器（Task Verifiers）的“规划-检索-生成-迭代-摄入”（PRGII）工作流，任务验证器提供多维奖励信号；(3) 一种双结果经验记忆，采用结合了语义搜索、结构特征匹配和计划有向无环图（DAG）遍历的混合检索方式——实现了在词汇无重叠但操作结构相似的任务之间的跨领域迁移。成功的经验作为正向上下文示例；失败的经验则作为带有结构化错误标注的负面示例。我们使用 Claude Sonnet 4.5 和 Opus 4.5 在 BigCodeBench、KGQAGen-10k 和 Humanity's Last Exam (HLE) 上进行了评估。在 KGQAGen-10k 上，APEX-EM 达到了 89.6% 的准确率，而无记忆情况下为 41.3%（提升了 48.3 个百分点），超过了真值检索（oracle-retrieval）的上限（84.9%）。在 BigCodeBench 上，其成功率从 53.9% 的基准提升至 83.3%（提升了 29.4 个百分点），在骨干网络冻结的同等条件下，超过了 MemRL 11.0 个百分点的增益（注：我们在分析中对骨干网络的差异进行了控制）。在 HLE 上，实体图检索从 25.2% 提升至 48.0%（提升了 22.8 个百分点）。消融实验表明，各组件的价值取决于具体任务：丰富的评判反馈对代码生成的影响微乎其微，但对结构化查询至关重要（提升了 10.3 个百分点），而基于二元信号的迭代可以部分补偿较弱的反馈。

## Abstract
LLM-based autonomous agents lack persistent procedural memory: they re-derive solutions from scratch even when structurally identical tasks have been solved before. We present APEX-EM, a non-parametric online learning framework that accumulates, retrieves, and reuses structured procedural plans without modifying model weights. APEX-EM introduces: (1) a structured experience representation encoding the full procedural-episodic trace of each execution -- planning steps, artifacts, iteration history with error analysis, and quality scores; (2) a Plan-Retrieve-Generate-Iterate-Ingest (PRGII) workflow with Task Verifiers providing multi-dimensional reward signals; and (3) a dual-outcome Experience Memory with hybrid retrieval combining semantic search, structural signature matching, and plan DAG traversal -- enabling cross-domain transfer between tasks sharing no lexical overlap but analogous operational structure. Successful experiences serve as positive in-context examples; failures as negative examples with structured error annotations.   We evaluate on BigCodeBench, KGQAGen-10k, and Humanity's Last Exam using Claude Sonnet 4.5 and Opus 4.5. On KGQAGen-10k, APEX-EM achieves 89.6% accuracy versus 41.3% without memory (+48.3pp), surpassing the oracle-retrieval upper bound (84.9%). On BigCodeBench, it reaches 83.3% SR from a 53.9% baseline (+29.4pp), exceeding MemRL's +11.0pp gain under comparable frozen-backbone conditions (noting backbone differences controlled for in our analysis). On HLE, entity graph retrieval reaches 48.0% from 25.2% (+22.8pp). Ablations show component value is task-dependent: rich judge feedback is negligible for code generation but critical for structured queries (+10.3pp), while binary-signal iteration partially compensates for weaker feedback.

#### 1. 核心问题与研究动机
*   **核心问题**：当前的 LLM 自主智能体在程序层面是“无状态”的。即使面对结构完全相同的任务，智能体每次都会从零开始推导方案，无法积累和复用过去的执行经验。
*   **研究动机**：现有的记忆机制（如 RAG 或 MemRL）多存储非结构化的文本摘要或简单的标量效用值（Q-value），丢失了执行过程中的程序化结构（如规划步骤、实体发现、错误轨迹等）。此外，现有系统难以实现跨领域的经验迁移（例如，将查询体育数据库的逻辑迁移到财务数据库）。

#### 2. 方法论：核心思想与技术细节
APEX-EM 提出了一种非参数化在线学习框架，其核心由以下三部分组成：
*   **程序化知识图谱 (PKG)**：将经验存储为具有五层反射结构的节点：
    1.  **目标反射 (RG)**：任务描述、约束和验证契约。
    2.  **程序反射 (RP)**：可复用的步骤模板。
    3.  **证据反射 (RE)**：执行过程中使用的工具输出或数据摘要。
    4.  **错误注册表 (Rerr)**：结构化的错误分类、根因分析及恢复策略。
    5.  **补丁反射 (Rpatch)**：针对特定错误的修复逻辑。
*   **PRGII 工作流**：
    1.  **规划 (Plan)**：分解任务，提取“结构化特征签名”（Structural Signature，即抽象操作序列，如 `解析实体 -> 时间过滤 -> 聚合`）。
    2.  **检索 (Retrieve)**：结合语义搜索、结构签名匹配（使用最长公共子序列 LCS）和图遍历，实现跨领域迁移。
    3.  **生成 (Generate)**：基于检索到的正向（成功）和负向（失败）示例生成代码或查询。
    4.  **迭代 (Iterate)**：通过验证器进行自我修正。
    5.  **摄入 (Ingest)**：由 Teacher 模型评分并存入记忆库。
*   **双结果记忆**：成功经验作为正向 Few-shot 示例，失败经验则作为带标注的负面示例，防止智能体重复同样的错误。

#### 6. 主要结论与发现
*   **性能飞跃**：在 KGQA 任务上，准确率从 41.3% 提升至 89.6%（+48.3pp），超过了拥有真值子图的 Oracle 基准。在 BCB 上提升了 29.4pp。

*   **跨领域迁移**：结构化签名匹配使得智能体能在词汇完全不同的任务间迁移逻辑，解决了纯语义检索在低相似度场景下的失效问题。

#### 7. 优点
*   **结构化深度**：不同于以往的扁平化记忆，PKG 提供了极高的可解释性和可审计性。
*   **跨领域能力**：通过抽象操作序列（Signature）实现了真正的程序逻辑迁移。
*   **双向学习**：系统地利用失败经验作为负面约束，这在 LLM 智能体领域较为少见且有效。
*   **非参数化**：无需微调模型即可实现持续在线学习，避免了灾难性遗忘。

#### 8. 不足与局限
*   **骨干网络差异**：虽然对比了 MemRL，但由于使用了不同的骨干网络（Claude vs GPT/Gemini），数值上的直接对比存在一定偏差，主要应看相对增益。

*   **收敛性证明**：作为非参数化方法，缺乏像强化学习那样的理论收敛性保证。
