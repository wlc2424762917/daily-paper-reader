## 摘要
多模态大语言模型（MLLMs）正从被动观察者演变为主动智能体，通过视觉扩展（调用视觉工具）和知识扩展（开放网络搜索）来解决问题。然而，现有的评估方法存在不足：缺乏灵活的工具集成，将视觉工具和搜索工具分开测试，且主要根据最终答案进行评估。因此，这些评估无法验证工具是否被实际调用、应用是否正确或使用是否高效。为了解决这一问题，我们推出了 Agentic-MME，这是一个针对多模态智能体能力的过程验证基准。它包含跨 6 个领域和 3 个难度级别的 418 个真实世界任务，用于评估能力协同，并包含超过 2,000 个逐步检查点，每个任务平均耗费 10 小时以上的人工标注。每个任务都包含一个支持沙箱代码和 API 的统一评估框架，以及一条沿 S 轴和 V 轴双轴标注了逐步检查点的人类参考轨迹。为了实现真正的过程级验证，我们审计细粒度的中间状态而非仅关注最终答案，并根据相对于人类轨迹的“过度思考”指标来量化效率。实验结果显示，表现最好的模型 Gemini3-pro 实现了 56.3% 的总准确率，但在 Level-3 任务上显著下降至 23.0%，这凸显了现实世界多模态智能体问题解决的难度。

## Abstract
Multimodal Large Language Models (MLLMs) are evolving from passive observers into active agents, solving problems through Visual Expansion (invoking visual tools) and Knowledge Expansion (open-web search). However, existing evaluations fall short: they lack flexible tool integration, test visual and search tools separately, and evaluate primarily by final answers. Consequently, they cannot verify if tools were actually invoked, applied correctly, or used efficiently. To address this, we introduce Agentic-MME, a process-verified benchmark for Multimodal Agentic Capabilities. It contains 418 real-world tasks across 6 domains and 3 difficulty levels to evaluate capability synergy, featuring over 2,000 stepwise checkpoints that average 10+ person-hours of manual annotation per task. Each task includes a unified evaluation framework supporting sandboxed code and APIs, alongside a human reference trajectory annotated with stepwise checkpoints along dual-axis: S-axis and V-axis. To enable true process-level verification, we audit fine-grained intermediate states rather than just final answers, and quantify efficiency via an overthinking metric relative to human trajectories. Experimental results show the best model, Gemini3-pro, achieves 56.3% overall accuracy, which falls significantly to 23.0% on Level-3 tasks, underscoring the difficulty of real-world multimodal agentic problem solving.

#### 1. 核心问题与整体含义（研究动机和背景）

*   **工具集成缺乏灵活性：** 往往将视觉工具与搜索工具割裂开来，无法测试两者的协同效应。
*   **评估维度单一：** 仅关注最终答案的正确性，无法判断工具是否被正确调用、执行是否高效。
*   **缺乏过程验证：** 无法诊断失败是源于感知错误、规划缺失还是工具误用。
**Agentic-MME** 旨在通过一个“过程验证”的基准，深入探究智能体能力究竟为多模态智能带来了什么，并量化其在复杂现实任务中的表现。

#### 2. 论文提出的方法论
核心思想是构建一个包含**细粒度过程检查点**的评估框架，支持视觉与知识扩展的深度协同。
*   **三级难度体系：**
    *   **Level 1：** 单步视觉操作（如一次裁剪）。
    *   **Level 2：** 简单工具链（如裁剪+搜索）。
    *   **Level 3：** 深度协同（如模糊搜索+跨模态验证），需要模型在视觉反馈和外部知识间进行多次迭代。
*   **双轴评估协议：**
    *   **S轴（Strategy/Strategy）：** 审计搜索策略，验证关键词、URL和中间答案的正确性。
    *   **V轴（Visual Evidence）：** 审计视觉操作，验证工具调用意图及生成的中间图像（Artifacts）是否包含解题所需的关键证据。
*   **统一执行环境：** 开发了标准化框架，同时支持**沙箱 Python 代码执行**（通过 AST 追踪器提取操作）和**结构化函数调用（API）**。
*   **效率指标：** 引入 **Overthink（过度思考）** 指标，通过对比模型轨迹与人类最短参考轨迹的步数差来量化冗余操作。

*   **对比方法：**

#### 6. 论文的主要结论与发现
*   **性能鸿沟：** 所有模型均远低于人类水平（人类 93.8% vs 最强模型 56.3%），在 Level 3 任务中，模型准确率骤降（最高仅 33.3%）。
*   **开源 vs 闭源：** 开源模型在搜索规划和多步推理上存在显著短板，S 轴得分极低。
*   **接口偏好：** 结构化 API 模式（Atm）普遍优于代码生成模式（Gen），因为代码生成增加了模型在语法和 I/O 处理上的认知负担。
*   **行为缺陷：** 模型普遍存在“行动迟缓”（倾向于盲猜而非用工具）或“过度思考”（陷入无效的工具调用循环）的问题。

#### 7. 优点
*   **过程透明化：** 首次通过双轴检查点实现了对多模态 Agent 中间推理过程的细粒度审计。
*   **协同评估：** 填补了视觉操作与网络搜索协同测试的空白，更贴近真实“深研”场景。
*   **高质量标注：** 极高的人工标注投入确保了基准的权威性和挑战性。
*   **统一框架：** 解决了不同模型代码风格不一、沙箱环境不兼容导致的评估不公问题。

#### 8. 不足与局限
*   **任务规模：** 418 个任务虽然标注深，但绝对数量相对较小，可能无法覆盖所有长尾场景。
*   **裁判偏见：** 评估高度依赖 LLM-as-a-Judge，尽管经过了人类校验，仍可能存在潜在的评估偏见。
*   **动态性挑战：** 网络搜索结果随时间变化，虽然论文使用了缓存机制，但如何保持基准的长期时效性是一个挑战。
