## 摘要
桌面交互流提供了交织在一起的用户任务的持续且隐私敏感的记录。在设备端将这些流转换为按任务组织的个人日志面临两个主要挑战：详尽的视觉语言模型（VLM）处理会耗尽本地资源，而全局流处理会导致跨任务的上下文污染。我们提出了 FOCAL（过滤式设备端持续活动日志记录），这是一个采用统一“过滤-规划-记录”架构的隐私优先多智能体系统。它级联了一个用于噪声抑制的轻量级过滤智能体（Filter Agent）、一个用于任务归因的纯文本大脑智能体（Brain Agent）、一个用于选择性视觉推理的记录智能体（Record Agent），以及一个用于上下文连贯摘要的任务隔离记忆智能体（Memory Agent）。在 DesktopBench（包含 420 个复杂会话中的 2,572 张截图）上的实验表明，与基准模型相比，FOCAL 将总 Token 消耗降低了 60.4%，VLM 调用次数减少了 72.3%，同时将关键信息召回率（KIR）从 0.38 提升至 0.61。至关重要的是，在 A→B→A 任务中断情况下，FOCAL 保持了 0.81 的任务准确率（Task Acc）和 0.80 的 KIR，而基准模型的任务准确率则崩溃至 0.03。FOCAL 开创了将无指令桌面流高效、在设备端摘要为多视角个人日志的先河。

## Abstract
Desktop interaction streams provide a continuous, privacy-sensitive record of interleaved user tasks. Transforming these streams into task-organized personal logs on-device faces two main challenges: exhaustive Vision-Language Model (VLM) processing strains local resources, and global stream processing causes cross-task context pollution. We present FOCAL (Filtered On-device Continuous Activity Logging), a privacy-first multi-agent system utilizing a unified filter-plan-log architecture. It cascades a lightweight Filter Agent for noise suppression, a text-only Brain Agent for task attribution, a Record Agent for selective visual reasoning, and a task-isolated Memory Agent for context-coherent summarization. Experiments on DesktopBench (comprising 2,572 screenshots across 420 complex sessions) show FOCAL reduces total token consumption by 60.4% and VLM call count by 72.3% versus a baseline, while boosting Key Information Recall (KIR) from 0.38 to 0.61. Crucially, under $A{\to}B{\to}A$ task interruptions, FOCAL maintains Task Acc 0.81 and KIR 0.80, whereas the baseline collapses to Task Acc 0.03. FOCAL pioneers the efficient, on-device summarization of instruction-free desktop streams into multi-perspective personal logs.

*   **研究背景**：个人桌面活动包含高度隐私敏感的信息（如邮件、代码、凭证），且用户在工作中经常在多个任务间频繁切换（交织任务流）。
*   **核心问题**：在设备端（On-device）实现持续的桌面活动摘要面临两大挑战：
    1.  **资源侵入性**：详尽的视觉语言模型（VLM）推理会消耗大量本地 CPU/GPU 资源，干扰用户正常工作。
    2.  **跨任务上下文污染**：传统的全局流处理会将不同任务的描述混在一起，导致摘要逻辑混乱，尤其是在任务中断并恢复（$A \to B \to A$）的情况下。
*   **研究目标**：开发一种隐私优先、低功耗且能准确区分任务边界的端侧桌面日志系统。

### 2. 方法论：FOCAL 系统架构
FOCAL 采用了一种**“过滤-规划-记录”（Filter-Plan-Log）**的多智能体协作架构，核心思想是**在昂贵的视觉推理之前进行任务感知的控制**。

*   **五大智能体分工**：
    1.  **过滤智能体 (Filter Agent)**：实时捕获元数据和截图，抑制无意义的低信息量事件（如单纯的鼠标移动）。
    2.  **大脑智能体 (Brain Agent)**：**关键创新点**。仅利用轻量级的文本元数据（应用名、窗口标题）进行一次性规划，决定哪些帧需要 VLM 处理（自适应采样），并将动作归类到特定任务 ID。
    3.  **记录智能体 (Record Agent)**：仅对采样后的关键帧调用 VLM，结合当前任务的局部记忆生成语义描述。
    4.  **记忆智能体 (Memory Agent)**：实现**任务隔离存储**。每个任务拥有独立的存储空间，防止不同任务间的上下文信息相互干扰。
    5.  **摘要智能体 (Summary Agent)**：在会话结束时，将各隔离存储空间的内容整合为结构化的多视角个人日志。
