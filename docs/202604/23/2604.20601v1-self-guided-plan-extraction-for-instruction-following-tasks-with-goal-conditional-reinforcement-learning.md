## 摘要
我们介绍了 SuperIgor，这是一个用于指令遵循任务的框架。与以往依赖预定义子任务的方法不同，SuperIgor 通过自学习机制使语言模型能够生成并优化高层计划，从而减少了对手动数据集标注的需求。我们的方法涉及迭代协同训练：训练强化学习（RL）智能体以遵循生成的计划，同时语言模型根据 RL 的反馈和偏好调整并修改这些计划。这形成了一个反馈循环，使得智能体和规划器能够共同提升。我们在具有丰富动力学特性和随机性的环境中验证了我们的框架。结果表明，SuperIgor 智能体比基准方法更严格地遵循指令，同时也展现出对先前未见指令的强大泛化能力。

## Abstract
We introduce SuperIgor, a framework for instruction-following tasks. Unlike prior methods that rely on predefined subtasks, SuperIgor enables a language model to generate and refine high-level plans through a self-learning mechanism, reducing the need for manual dataset annotation. Our approach involves iterative co-training: an RL agent is trained to follow the generated plans, while the language model adapts and modifies these plans based on RL feedback and preferences. This creates a feedback loop where both the agent and the planner improve jointly. We validate our framework in environments with rich dynamics and stochasticity. Results show that SuperIgor agents adhere to instructions more strictly than baseline methods, while also demonstrating strong generalization to previously unseen instructions.

#### 1. 核心问题与整体含义（研究动机和背景）
在具身智能领域，让 AI 代理遵循自然语言指令（Instruction-Following）是一个核心挑战。现有方法通常分为两类：**基于演示的学习**（成本高、扩展性差）和**基于经验的强化学习**（面临稀疏奖励和长程任务理解困难）。
目前的规划方法大多依赖于**预定义的子任务库**，这限制了代理在没有先验技能环境中的表现。本文提出了 **SuperIgor** 框架，旨在解决如何在没有预定义技能、没有手动标注数据集的情况下，利用大语言模型（LLM）自主生成、验证并优化高层计划，从而引导强化学习代理完成复杂任务。

#### 2. 方法论：核心思想与关键技术
SuperIgor 采用了一种**迭代协同训练（Co-training）**的闭环机制，包含四个阶段：
*   **计划生成（Plan Generation）：** 
    *   利用 LLM 从指令中提取子任务并建立“子任务库”。
    *   通过多次查询诱导子任务之间的依赖关系，构建**本体图（Ontology Graph）**。
    *   基于本体图将复杂指令展开为线性化的子任务序列（计划）。
*   **策略学习（Policy Learning）：** 
    *   使用 PPO 算法训练 RL 代理。
    *   引入**技能课程学习（Skill Curriculum Learning）**：代理先学习单步子任务，只有当某技能成功率超过阈值 $\tau$ 时，才将其加入更复杂的组合计划中，缓解稀疏奖励问题。
*   **计划验证（Plan Validation）：** 在随机环境中多次运行代理，评估生成计划的实际成功率。
*   **LLM 微调（LLM Fine-Tuning）：** 
    *   使用**直接偏好优化（DPO）**，根据代理的执行反馈（成功与否）作为偏好信号，微调 LLM 规划器，使其倾向于生成代理更容易执行的高质量计划。

*   **结构化规划：** 通过本体图引入了环境逻辑约束，使得 LLM 生成的计划更具物理合理性。

*   **初始计划依赖：** 框架性能高度依赖于 LLM 的初始零样本规划能力，若 LLM 无法理解环境逻辑，后续优化将难以进行。

*   **计算成本：** 相比端到端 RL，增加了 LLM 推理和微调的开销。
*   **解释性挑战：** 当训练停滞时，难以明确区分是 RL 策略探索不足还是 LLM 计划空间设计有误。
