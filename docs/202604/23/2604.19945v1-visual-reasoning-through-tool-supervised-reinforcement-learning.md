## 摘要
在本文中，我们研究了多模态大语言模型如何有效掌握工具使用，以解决复杂的视觉推理任务。为此，我们提出了一种新颖的工具监督强化学习（ToolsRL）框架，通过直接的工具监督来实现更有效的工具使用学习。我们专注于一系列简单、原生且可解释的视觉工具，包括放大、旋转、翻转以及绘制点/线，这些工具的监督信号易于收集。我们开发了一种强化学习课程方案，其中第一阶段仅由一组动机明确的工具特定奖励进行优化，第二阶段则在允许调用工具的同时，使用以准确率为目标的奖励进行训练。通过这种方式，模型在利用工具完成视觉推理任务之前就已掌握了工具调用能力，从而避免了这些异构任务之间潜在的优化冲突。实验表明，工具监督的课程训练是高效的，且 ToolsRL 能够在复杂的视觉推理任务中实现强大的工具使用能力。

## Abstract
In this paper, we investigate the problem of how to effectively master tool-use to solve complex visual reasoning tasks for Multimodal Large Language Models. To achieve that, we propose a novel Tool-supervised Reinforcement Learning (ToolsRL) framework, with direct tool supervision for more effective tool-use learning. We focus on a series of simple, native, and interpretable visual tools, including zoom-in, rotate, flip, and draw point/line, whose tool supervision is easy to collect. A reinforcement learning curriculum is developed, where the first stage is solely optimized by a set of well motivated tool-specific rewards, and the second stage is trained with the accuracy targeted rewards while allowing calling tools. In this way, tool calling capability is mastered before using tools to complete visual reasoning tasks, avoiding the potential optimization conflict among those heterogeneous tasks. Our experiments have shown that the tool-supervised curriculum training is efficient and ToolsRL can achieve strong tool-use capabilities for complex visual reasoning tasks.

### 1. 核心问题与研究动机
*   **核心问题**：多模态大语言模型（MLLMs）在处理复杂视觉推理任务（如识别旋转文本、定位微小物体）时，仅靠文本推理（thinking-with-text）能力不足。虽然引入视觉工具（缩放、旋转、绘图）能提供中间证据，但如何让模型**自主、高效地掌握工具调用策略**（何时用、怎么用）仍是巨大挑战。
*   **研究背景**：
    *   **SFT（监督微调）**：依赖高质量专家轨迹，构建成本高且易过拟合。
    *   **标准 RL（强化学习）**：通常仅依赖最终答案的稀疏奖励，导致工具调用频率低、难以形成多步推理链。
*   **研究动机**：通过引入易于获取的“工具监督信号”和“两阶段课程学习”，解决工具学习效率低下及异构任务（工具操作 vs. 答案准确率）之间的优化冲突。

### 2. 核心方法论：ToolsRL 框架
ToolsRL 采用**两阶段强化学习课程**，将工具掌握与任务求解解耦：

*   **核心思想**：先让模型在直接监督下学会“玩转”工具，再让其利用工具去“解决”问题。
*   **关键技术细节**：
    *   **原生工具集**：包括 Zoom-in（缩放）、Rotate/Flip（旋转/翻转）、Draw Line/Point（画线/点）。
    *   **第一阶段：工具监督阶段（Tool Supervision）**：
        *   **目标**：仅优化工具调用的准确性。
        *   **奖励设计**：
            *   **Zoom-in**：使用 **ModF1 奖励**，通过调整假阳性（FP）权重（$w_{fp}=0.1$）来鼓励模型进行探索性缩放，只要覆盖目标区域即可。
            *   **Rotate/Flip**：基于图像是否恢复到标准朝向的二值奖励。
            *   **Draw**：基于坐标的统一奖励，利用 **匈牙利匹配（Hungarian matching）** 计算预测点/线与真值的相似度。
        *   **组合奖励**：结合全局工具奖励（鼓励探索）和答案关联工具奖励（确保相关性）。
    *   **第二阶段：任务准确率阶段（Task Accuracy）**：
        *   **目标**：优化最终答案的准确率。
        *   **机制**：允许模型自由调用已掌握的工具，使用 LLM 作为裁判（LLM-Judge）或数值对比来提供奖励。
*   **算法流程**：基于 **GRPO（组相对策略优化）** 算法进行大规模采样和策略更新。

### 6. 主要结论与发现
*   **课程学习至关重要**：直接在第二阶段训练会导致模型倾向于“走捷径”（仅靠文本推理），而第一阶段的工具监督能强制模型建立视觉操作与信息获取的联系。
*   **性能领先**：ToolsRL 在文档理解（DocVQA-RF 提升约 16%）和高分辨率空间推理任务上表现尤为突出。
*   **涌现复杂行为**：模型自发学会了多步视觉搜索、视觉验证（通过打点标记）以及复合工具使用（缩放后绘图）。

### 7. 优点与亮点
*   **监督信号廉价**：相比昂贵的专家轨迹，ToolsRL 使用的边界框、旋转角度和坐标点等监督信号极易通过程序合成或现有检测数据集获取。
*   **奖励设计精巧**：ModF1 奖励解决了缩放任务中“过度惩罚非紧凑框”的问题，连续性绘图奖励加速了坐标预测的收敛。
*   **原生集成**：工具直接集成在模型推理流中，无需外部复杂的感知模型（如检测器或分割模型）。

### 8. 不足与局限
*   **工具范围有限**：目前仅限于基础的几何变换和绘图，尚未涉及更复杂的外部工具（如计算器、搜索引擎或专门的 OCR 引擎）。
*   **合成数据依赖**：在图表绘图任务中高度依赖合成数据集，在真实复杂场景下的泛化能力仍有待进一步验证。
*   **推理开销**：由于模型倾向于进行多步工具调用（平均 3.4 次），相比单次推理，其计算延迟和 Token 消耗会显著增加。
