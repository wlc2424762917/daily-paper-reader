## 摘要
长期对话大语言模型（LLM）智能体需要能够从历史交互中恢复相关证据的记忆系统，同时避免无关上下文在回答阶段造成负担。然而，现有的记忆系统（包括分层系统）通常仍仅依赖向量相似度进行检索。这往往会导致证据集臃肿：添加许多表面相似的对话轮次几乎不会增加召回率，反而会降低检索精度，增加回答阶段的上下文成本，并使检索到的记忆难以检查和管理。为了解决这一问题，我们提出了 HiGMem（分层且由 LLM 引导的记忆系统），这是一种双层“事件-轮次”记忆系统，允许 LLM 将事件摘要作为语义锚点，以预测哪些相关的对话轮次值得阅读。这使得模型能够先检查高层事件摘要，然后专注于较小的一组潜在有用轮次，通过推理提供简洁可靠的证据集，同时避免了与向量检索相比过高的检索开销。在 LoCoMo10 基准测试中，HiGMem 在五个问题类别中的四个上实现了最佳 F1 分数，并将对抗性 F1 分数从 A-Mem 的 0.54 提升至 0.78，同时检索的轮次减少了一个数量级。代码已在 https://github.com/ZeroLoss-Lab/HiGMem 公开。

## Abstract
Long-term conversational large language model (LLM) agents require memory systems that can recover relevant evidence from historical interactions without overwhelming the answer stage with irrelevant context. However, existing memory systems, including hierarchical ones, still often rely solely on vector similarity for retrieval. It tends to produce bloated evidence sets: adding many superficially similar dialogue turns yields little additional recall, but lowers retrieval precision, increases answer-stage context cost, and makes retrieved memories harder to inspect and manage. To address this, we propose HiGMem (Hierarchical and LLM-Guided Memory System), a two-level event-turn memory system that allows LLMs to use event summaries as semantic anchors to predict which related turns are worth reading. This allows the model to inspect high-level event summaries first and then focus on a smaller set of potentially useful turns, providing a concise and reliable evidence set through reasoning, while avoiding the retrieval overhead that would be excessively high compared to vector retrieval. On the LoCoMo10 benchmark, HiGMem achieves the best F1 on four of five question categories and improves adversarial F1 from 0.54 to 0.78 over A-Mem, while retrieving an order of magnitude fewer turns. Code is publicly available at https://github.com/ZeroLoss-Lab/HiGMem.

*   **研究动机**：长期对话智能体需要从海量历史交互中提取相关证据。现有的记忆系统（如 MemGPT、A-Mem）主要依赖**向量相似度检索**。这种“被动检索”方式存在严重缺陷：为了保证召回率，往往会检索出大量表面相似但实际无关的对话片段（证据膨胀），导致检索精度低、下游模型上下文成本激增，且难以有效管理长程上下文中的逻辑关系。

### 2. 方法论
HiGMem 提出了一个**双层（事件-轮次）架构**，核心思想是利用 LLM 的推理能力，将高层事件摘要作为“语义锚点”来引导细粒度证据的检索。
*   **层级架构**：
    *   **轮次层 (Turn Layer)**：存储原始对话及 LLM 生成的元数据（关键词、标签、时间戳、上下文）。
    *   **事件层 (Event Layer)**：将相关的轮次聚合成“事件节点”，包含简短摘要和结构化事实表，并维护指向底层轮次节点的索引链接。
*   **记忆构建机制**：
    *   **实时分析**：新对话进入时，LLM 结合滑动窗口提取元数据。
    *   **动态关联**：通过向量编码计算新轮次与现有事件的相似度，由 LLM 决定将其归入现有事件或创建新事件，并动态更新事件摘要。
*   **LLM 引导检索流程**：
    1.  **双路检索**：根据查询关键词，同时从事件层和轮次层检索 Top-K 候选节点。
    2.  **推理预测**：LLM 检查检索到的“事件摘要”，预测该事件下哪些关联的“轮次”真正有助于回答问题。
    3.  **精细过滤**：将预测的轮次与初始检索的轮次合并，再次由 LLM 过滤，形成极简且高精度的最终证据集。

### 6. 主要结论与发现
*   **性能领先**：HiGMem 在 LoCoMo10 的五个类别中的四个取得了最高 F1 分数。

### 7. 优点
*   **精准定位**：通过“先看摘要，再选轮次”的策略，解决了传统 RAG 在长文本中“大海捞针”导致的噪声问题。
*   **成本优化**：将计算压力前置到廉价模型，显著降低了高质回答阶段的 Token 成本。
*   **可解释性**：层级化的事件-轮次结构使得记忆的存储和检索过程更易于人类检查和管理。

### 8. 不足与局限
*   **推理延迟**：由于在检索阶段引入了额外的 LLM 推理步骤，单次问答的响应时间（QA Time）比传统方法略长（约增加 1.6 倍）。
*   **多方对话挑战**：在 DialSim（多方对话）实验中，F1 分数略低于 A-Mem，表明目前的“事件-轮次”结构在处理复杂的人际关系和多线索对话时仍有优化空间。
*   **依赖模型能力**：系统的有效性高度依赖于中间步骤 LLM 的判断力，若中间模型出现幻觉，可能导致关键证据丢失。
