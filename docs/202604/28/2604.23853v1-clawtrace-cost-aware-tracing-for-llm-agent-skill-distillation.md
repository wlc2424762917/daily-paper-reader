## 摘要
技能蒸馏流水线从大语言模型（LLM）智能体的轨迹中学习可重用的规则，但它们缺乏一个关键信号：每一步的成本。如果没有单步成本信息，流水线就无法区分是为了修复错误而添加缺失步骤，还是为了移除从未影响结果的高昂步骤。我们推出了 ClawTrace，这是一个智能体追踪平台，它记录了智能体会话期间的每一次 LLM 调用、工具使用和子智能体生成，并将每个会话编译成 TraceCard：一个包含单步美元成本、Token 计数和冗余标志的紧凑 YAML 摘要。基于 ClawTrace，CostCraft 是一种蒸馏流水线，它通过读取 TraceCard 来生成三种类型的技能补丁。“保留”（Preserve）补丁保留了导致成功的行为；“剪枝”（Prune）补丁移除了无关紧要的高昂步骤，每个补丁都由针对特定高成本步骤的反事实论据支持；“修复”（Repair）补丁则基于真值证据修复失败。在 30 个预留的 SpreadsheetBench 任务上的消融实验表明，成本归因和剪枝补丁都能独立减少质量退化。当相同的技能应用于 30 个不相关的 SkillsBench 任务时，出现了一种意想不到的不对称性：剪枝规则可以跨基准测试迁移，并将中位数成本降低了 32%，而基于特定基准惯例训练的保留规则则在新的任务类型上导致了性能退化。我们发布了 ClawTrace 和 TraceCards，作为成本感知智能体研究的开放基础设施。

## Abstract
Skill-distillation pipelines learn reusable rules from LLM agent trajectories, but they lack a key signal: how much each step costs. Without per-step cost, a pipeline cannot distinguish adding a missing step to fix a bug from removing an expensive step that never affected the outcome. We introduce ClawTrace, an agent tracing platform that records every LLM call, tool use, and sub-agent spawn during an agent session and compiles each session into a TraceCard: a compact YAML summary with per-step USD cost, token counts, and redundancy flags. Built on ClawTrace, CostCraft is a distillation pipeline that reads TraceCards and produces three types of skill patches. Preserve patches keep behaviors that led to success. Prune patches remove expensive steps that did not matter, each backed by a counterfactual argument against a named high-cost step. Repair patches fix failures grounded in oracle evidence. Ablations on 30 held-out SpreadsheetBench tasks show that both cost attribution and prune patches independently reduce quality regressions. When the same skill is applied to 30 unrelated SkillsBench tasks, an unexpected asymmetry emerges: prune rules transferred across benchmarks and cut median cost by 32%, while preserve rules, trained on benchmark-specific conventions, caused regressions on new task types. We release ClawTrace and TraceCards as open infrastructure for cost-aware agent research.

在 LLM 智能体（Agent）领域，通过从执行轨迹中提取“技能”（结构化指令包）来优化智能体行为已成为主流趋势。然而，现有的蒸馏流水线（如 Trace2Skill）存在一个关键缺陷：**缺乏成本信号**。

*   **背景**：现有的观测工具（如 LangSmith）主要面向人类开发者，其输出格式不便于自动化蒸馏流水线直接消费。因此，需要一种能够量化每一步成本、识别冗余并生成紧凑摘要的工具，以实现“成本-质量”双优化的技能学习。

#### 6. 主要结论与发现
*   **成本归因至关重要**：剥离成本信息后，成功任务的成本涨幅从 +22% 飙升至 +49%，且灾难性失败（无输出）显著增加。
*   **剪枝补丁的保护作用**：意外发现剪枝补丁不仅能省钱，更像是一个“护栏”。移除剪枝补丁后，质量退化率增加了三倍，因为它能防止其他技能规则过度引导智能体进入无效的冗余循环。
*   **迁移性不对称**：在跨基准测试中，**剪枝规则**表现出极强的通用性，使中位成本降低了 **32%**；而**保留规则**由于带有特定基准的惯例，在处理新任务时反而导致了性能下降。

#### 7. 优点
*   **创新性引入成本维度**：填补了智能体技能蒸馏中长期缺失的成本信号，使“效率”成为可优化的目标。
*   **紧凑的中间表示**：TraceCard 的设计非常巧妙，将海量日志压缩为 LLM 可理解的紧凑 YAML，极大地节省了 Context Window。
*   **开源贡献**：提供了 ClawTrace 平台和 TraceCards 基础设施，对 AgentOps 社区具有实用价值。

#### 8. 不足与局限
*   **样本量限制**：训练集仅 10 个任务，导致学习到的剪枝规则种类较少（仅覆盖了文件重读等模式）。
*   **随机性风险**：单种子实验无法完全排除 LLM 推理的随机性对结果的影响。
*   **模型依赖**：所有实验均基于单一的高级模型，尚未验证该框架在较弱模型（如 7B 参数量级）上的表现。
*   **启发式算法依赖**：TraceCard 中的冗余检测和子智能体评估依赖于启发式阈值（如 Levenshtein 距离 0.8），在复杂场景下可能存在漏报或误报。
