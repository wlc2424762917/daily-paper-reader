## 摘要
AI 智能体的兴起为可视化分析（VA）带来了根本性的转变，智能体开始作为一个新的用户群体出现。目前的智能体方法——基于计算机视觉和原始 DOM 访问——无法准确且高效地执行 VA 任务。本文介绍了可视化分析上下文协议（VACP），这是一个旨在使 VA 应用程序具备“智能体就绪”能力的框架，它通过显式暴露应用程序状态、可用交互以及直接执行机制，扩展了通用协议。为了支持该上下文协议，我们提出了 VA 界面中 AI 智能体需求和知识表示的形式化规范。我们将 VACP 实例化为一个兼容主流可视化语法和 Web 框架的库，从而能够增强现有系统并开发新系统。我们在代表性 VA 任务上的评估表明，与目前的智能体方法相比，启用 VACP 的智能体在界面解释和执行方面实现了更高的成功率，同时降低了 Token 消耗和延迟。VACP 弥合了以人为中心的可视化分析界面与机器可感知性之间的鸿沟，确保智能体能够可靠地作为可视化分析系统中的协作用户。

## Abstract
The rise of AI agents introduces a fundamental shift in Visual Analytics (VA), in which agents act as a new user group. Current agentic approaches - based on computer vision and raw DOM access - fail to perform VA tasks accurately and efficiently. This paper introduces the Visual Analytics Context Protocol (VACP), a framework designed to make VA applications "agent-ready" that extends generic protocols by explicitly exposing application state, available interactions, and mechanisms for direct execution. To support our context protocol, we contribute a formal specification of AI agent requirements and knowledge representations in VA interfaces. We instantiate VACP as a library compatible with major visualization grammars and web frameworks, enabling augmentation of existing systems and the development of new ones. Our evaluation across representative VA tasks demonstrates that VACP-enabled agents achieve higher success rates in interface interpretation and execution compared to current agentic approaches, while reducing token consumption and latency. VACP closes the gap between human-centric VA interfaces and machine perceivability, ensuring agents can reliably act as collaborative users in VA systems.

### 2. 方法论：VACP 协议架构
VACP 的核心思想是将 VA 界面抽象为一套**语义图（Semantic Graph）**，解耦了交互意图与物理实现。
*   **四层知识表示**：
    *   **L1 像素层**：原始视觉输出。
    *   **L2 DOM 层**：浏览器逻辑树。
    *   **L3 声明式语法层**：如 Vega-Lite 配置，定义了数据到视觉的映射。
    *   **L4 语义逻辑层**：开发者的设计意图和业务逻辑。
*   **三大核心组件**：
    1.  **应用状态表示（Application State）**：通过 `get_state()` 暴露分层的语义节点，每个节点有稳定 ID，避免 Agent 在页面刷新后丢失目标。
    2.  **交互能力暴露（Available Interactions）**：动态列出当前合法的操作（如缩放、过滤、刷选），Agent 通过 `get_capabilities()` 获知。
    3.  **执行网关（Execution Gateway）**：Agent 通过 `execute_interaction()` 直接触发语义层动作，无需模拟鼠标拖拽，系统返回结构化反馈（成功/错误信息）。
*   **关键技术细节**：基于 Anthropic 的 **MCP（Model Context Protocol）** 构建，支持按需查询数据（`inspect_data()`），避免将海量原始数据塞入 Agent 的上下文窗口。

### 6. 主要结论与发现
*   **性能飞跃**：启用 VACP 后，Agent 的任务成功率从基准的 28%-51% 提升至接近 **100%**。
*   **效率提升**：S3（纯 VACP）场景下的 Token 消耗和执行时间显著低于视觉方案，Claude 4.5 在 VACP 下的 Token 消耗甚至不到 GPT-5.2 的十分之一。
*   **验证循环**：在 S4（全能力）场景中，Agent 表现出“语义操作 + 视觉验证”的行为模式，即先通过 API 精准操作，再通过截图确认视觉反馈，这被证明是最稳健的策略。
*   **模型差异**：Claude 4.5 在处理 VACP 协议时表现出最高的推理效率。

### 7. 优点
*   **协议标准化**：首次为 VA 领域提出了 Agent 专用的上下文协议，填补了行业空白。
*   **细节按需获取**：通过 `inspect_data` 解决了长上下文窗口的性能衰减问题。
*   **框架兼容性**：提供的 TypeScript 库能无缝适配 Vega-Lite 等主流工具，具有很强的实用性。
*   **减少幻觉**：通过直接操作数据状态，消除了 Agent 在像素层定位时的坐标计算错误。

### 8. 不足与局限
*   **开发成本**：现有的自定义 VA 系统需要开发者手动映射状态和交互到 VACP 协议，存在一定的适配工作量。
*   **视觉直觉缺失**：纯语义协议（S3）虽然高效，但会丢失如“格式塔原则”（Gestalt principles）等人类直观感知的视觉模式，因此仍需视觉辅助。
*   **动态性挑战**：对于高度动态、实时更新的流式可视化系统，协议的同步频率和延迟可能成为瓶颈。
*   **模型依赖**：协议的效果高度依赖于底层 LLM 的工具调用（Tool-use）能力。
