## 摘要
大语言视觉模型赋予了 GUI 智能体强大的界面理解与交互通用能力。然而，由于训练过程中缺乏特定领域软件操作数据的暴露，这些智能体表现出显著的领域偏差——它们缺乏对特定应用程序操作工作流（规划）和 UI 元素布局（定位）的熟悉度，从而限制了其在真实世界任务中的表现。在本文中，我们提出了 GUIDE（通过教学视频驱动的专业知识消除 GUI 偏差），这是一个无需训练、即插即用的框架，通过检索增强的自动标注流水线，从网络教学视频中自主获取领域特定专业知识，从而解决 GUI 智能体的领域偏差。GUIDE 引入了两项关键创新：首先，字幕驱动的 Video-RAG 流水线通过字幕分析解锁视频语义，执行渐进式三阶段检索（领域分类、主题提取和相关性匹配），以识别与任务相关的教学视频；其次，基于逆动力学范式构建的全自动标注流水线将经过 UI 元素检测增强的连续关键帧输入到视觉语言模型（VLM）中，推断出所需的规划和定位知识，并将其注入智能体的相应模块中，以解决领域偏差的两种表现形式。在 OSWorld 上的广泛实验证明了 GUIDE 作为多智能体系统和单模型智能体即插即用组件的通用性。在不修改任何模型参数或架构的情况下，它持续带来超过 5% 的性能提升并减少了执行步骤，验证了 GUIDE 作为一种与架构无关的增强方案，能够有效弥合 GUI 智能体的领域偏差。

## Abstract
Large vision-language models have endowed GUI agents with strong general capabilities for interface understanding and interaction. However, due to insufficient exposure to domain-specific software operation data during training, these agents exhibit significant domain bias - they lack familiarity with the specific operation workflows (planning) and UI element layouts (grounding) of particular applications, limiting their real-world task performance. In this paper, we present GUIDE (GUI Unbiasing via Instructional-Video Driven Expertise), a training-free, plug-and-play framework that resolves GUI agent domain bias by autonomously acquiring domain-specific expertise from web tutorial videos through a retrieval-augmented automated annotation pipeline. GUIDE introduces two key innovations. First, a subtitle-driven Video-RAG pipeline unlocks video semantics through subtitle analysis, performing progressive three-stage retrieval - domain classification, topic extraction, and relevance matching - to identify task-relevant tutorial videos. Second, a fully automated annotation pipeline built on an inverse dynamics paradigm feeds consecutive keyframes enhanced with UI element detection into VLMs, inferring the required planning and grounding knowledge that are injected into the agent's corresponding modules to address both manifestations of domain bias. Extensive experiments on OSWorld demonstrate GUIDE's generality as a plug-and-play component for both multi-agent systems and single-model agents. It consistently yields over 5% improvements and reduces execution steps - without modifying any model parameters or architecture - validating GUIDE as an architecture-agnostic enhancement to bridge GUI agent domain bias.

#### 1. 核心问题与整体含义（研究动机和背景）
当前的 GUI 智能体（如基于 VLM 的 Agent）虽然具备强大的通用界面理解能力，但在面对特定专业软件（如 GIMP、LibreOffice）时，往往表现出显著的**领域偏差（Domain Bias）**。这种偏差主要体现在两个层面：
*   **规划层面（Planning-level）**：不熟悉特定软件的操作逻辑和工作流（例如，知道要调亮度，但不知道在 GIMP 中路径是“Colors -> Brightness”而非“Image -> Adjustments”）。
*   **定位层面（Grounding-level）**：不熟悉特定软件的 UI 布局和控件样式，难以在复杂界面中精准找到目标元素。

由于手动标注特定领域的演示数据成本极高且难以覆盖所有软件更新，本文提出了 **GUIDE** 框架。其核心动机是利用互联网上（如 YouTube）海量的教学视频作为知识库，通过自动化的检索和标注，实时为 Agent 提供“专家指导”，从而弥补通用模型与特定领域任务之间的代差。

#### 2. 方法论：核心思想与技术细节
GUIDE 是一个**无需训练（Training-free）、即插即用**的框架，由三个协同工作的 Agent 组成：

*   **字幕驱动的 Video-RAG 流水线（检索 Agent）**：
    *   利用视频字幕（CC）作为视觉像素与任务语义之间的桥梁。
    *   **三阶段过滤**：1. **领域分类**（通过字幕判断是否为真实的 GUI 操作演示）；2. **主题提取**（结合标题和字幕提炼核心操作步骤）；3. **相关性匹配**（使用双锚点 Prompt 计算视频与任务的语义匹配度）。
*   **全自动标注流水线（标注 Agent）**：
    *   采用**逆动力学（Inverse Dynamics）范式**：给定连续的关键帧 $s_t$ 和 $s_{t+1}$，推断中间发生的动作 $a_t$。
    *   **感知增强**：利用 OmniParser 提取 UI 元素图（Bounding Boxes 和标签），辅助 VLM 理解界面。
    *   **知识分解**：将视频演示转化为结构化的**规划知识**（操作流、注意事项）和**定位知识**（关键元素的视觉特征、位置和功能描述）。
*   **即插即用集成（GUI Agent 集成）**：
    *   将提取的自然语言知识注入 Agent 的系统提示词（System Prompt）中。
    *   支持**单模型架构**（如 Qwen3-VL）和**多智能体架构**（如 AgentS3），将知识分别引导至规划模块和坐标生成模块。

#### 6. 主要结论与发现
*   **显著提升**：GUIDE 在不同架构的 Agent 上均实现了 **4.5% - 7.5%** 的绝对性能提升。
*   **规划知识是核心**：消融实验显示，规划知识贡献了约 85%-91% 的性能增益，说明“知道怎么做”是解决领域偏差的首要瓶颈。
*   **定位知识是补充**：在 GIMP 和 Calc 等 UI 极其复杂的领域，定位知识能显著减少 Agent 的探索步数，提高执行效率。
*   **架构无关性**：证明了通过自然语言注入知识是一种通用的增强手段，无需修改模型权重即可提升 Agent 的专业能力。

#### 7. 优点（亮点）
*   **资源利用创新**：将 YouTube 视频转化为可搜索、可理解的结构化知识库，解决了数据稀缺问题。
*   **字幕驱动检索**：相比单纯的标题搜索，利用字幕语义极大地提高了检索的查准率，避免了非 GUI 视频的干扰。
*   **即插即用**：不改变 Agent 原有架构，具有极强的通用性和可扩展性。
*   **逆动力学标注**：通过对比前后帧自动推断操作意图，比传统的单帧标注更具逻辑深度。

#### 8. 不足与局限
*   **检索依赖性**：如果 YouTube 上缺乏相关视频（如跨应用复杂流），系统会退化为普通 Agent。
*   **版本漂移风险**：视频中的软件版本可能与测试环境不一致，导致 UI 描述失效（虽然文中提到 Agent 会优先信任当前截图，但仍存在干扰风险）。
*   **推理延迟**：注入长文本知识会增加 Prompt 长度，导致单步推理延迟增加（约增加 2.1 秒/步）。
*   **视频格式限制**：对于纯动画演示（非实操录屏）或无字幕视频，标注质量会大幅下降。
