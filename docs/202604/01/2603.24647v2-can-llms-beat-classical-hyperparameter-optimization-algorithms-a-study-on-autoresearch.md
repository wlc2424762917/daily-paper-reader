## 摘要
autoresearch 仓库允许 LLM 智能体通过直接编辑训练代码，在不受限的搜索空间中搜索最优超参数配置。在给定的固定计算预算和约束条件下，我们以 autoresearch 为实验平台，在微调小语言模型的超参数任务上，对比了经典超参数优化（HPO）算法与基于 LLM 的方法。在固定的超参数搜索空间内，CMA-ES 和 TPE 等经典 HPO 方法的表现始终优于基于 LLM 的智能体。然而，尽管仅使用自托管的开源权重 27B 模型，在不受限搜索空间中直接编辑训练源码的 LLM 智能体仍大幅缩小了与经典方法之间的差距。能够避免显存溢出故障的方法优于搜索多样性更高的方法，这表明可靠性比探索广度更重要。虽然中小型 LLM 难以在多次试验中追踪优化状态，但经典方法缺乏领域知识。为了弥补这一差距，我们引入了 Centaur，这是一种将 CMA-ES 的内部状态（包括均值向量、步长和协方差矩阵）与 LLM 共享的混合方法。Centaur 在我们的实验中取得了最佳结果，其 0.8B 变体的表现优于 27B 变体，这表明当与强大的经典优化器配合使用时，廉价的 LLM 就已足够。0.8B 模型不足以进行不受限的代码编辑，但足以进行混合优化；而对于固定搜索空间的方法，将模型规模扩大到 27B 并未带来优势。使用前沿模型 Gemini 3.1 Pro Preview 进行的初步实验未能消除与经典方法之间的差距。代码可在 https://github.com/ferreirafabio/autoresearch-automl 获取。

## Abstract
The autoresearch repository enables an LLM agent to search for optimal hyperparameter configurations on an unconstrained search space by editing the training code directly. Given a fixed compute budget and constraints, we use autoresearch as a testbed to compare classical hyperparameter optimization (HPO) algorithms against LLM-based methods on tuning the hyperparameters of a small language model. Within a fixed hyperparameter search space, classical HPO methods such as CMA-ES and TPE consistently outperform LLM-based agents. However, an LLM agent that directly edits training source code in an unconstrained search space narrows the gap to classical methods substantially despite using only a self-hosted open-weight 27B model. Methods that avoid out-of-memory failures outperform those with higher search diversity, suggesting that reliability matters more than exploration breadth. While small and mid-sized LLMs struggle to track optimization state across trials, classical methods lack domain knowledge. To bridge this gap, we introduce Centaur, a hybrid that shares CMA-ES's internal state, including mean vector, step-size, and covariance matrix, with an LLM. Centaur achieves the best result in our experiments, with its 0.8B variant outperforming the 27B variant, suggesting that a cheap LLM suffices when paired with a strong classical optimizer. The 0.8B model is insufficient for unconstrained code editing but sufficient for hybrid optimization, while scaling to 27B provides no advantage for fixed search space methods. Preliminary experiments with the frontier model Gemini 3.1 Pro Preview do not close the gap to classical methods. Code is available at https://github.com/ferreirafabio/autoresearch-automl.

### 2. 方法论
*   **核心思想**：对比四类方法：经典 HPO、纯 LLM 优化（固定空间）、LLM 代码编辑（无约束空间）以及作者提出的 **Centaur（半人马）** 混合优化方法。

    *   **算法流程**：在 Centaur 中，CMA-ES 负责维护全局优化状态。在 30% 的试验中，LLM 会接收到 CMA-ES 的建议值及其内部数学状态，并结合历史最优配置进行“知情”后的修改（Override）。这种方式结合了经典算法的数值搜索能力和 LLM 的领域知识（如对 Transformer 架构的直觉）。

### 3. 实验设计
*   **场景与任务**：优化一个约 50M 参数的小型 Transformer 模型（nanochat），在 FineWeb 数据集上训练，目标是最小化验证集的 bits-per-byte (val_bpb)。
*   **Benchmark 方法**：
    *   **经典方法**：TPE、CMA-ES、SMAC、随机搜索。
    *   **LLM 方法**：LLAMBO（两种版本）、Karpathy Agent（固定 14 个 HP）、Karpathy Agent（直接编辑代码）。
    *   **混合方法**：Centaur。
*   **对比模型**：主要使用开源的 Qwen3.5（0.8B 和 27B），并与闭源前沿模型 Gemini 3.1 系列进行对比。

### 4. 资源与算力
*   **硬件**：单台 NVIDIA **H200 GPU** (141 GB HBM3e)。
*   **预算**：每种方法分配 **24 小时** 的 GPU 训练时长。

*   **实验规模**：每种方法运行 **3 个随机种子**，报告均值和标准差。

### 6. 主要结论与发现
*   **经典算法依然强劲**：在固定的搜索空间内，TPE 和 CMA-ES 的表现始终优于纯 LLM 方法。
*   **代码编辑的潜力**：直接编辑源码的 LLM 智能体表现优于固定空间的 LLM，且能显著缩小与经典算法的差距，展现了 LLM 处理非结构化搜索空间的优势。
*   **Centaur 夺冠**：混合方法 Centaur 取得了最佳结果。有趣的是，**0.8B 的小模型在 Centaur 框架下优于 27B 模型**，说明在有强力经典优化器辅助时，轻量级 LLM 的领域直觉已足够。
*   **可靠性胜过多样性**：实验发现，能够学习避免显存溢出（OOM）故障的方法（如经典算法）比搜索多样性高但频繁失败的方法（如 LLAMBO）表现更好。

### 7. 优点
*   **创新性结合**：Centaur 巧妙地解决了 LLM 难以追踪长期数值优化状态的弱点，同时弥补了经典算法缺乏“Transformer 训练常识”的缺陷。
*   **工程实践性强**：展示了在有限算力下，通过自托管开源模型即可实现超越或匹配前沿闭源模型（如 Gemini Pro）的优化效果。

### 8. 不足与局限
*   **任务单一**：实验仅基于 `nanochat` 一个任务，尚未在更大规模模型或不同领域的任务上验证泛化性。
*   **LLM 局限性**：当前的 LLM 优化器没有配备外部工具（如搜索论文、查阅文档），限制了其发挥更强的 Agent 能力。
