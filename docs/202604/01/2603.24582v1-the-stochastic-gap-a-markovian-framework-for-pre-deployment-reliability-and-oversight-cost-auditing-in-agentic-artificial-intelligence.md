## 摘要
组织中的代理式人工智能（Agentic AI）是一个受可靠性和监管成本约束的序列决策问题。当确定性工作流被动作和工具调用的随机策略取代时，核心问题不在于下一步动作是否看起来合理，而在于生成的轨迹是否在统计上得到支持、在局部上无歧义且在经济上可治理。我们为此场景开发了一个测度论马尔可夫框架。其核心指标包括状态盲区测度 B_n(τ)、状态-动作盲区测度 B^{SA}_{π,n}(τ)、基于熵的人在回路（human-in-the-loop）升级门控，以及工作流访问测度上的预期监管成本恒等式。我们在 2019 年业务流程智能挑战赛（BPIC 2019）的采购到付款日志（包含 251,734 个案例、1,595,923 个事件、42 个不同的工作流动作）上实例化了该框架，并根据同一流程的 80/20 时间顺序划分构建了一个日志驱动的模拟智能体。主要实证结果表明，大型工作流在状态层面可能看起来得到了良好支持，但在下一步决策中仍保留了大量的盲区测度：通过将运营状态细化以包含案例上下文、经济规模和执行者类别，状态空间从 42 扩展到 668，状态-动作盲区测度在 τ=50 时从 0.0165 上升到 τ=1000 时的 0.1253。在留出集上，m(s) = max_a π-hat(a|s) 追踪到的实际自主步骤准确率平均误差在 3.4 个百分点以内。界定统计可信自主性的这些指标同时也决定了预期的监管负担。该框架在大型企业采购工作流中得到了验证，旨在直接应用于具备运营事件日志的工程流程。

## Abstract
Agentic artificial intelligence (AI) in organizations is a sequential decision problem constrained by reliability and oversight cost. When deterministic workflows are replaced by stochastic policies over actions and tool calls, the key question is not whether a next step appears plausible, but whether the resulting trajectory remains statistically supported, locally unambiguous, and economically governable. We develop a measure-theoretic Markov framework for this setting. The core quantities are state blind-spot mass B_n(tau), state-action blind mass B^SA_{pi,n}(tau), an entropy-based human-in-the-loop escalation gate, and an expected oversight-cost identity over the workflow visitation measure.   We instantiate the framework on the Business Process Intelligence Challenge 2019 purchase-to-pay log (251,734 cases, 1,595,923 events, 42 distinct workflow actions) and construct a log-driven simulated agent from a chronological 80/20 split of the same process. The main empirical finding is that a large workflow can appear well supported at the state level while retaining substantial blind mass over next-step decisions: refining the operational state to include case context, economic magnitude, and actor class expands the state space from 42 to 668 and raises state-action blind mass from 0.0165 at tau=50 to 0.1253 at tau=1000. On the held-out split, m(s) = max_a pi-hat(a|s) tracks realized autonomous step accuracy within 3.4 percentage points on average.   The same quantities that delimit statistically credible autonomy also determine expected oversight burden. The framework is demonstrated on a large-scale enterprise procurement workflow and is designed for direct application to engineering processes for which operational event logs are available.

### 论文总结：随机性差距（The Stochastic Gap）——代理式 AI 部署前审计框架

#### 1. 核心问题与整体含义（研究动机和背景）
随着企业将传统的确定性工作流（如 ERP 系统中的审批流）替换为基于大语言模型（LLM）的代理式 AI（Agentic AI），执行逻辑从“规则驱动”转向了“概率采样”。这种转变引入了**“随机性差距”（Stochastic Gap）**：即 Agent 在复杂、长程的工作流中，其决策是否具备足够的历史数据支持，以及由此产生的监管成本是否会让项目失去经济价值。论文旨在解决企业在部署 Agent 之前的核心疑虑：**系统是否可信？人类监管的成本是否可控？**

#### 2. 方法论：核心思想与技术细节
论文提出了一个基于测度论的马尔可夫框架，将工作流建模为有限状态马尔可夫决策过程（MDP）。
*   **核心指标：**
    *   **状态盲点质量（State Blind-spot Mass, $B_n(\tau)$）：** 衡量部署时可能遇到的低频或未见状态的比例。
    *   **状态-动作盲点质量（State-Action Blind Mass, $BSA_{\pi,n}(\tau)$）：** 衡量 Agent 需要做出决策时，缺乏历史数据支持的比例。这是评估 Agent 可靠性的关键，因为即使状态常见，特定的“状态-动作”组合可能极罕见。
*   **人在回路（HITL）升级门控：** 设计了一个自动触发机制，当满足以下任一条件时，Agent 必须将决策权移交给人类：
    1.  **支持度不足：** 历史样本数低于阈值 $\tau$。
    2.  **决策熵过高：** 动作分布的香农熵（Shannon Entropy）超过阈值 $h_0$，表示决策存在歧义。
    3.  **风险权重过高：** 涉及高额经济价值或敏感操作（如取消发票）。
*   **成本恒等式：** 建立了一个数学模型，将自主执行成本、人类监管成本与工作流的访问测度（Visitation Measure）挂钩，证明了可靠性边界直接决定了部署的经济可行性。

#### 3. 实验设计
*   **数据集：** 使用了真实的 **BPI Challenge 2019 (BPIC 2019)** 采购到付款（Purchase-to-Pay）事件日志。包含 251,734 个案例、近 160 万个事件和 42 种动作。
*   **实验场景：** 
    *   **描述性审计：** 在全量日志上分析不同状态抽象级别（从仅看“活动”到加入“金额、执行者类别、上下文”）对盲区测度的影响。
    *   **代理验证：** 采用 80/20 时间顺序切分数据，用前 80% 数据训练一个模拟 Agent，在后 20% 的留出集上验证理论指标（如最大概率 $m(s)$）与实际准确率的匹配度。
*   **对比方法：** 对比了三种不同精细度的状态抽象模型，观察随着上下文信息的增加，统计支持度如何急剧下降。

#### 5. 实验数量与充分性
*   **实验规模：** 实验基于超过 150 万条真实业务记录，样本量极其充分，具有很强的统计说服力。
*   **消融与对比：** 论文通过调整支持度阈值 $\tau$、熵阈值 $h_0$ 和风险权重 $w_0$ 进行了多组灵敏度分析。
*   **客观性：** 实验采用了严格的时间顺序切分（Chronological Split），模拟了真实的部署前预测场景，避免了随机切分带来的数据泄露问题，实验设计客观公平。

#### 6. 主要结论与发现
*   **状态覆盖不等于动作可靠：** 即使 99% 的状态在历史中出现过，仍有超过 12% 的决策动作（State-Action pairs）缺乏足够支持。
*   **上下文导致“支持度坍塌”：** 当状态定义加入经济金额和执行者类别后，状态空间从 42 扩展到 668，盲区测度显著上升。
*   **路径依赖效应：** 局部决策的微小不确定性会在长程工作流中累积。在某些设定下，虽然 72% 的步骤可以自主完成，但仅有 49% 的案例能实现全流程端到端自主。
*   **预测准确性：** 理论预测的自主步骤准确率与实际观测值的平均偏差仅为 **3.4 个百分点**，证明了该框架作为部署前审计工具的有效性。

#### 7. 优点（亮点）
*   **理论与经济结合：** 首次将 Agent 的统计可靠性（熵、支持度）与企业的监管成本（ROI）通过数学恒等式统一起来。
*   **直接可用性：** 框架不依赖于特定的 LLM 架构，只要企业拥有符合标准的事件日志（Event Logs），即可进行审计。
*   **风险感知：** 引入了风险权重，使得审计不仅关注“能不能做对”，还关注“做错的代价”。

#### 8. 不足与局限
*   **马尔可夫假设：** 采用了一阶马尔可夫近似，可能忽略了极长距离的上下文依赖。
*   **观测性偏差：** 实验基于历史日志，无法模拟 Agent 采取历史记录中从未出现过的“创新性”动作后的反馈（即缺乏反事实评估）。
*   **状态抽象的艺术：** 状态定义的精细程度高度依赖领域专家的经验，错误的抽象可能导致盲区指标失真。
*   **应用限制：** 主要适用于结构化或半结构化的企业业务流程，对于完全开放域、无日志记录的 Agent 场景（如创意写作 Agent）适用性较弱。
