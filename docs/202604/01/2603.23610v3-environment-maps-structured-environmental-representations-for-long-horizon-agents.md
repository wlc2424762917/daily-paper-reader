## 摘要
尽管大语言模型（LLMs）发展迅速，但复杂软件工作流的鲁棒自动化仍是一个悬而未决的问题。在长程任务场景中，智能体经常面临级联错误和环境随机性的困扰；动态界面中的一次失误就可能导致任务失败，进而引发幻觉或陷入盲目的试错。本文引入了“环境地图”（Environment Maps）：一种持久的、与智能体无关的表示方式，通过将屏幕录像和执行轨迹等异构证据整合进一个结构化图中，来缓解上述失败问题。该表示由四个核心组件组成：(1) 上下文（抽象位置），(2) 动作（参数化示能），(3) 工作流（观测到的轨迹），以及 (4) 隐性知识（领域定义和可重用程序）。我们在 WebArena 基准测试的五个领域中评估了该框架。配备环境地图的智能体实现了 28.2% 的成功率，几乎是仅限于会话绑定上下文的基准模型（14.2%）的两倍，且优于能够访问用于生成环境地图的原始轨迹数据的智能体（23.3%）。通过在模型与环境之间提供结构化接口，环境地图为长程规划建立了一个持久的基础，且具有人类可解释性、可编辑性和增量改进的特点。

## Abstract
Although large language models (LLMs) have advanced rapidly, robust automation of complex software workflows remains an open problem. In long-horizon settings, agents frequently suffer from cascading errors and environmental stochasticity; a single misstep in a dynamic interface can lead to task failure, resulting in hallucinations or trial-and-error. This paper introduces $\textit{Environment Maps}$: a persistent, agent-agnostic representation that mitigates these failures by consolidating heterogeneous evidence, such as screen recordings and execution traces, into a structured graph. The representation consists of four core components: (1) Contexts (abstracted locations), (2) Actions (parameterized affordances), (3) Workflows (observed trajectories), and (4) Tacit Knowledge (domain definitions and reusable procedures). We evaluate this framework on the WebArena benchmark across five domains. Agents equipped with environment maps achieve a 28.2% success rate, nearly doubling the performance of baselines limited to session-bound context (14.2%) and outperforming agents that have access to the raw trajectory data used to generate the environment maps (23.3%). By providing a structured interface between the model and the environment, Environment Maps establish a persistent foundation for long-horizon planning that is human-interpretable, editable, and incrementally refinable.

#### 1. 核心问题与研究动机
*   **核心问题**：尽管大语言模型（LLM）在短期任务上表现出色，但在处理**长程（Long-horizon）复杂软件工作流**时，极易受到“级联错误”的影响。一旦在动态界面中发生一次误操作，智能体往往无法自我纠正，导致幻觉或无效的试错。
*   **研究动机**：现有的智能体通常缺乏对环境的持久认知。UI 的漂移、领域特定的语义以及缺乏对动作后果的预测，使得智能体在复杂系统中如同“盲人摸象”。作者认为，需要一种**持久的、结构化的、与具体智能体无关**的环境表示，来连接原始观测与高层规划。

#### 2. 方法论：环境地图（Environment Maps）
该方法的核心思想是将异构的执行证据（如屏幕录像、DOM 树、执行轨迹）转化为一个结构化的图表示 $M = (C, A, W, K)$。

*   **核心组件**：
    1.  **Contexts (C, 上下文)**：抽象的页面位置，通过 URL 模式识别（如 `/users/{id}`）。
    2.  **Actions (A, 动作)**：参数化的示能（Affordances），例如将“点击张三”抽象为“点击 {用户名}”。
    3.  **Workflows (W, 工作流)**：观察到的成功操作序列。
    4.  **Tacit Knowledge (K, 隐性知识)**：领域定义（如缩写含义）和常用操作规程。
*   **构建流程（5个阶段）**：
    *   **阶段1：统一步骤序列**。解析原始观测，提取动作名称、描述和结果。
    *   **阶段2：动作提取**。从轨迹中提取已执行动作，并从 DOM 中识别潜在动作。
    *   **阶段3：动作泛化**。利用 LLM 识别模式，将具体动作抽象为带参数的模板。
    *   **阶段4：上下文与知识提取**。通过 URL 归一化定义页面，并提取操作规程。
    *   **阶段5：集成与合并**。去重并建立实体间的追溯链接，生成最终的 JSON 格式地图。

#### 3. 实验设计
*   **数据集/场景**：使用 **WebArena** 基准测试，涵盖 5 个真实交互环境：电子商务（Magento）、管理后台（CMS）、代码托管（GitLab）、论坛（Reddit）和地图（OpenStreetMap）。
*   **评估框架**：采用 **WebArena Verified**，通过 HAR 追踪检查确保评估的确定性和可重复性。
*   **对比方法**：
    1.  **Baseline (No Map)**：仅提供当前页面观测（可访问性树）和任务意图。
    2.  **Trajectory Access**：允许智能体访问原始的人类操作轨迹文件（JSONL 格式），但未经过结构化处理。
    3.  **Environment Map Access**：提供基于相同轨迹数据生成的结构化环境地图。
*   **智能体配置**：基于 `claude-3-5-sonnet` 的 Claude Agent SDK。

#### 4. 资源与算力
*   **训练算力**：论文**未提及**模型训练的具体 GPU 型号或时长，因为该方法主要基于 Prompting 和外部知识检索，而非模型微调。
*   **构建成本**：论文明确指出，为 5 个环境构建地图的成本极低，每个环境仅需 **1-4 美元** 的 API 费用，耗时 **13-31 分钟**。这与动辄消耗数百 GPU 小时的微调（SFT）相比，具有极高的成本优势。

#### 5. 实验数量与充分性
*   **实验规模**：在 5 个领域共进行了 **812 个任务** 的测试，样本量充足。
*   **消融与对比**：通过对比“原始轨迹”和“结构化地图”，有效地证明了**知识的组织形式**（而非数据本身）是性能提升的关键。
*   **客观性**：使用了 95% 置信区间进行统计分析，并采用了比原始 WebArena 更严格的 Verified 评估标准，减少了因智能体幻觉导致的误判。

#### 6. 主要结论与发现
*   **性能翻倍**：配备环境地图的智能体成功率达到 **28.2%**，远高于基准的 **14.2%**。
*   **结构化优于原始数据**：即使拥有相同的数据源，结构化的地图（28.2%）也显著优于原始轨迹（23.3%）。
*   **泛化能力**：在人类轨迹未覆盖的任务上，环境地图依然带来了显著的性能提升（从 12.6% 提升至 26.9%），证明了地图的抽象泛化价值。
*   **复杂环境受益更大**：在 GitLab 和 CMS 等 UI 密集、分支复杂的环境中，地图带来的增益最为明显。

#### 7. 优点
*   **智能体无关性**：地图是独立的知识库，可以适配不同的 LLM 或 Prompt 策略。
*   **人类可读与可编辑**：JSON 格式的地图允许人类专家进行干预、修正或更新，具有极高的可解释性。
*   **低成本维护**：无需重新训练模型即可应对 UI 变更，只需更新地图中的特定节点。
*   **减少回溯**：HAR 分析显示，使用地图的智能体在操作中的回溯（Backtracking）明显减少，效率更高。

#### 8. 不足与局限
*   **跨站点任务瓶颈**：目前的地图主要针对单一环境，对于跨站点的复杂工作流（Multi-site tasks）几乎没有提升。
*   **时效性挑战**：随着 Web 环境的演进，地图可能会过时，如何自动检测并修复地图的“陈旧性”仍需研究。
*   **状态验证不足**：目前的动作前置条件和效果主要是描述性的，缺乏基于实时状态观测的硬性约束验证。
*   **覆盖范围**：实验未涵盖 Wikipedia 等信息检索类任务，主要侧重于交互式操作。
