## 摘要
强化学习是真正扩展了大语言模型（LLM）智能体的能力，还是仅仅提高了它们的可靠性？对于静态推理，最近的研究倾向于后者：基础模型和强化学习（RL）模型的 pass@k 曲线在 k 较大时趋于收敛。我们探究这是否适用于智能体工具使用场景，在这种场景下，T 轮交互实现了重采样无法恢复的组合策略。我们引入了 PASS@(k,T)，这是一种同时改变采样预算 k 和交互深度 T 的二维指标，旨在将能力扩展与效率提升区分开来。我们的主要发现是，与静态推理的结果相反，工具使用的强化学习确实扩大了能力边界：RL 智能体的 pass 曲线位于基础模型之上，且差距在 k 较大时会扩大而非收敛。这种扩展专门针对组合式、序列化的信息采集；在更简单的任务上，强化学习的表现与先前研究的预测一致。在训练数据匹配的情况下，有监督微调（SFT）在相同的组合任务上反而导致了边界退化，从而确定了自主探索是其中的因果因素。机制分析表明，强化学习重新调整了基础策略分布的权重，使其向那些下游推理更常产生正确答案的子集倾斜，改进主要集中在智能体如何整合检索到的信息。这些结果调和了对 LLM 强化学习的乐观和悲观解读：两者在不同的任务类型上都是正确的。

## Abstract
Does reinforcement learning genuinely expand what LLM agents can do, or merely make them more reliable? For static reasoning, recent work answers the second: base and RL pass@k curves converge at large k. We ask whether this holds for agentic tool use, where T rounds of interaction enable compositional strategies that re-sampling cannot recover. We introduce PASS@(k,T), a two-dimensional metric that jointly varies sampling budget k and interaction depth T, separating capability expansion from efficiency improvement. Our main finding is that, contrary to the static-reasoning result, tool-use RL genuinely enlarges the capability boundary: the RL agent's pass-curve pulls above the base model's and the gap widens at large k rather than converging. The expansion is specific to compositional, sequential information gathering; on simpler tasks RL behaves as prior work predicts. Under matched training data, supervised fine-tuning regresses the boundary on the same compositional tasks, isolating self-directed exploration as the causal factor. Mechanism analysis shows RL reweights the base strategy distribution toward the subset whose downstream reasoning more often yields a correct answer, with the improvement concentrated on how the agent integrates retrieved information. These results reconcile optimistic and pessimistic readings of RL for LLMs: both are correct, on different task types.

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题**：强化学习是真正扩展了 LLM 智能体的“能力边界”（即让它学会以前完全做不到的事），还是仅仅提高了“效率/可靠性”（即让它更稳健地完成以前偶尔能做到的事）？
*   **背景**：在静态数学推理领域，近期研究（如 Yue et al., 2025）认为 RL 只是重新分配了概率质量，其能力上限受限于基础模型。本论文质疑这一悲观结论是否适用于“智能体工具使用”场景，因为智能体与环境的多轮交互（T）可能产生重采样（k）无法替代的组合策略。

### 2. 方法论：PASS@(k, T) 框架
*   **核心思想**：提出一个二维评估指标 **PASS@(k, T)**，将智能体的表现分解为两个维度：
    *   **k（采样预算）**：独立尝试的次数，用于衡量采样效率和可靠性。
    *   **T（交互深度）**：每轮尝试中允许与环境交互的最大轮数，用于衡量处理复杂组合任务的能力。
*   **关键定义**：
    *   **能力边界 (Capability Boundary)**：在 $k \to \infty$ 时，模型能够解决的问题集合。
    *   **能力扩展 (Capability Expansion)**：RL 模型解决了一些基础模型在任何采样预算下都无法解决的问题。
    *   **效率提升 (Efficiency Improvement)**：RL 模型在基础模型本就能解决的问题上，提高了单次尝试（pass@1）的成功率。
*   **算法流程**：通过在不同的 $(k, T)$ 组合下进行 $n$ 次独立采样，使用无偏超几何估计器计算成功概率。

### 3. 实验设计
*   **任务分类**：
    *   **Category A (纯推理)**：MATH-500，不使用工具（对照组）。
    *   **Category B (独立检索)**：HotPotQA 比较类问题，需两次独立检索。
    *   **Category C (序列检索)**：HotPotQA 桥接类问题，需多步组合检索（第二步依赖第一步结果）。
*   **对比方法**：
    *   **$\pi_{base}$**：基础模型（Qwen2.5-7B-Instruct）。
    *   **$\pi_{SFT}$**：在 200 个专家轨迹上进行有监督微调。
    *   **$\pi_{RL}$**：使用 **GRPO** 算法在相同的 200 个问题上进行强化学习训练。
    *   **$\pi_{RL+explore}$**：加入探索奖励的 RL 变体。
*   **环境**：基于 BM25 的文档检索工具。

### 4. 资源与算力
*   **硬件**：单张 **NVIDIA RTX 6000 Ada (48GB)** GPU。
*   **训练时长**：$\pi_{RL}$ 训练约 **55 小时**（1000 个梯度步，LoRA 秩 16）。$\pi_{SFT}$ 训练约 12 分钟。
*   **推理成本**：全量评估（4 个模型 × 3 个类别 × 32,000 条轨迹）约耗时 8 GPU 小时。

### 5. 实验数量与充分性
*   **实验规模**：每个类别 100 个测试问题，每个 $(q, T)$ 组合进行 $n=64$ 次采样。
*   **充分性**：
    *   涵盖了从简单推理到复杂组合检索的不同难度梯度。
    *   进行了**机制分析**（困惑度分解、策略多样性测量、跨策略交换实验），从因果角度解释了性能提升的来源。
    *   通过自助法（Bootstrap）计算置信区间，验证了结果的统计显著性。
    *   **公平性**：SFT 和 RL 使用完全相同的 200 个训练问题，排除了数据暴露带来的偏差。

### 6. 主要结论与发现
*   **RL 确实扩展了能力边界**：在复杂的序列检索任务（Category C）中，RL 模型的 pass 曲线在 $k$ 较大时与基础模型的差距拉大，而非收敛。这证明了 RL 教授了基础模型无法通过简单重采样获得的新能力。
*   **SFT 的负面效应**：在相同的复杂任务上，SFT 反而导致了能力边界的退化（丢失了基础模型原本能解决的问题），这表明自主探索（Exploration）是 RL 成功的关键。
*   **任务依赖性**：在简单任务或纯推理任务上，RL 的表现符合“效率提升”模型（与前人研究一致）；但在需要组合策略的智能体任务中，RL 表现出独特的能力扩展。
*   **机制解释**：RL 并非发明了全新的搜索词，而是通过**重新加权**，优化了模型如何整合检索到的信息并进行后续推理的策略分布。

### 7. 优点
*   **维度创新**：引入 $T$ 轴打破了以往对 pass@k 的一维认知，为智能体评估提供了更科学的工具。
*   **因果分离**：通过匹配数据的 SFT vs RL 实验，清晰地隔离出“学习信号”而非“数据量”是能力提升的主因。
*   **调和矛盾**：解释了为什么关于 RL 的研究会出现“乐观”与“悲观”两种截然不同的结论——关键在于任务是否具有组合复杂性。

### 8. 不足与局限
*   **模型规模限制**：主要实验基于 7B 模型，尚未在大规模模型（如 70B+）上验证该结论的普适性。
*   **工具单一性**：仅使用了简单的检索工具，未涉及更复杂的 API 调用或多工具协作场景。
*   **训练集较小**：仅使用了 200 个问题的微型训练集，虽然足以证明原理，但可能限制了模型潜力的完全释放。
*   **检索环境**：使用的是静态的 BM25 检索而非动态的互联网搜索，环境反馈的丰富度有限。
