## 摘要
随着 Web 智能体的快速演进，越来越多的研究已超越了传统的原子级浏览器交互，开始探索将工具使用作为一种更高层级的动作范式。尽管先前的研究展示了工具的潜力，但其结论往往基于有限的实验规模，且有时处于不可比的设置之下。因此，几个基本问题仍不明确：i) 工具是否能为 Web 智能体带来持续的增益；ii) 有效工具的实际设计原则是什么；iii) 工具使用可能会引入哪些副作用。为了给未来的研究奠定更坚实的实证基础，我们通过一项广泛且经过严格控制的研究，重新审视了 Web 智能体中的工具使用，涵盖了多种工具来源、骨干模型、工具使用框架和评估基准。我们的发现既修正了先前的一些结论，又通过更广泛的证据补充了其他结论。我们希望这项研究能提供更可靠的实证基础，并启发未来关于工具使用型 Web 智能体的研究。

## Abstract
As web agents rapidly evolve, an increasing body of work has moved beyond conventional atomic browser interactions and explored tool use as a higher-level action paradigm. Although prior studies have shown the promise of tools, their conclusions are often drawn from limited experimental scales and sometimes non-comparable settings. As a result, several fundamental questions remain unclear: i) whether tools provide consistent gains for web agents, ii) what practical design principles characterize effective tools, and iii) what side effects tool use may introduce. To establish a stronger empirical foundation for future research, we revisit tool use in web agents through an extensive and carefully controlled study across diverse tool sources, backbone models, tool-use frameworks, and evaluation benchmarks. Our findings both revise some prior conclusions and complement others with broader evidence. We hope this study provides a more reliable empirical basis and inspires future research on tool-use web agents.

论文并未提出单一的新算法，而是构建了一个**多维度的实证研究框架**，核心思想包括：
*   **多框架对比**：对比了三种代表性框架：*Hybrid-Agent*（使用人工编写的 REST API）、*SkillWeaver*（LLM 自主合成复杂函数）和 *WALT*（LLM 合成且侧重 URL 操作的工具）。

*   **语义技能（Semantic Skills）转换**：提出将“黑盒”的可执行代码工具转换为“白盒”的自然语言描述（即技能），让智能体通过阅读操作指南而非直接调用函数来完成任务。
*   **能力蒸馏视角**：将 LLM 合成工具的过程视为一种从强模型到弱模型的能力蒸馏。

### 3. 实验设计
*   **数据集 / 场景**：使用了 **WebArena** 和 **VisualWebArena** 两个主流基准测试，涵盖购物、内容管理（CMS）、GitLab、Reddit 和地图等真实 Web 环境。
*   **对比方法**：

*   **骨干模型（Backbone Models）**：测试了 5 种不同规模和家族的模型：GPT-5、GPT-5-mini、GPT-5-nano、Grok-4.1-Fast-Reasoning 和 Mistral-Large-3.1。

### 4. 资源与算力
*   **算力说明**：文中未明确提及具体的 GPU 硬件型号或训练时长。
*   **资源消耗**：实验主要通过 Azure API 服务调用模型。作者特别提到了“工具税（Tool Tax）”，即使用工具会显著增加 **Token 消耗**（尤其是 Hybrid-Agent 框架，因其庞大的 API 库导致 Prompt 极长）。此外，工具合成过程本身也被描述为“昂贵的（expensive）”。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了 3 个框架、5 种骨干模型、2 个大型基准测试，并针对工具复杂度、调用频率、Token 成本、视觉消融等进行了多项专项分析。
*   **充分性与客观性**：实验设计非常充分且严谨。通过交叉验证“工具开发者”与“工具使用者”的强弱关系，消除了单一模型带来的偏差。对比了人工工具与合成工具的差异，结论具有较强的说服力和客观性。

### 6. 论文的主要结论与发现
*   **单向能力蒸馏**：LLM 合成的工具主要表现为“强对弱”的增益。只有当工具合成者明显强于使用者时，增益才稳定；强模型使用弱模型合成的工具往往会导致性能下降。
*   **复杂度陷阱**：并非工具越复杂越好。过度针对特定任务的复杂工具会降低通用性，且在实际中很少被调用。
*   **工具税**：工具使用并不总是提高效率。大型工具库会增加 Token 成本，且由于工具选择、错误恢复等环节，动作步数（Action Overhead）有时反而会增加。
*   **语义技能的优势**：对于强模型，透明的自然语言“技能指南”比黑盒代码工具更有效，因为其容错性更高。
*   **视觉的重要性**：即使有了工具，视觉信息（截图）依然能提供互补增益，使智能体更具鲁棒性。

### 7. 优点
*   **去伪存真**：挑战了“工具越多越好”的直觉，揭示了工具使用中的“幻觉”和潜在成本。
*   **设计原则明确**：提出了“解耦推理与执行”的原则，建议工具应专注于确定性的 UI 操作，而将推理留给智能体。
*   **实证深度**：不仅关注成功率，还深入分析了 Token 成本、步数效率和工具组合性，提供了全方位的评估。

### 8. 不足与局限
*   **基准局限性**：实验主要集中在 WebArena 系列，虽然具有代表性，但尚未涵盖所有类型的 Web 交互场景。
*   **合成成本**：虽然分析了工具的副作用，但对于如何降低“工具合成”阶段的高昂成本缺乏深入探讨。
*   **模型依赖**：结论高度依赖于当前最先进模型的表现，随着模型能力的进一步演进，某些结论（如语义技能与代码工具的界限）可能需要持续更新。
