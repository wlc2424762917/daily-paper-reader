## 摘要
我们提出了 ATLAS-RTC，这是一种用于自回归语言模型的运行时控制系统，可在解码过程中强制执行结构化输出。ATLAS-RTC 在每一步监控生成过程，利用轻量级信号检测与输出契约的偏离，并应用针对性的干预措施，如偏置（biasing）、掩码（masking）和回滚（rollback）。与事后验证或静态约束解码不同，它以闭环方式运行，能够在错误发生前进行修正。在结构化生成和工具调用任务中，ATLAS-RTC 将首次尝试成功率提高了 20 到 37.8 个百分点，在故障主导的场景下，延迟降低了高达 88%。结果表明，许多失败源于解码伪影而非对任务的误解，这促使运行时控制成为大语言模型系统中一个独立的层级。

## Abstract
We present ATLAS-RTC, a runtime control system for autoregressive language models that enforces structured output during decoding. ATLAS-RTC monitors generation at each step, detects drift from output contracts using lightweight signals, and applies targeted interventions such as biasing, masking, and rollback. Unlike post-hoc validation or static constrained decoding, it operates in a closed loop, enabling correction before errors materialize. Across structured generation and tool-calling tasks, ATLAS-RTC improves first-attempt success rates by 20 to 37.8 percentage points, with up to 88% latency reduction in failure-dominated settings. Results show that many failures arise from decoding artifacts rather than task misunderstanding, motivating runtime control as a distinct layer in LLM systems.

    *   **阶段感知契约（Stage-aware Contracts）**：将生成划分为不同阶段（如：键名区、分隔符区、值生成区）。在结构决策点实施严格限制，在值生成区放宽限制以保持模型表达力。
    *   **梯度干预阶梯（Ladder Policy）**：根据风险值从低到高采取：无操作 -> Logit 偏置 -> 温度调节 -> 令牌掩码 -> **中途回滚与重定向（Rollback & Re-steering）**。
    *   **KV 缓存回滚**：当检测到严重偏离时，截断 KV 缓存并回退 $n$ 个令牌，强制重新生成符合契约的路径。

### 7. 优点
*   **动态性**：相比静态语法约束，它能感知生成阶段，避免在需要模型自由发挥（如生成字符串值）时过度干预。
*   **闭环纠错**：引入了回滚机制，允许模型在发现走错路时“撤回”并修正，而不是直接报错。
*   **非侵入性**：无需修改模型权重，作为推理侧的插件层运行。

### 8. 不足与局限
*   **语义与结构的脱节**：ATLAS-RTC 只能保证“格式正确”，无法保证“内容正确”。如果模型漏掉了关键信息（如邮件摘要），系统无法凭空补全。
*   **计算开销**：逐令牌的监控和潜在的回滚会增加计算复杂度，在模型本身表现已经很好的简单任务中，可能会引入额外的延迟。
*   **复杂 Schema 挑战**：在处理多行字符串或极其复杂的嵌套结构时，性能会有所下降。
*   **漂移检测限制**：目前的检测主要基于启发式信号，尚未能捕捉深层的语义偏离。
