## 摘要
GUI 流程自动化 (GPA) 是一种轻量级且通用的基于视觉的机器人流程自动化 (RPA)，仅需单次演示即可实现快速且稳定的流程回放。针对传统 RPA 的脆弱性以及当前基于视觉语言模型 (VLM) 的 GUI 智能体的不确定性风险，GPA 引入了三大核心优势：(1) 鲁棒性：通过基于顺序蒙特卡洛 (Sequential Monte Carlo) 的定位来处理缩放和检测的不确定性；(2) 确定性与可靠性：通过就绪状态校准 (readiness calibration) 提供保障；(3) 隐私性：通过快速、完全本地的执行来确保。该方法提供了企业级工作流所需的适应性、鲁棒性和安全性。它还可以作为 MCP/CLI 工具供其他具备编程能力的智能体使用，使智能体仅负责推理和编排，而由 GPA 处理 GUI 执行。我们进行了一项初步实验，将 GPA 与 Gemini 3 Pro（配备 CUA 工具）进行对比，发现 GPA 在完成长程 GUI 任务时，不仅成功率更高，且执行速度快了 10 倍。

## Abstract
GUI Process Automation (GPA) is a lightweight but general vision-based Robotic Process Automation (RPA), which enables fast and stable process replay with only a single demo. Addressing the fragility of traditional RPA and the non-deterministic risks of current vision language model-based GUI agents, GPA introduces three core benefits: (1) Robustness via Sequential Monte Carlo-based localization to handle rescaling and detection uncertainty; (2) Deterministic and Reliability safeguarded by readiness calibration; and (3) Privacy through fast, fully local execution. This approach delivers the adaptability, robustness, and security required for enterprise workflows. It can also be used as an MCP/CLI tool by other agents with coding capabilities so that the agent only reasons and orchestrates while GPA handles the GUI execution. We conducted a pilot experiment to compare GPA with Gemini 3 Pro (with CUA tools) and found that GPA achieves higher success rate with 10 times faster execution speed in finishing long-horizon GUI tasks.

### 1. 核心问题与研究背景
*   **研究动机**：传统的机器人流程自动化（RPA）依赖于硬编码的选择器（如HTML标签），一旦UI微调或分辨率改变就会失效（脆弱性）。而新兴的基于大视觉语言模型（VLM）的GUI智能体虽然灵活，但存在非确定性（容易产生幻觉）、高延迟以及隐私风险（需上传截图到云端）。
*   **核心问题**：如何构建一个既能像VLM一样从演示中学习，又能像传统RPA一样确定、高效且保护隐私的GUI自动化系统？
*   **整体含义**：GPA（GUI Process Automation）通过“单次演示+几何匹配”的思路，将GUI自动化建模为视觉图匹配和定位问题，实现了在本地环境下的稳健回放。

### 2. 方法论
GPA 分为**演示阶段**和**执行阶段**，核心思想是将UI状态表示为图，并将定位视为概率推理问题。
*   **UI 图表示**：将截图解析为图 $G=(V, E)$。节点 $V$ 包含边界框、OCR文本和由 IconCLIP 提取的视觉特征；边 $E$ 通过空间近邻（k-NN）连接。
*   **关键技术：序贯蒙特卡洛（SMC）定位**：
    *   **建模**：将目标定位建模为后验概率 $p(\theta | Z)$，其中 $\theta = [x, y, s_x, s_y]$ 包含坐标和水平/垂直缩放比例。
    *   **几何上下文**：不仅匹配目标元素，还利用演示中的邻居节点作为参照。即使目标元素外观改变或检测器漏检，也能通过邻居的相对位置推算目标位置。
    *   **算法流程**：使用粒子滤波（Particle Filter）近似后验分布。通过似然回火（Likelihood Tempering）和 MCMC 步进行粒子重采样和复原，防止粒子退化并处理多峰分布。
*   **就绪校准（Readiness Checking）**：引入置信度评分 $C = \tilde{p}(Z | \theta) \times C_{spatial}$。只有当观测似然和空间一致性均超过阈值时才执行动作，否则进行重试。
*   **有限状态机（FSM）**：通过 FSM 管理执行逻辑（执行、重试、失败、完成），并包含“滚动查找”等特殊逻辑。

### 6. 主要结论与发现
*   **性能卓越**：GPA 在所有任务中实现了 **100% 的成功率**，而 Gemini 在长程任务中成功率降至 87.6%。
*   **速度极快**：GPA 的平均执行速度比 Gemini **快约 10 倍**（33.7秒 vs 329.3秒）。
*   **可靠性高**：GPA 的确定性回放和就绪检查机制有效避免了 VLM 常见的“幻觉”点击和步骤漂移。

### 7. 优点与亮点
*   **鲁棒的定位算法**：引入 SMC 处理 UI 缩放和检测不确定性，比简单的坐标匹配或模板匹配更具适应性。
*   **隐私与效率兼顾**：完全本地化运行，无需将敏感截图上传至云端，且延迟极低。
*   **易用性**：仅需一次演示即可生成自动化脚本，无需编写代码或手动标注选择器。
*   **预检查机制**：通过后台预处理下一阶段任务，进一步降低了感知延迟。

*   **缺乏推理能力**：GPA 本质上是“记录与回放”系统，无法处理需要逻辑判断的任务（例如：根据当前日期在日历中选择特定日期，若演示和执行时的月份不同，GPA 将失效）。
*   **应用限制**：对于高度动态、布局完全重构的 UI 适应性有限。
