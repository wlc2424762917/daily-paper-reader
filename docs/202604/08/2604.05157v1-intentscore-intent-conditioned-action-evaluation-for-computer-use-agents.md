## 摘要
计算机使用智能体（CUAs）利用大语言模型在桌面环境中执行 GUI 操作，但它们在生成动作时往往不评估动作质量，导致不可逆的错误在后续步骤中产生级联影响。我们提出了 IntentScore，这是一种感知计划的奖励模型，它从跨越三个操作系统的 39.8 万个离线 GUI 交互步骤中学习对候选动作进行评分。IntentScore 采用两个互补的目标进行训练：用于状态-动作相关性的对比对齐和用于动作正确性的边际排名。在架构上，它将每个候选动作的规划意图嵌入到动作编码器中，从而能够区分具有相似动作但逻辑不同的候选动作。IntentScore 在留出评估中实现了 97.5% 的成对判别准确率。作为 Agent S3 在 OSWorld（一个训练期间完全未见的环境）上的重排序器部署时，IntentScore 将任务成功率提高了 6.9 个百分点，证明了从异构离线轨迹中学习到的奖励估计可以泛化到未见的智能体和任务分布。

## Abstract
Computer-Use Agents (CUAs) leverage large language models to execute GUI operations on desktop environments, yet they generate actions without evaluating action quality, leading to irreversible errors that cascade through subsequent steps. We propose IntentScore, a plan-aware reward model that learns to score candidate actions from 398K offline GUI interaction steps spanning three operating systems. IntentScore trains with two complementary objectives: contrastive alignment for state-action relevance and margin ranking for action correctness. Architecturally, it embeds each candidate's planning intent in the action encoder, enabling discrimination between candidates with similar actions but different rationales. IntentScore achieves 97.5% pairwise discrimination accuracy on held-out evaluation. Deployed as a re-ranker for Agent S3 on OSWorld, an environment entirely unseen during training, IntentScore improves task success rate by 6.9 points, demonstrating that reward estimation learned from heterogeneous offline trajectories generalizes to unseen agents and task distributions.

### 1. 核心问题与整体含义（研究动机和背景）
目前的计算机使用智能体（CUAs）主要依赖大语言模型（LLM）来规划和执行图形用户界面（GUI）操作。然而，这些智能体通常只生成单一动作或简单选择最频繁的候选动作，缺乏对动作质量的评估机制。在 GUI 环境中，一旦点击错误按钮或关闭对话框，环境状态往往不可逆转，导致错误级联。
**核心问题：** 如何在不增加昂贵的 LLM 调用或环境副本的前提下，构建一个轻量级、独立的奖励模型，从多个候选动作中识别出最正确的动作？

### 2. 方法论
论文提出了 **IntentScore**，一个感知计划（Plan-aware）的双编码器奖励模型。
*   **核心思想：** 利用大规模离线 GUI 交互轨迹，学习一个能够区分“相关动作”与“正确动作”的奖励估计器，并将智能体的“规划意图”（Thought）作为评估的关键条件。
*   **关键技术细节：**
    *   **双编码器架构：** 包含状态编码器（处理截图、文本上下文、历史记录）和**意图感知动作编码器**。
    *   **意图嵌入：** 将候选动作的“思考过程”（Rationale/Thought）嵌入动作编码器而非状态编码器。这使得模型能区分动作表现相似但背后逻辑不同的候选方案。
    *   **双目标训练框架：**
        1.  **对比对齐（InfoNCE）：** 学习状态与动作的相关性（即“这个动作是否属于这个界面”）。
        2.  **边际排名（Margin Ranking）：** 引入硬负样本（如同一任务中相邻步骤的动作），学习动作的正确性（即“这个动作是否是当前步骤的最佳选择”）。
    *   **两阶段训练：** 先在跨操作系统（Windows, Mac, Ubuntu）的 39.8 万步数据上预训练，再在目标领域（Ubuntu）微调。

### 6. 主要结论与发现
*   **性能提升：** IntentScore 在离线判别任务中达到 **97.5%** 的准确率；在 OSWorld 在线测试中，将 Agent S3 的成功率从 45.2% 提升至 **52.1%**（+6.9 个百分点）。

*   **对齐不等于正确：** 仅靠对比对齐（InfoNCE）无法区分正确动作和语义相关的错误动作，必须引入边际排名损失来注入正确性信号。

### 7. 优点
*   **轻量高效：** 13M 参数量使其极易部署，且不增加 LLM 推理成本。
*   **泛化性强：** 能够处理未见过的操作系统、智能体架构和任务分布。
*   **设计巧妙：** 通过将意图放入动作编码器，解决了 GUI 动作中“看似相同、意图不同”的判别难题。

### 8. 不足与局限
*   **离线-在线差距：** 尽管离线准确率极高（97.5%），但在线成功率提升（+6.9%）相对温和，反映了部署时的分布偏移（Distribution Shift）问题。

*   **单步评估：** 目前仅针对单步动作进行评分，尚未扩展到多步前瞻（Look-ahead）或长期价值估计。
