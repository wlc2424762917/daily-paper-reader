## 摘要
现代任务导向型聊天机器人将图形用户界面（GUI）元素与自然语言对话结合呈现，但代理的角色在很大程度上仍局限于将自然语言输入解释为 GUI 操作并遵循线性工作流。在诸如预订机票或餐厅等由偏好驱动的多步骤任务中，早期的选择会限制后续选项，并可能迫使用户从头开始。用户偏好是这些决策的关键标准，但现有的代理并未系统地利用它们。我们提出了 MAESTRO，它将代理的角色从执行扩展到了决策支持。MAESTRO 维护一个共享的偏好记忆库，从自然语言话语中提取偏好及其强度，并提供两种机制：“基于偏好的 GUI 自适应”根据偏好强度对现有 GUI 应用原位操作符（增强、排序、过滤和高亮），支持阶段内比较；“偏好引导的工作流导航”检测偏好与可用选项之间的冲突，建议回溯，并记录失败路径以避免重复进入死胡同。我们通过一项包含 33 名参与者的受试者内研究，在电影预订带 GUI 的对话代理（CAG）中评估了 MAESTRO，研究包含两种实验条件（基准 vs. MAESTRO）和两种模式（文本 vs. 语音）。

## Abstract
Modern task-oriented chatbots present GUI elements alongside natural-language dialogue, yet the agent's role has largely been limited to interpreting natural-language input as GUI actions and following a linear workflow. In preference-driven, multi-step tasks such as booking a flight or reserving a restaurant, earlier choices constrain later options and may force users to restart from scratch. User preferences serve as the key criteria for these decisions, yet existing agents do not systematically leverage them. We present MAESTRO, which extends the agent's role from execution to decision support. MAESTRO maintains a shared preference memory that extracts preferences from natural-language utterances with their strength, and provides two mechanisms. Preference-Grounded GUI Adaptation applies in-place operators (augment, sort, filter, and highlight) to the existing GUI according to preference strength, supporting within-stage comparison. Preference-Guided Workflow Navigation detects conflicts between preferences and available options, proposes backtracking, and records failed paths to avoid revisiting dead ends. We evaluated MAESTRO in a movie-booking Conversational Agent with GUI (CAG) through a within-subjects study with two conditions (Baseline vs. MAESTRO) and two modes (Text vs. Voice), with N = 33 participants.

*   **研究动机**：在预订机票或电影票等偏好驱动的多步任务中，早期的选择会限制后续选项。如果发生冲突（如选定的场次没有座位），用户往往被迫从头开始。现有的系统缺乏对用户偏好的系统化记录，无法根据偏好动态调整界面或智能引导导航，导致决策效率低下。

### 2. 方法论
MAESTRO 的核心思想是将代理的角色从单纯的“执行者”扩展为“决策支持者”，其核心组件包括：
*   **偏好记忆库 (Preference Memory)**：从对话中提取结构化偏好记录，包含描述、强度（硬约束 vs. 软偏好）和相关阶段。
*   **基于偏好的 GUI 自适应 (Preference-Grounded GUI Adaptation)**：提供四种原位（In-place）操作符，在不改变 GUI 结构的前提下调整信息呈现：
    *   **Augment（增强）**：在按钮上添加额外元数据（如评分、距离）。
    *   **Filter（过滤）**：隐藏不符合硬约束的选项。
    *   **Sort（排序）**：根据数值属性（如评分、时长）重新排列选项。
    *   **Highlight（高亮）**：视觉强调符合软偏好的最佳选项。
*   **偏好引导的工作流导航 (Preference-Guided Workflow Navigation)**：
    *   **冲突检测与回溯建议**：当当前阶段无可用选项时，根据各阶段剩余备选项数量，建议用户回溯到最合适的历史步骤。
    *   **死胡同记录 (Dead-End Recording)**：记录导致失败的路径，防止用户在尝试新路径时重复进入已知的死胡同。
*   **代理架构**：基于大语言模型（文中提到 GPT-4o/5.4 级别模型）进行推理，通过工具调用（Tool Calling）执行 GUI 适配和导航指令。

### 6. 主要结论与发现
*   **提升决策质量**：MAESTRO 显著减少了最终提交结果中的偏好违规次数（Violation Count），并降低了过程中选择错误选项的频率。
*   **改变交互模式**：MAESTRO 鼓励用户更多地使用自然语言表达偏好和请求操作，而不仅仅是点击 GUI。
*   **语音模态的双刃剑**：语音模式虽然能促进用户表达更多偏好，但由于延迟和轮循（Turn-taking）限制，在 MAESTRO 这种反馈较多的系统中会增加用户的感知负担。
*   **用户偏好**：绝大多数用户（尤其是文本模式下）更倾向于使用 MAESTRO，认为其侧边栏 GUI 适配和回溯功能降低了认知负荷。

### 7. 优点
*   **原位适配机制**：通过四种简单的 GUI 操作符（过滤、排序等）实现了强大的决策支持，避免了重新生成整个界面的开销和用户迷失感。
*   **结构化偏好管理**：将模糊的对话转化为结构化记忆，解决了长对话中 LLM 容易遗忘偏好的问题。
*   **智能导航**：引入了“死胡同记录”和“备选计数”概念，使代理能够像人类助手一样引导用户避开无效路径。

### 8. 不足与局限
*   **领域局限性**：仅在电影预订单一领域进行了验证，其通用性（如处理更异构或更复杂的任务）有待进一步研究。
*   **功能耦合**：实验将 GUI 适配和导航引导作为一个整体包（Bundle）进行测试，无法区分各子功能对性能提升的具体贡献比例。
*   **前瞻能力缺失**：用户期望代理能具备“前瞻”能力（例如在选电影时就知道后面哪个场次有连座），但目前 MAESTRO 主要基于历史记录进行反应。
*   **语音延迟**：在语音模式下，代理的反馈过于冗长且不可中断，导致用户体验下降。
