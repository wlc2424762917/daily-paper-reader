## 摘要
大语言模型（LLM）智能体在复杂的交互式决策任务中展现了强大的能力。然而，现有的 LLM 智能体通常依赖于日益增长的交互历史，导致了高昂的计算成本和有限的可扩展性。在本文中，我们提出了 STEP-HRL，这是一种分层强化学习（HRL）框架，它通过仅以单步转移而非完整交互历史为条件，实现了步级学习。STEP-HRL 对任务进行分层构建，利用已完成的子任务来表示整个任务的全局进度。通过引入局部进度模块，它还迭代且有选择性地总结每个子任务内的交互历史，以生成局部进度的紧凑摘要。这些组件共同为高层和底层策略生成了增强的步级转移。在 ScienceWorld 和 ALFWorld 基准测试上的实验结果一致表明，STEP-HRL 在性能和泛化能力方面显著优于基准模型，同时减少了 token 的使用量。我们的代码可在 https://github.com/TonyStark042/STEP-HRL 获取。

## Abstract
Large language model (LLM) agents have demonstrated strong capabilities in complex interactive decision-making tasks. However, existing LLM agents typically rely on increasingly long interaction histories, resulting in high computational cost and limited scalability. In this paper, we propose STEP-HRL, a hierarchical reinforcement learning (HRL) framework that enables step-level learning by conditioning only on single-step transitions rather than full interaction histories. STEP-HRL structures tasks hierarchically, using completed subtasks to represent global progress of overall task. By introducing a local progress module, it also iteratively and selectively summarizes interaction history within each subtask to produce a compact summary of local progress. Together, these components yield augmented step-level transitions for both high-level and low-level policies. Experimental results on ScienceWorld and ALFWorld benchmarks consistently demonstrate that STEP-HRL substantially outperforms baselines in terms of performance and generalization while reducing token usage. Our code is available at https://github.com/TonyStark042/STEP-HRL.

*   **研究动机**：现有的 LLM 智能体在处理复杂决策任务时，通常依赖于“历史条件化（History-conditioned）”的策略，即需要输入越来越长的交互历史。这导致了两个核心问题：一是计算成本随上下文长度呈平方级增长；二是冗余信息会掩盖关键决策信号，导致长程信用分配（Credit Assignment）困难。
*   **核心目标**：提出一种名为 **STEP-HRL** 的框架，旨在打破对全量历史的依赖，通过分层结构和进度建模，实现仅基于“单步转移（Step-level transitions）”的高效决策。

#### 2. 方法论
STEP-HRL 的核心思想是将任务分解为分层结构，并引入“局部进度”模块来压缩信息。
*   **核心组件**：
    *   **高层策略 ($\pi_h$)**：负责生成子任务（Subtasks）。它以全局进度（已完成的子任务序列）和前一子任务的最终局部进度为输入。
    *   **底层策略 ($\pi_l$)**：负责执行具体动作。它仅以当前子任务、当前观察和局部进度摘要为输入。
    *   **局部进度模块 ($\pi_p$)**：这是关键创新。它是一个迭代更新的策略，负责将子任务内的交互历史有选择地总结为紧凑的文本表示，从而实现恒定大小的输入。
*   **训练流程**：
    1.  **行为克隆 (BC) 初始化**：利用专家演示数据（包含子任务和进度标注）对三个策略进行联合初始化。
    2.  **步级离线强化学习 (Offline RL)**：采用类似 IQL (Implicit Q-Learning) 的架构。通过值函数网络（Critic）评估状态价值，并使用优势加权回归（Advantage-Weighted Regression）优化策略。所有策略共享同一个 LLM 主干网络，但拥有独立的 Critic 网络。

*   **性能卓越**：STEP-HRL 在 ScienceWorld 和 ALFWorld 上均显著超过了所有基准模型。例如，在 ScienceWorld 的未见任务上，相比 GLIDER 提升了约 13%-26%。

#### 7. 优点
*   **架构创新**：通过“局部进度”模块巧妙地解决了 LLM 智能体对长上下文依赖的顽疾。
*   **推理友好**：恒定的输入长度使得模型在处理极长任务时不会出现显存溢出或推理变慢的问题。
*   **参数高效**：高、低层策略共享主干，既促进了知识迁移，又减少了训练开销。

#### 8. 不足与局限
*   **专家数据依赖**：该方法高度依赖高质量的专家演示，特别是需要人工或强模型（如 DeepSeek）预先生成子任务和进度标注，这在某些缺乏标注的环境中难以实施。
*   **子任务终止风险**：子任务的终止预测与动作生成耦合，如果终止判断失误（过早或过晚），会直接影响后续高层规划的准确性。
*   **环境反馈要求**：虽然实现了步级学习，但仍需要环境提供相对明确的状态反馈来更新局部进度。
