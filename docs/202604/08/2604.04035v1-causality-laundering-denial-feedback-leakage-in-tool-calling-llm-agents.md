## 摘要
调用工具的 LLM 智能体可以读取私有数据、调用外部服务并触发真实世界的动作，从而在工具执行点产生安全问题。我们识别出一种拒绝反馈泄露模式，并将其称为“因果洗钱”（causality laundering）。在这种模式中，攻击者探测受保护的操作，从拒绝结果中学习，并通过随后看似无害的工具调用将推断出的信息外泄。这种攻击无法仅通过扁平的溯源追踪（flat provenance tracking）来捕获，因为泄露的信息源于被拒绝操作的因果影响，而非直接的数据流。我们提出了智能体引用监视器（Agentic Reference Monitor, ARM），这是一个运行时强制执行层，通过咨询涵盖工具调用、返回数据、字段级溯源和被拒绝操作的溯源图来调解每一次工具调用。ARM 通过完整性格（integrity lattice）传播信任，并使用来自被拒绝操作节点的反事实边（counterfactual edges）来增强溯源图，从而实现对传递性数据依赖和拒绝诱导的因果影响的强制执行。在针对三个代表性攻击场景的受控评估中，ARM 拦截了扁平溯源基准方案所遗漏的因果洗钱、传递性污点传播以及混合溯源字段误用，同时仅增加了亚毫秒级的策略评估开销。这些结果表明，拒绝感知的因果溯源是保障调用工具的智能体系统安全的一种有用抽象。

## Abstract
Tool-calling LLM agents can read private data, invoke external services, and trigger real-world actions, creating a security problem at the point of tool execution. We identify a denial-feedback leakage pattern, which we term causality laundering, in which an adversary probes a protected action, learns from the denial outcome, and exfiltrates the inferred information through a later seemingly benign tool call. This attack is not captured by flat provenance tracking alone because the leaked information arises from causal influence of the denied action, not direct data flow. We present the Agentic Reference Monitor (ARM), a runtime enforcement layer that mediates every tool invocation by consulting a provenance graph over tool calls, returned data, field-level provenance, and denied actions. ARM propagates trust through an integrity lattice and augments the graph with counterfactual edges from denied-action nodes, enabling enforcement over both transitive data dependencies and denial-induced causal influence. In a controlled evaluation on three representative attack scenarios, ARM blocks causality laundering, transitive taint propagation, and mixed-provenance field misuse that a flat provenance baseline misses, while adding sub-millisecond policy evaluation overhead. These results suggest that denial-aware causal provenance is a useful abstraction for securing tool-calling agent systems.

### 1. 论文的核心问题与整体含义
*   **研究动机**：随着 LLM 智能体（Agents）被授权调用工具（如读取文件、发送邮件），安全边界从文本生成转向了工具执行。
*   **核心问题**：作者发现了一种新型攻击模式——**“因果洗钱”（Causality Laundering）**。传统的安全防御（如污点追踪或数据流分析）通常假设：如果一个工具调用被拒绝（Deny），攻击就结束了。然而，智能体可以从“被拒绝”这一反馈中推断出敏感信息（例如：文件存在但无权访问，而非文件不存在），并随后通过看似无害的工具调用将此推论外泄。
*   **整体含义**：现有的防御机制忽略了“拒绝事件”作为一种隐式信息流的因果影响。本论文旨在填补这一空白，通过追踪拒绝操作的因果关系来保护智能体系统。

### 2. 方法论：智能体引用监视器（ARM）
*   **核心思想**：将“被拒绝的操作”视为溯源图（Provenance Graph）中的一等公民，通过引入“反事实边”来捕捉拒绝反馈对后续行为的潜在因果影响。
*   **关键技术细节**：
    *   **分层策略管道**：ARM 包含四层过滤：
        1.  **硬边界（L1）**：无条件检查（如敏感路径屏蔽、凭据检测、负载大小限制）。
        2.  **图感知溯源（L2G）**：核心层，通过查询溯源图判断当前调用是否受不可信数据或拒绝事件的影响。
        3.  **模式衍生约束（L3）**：根据工具的 JSON Schema 自动生成防御规则（如限制可选参数长度）。
        4.  **手动策略（L4）**：操作员定义的权限令牌（Capability Tokens）。
    *   **溯源图结构**：节点包括工具调用、返回数据、数据字段和**拒绝动作**。边包括直接输出、输入到、字段所属以及关键的**反事实边（Counterfactual Edge）**。
    *   **完整性晶格（Integrity Lattice）**：定义了五个信任等级（系统指令 > 用户输入 > 可信工具 > 不可信工具 > 工具描述），信任度在图中按最小值原则传播。
*   **算法流程**：当工具调用被拒绝时，ARM 记录一个“拒绝动作节点”。随后的下一个工具调用会自动建立一条指向该节点的反事实边。在评估后续调用时，若发现其溯源路径中包含反事实边或信任度低于阈值，则予以拦截。

### 3. 实验设计
*   **实验场景**：设计了三个具有代表性的攻击场景：
    1.  **因果洗钱（拒绝推论外泄）**：智能体试图读取敏感文件被拒，随后发送邮件告知攻击者该文件存在。
    2.  **传递性污点链**：攻击者通过多步工具调用（如格式化工具）来“洗白”不可信数据的来源。
    3.  **混合溯源字段误用**：利用结构化输出中某些受污染的特定字段（如联系人列表中的恶意邮箱地址）。
*   **基准测试（Benchmark）**：使用“扁平溯源基准（Flat Baseline）”，即仅检查参数值是否出现在历史记录中，而不追踪复杂的因果或字段级关系。
*   **对比方法**：主要对比了 L2（扁平溯源）与 L2G（图感知溯源）在相同 L1/L3/L4 配置下的表现。

### 4. 资源与算力
*   **硬件环境**：实验在配备 **Apple M4 Pro 芯片、48 GB RAM** 的设备上运行。
*   **性能表现**：策略评估的延迟在**亚毫秒级（< 1ms）**。
*   **训练时长**：由于 ARM 是一个确定性的运行时监控层，不涉及模型训练，因此没有训练时长或 GPU 算力消耗的相关数据。

### 5. 实验数量与充分性
*   **实验规模**：
    *   针对 3 个核心攻击场景进行了受控差异化实验。
    *   包含 45 组单元测试，涵盖溯源图构建、信任传播和端到端场景。
    *   性能测试进行了 100 次运行取中位数。
*   **充分性评价**：实验在验证“功能性”和“防御有效性”方面是充分的，证明了图模型优于扁平模型。但由于是原型验证，缺乏大规模、自动化的基准测试（如 AgentDojo）和真实世界复杂工作流的广泛覆盖。

### 6. 主要结论与发现
*   **拒绝感知是必要的**：忽略拒绝反馈会导致严重的隐私泄露，ARM 证明了将拒绝事件纳入溯源模型的可行性。
*   **图模型更精准**：相比扁平的引用检查，基于图的溯源能有效识别传递性污染和字段级风险。
*   **开销极低**：确定性的图遍历算法（基于 rustworkx）引入的延迟远小于 LLM 本身的推理延迟，适合实时部署。

### 7. 优点
*   **创新性**：首次形式化定义了“因果洗钱”攻击，并提出了处理拒绝反馈的“反事实边”概念。
*   **确定性防御**：不依赖 LLM 来检查安全性（避免了“用受攻击的模型保护自己”的悖论），而是采用确定性的图算法。
*   **细粒度控制**：支持字段级（Field-level）的溯源追踪，能够处理复杂的 JSON 结构化数据。

### 8. 不足与局限
*   **启发式假设**：反事实边目前基于简单的“时间相邻”启发式方法，可能导致误报（拦截了紧随拒绝事件后的正常操作）或漏报（攻击者故意延迟外泄步骤）。
*   **依赖人工标注**：字段级的保护需要集成者手动指定某些字段的信任覆盖，否则会退化为对象级追踪。
*   **评估范围有限**：尚未在多智能体协作场景或超长历史记录的复杂任务中进行压力测试。
*   **非完全不干扰**：虽然缓解了因果泄露，但不能完全消除所有隐式流（如定时攻击）。
