## 摘要
基于大语言模型（LLM）的演化是开放式发现的一种极具前景的方法，这类发现的进展需要持续的搜索和知识积累。现有方法仍严重依赖固定的启发式方法和硬编码的探索规则，这限制了 LLM 智能体的自主性。我们提出了 CORAL，这是首个针对开放式问题的自主多智能体演化框架。CORAL 用长期运行的智能体取代了僵化的控制，这些智能体通过共享持久化记忆、异步多智能体执行和基于心跳的干预进行探索、反思和协作。它还提供了实用的安全保障措施，包括隔离的工作空间、评估器分离、资源管理以及智能体会话和健康管理。在多样化的数学、算法和系统优化任务上的评估结果显示，CORAL 在 10 个任务上刷新了最先进（SOTA）结果，在评估次数远少于固定演化搜索基线的情况下，实现了 3 到 10 倍的改进率。在 Anthropic 的内核工程任务中，四个协同演化的智能体将已知最佳分数从 1363 个周期提升至 1103 个周期。机制分析进一步表明，这些提升源于知识重用以及多智能体的探索与通信。总之，这些结果表明，更强的智能体自主性和多智能体演化可以显著提升开放式发现的能力。代码可在 https://github.com/Human-Agent-Society/CORAL 获取。

## Abstract
Large language model (LLM)-based evolution is a promising approach for open-ended discovery, where progress requires sustained search and knowledge accumulation. Existing methods still rely heavily on fixed heuristics and hard-coded exploration rules, which limit the autonomy of LLM agents. We present CORAL, the first framework for autonomous multi-agent evolution on open-ended problems. CORAL replaces rigid control with long-running agents that explore, reflect, and collaborate through shared persistent memory, asynchronous multi-agent execution, and heartbeat-based interventions. It also provides practical safeguards, including isolated workspaces, evaluator separation, resource management, and agent session and health management. Evaluated on diverse mathematical, algorithmic, and systems optimization tasks, CORAL sets new state-of-the-art results on 10 tasks, achieving 3-10 times higher improvement rates with far fewer evaluations than fixed evolutionary search baselines across tasks. On Anthropic's kernel engineering task, four co-evolving agents improve the best known score from 1363 to 1103 cycles. Mechanistic analyses further show how these gains arise from knowledge reuse and multi-agent exploration and communication. Together, these results suggest that greater agent autonomy and multi-agent evolution can substantially improve open-ended discovery. Code is available at https://github.com/Human-Agent-Society/CORAL.

在科学发现、算法设计和系统优化等“开放式发现”领域，目标明确但最优解未知。现有的基于大语言模型（LLM）的演化方法（如 FunSearch, AlphaEvolve）通常将 LLM 嵌入到固定的进化循环中，由硬编码的启发式规则决定如何选择父代、何时变异以及保留哪些知识。这种“固定搜索”范式限制了智能体的自主性，使其无法根据反馈灵活调整策略。
**论文的核心问题是：** 能否通过赋予智能体更高的自主权，并利用多智能体异步协作（水平扩展），在无需人工预设复杂规则的情况下，实现更高效的开放式知识发现和性能优化？

#### 2. 方法论：核心思想与关键技术
CORAL 框架将进化过程中的决策权从固定算法转移到自主智能体手中，其核心思想是**“自主多智能体演化”**。
*   **共享持久化存储（Shared Persistent Memory）：** 采用类文件系统的结构，将知识分为三类：
    *   `attempts/`：记录历史尝试、代码及评估分数。
    *   `notes/`：记录智能体的观察、反思和学习心得。
    *   `skills/`：提炼可复用的代码片段、工具或操作流程。
*   **异步多智能体组织：** 多个智能体在各自隔离的工作区（Git Worktree）运行，不直接通信，而是通过读写共享存储进行间接协作（如借鉴他人的成功代码或笔记）。
*   **心跳机制（Heartbeat）：** 类似于“提醒应用”，定期干预智能体行为：
    *   **反思心跳：** 强制记录当前实验的笔记。
    *   **整合心跳：** 定期汇总零散笔记，提炼为系统性知识或技能。
    *   **重定向心跳：** 当性能陷入停滞（Plateau）时，提示智能体放弃当前路径，尝试全新方向。
*   **安全与执行保障：** 实现了评估器隔离（防止智能体作弊）、资源配额管理以及会话状态持久化（支持中断后恢复）。

#### 7. 优点
*   **范式创新：** 首次将进化算法的控制权完全交给自主智能体，摆脱了硬编码规则的束缚。
*   **知识管理系统：** 引入了结构化的笔记和技能提炼机制，使 LLM 能够像人类研究员一样积累经验。
*   **异步扩展性：** 框架设计简洁，通过文件系统实现多智能体解耦，易于扩展到更多智能体或更复杂的任务。

#### 8. 不足与局限
*   **模型依赖：** 实验证明该框架高度依赖于像 Claude Opus 4.6 这样具备极强推理和代码能力的顶级模型，在弱模型上的表现尚待验证。
*   **初始化同质性：** 目前所有智能体初始化相同，缺乏角色分工（如有的负责探索，有的负责精化），可能限制了多样性。
