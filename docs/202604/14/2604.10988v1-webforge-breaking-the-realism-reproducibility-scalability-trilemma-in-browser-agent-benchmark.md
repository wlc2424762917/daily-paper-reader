## 摘要
现有的浏览器智能体基准测试面临一个根本性的三难困境：基于真实网站的基准测试由于内容漂移（content drift）而缺乏可复现性，受控环境因忽略真实网络噪声而牺牲了真实性，且两者都需要昂贵的人工策展，限制了可扩展性。我们提出了 WebForge，这是首个通过“规划、生成、细化和验证”四智能体流水线解决该困境的全自动框架，能够端到端地生成交互式、自包含的网页环境，无需人工标注。一个七维难度控制框架从导航深度、视觉复杂度、推理难度等方面构建任务设计，实现了超越单一综合评分的系统化能力画像。利用 WebForge，我们构建了 WebForge-Bench，这是一个包含跨越 7 个领域和 3 个难度级别的 934 个任务的基准测试。多模型实验表明，难度分层有效地区分了模型能力，而跨领域分析揭示了综合指标无法察觉的能力偏差。总之，这些结果证实了多维度评估能够揭示单一综合评分无法捕捉的独特能力特征。代码和基准测试已在 https://github.com/yuandaxia2001/WebForge 公开。

## Abstract
Existing browser agent benchmarks face a fundamental trilemma: real-website benchmarks lack reproducibility due to content drift, controlled environments sacrifice realism by omitting real-web noise, and both require costly manual curation that limits scalability. We present WebForge, the first fully automated framework that resolves this trilemma through a four-agent pipeline -- Plan, Generate, Refine, and Validate -- that produces interactive, self-contained web environments end-to-end without human annotation. A seven-dimensional difficulty control framework structures task design along navigation depth, visual complexity, reasoning difficulty, and more, enabling systematic capability profiling beyond single aggregate scores. Using WebForge, we construct WebForge-Bench, a benchmark of 934 tasks spanning 7 domains and 3 difficulty levels. Multi-model experiments show that difficulty stratification effectively differentiates model capabilities, while cross-domain analysis exposes capability biases invisible to aggregate metrics. Together, these results confirm that multi-dimensional evaluation reveals distinct capability profiles that a single aggregate score cannot capture. Code and benchmark are publicly available at https://github.com/yuandaxia2001/WebForge.

#### 1. 核心问题与整体含义（研究动机和背景）
浏览器智能体（Browser Agent）的发展受限于现有的评测基准。论文指出，现有基准测试面临**“三难困境”（Trilemma）**：
*   **真实性（Realism）**：真实网站环境复杂，但存在“内容漂移”（Content Drift），导致任务随时间失效。
*   **可复现性（Reproducibility）**：受控沙盒环境虽可复现，但往往过于“干净”，缺乏真实网页的噪声（如弹窗、Cookie 声明、网络延迟）。
*   **可扩展性（Scalability）**：无论是维护真实网站还是构建受控环境，都需要昂贵的人工策展和标注。
WebForge 的核心目标是创建一个**全自动化**的框架，生成既真实、可复现又具备大规模扩展能力的浏览器智能体基准测试。

#### 2. 方法论：核心思想与技术细节
WebForge 提出了一个由四个专门智能体组成的自动化流水线（Plan → Generate → Refine → Validate）：
*   **规划智能体（Plan Agent）**：采用双阶段生成。首先由创意模型在高温度设置下草拟任务，再由精密模型在低温度下进行逻辑验证和难度校准。它根据**七维难度向量**（导航深度、跳转广度、页面交互、视觉复杂度、信息复杂度、推理/计算、风险因素）设计任务蓝图。
*   **生成智能体（Generation Agent）**：根据蓝图构建功能完整的静态网站。它会参考真实网站的设计模式，抓取真实数据（图像、产品信息），并利用 `localStorage` 实现状态管理（如购物车）。
*   **精炼智能体（Refinement Agent）**：负责质量保证和“注入噪声”。它会检查死链、修复交互漏洞，并故意加入真实世界的干扰项（如广告弹窗、Cookie 确认框、模拟网络延迟）。
*   **验证智能体（Validation Agent）**：作为最后的质量关卡。它在真实的 Chromium 浏览器中运行，通过“观察-推理-动作”循环尝试解决生成的任务。只有被机器验证为可解的任务才会进入最终基准。
*   **反作弊机制**：为了防止模型通过读取源码走捷径，WebForge 对关键答案数据进行 Base64 加密，并设计了“误导性代码”，即针对常见的错误操作返回看似合理但错误的确认码。

*   **首个全自动化框架**：实现了从任务设计到环境部署再到解法验证的零人工参与。

#### 8. 不足与局限
*   **后端模拟限制**：由于采用静态网页技术，无法模拟需要真实服务器交互的场景（如实时股价变动、多用户实时协作、复杂的数据库回滚语义）。
