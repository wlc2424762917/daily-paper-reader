## 摘要
GUI 智能体通过视觉界面而非编程 API 驱动应用程序，通过点击、滑动和按键与任意软件交互，触及了基于命令行界面（CLI）的智能体无法覆盖的长尾应用。然而，该领域的进展瓶颈与其说是建模能力，不如说是缺乏连贯的全栈基础设施：在线强化学习（RL）训练受困于环境不稳定和封闭的流水线，评估协议在不同研究之间悄然发生偏移，且训练好的智能体很少能触及真实设备上的真实用户。我们提出了 ClawGUI，这是一个在单一体系内解决上述三个差距的开源框架。ClawGUI-RL 提供了首个开源的 GUI 智能体强化学习基础设施，并经过验证支持并行虚拟环境和真实物理设备，将 GiGPO 与过程奖励模型（PRM）集成，以实现密集步级监督。ClawGUI-Eval 在 6 个基准测试和 11 个以上模型中强制执行完全标准化的评估流水线，在官方基准测试中实现了 95.8% 的复现率。ClawGUI-Agent 通过 12 个以上的聊天平台将训练好的智能体引入 Android、HarmonyOS 和 iOS，具备混合 CLI-GUI 控制和持久化个性化记忆功能。在该流水线中进行端到端训练后，ClawGUI-2B 在 MobileWorld GUI-Only 任务上实现了 17.1% 的成功率，比同规模的 MAI-UI-2B 基准模型高出 6.0%。

## Abstract
GUI agents drive applications through their visual interfaces instead of programmatic APIs, interacting with arbitrary software via taps, swipes, and keystrokes, reaching a long tail of applications that CLI-based agents cannot. Yet progress in this area is bottlenecked less by modeling capacity than by the absence of a coherent full-stack infrastructure: online RL training suffers from environment instability and closed pipelines, evaluation protocols drift silently across works, and trained agents rarely reach real users on real devices. We present \textbf{ClawGUI}, an open-source framework addressing these three gaps within a single harness. \textbf{ClawGUI-RL} provides the first open-source GUI agent RL infrastructure with validated support for both parallel virtual environments and real physical devices, integrating GiGPO with a Process Reward Model for dense step-level supervision. \textbf{ClawGUI-Eval} enforces a fully standardized evaluation pipeline across 6 benchmarks and 11+ models, achieving 95.8\% reproduction against official baselines. \textbf{ClawGUI-Agent} brings trained agents to Android, HarmonyOS, and iOS through 12+ chat platforms with hybrid CLI-GUI control and persistent personalized memory. Trained end to end within this pipeline, \textbf{ClawGUI-2B} achieves 17.1\% Success Rate on MobileWorld GUI-Only, outperforming the same-scale MAI-UI-2B baseline by 6.0\%.

### 1. 核心问题与整体含义（研究动机和背景）
GUI 智能体（GUI Agents）能够像人类一样通过点击、滑动和输入来操作任意软件，具有极高的通用性。然而，该领域目前面临三大瓶颈：
*   **训练基础设施封闭且不稳定：** 在线强化学习（RL）训练通常局限于虚拟沙盒，环境易崩溃，且缺乏支持物理设备训练的开源框架。
*   **评估标准混乱：** 不同研究在提示词格式、分辨率、坐标归一化等细节上不一致，导致实验结果难以公平比较和复现。
*   **部署脱节：** 实验室训练的模型很少能真正部署到真实用户的移动设备（如 Android、iOS）上并具备持久记忆。

**ClawGUI** 的提出旨在通过一个统一的开源全栈框架，打通 GUI 智能体从训练、评估到真实部署的全流程。

### 2. 方法论：核心思想与关键技术
ClawGUI 由三个紧密集成的模块组成：
*   **ClawGUI-RL（可扩展在线 RL 训练）：**
    *   **环境管理：** 支持 Docker 化的 Android 模拟器并行运行，具备健康检查、崩溃恢复和备用服务器轮换机制；同时支持物理设备的接入。
    *   **密集奖励设计：** 结合了二元结果奖励（成功/失败）和**过程奖励模型（PRM）**。PRM 对每一步操作进行评分，解决 GUI 任务中奖励稀疏的问题。
    *   **GiGPO 算法：** 采用“组内组策略优化”（Group-in-Group Policy Optimization），通过锚点状态分组实现细粒度的步级信用分配，比传统的 GRPO 更适合长序列 GUI 交互。
*   **ClawGUI-Eval（可复现评估）：**
    *   采用“推理（Infer）→ 评判（Judge）→ 指标（Metric）”的三阶段标准化流水线。
    *   固定了每个模型的配置参数，确保评估的严谨性。
*   **ClawGUI-Agent（个人 GUI 助手）：**
    *   **混合控制：** 结合 CLI（命令行）的高效和 GUI（图形界面）的通用性。
    *   **个性化记忆：** 使用向量数据库存储用户习惯和事实，实现长期的个性化适配。
    *   **多平台支持：** 通过 12 种以上的聊天平台（如 Telegram、飞书）远程或本地控制 Android、iOS 和鸿蒙设备。

### 3. 实验设计
*   **数据集/基准测试：** 涵盖了 6 个主流基准，包括 ScreenSpot-Pro、ScreenSpot-V2、UI-Vision、MMBench-GUI、OSWorld-G 和 AndroidControl。
*   **核心 Benchmark：** 使用 **MobileWorld**（GUI-Only 任务，共 117 个任务）评估端到端任务成功率。
*   **对比方法：**
    *   **同规模基线：** MAI-UI-2B。
    *   **大规模模型：** Qwen3-VL (8B/32B/235B)、UI-Venus (72B)、UI-TARS 等。
    *   **Agent 框架：** Claude-4.5-Sonnet、GPT-5 等闭源模型驱动的框架。

### 4. 资源与算力
*   **硬件设备：** 训练 ClawGUI-2B 使用了 **8 张 NVIDIA A6000 (48GB) GPU**。
*   **并行规模：** 启动了 **64 个并行虚拟环境**（Docker 容器）。
*   **训练细节：** 学习率为 1e-6，训练 3 个 Epoch，Rollout Group Size 为 8。
*   **判别模型：** 使用 Qwen3.5-72B 作为 PRM 的判别器模型。

### 5. 实验数量与充分性
*   **实验规模：** 论文对 11 个以上的模型在 6 个基准测试上进行了复现实验，并发布了所有推理预测结果。
*   **消融实验：** 专门对比了 GRPO（稀疏奖励）与 GiGPO（密集奖励）的效果。
*   **充分性与客观性：** 实验非常充分。作者不仅展示了自家模型的优越性，还通过复现其他模型达成了 95.8% 的复现率，证明了框架的客观性和标准化能力。

### 6. 主要结论与发现
*   **训练框架的威力：** ClawGUI-2B 在 MobileWorld 上达到 17.1% 的成功率，显著超过同规模基线 MAI-UI-2B（11.1%），甚至超过了参数量大得多的 Qwen3-VL-32B（11.9%）。
*   **密集奖励至关重要：** 使用 GiGPO（步级奖励）比 GRPO（回合级奖励）带来了 17.9% 的相对性能提升，证明在 GUI 领域“每一步都算数”。
*   **评估标准化是必要的：** 发现之前研究中 2% 左右的性能波动往往源于提示词或分辨率等基础设施配置的差异，而非模型核心能力的改变。

### 7. 优点（亮点）
*   **全栈统一：** 首次将 RL 训练、标准化评估和多端真实部署整合进一个开源框架。
*   **支持物理设备：** 填补了开源界在真实物理设备上进行 RL 训练的空白。
*   **高复现性：** 提供了极高复现率的评估工具，有助于净化 GUI 领域的研究环境。
*   **实用性强：** ClawGUI-Agent 已经支持多种真实操作系统和社交软件接口，具备落地潜力。

### 8. 不足与局限
*   **物理设备任务获取：** 物理设备上的任务仍需人工编写和 MLLM 评判，无法像虚拟环境那样实现完全自动化的海量任务生成。
*   **闭源模型依赖：** 评估闭源模型（如 Gemini）时需采用特殊的“Zoom”范式，这可能无法完全体现模型在原生高分辨率下的表现。
*   **算力门槛：** 虽然模型只有 2B，但进行大规模并行 RL 训练仍需要较多显存（8x A6000），对个人开发者仍有一定门槛。
