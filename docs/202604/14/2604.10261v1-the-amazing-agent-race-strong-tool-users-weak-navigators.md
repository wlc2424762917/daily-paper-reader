## 摘要
现有的 LLM 智能体工具使用基准测试绝大多数是线性的：我们对六个基准测试的分析显示，55% 到 100% 的实例是 2 到 5 步的简单链条。我们推出了 The Amazing Agent Race (AAR)，这是一个以有向无环图 (DAG) 谜题（或称“赛段”）为特色的基准测试，包含分叉-合并工具链。我们发布了两种变体的 1,400 个实例：顺序型（800 个赛段）和组合型（600 个 DAG 赛段）。智能体必须在维基百科中导航，执行多步工具链，并将结果汇总为可验证的答案。赛段是根据维基百科种子程序化生成的，涵盖四个难度级别，并带有实时 API 验证。三个互补的指标（终点准确率、维修站访问率和路障完成率）分别诊断导航、工具使用和算术故障。在 1,400 个赛段上评估了三个智能体框架，表现最好的仅达到 37.2% 的准确率。导航错误占主导地位（占试验的 27% 到 52%），而工具使用错误保持在 17% 以下，且智能体架构与模型规模同样重要（Claude Code 以 6 倍少的 token 消耗达到了与 Codex CLI 相当的 37% 准确率）。AAR 的组合结构揭示了智能体的失败不在于调用工具，而在于导航到正确的页面，这是线性基准测试无法察觉的盲点。项目页面可通过以下网址访问：https://minnesotanlp.github.io/the-amazing-agent-race

## Abstract
Existing tool-use benchmarks for LLM agents are overwhelmingly linear: our analysis of six benchmarks shows 55 to 100% of instances are simple chains of 2 to 5 steps. We introduce The Amazing Agent Race (AAR), a benchmark featuring directed acyclic graph (DAG) puzzles (or "legs") with fork-merge tool chains. We release 1,400 instances across two variants: sequential (800 legs) and compositional (600 DAG legs). Agents must navigate Wikipedia, execute multi-step tool chains, and aggregate results into a verifiable answer. Legs are procedurally generated from Wikipedia seeds across four difficulty levels with live-API validation. Three complementary metrics (finish-line accuracy, pit-stop visit rate, and roadblock completion rate) separately diagnose navigation, tool-use, and arithmetic failures. Evaluating three agent frameworks on 1,400 legs, the best achieves only 37.2% accuracy. Navigation errors dominate (27 to 52% of trials) while tool-use errors remain below 17%, and agent architecture matters as much as model scale (Claude Code matches Codex CLI at 37% with 6x fewer tokens). The compositional structure of AAR reveals that agents fail not at calling tools but at navigating to the right pages, a blind spot invisible to linear benchmarks. The project page can be accessed at: https://minnesotanlp.github.io/the-amazing-agent-race

### 论文总结：The Amazing Agent Race (AAR)

#### 1. 论文的核心问题与整体含义
当前大型语言模型（LLM）智能体在工具使用和任务规划方面取得了显著进展，但现有的评估基准（如 ToolBench, GAIA 等）存在严重的**“线性化”缺陷**。作者分析发现，现有基准中 55% 到 100% 的任务仅为 2 到 5 步的简单线性链条，缺乏复杂的逻辑分支和信息聚合。

本研究提出了 **The Amazing Agent Race (AAR)** 基准，旨在测试智能体在非线性、有向无环图（DAG）结构任务中的表现。核心问题是：当智能体必须通过导航获取信息、将信息分叉到并行工具分支并最终合并结果时，它们会在哪个环节崩溃？

#### 2. 论文提出的方法论
*   **核心思想**：受真人秀《极速前进》启发，将评估建模为在维基百科上的“赛段”（Legs）。每个赛段是一个 DAG 结构，包含多个“维修站”（Pit Stops）。
*   **关键技术细节**：
    *   **任务结构**：包含四种类型的维修站：**路线信息**（从维基百科提取事实）、**路障**（执行多步工具链）、**绕道**（分析推理/转换）、**终点线**（计算最终代码）。
    *   **钻石模式（Diamond Patterns）**：通过“分叉-合并”结构创建非线性依赖，例如从一个实体出发，并行调用海拔 API 和人口 API，最后合并计算。
    *   **自动化流水线**：包含 8 个步骤（抓取、规划、构建、预验证、链接、增强、执行、口语化），从维基百科种子页面自动生成任务。
    *   **诊断指标**：
        1.  **FA (Finish-line Accuracy)**：终点准确率（最终答案是否正确）。
        2.  **PVR (Pit-stop Visit Rate)**：维修站访问率（衡量导航能力）。
        3.  **RCR (Roadblock Completion Rate)**：路障完成率（衡量工具调用能力）。

#### 3. 实验设计
*   **数据集**：发布了 1,400 个赛段，包括 **AAR-Linear**（800 个顺序任务）和 **AAR-DAG**（600 个复杂分支任务）。
*   **工具集**：提供 19 种工具，涵盖地图、天气、金融、代码执行等 8 个类别。
*   **对比方法（智能体框架）**：
    *   **Codex CLI**：OpenAI 的编程助手框架。
    *   **Claude Code**：Anthropic 的自主规划与执行框架。
    *   **mini-swe-agent**：轻量级 ReAct 风格框架。
*   **底层模型**：GPT-5.4, GPT-5.4-mini, Claude Sonnet 4, 以及开源推理模型 GPT-OSS-120B。

#### 4. 资源与算力
*   **总计算耗时**：全量评估（7,000 次试验，涵盖 10 种配置）共消耗了 **286 个计算小时**（Compute-hours）。
*   **环境**：所有评估均在 Harbor 框架下的 Docker 容器中运行，每个赛段限时 600 秒。
*   **Token 消耗**：Codex CLI 平均每场试验消耗 1.4M-1.8M tokens；Claude Code 效率更高，消耗仅为前者的 1/6（约 114K-225K tokens）。

#### 5. 实验数量与充分性
*   **实验规模**：进行了 7,000 次独立试验，样本量充足。
*   **维度覆盖**：涵盖了 4 个难度级别（Easy 到 Extreme），并对比了线性与 DAG 结构的差异。
*   **客观性**：通过实时 API 验证和缓存的黄金执行轨迹（Golden Traces）确保结果可复现且不受 API 波动影响。引入随机基准（10% 准确率）作为参照，实验设计严谨、公平。

#### 6. 论文的主要结论与发现
*   **导航是核心瓶颈**：最强智能体的综合准确率仅为 **37.2%**。导航错误（PVR 低）占主导地位（27%-52%），而工具使用错误（RCR）相对较少（<17%）。
*   **组合结构挑战导航而非工具**：从线性转为 DAG 结构时，导航得分下降了 13-18 个百分点，但工具调用成功率保持稳定。
*   **架构与规模同等重要**：Claude Code 凭借优秀的架构，在 token 消耗极低的情况下达到了与 frontier 模型相当的性能。
*   **推理模型的局限**：具有“长思考”能力的推理模型（如 GPT-OSS-120B）在需要大量浅层工具调用的时间受限任务中表现极差（准确率仅 3.1%），因为它们将过多预算花在了内部思考而非执行上。

#### 7. 优点
*   **结构创新**：打破了现有基准的线性束缚，引入 DAG 结构更真实地模拟了复杂任务。
*   **诊断深入**：通过解耦的三个指标（FA, PVR, RCR），精准定位了智能体是“找不到路”还是“不会用工具”。
*   **抗污染设计**：利用实时 API、谜题式描述（不直接提及标题）和程序化生成，有效防止模型通过记忆训练数据来“刷分”。

#### 8. 不足与局限
*   **领域局限性**：目前主要依赖维基百科作为导航源，尚未扩展到更广泛的 Web 环境或私有数据库。
*   **捷径风险**：在极端难度下，部分智能体可能通过线索中的信息泄露直接推断出工具参数，从而绕过导航步骤（虽然 PVR 指标能捕捉到这种行为）。
*   **时间敏感性**：固定 600 秒的超时限制可能对某些需要深度推理的模型不公平，尽管分析显示大多数错误并非由超时引起。
