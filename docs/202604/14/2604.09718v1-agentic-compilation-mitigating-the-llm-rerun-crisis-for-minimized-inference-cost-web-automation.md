## 摘要
通过连续推理循环运行的 LLM 驱动网页智能体——即通过重复查询模型来评估浏览器状态并选择动作——在处理重复性任务时面临根本性的可扩展性约束。我们将此定义为“重复运行危机”（Rerun Crisis）：Token 支出和 API 延迟随执行频率线性增长。对于一个包含 5 个步骤且迭代 500 次的工作流，连续型智能体会产生约 150.00 美元的推理成本；即使采用激进的缓存策略，成本仍接近 15.00 美元。我们提出了一种“编译并执行”（Compile-and-Execute）架构，将 LLM 推理与浏览器执行解耦，从而将每个工作流的推理成本降低至 0.10 美元以下。单次 LLM 调用处理来自 DOM 清洗模块（DSM）的高 Token 效率语义表示，并生成确定性的 JSON 工作流蓝图。随后，轻量级运行时在无需进一步模型查询的情况下驱动浏览器。我们将这种成本降低形式化为从 O(M x N) 到摊销 O(1) 的推理扩展，其中 M 为重复运行次数，N 为顺序动作数量。在数据提取、表单填写和指纹识别任务中的实证评估表明，零样本编译成功率达到 80-94%。至关重要的是，JSON 中间表示的模块化允许通过极少的人机回环（HITL）修补，将执行可靠性提升至接近 100%。在五种前沿模型上，单次编译成本仅为 0.002 美元至 0.092 美元，这些结果证明确定性编译作为一种新范式，能够实现在以往连续架构下经济上不可行的大规模自动化。

## Abstract
LLM-driven web agents operating through continuous inference loops -- repeatedly querying a model to evaluate browser state and select actions -- exhibit a fundamental scalability constraint for repetitive tasks. We characterize this as the Rerun Crisis: the linear growth of token expenditure and API latency relative to execution frequency. For a 5-step workflow over 500 iterations, a continuous agent incurs approximately 150.00 USD in inference costs; even with aggressive caching, this remains near 15.00 USD. We propose a Compile-and-Execute architecture that decouples LLM reasoning from browser execution, reducing per-workflow inference cost to under 0.10 USD. A one-shot LLM invocation processes a token-efficient semantic representation from a DOM Sanitization Module (DSM) and emits a deterministic JSON workflow blueprint. A lightweight runtime then drives the browser without further model queries. We formalize this cost reduction from O(M x N) to amortized O(1) inference scaling, where M is the number of reruns and N is the sequential actions. Empirical evaluation across data extraction, form filling, and fingerprinting tasks yields zero-shot compilation success rates of 80-94%. Crucially, the modularity of the JSON intermediate representation allows minimal Human-in-the-Loop (HITL) patching to elevate execution reliability to near-100%. At per-compilation costs between 0.002 USD and 0.092 USD across five frontier models, these results establish deterministic compilation as a paradigm enabling economically viable automation at scales previously infeasible under continuous architectures.

### 1. 核心问题与整体含义
*   **研究动机**：现有的网页智能体（如基于 ReAct 或 CoT 架构）通常采用“连续推理循环”模式，即每执行一个动作都要调用一次 LLM 来观察状态并决策。
*   **重运行危机（Rerun Crisis）**：在企业级大规模重复任务中，这种模式导致推理成本和延迟随执行次数（M）和步骤数（N）线性增长（复杂度为 $O(M \times N)$）。例如，执行 500 次简单的 5 步工作流可能耗资 150 美元，这使得 AI 自动化在经济上难以落地。
*   **核心目标**：将 LLM 从“实时决策者”转变为“一次性编译器”，实现推理与执行的解耦。

### 2. 方法论
*   **核心思想**：模仿传统编译器的原理，将自然语言意图（源代码）通过 LLM（编译器）转化为确定性的 JSON 蓝图（中间表示/字节码），最后由轻量级引擎（运行时）在无需 LLM 的情况下重复执行。
*   **关键技术细节**：
    *   **DOM 脱敏模块 (DSM)**：通过消除噪声（删除 script/style/base64）、提取可见信号（过滤隐藏元素）和清洗属性（保留语义化的 data-* 属性），将原始 HTML 压缩高达 85%，显著降低 Token 消耗。
    *   **单次编译协议 (One-Shot Compilation)**：LLM 接收脱敏后的 DOM 骨架，一次性生成包含选择器、分页逻辑和循环结构的 JSON 蓝图。
    *   **确定性执行引擎**：一个不依赖 AI 的运行时，根据 JSON 指令驱动浏览器，利用动态等待启发式算法处理异步渲染。
    *   **延迟重规划 (Lazy Replanning)**：仅在 UI 发生重大变化导致执行失败时，才触发 LLM 进行局部的“选择器修复”，而非全程参与。

### 6. 主要结论与发现
*   **成本骤降**：推理成本从 $O(M \times N)$ 降至摊销后的 $O(1)$。单次任务成本从约 150 美元降至 0.1 美元以下，降幅达 1500 倍。
*   **高成功率**：零样本（Zero-shot）编译成功率在 80% 到 94% 之间。
*   **可靠性**：通过 JSON 蓝图的模块化特性，结合人机回环（HITL）微调，可以将执行可靠性提升至接近 100%。
*   **模型差异**：发现轻量级模型（如 Gemini Flash）在处理复杂的 Schema 约束生成时容易失败，而前沿模型表现优异。

### 7. 优点
*   **经济可行性**：彻底解决了 LLM 智能体在生产环境中“跑不起”的问题。
*   **鲁棒性设计**：DSM 模块强制 LLM 使用稳定的语义选择器（如 ARIA 角色），而非脆弱的 CSS 路径，增强了对 UI 微调的抵抗力。
*   **可维护性**：生成的 JSON 蓝图是人类可读的，方便非技术人员通过自然语言或简单修改进行维护。

### 8. 不足与局限
*   **环境限制**：该方法高度依赖 DOM 树结构，对于 Canvas 渲染、WebGL 或高度混淆的无 DOM 界面（如某些复杂游戏或安全严密的金融应用）无效。
*   **动态逻辑挑战**：如果网页的交互逻辑随操作状态发生剧烈且不可预测的变化，静态的 JSON 蓝图可能难以应对，仍需回归连续推理模式。
*   **实验偏差**：实验主要在企业级稳定网页上进行，对于互联网上极度不规范或频繁变动的“野外”网页，其重编译频率（及由此产生的成本）尚需进一步研究。
