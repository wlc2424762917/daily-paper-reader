## 摘要
人们对于构建能够与数字平台交互以自主执行有意义的企业任务的智能体表现出日益增长的兴趣。在已探索的方法中，包括基于模型上下文协议（MCP）等抽象构建的工具增强型智能体，以及通过图形界面操作的网页智能体。然而，考虑到成本和运行开销，目前尚不清楚这些复杂的智能体系统是否必要。我们认为，仅配备终端和文件系统的编程智能体，通过直接与平台 API 交互，可以更有效地解决许多企业任务。我们在多种真实世界系统中评估了这一假设，结果表明这些底层终端智能体的表现与更复杂的智能体架构相当，甚至更优。我们的研究结果表明，简单的编程接口结合强大的基础模型，足以满足实际的企业自动化需求。

## Abstract
There has been growing interest in building agents that can interact with digital platforms to execute meaningful enterprise tasks autonomously. Among the approaches explored are tool-augmented agents built on abstractions such as Model Context Protocol (MCP) and web agents that operate through graphical interfaces. Yet, it remains unclear whether such complex agentic systems are necessary given their cost and operational overhead. We argue that a coding agent equipped only with a terminal and a filesystem can solve many enterprise tasks more effectively by interacting directly with platform APIs. We evaluate this hypothesis across diverse real-world systems and show that these low-level terminal agents match or outperform more complex agent architectures. Our findings suggest that simple programmatic interfaces, combined with strong foundation models, are sufficient for practical enterprise automation.

#### 6. 主要结论与发现
*   **性能领先**：终端代理在成功率上匹配或超越了复杂的 Web 代理，且远优于 MCP 代理。
*   **成本极低**：终端代理的推理成本仅为 Web 代理的一小部分（例如在 ServiceNow 上，成本差异达 4-6 倍）。
*   **MCP 的局限性**：预定义的工具接口过于僵化，无法处理 API 支持但工具未暴露的复杂查询或字段。
*   **文档的双刃剑效应**：文档对 ERPNext 等逻辑清晰的平台有帮助，但对 ServiceNow 等面向 UI 的文档反而可能误导智能体采取复杂的 API 路径。
*   **技能积累的价值**：允许智能体保存“技能”能显著降低后续任务的探索开销，减少 API 调用次数。

#### 7. 优点
*   **极简高效**：证明了“少即是多”，通过回归底层 API 交互解决了复杂系统带来的脆弱性和高延迟。
*   **鲁棒性强**：相比 Web 代理，终端代理不受 UI 布局更改的影响；相比 MCP，它具有无限的表达能力。
*   **真实性高**：实验基于真实的生产级企业平台，而非简化的模拟环境。
*   **可扩展性**：提出的“技能积累”机制为智能体的自我进化提供了简单可行的路径。

#### 8. 不足与局限
*   **视觉/UI 盲区**：无法处理必须通过图形界面完成的任务（如拖拽式工作流编辑器、读取复杂的渲染图表）。
*   **API 依赖**：如果企业平台未暴露某些功能的 API（如某些身份模拟操作），终端代理将彻底失效。
*   **安全风险**：终端代理具有广泛的执行权限（Shell 访问），在实际部署中需要极严苛的沙箱环境和权限控制。
*   **模型依赖**：该方法高度依赖 LLM 的编码和推理能力，在较弱的模型上可能无法正确构造复杂的 API 请求。
