## 摘要
大语言模型的最新进展提升了代码智能体（coding agents）的能力，但对复杂的端到端网站开发的系统性评估仍然有限。为了填补这一空白，我们推出了 Vision2Web，这是一个用于视觉网站开发的分层基准，涵盖了从静态 UI 到代码生成、交互式多页面前端复现，到长程全栈网站开发的各个阶段。该基准基于真实世界的网站构建，包含 16 个类别的共 193 个任务，拥有 918 张原型图和 1,255 个测试用例。为了支持灵活、全面且可靠的评估，我们提出了一种基于工作流的智能体验证范式，该范式由两个互补组件组成：GUI 智能体验证器和基于视觉语言模型（VLM）的裁判。我们评估了在不同代码智能体框架下实例化的多个视觉语言模型，揭示了在所有任务层级上存在的显著性能差距，即使是目前最先进的模型在全栈开发方面仍面临挑战。

## Abstract
Recent advances in large language models have improved the capabilities of coding agents, yet systematic evaluation of complex, end-to-end website development remains limited. To address this gap, we introduce Vision2Web, a hierarchical benchmark for visual website development, spanning from static UI-to-code generation, interactive multi-page frontend reproduction, to long-horizon full-stack website development. The benchmark is constructed from real-world websites and comprises a total of 193 tasks across 16 categories, with 918 prototype images and 1,255 test cases. To support flexible, thorough and reliable evaluation, we propose workflow-based agent verification paradigm based on two complementary components: a GUI agent verifier and a VLM-based judge. We evaluate multiple visual language models instantiated under different coding-agent frameworks, revealing substantial performance gaps at all task levels, with state-of-the-art models still struggling on full-stack development.

### 2. 方法论：核心思想与技术细节
论文提出了 **Vision2Web** 基准测试框架，其核心由两部分组成：

#### **分层任务设计 (Hierarchical Task Design)**
将网站开发拆解为三个难度递增的层级：
*   **Level 1: 静态网页 (Static Webpage)**：要求根据不同设备（桌面、平板、手机）的原型图生成响应式代码，重点考察视觉还原。
*   **Level 2: 交互式前端 (Interactive Frontend)**：提供多张原型图和页面逻辑描述，要求实现具有导航流和结构一致性的多页面前端。
*   **Level 3: 全栈网站 (Full-Stack Website)**：提供结构化需求文档和原型图，要求管理应用状态、实现后端逻辑、数据库交互及端到端调试。

#### **基于工作流的智能体验证 (Workflow-Based Agent Verification)**
为了解决端到端评估中实现多样性导致难以测试的问题，论文放弃了传统的单元测试，采用了：
*   **GUI 智能体验证器 (GUI Agent Verifier)**：使用一个预定义的测试工作流（包含目标、引导动作和验证准则），驱动一个 GUI 智能体（如 GLM-4.6V）在生成的网站上执行操作，计算**功能得分 (FS)**。
*   **VLM 评判器 (VLM-based Judge)**：利用视觉语言模型（如 Gemini-3-Pro）对比原型图与生成网页的截图，按组件进行细粒度评分，计算**视觉得分 (VS)**。

### 6. 主要结论与发现
*   **性能随复杂度剧降**：所有智能体在全栈任务上的表现远逊于静态网页任务。例如，Gemini-3-Pro 在静态网页上表现强劲，但在全栈任务的功能得分仅为 22.6。
*   **Claude-Opus-4.5 领跑**：在所有层级和框架中，Claude-Opus-4.5 表现最稳健，尤其在处理长程规划和多页面集成方面显著优于其他模型。
*   **系统性弱点**：智能体在**状态管理 (State Management)**、**CRUD 操作**和**文件/媒体处理**方面表现最差，反映出其在处理持久化数据流和系统级资源协调时的局限。
*   **视觉退化**：随着页面深度增加，视觉一致性显著下降；在移动端等小屏幕设备上的适配能力普遍弱于桌面端。

### 7. 优点
*   **层级化解耦**：通过三个层级的设计，能够清晰地定位智能体是在视觉理解、交互逻辑还是系统架构层面发生失败。
*   **真实性与多样性**：数据源自真实网站而非合成数据，涵盖了复杂的 SaaS 和交易场景，具有极高的实用参考价值。
*   **创新的验证范式**：引入 GUI 智能体作为“黑盒测试员”，解决了端到端开发中代码实现路径不唯一导致的自动化测试难题。

### 8. 不足与局限
*   **验证器幻觉风险**：GUI 验证器本身基于模型，可能存在推理幻觉，导致评分偏差（尽管人工校验显示准确率较高）。
*   **API 成本与时效性**：依赖闭源 SOTA 模型进行评估，成本较高且模型更新可能影响基准测试的长期稳定性。
*   **领域局限**：目前仅专注于 Web 开发，尚未扩展到移动 App 或桌面软件等其他视觉驱动的软件工程领域。
