## 摘要
自主智能体越来越多地与 Web 交互，但大多数网站仍是为人类浏览器设计的——这是新兴的“智能体 Web”（Agentic Web）必须解决的根本性不匹配。智能体必须重复浏览页面、检查 DOM 并逆向工程可调用路由，这一过程缓慢、脆弱，且在不同智能体之间冗余重复。我们观察到，每个现代网站在其用户界面背后都已经暴露了内部 API（有时称为“影子 API”），即驱动网站自身功能的第一方端点。我们提出了 Unbrowse，这是一个共享路由图，它将基于浏览器的路由发现转化为这些可调用的第一方接口的集体维护索引。该系统从真实的浏览流量中被动学习路由，并通过直接 API 调用提供缓存路由。在跨 94 个域的等效信息检索任务的单主机实时 Web 基准测试中，完全预热的缓存执行平均耗时 950 毫秒，而 Playwright 浏览器自动化为 3,404 毫秒（平均加速 3.6 倍，中位数加速 5.4 倍），且缓存良好的路由在 100 毫秒内即可完成。三路径执行模型（本地缓存、共享图或浏览器回退）确保了系统的自愿性和自我修正。通过 x402 协议建立的三层微支付模型对图查询收取单次查询搜索费（第 3 层），对发现文档收取一次性安装费（第 1 层），并为选择加入的网站所有者提供可选的单次执行费（第 2 层）。所有层级都基于理性采用的必要条件：只有当总费用低于浏览器重新发现的预期成本时，智能体才会使用共享图。

## Abstract
Autonomous agents increasingly interact with the web, yet most websites remain designed for human browsers -- a fundamental mismatch that the emerging ``Agentic Web'' must resolve. Agents must repeatedly browse pages, inspect DOMs, and reverse-engineer callable routes -- a process that is slow, brittle, and redundantly repeated across agents. We observe that every modern website already exposes internal APIs (sometimes called \emph{shadow APIs}) behind its user interface -- first-party endpoints that power the site's own functionality. We present Unbrowse, a shared route graph that transforms browser-based route discovery into a collectively maintained index of these callable first-party interfaces. The system passively learns routes from real browsing traffic and serves cached routes via direct API calls. In a single-host live-web benchmark of equivalent information-retrieval tasks across 94 domains, fully warmed cached execution averaged 950\,ms versus 3{,}404\,ms for Playwright browser automation (3.6$\times$ mean speedup, 5.4$\times$ median), with well-cached routes completing in under 100\,ms. A three-path execution model -- local cache, shared graph, or browser fallback -- ensures the system is voluntary and self-correcting. A three-tier micropayment model via the x402 protocol charges per-query search fees for graph lookups (Tier~3), a one-time install fee for discovery documentation (Tier~1), and optional per-execution fees for site owners who opt in (Tier~2). All tiers are grounded in a necessary condition for rational adoption: an agent uses the shared graph only when the total fee is lower than the expected cost of browser rediscovery.

#### 1. 核心问题与整体含义
*   **研究动机**：当前的自主智能体（Autonomous Agents）主要通过模拟人类浏览器行为（如解析 DOM、点击 UI）与网页交互。这种“浏览器优先”的架构存在三大痛点：**速度慢**（需加载大量静态资源）、**易碎性高**（UI 微调即导致脚本失效）、**重复劳动**（不同智能体在同一网站上重复进行昂贵的逆向工程）。
*   **核心理念**：现代网站在 UI 背后普遍存在“影子 API”（Shadow APIs，即内部第一方接口）。论文提出 **Unbrowse** 系统，主张智能体应直接调用这些内部 API，并通过一个“共享路由图”将发现的 API 路径转化为集体维护的索引，从而消除智能体的“发现税”。

#### 2. 方法论
*   **核心思想**：构建一个共享的路由图（Shared Route Graph），通过被动观察真实浏览流量来学习可调用的 Web 接口，并将其封装为可复用的“技能”。
*   **三路径执行模型**：
    1.  **路径 1（本地缓存）**：优先检查本地是否有最近成功的路由记录。
    2.  **路径 2（共享图）**：若本地无缓存，则查询云端共享索引，通过 x402 协议支付微小的安装费获取 API 文档。
    3.  **路径 3（浏览器回退）**：若无现成路由，启动基于 Zig 编写的轻量级浏览器代理 **Kuri**，捕获流量并逆向工程出 API，随后将新路由发布至共享图。
*   **关键技术**：
    *   **流量过滤与提取**：基于 Content-Type（如 JSON/XML）和请求方法（POST/PUT 等）识别 API。
    *   **技能封装**：遵循 AgentSkills.io 标准，生成包含端点架构、认证模式和客户端代码的 `SKILL.md`。

#### 6. 主要结论与发现
*   **显著加速**：在预热缓存情况下，Unbrowse 平均延迟为 **950 ms**，而 Playwright 为 **3,404 ms**，平均加速 **3.6 倍**，中位数加速达 **5.4 倍**。

#### 7. 优点
*   **架构创新**：从“模拟人类操作”转向“直接机器交互”，极大地提升了智能体 Web 交互的效率和稳定性。
*   **激励机制**：首次引入基于 x402 协议的微支付模型，解决了路由维护的经济激励问题，使“智能体网页”生态具有可持续性。
*   **轻量化**：采用 Zig 编写的 Kuri 代理，比传统的 Playwright/Puppeteer 更轻量、启动更快。

#### 8. 不足与局限
*   **反爬虫限制**：对于部署了高级 Web 应用防火墙（如 Cloudflare 高级版）的站点，非浏览器请求仍可能被拦截，加速效果会打折扣。
*   **安全性风险**：恶意贡献者可能发布指向钓鱼端点的伪造路由，虽然有信任评分机制，但仍存在对抗风险。
*   **任务局限性**：目前主要验证了信息检索类任务，对于涉及复杂多步交互（如需人工干预的 OAuth 流程）的处理能力尚待进一步验证。
*   **实现完整性**：部分功能（如自动弃用连续失败的路由）在论文发表时仅为部分实现。
