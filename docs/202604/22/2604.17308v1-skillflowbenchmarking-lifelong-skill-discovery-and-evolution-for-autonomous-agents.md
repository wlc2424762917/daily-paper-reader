## 摘要
随着自主智能体能力边界的不断扩展，它们越来越能够通过即插即用的外部技能来完成专业化任务。然而，目前的基准测试大多仅测试模型是否能使用提供的技能，而未探讨它们是否能从经验中发现技能、在失败后修复技能，并随着时间的推移维护一个连贯的技能库。我们推出了 SkillFlow，这是一个包含 20 个系列、166 个任务的基准测试，其中每个系列内的任务构建遵循领域无关执行流（DAEF），该执行流定义了一个智能体工作流框架，使这些任务能够共享一致的工作流。智能体在智能体终身学习（Agentic Lifelong Learning）协议下进行评估：它们初始时没有技能，按顺序解决每个系列内的任务，通过轨迹驱动和准则驱动的技能补丁将经验教训外部化，并将更新后的技能库持续传递。实验揭示了显著的能力差距。对于 Claude Opus 4.6，终身技能演化将任务成功率从 62.65% 提升至 71.08%（+8.43 个百分点）。然而，高技能使用率并不一定意味着高实用性：尽管 Kimi K2.5 的技能使用率达到 66.87%，但仅获得了 +0.60 个百分点的提升，而 Qwen-Coder-Next 的任务完成率仅为 44.58%，且相对于原始设置仍有所退步。SkillFlow 为该研究方向提供了一个结构化的测试平台，并对终身评估下的技能发现、补丁修复、迁移及其失败模式进行了深入的实证分析。

## Abstract
As the capability frontier of autonomous agents continues to expand, they are increasingly able to complete specialized tasks through plug-and-play external skills. Yet current benchmarks mostly test whether models can use provided skills, leaving open whether they can discover skills from experience, repair them after failure, and maintain a coherent library over time. We introduce SkillFlow, a benchmark of 166 tasks across 20 families in which task construction within each family follows a Domain-Agnostic Execution Flow (DAEF) that defines an agent workflow framework, allowing these tasks to share a consistent workflow. Agents are evaluated under an Agentic Lifelong Learning protocol in which they begin without skills, solve tasks sequentially within each family, externalize lessons through trajectory- and rubric-driven skill patches, and carry the updated library forward. Experiments reveal a substantial capability gap. For Claude Opus 4.6, lifelong skill evolution improves task success from 62.65% to 71.08% (+8.43 points). However, high skill usage does not necessarily imply high utility: Kimi K2.5 gains only +0.60 points despite 66.87% skill usage, while Qwen-Coder-Next reaches only a 44.58% task completion rate and still regresses relative to the vanilla setting. SkillFlow contributes a structured testbed for this direction and an in-depth empirical analysis of skill discovery, patching, transfer, and their failure modes under lifelong evaluation.

### 论文总结：SkillFlow - 针对自主智能体终身技能发现与演化的基准测试

#### 1. 论文的核心问题与整体含义（研究动机和背景）
当前的自主智能体（Autonomous Agents）研究主要集中在模型如何使用预设的外部工具或技能。然而，在现实世界的长期应用中，智能体需要具备**终身学习**的能力，即：能否从过去的成功或失败经验中自主总结出可复用的技能？能否在遇到错误时修复这些技能？能否随着任务的更迭维护一个连贯且高效的技能库？

现有的基准测试（如 SkillsBench）大多是静态的，无法评估智能体在任务序列中的知识积累与演化。因此，本文提出了 **SkillFlow**，旨在填补这一空白，衡量智能体在跨任务、跨领域的场景下，进行技能发现、修复、迁移和维护的能力。

#### 2. 论文提出的方法论
论文的核心思想是构建一个具有一致工作流逻辑但领域背景各异的任务集合，并定义一套终身学习的评估协议。

*   **DAEF (Domain-Agnostic Execution Flow，领域无关执行流)**：
    这是 SkillFlow 的设计基石。研究者将任务抽象为一个工作流图 $T = (V, E, \lambda, \gamma)$，其中 $V$ 是操作节点，$E$ 是依赖关系，$\lambda$ 是操作类型（如读取、计算、验证），而 $\gamma$ 是具体的领域落地（如特定的文件名、实体）。通过移除 $\gamma$，得到抽象的 DAEF。这使得智能体在解决一个财务任务时学到的“技能”，可以被迁移到具有相同逻辑的医疗数据处理任务中。
*   **双智能体迭代构建流水线**：
    利用 Architect Agent（架构智能体）生成任务资产和环境，Critic Agent（评审智能体）在 Docker 环境中进行实测并提供反馈，通过多轮迭代确保任务的稳定性、难度梯度和 DAEF 的一致性。
*   **智能体终身学习协议 (Agentic Lifelong Learning Protocol)**：
    *   **初始状态**：智能体从零技能库开始。
    *   **序列执行**：按难度递增顺序解决同一系列内的任务。
    *   **技能补丁 (Skill Patch)**：任务完成后，智能体根据执行轨迹和验证器的反馈（Rubric），生成一个包含 `summary`、`upsert_files`（新增/更新）和 `delete_paths`（删除）的补丁，动态更新其外部技能库。

*   **数据集/场景**：SkillFlow 包含 20 个任务系列，共 166 个可运行任务，涵盖金融、供应链、医疗、政务、数据智能 5 大领域。
*   **Benchmark 构成**：每个系列包含 8-9 个任务，具有明确的难度梯度。
*   **对比方法/模型**：
    *   评估了 11 种主流模型，包括 Claude (Sonnet/Opus 4.5 & 4.6)、GPT (5.3 Codex/5.4)、MiniMax (M2.5/M2.7)、Qwen-Coder-Next、Qwen3-Coder-480B、Kimi K2.5。

*   **评估指标**：任务成功率（%comp）、效率（交互轮数、成本、Token 消耗）、技能生成数量与重用率。

#### 6. 论文的主要结论与发现
*   **能力差距显著**：最强模型 Claude Opus 4.6 表现出稳定的进化能力，成功率提升了 8.43%，且成本有所下降。
*   **技能通胀与低效**：弱模型（如 Qwen, 部分 MiniMax）虽然生成了大量技能（高重用率），但往往是任务轨迹的简单堆砌，导致“认知负荷”过重，成功率反而下降或停滞。
*   **修复比编写更关键**：模型之间的核心差距不在于能否写出技能，而在于能否根据失败反馈“修复”错误的逻辑。
*   **精简胜过冗余**：表现优秀的智能体倾向于维护一个紧凑、不断迭代的核心技能库，而非为每个任务创建独立技能。

#### 7. 优点
*   **DAEF 框架的创新**：通过抽象工作流逻辑，解决了智能体基准测试中“任务相关性”难以量化的难题，为技能迁移提供了科学依据。
*   **真实的终身学习模拟**：不同于以往的一次性测试，SkillFlow 模拟了知识积累和自我修正的过程，更符合未来自主智能体的发展方向。
*   **高质量的任务构建**：结合了自动化生成与严格的人工审核，确保了任务的真实性（Workplace-realistic）和可运行性。

#### 8. 不足与局限
*   **领域覆盖风险**：虽然涵盖了 5 大领域，但任务类型仍集中在数据处理、文档编辑和代码操作，对于具身智能或多模态交互的覆盖较少。
*   **技能格式限制**：目前采用基于 Markdown 和脚本的固定文件结构，可能限制了某些模型发挥更灵活的记忆管理能力。
*   **评估成本高昂**：由于涉及长序列任务和复杂的 Docker 环境，复现该基准测试需要较高的 API 费用和环境配置成本。
