Abstract —This paper considers multi-agent embodied question answering (MA-EQA), which aims to query robot teams on what they have seen over a long horizon. In contrast to existing edge resource management methods that emphasize sensing, communication, or computation performance metrics, MA-EQA emphasizes the memory qualities. To cope with this paradigm shift, we propose a quality of memory (QoM) model based on generative adversarial exam (GAE), which leverages forward simulation to assess memory retrieval and uses the resulting exam scores to compute QoM values. Then we propose memory centric power allocation (MCPA), which maximizes the QoM function under communication resource constraints. Through asymptotic analysis, it is found that the transmit powers are proportional to the GAE error probability, thus prioritizing towards high-QoM robots. Extensive experiments demonstrate that MCPA achieves significant improvements over extensive benchmarks in terms of diverse metrics in various scenarios. 

Index Terms —Embodied question answering, multi-agent memorization, power allocation, quality of memory. 

I. I NTRODUCTION 

Embodied questions answering (EQA) enables users to query the robots on what they have observed in their memories [1]. Standalone EQA solutions are constrained by limited memory capacity [2]. To address this limitation, multi-agent EQA (MA-EQA) aggregates the local memory of distributed robots over wireless networks, building a global memory with increased spatial and temporal coverage [3], [4]. However, in contrast to existing methods that adopt sensing coverage [5], communication throughput [6], computation efficiency [7], or their combinations [8], as design objectives, MA-EQA aims to maximize the memory qualities. Therefore, these schemes [5]–[8] may overlook the non-uniform memory contributions of different agents, resulting in degraded EQA accuracy. To prioritize transmissions based on data value, semantic communication (SemCom) [9], [10] reduces information re-dundancy while preserving semantic similarity (e.g., cosine similarity between embeddings). However, semantic similarity is an intermediate rather than end-to-end task metric for EQA, which may overlook subtle but critical differences in long memory contexts. On the other hand, task-oriented communi-cation [11]–[13] enables end-to-end optimization of inference performance within integrated sensing, communication, and computation systems. If the tasks are closed-set with well-defined object classes and labels [11]–[13], the objective function can be explicitly formulated. In MA-EQA, however, the task is open-set, and the memory must be constructed without prior knowledge of the questions [2], which makes it difficult to find a proper memory-oriented objective function. To fill the gap, we introduce memory centric power allo-cation (MCPA) that unifies memorization and communication into an end-to-end optimization framework. Our solution is inspired by a key observation: “While the specific questions users may ask are infinite and unpredictable, the patterns and templates of these questions are finite and predictable .” As such, we build a generative adversarial exam (GAE) system with proxy questioner and agentic verifier, enabling the forward simulation of potential EQA processes . The exam with fewer GAE scores represent higher memory values. This enables us to transform the implicit EQA accuracy into the explicit quality of memory (QoM), which is defined as a com-posite function of GAE scores and communication rates. As such, by maximizing the QoM under communication resource constraints, MCPA leads to a paradigm shift from sensing, communication, or computation centric towards memorization centric . To get deeper insights into MCPA, an analytical solu-tion is derived for the asymptotic case. The asymptotic optimal solution discloses that the transmit powers are proportional to the answer error probability of GAE. Extensive results in the high-fidelity Carla simulation show that MCPA achieves significant improvements over existing schemes in terms of both memory qualities and EQA accuracies. By analyzing the communication rate and sensing coverage performances, it is found that communication and sensing functions are not al-ways beneficial to memorization: their positive contribution is observed only when the associated QoM criteria are satisfied. In this section, we propose a novel QoM model to ap-proximate Ψ. The key insight is that the potential question cannot be an arbitrary question. The patterns and templates of questions are often known in advance. More importantly, to generate answers A for questions Q, at least one robot must have seen the associated scene; otherwise the question should be considered as invalid. Based on the above insight, we can generate in-context questions Qk from memory Mk, and use Qk to test the current memory M0. Because Qk is at least answerable by Mk, Qk is guaranteed to be a valid question. Furthermore, if there exists any question in Qk that cannot be answered by M0, then merging Mk is guaranteed to increase the question answering capability of the system. In contrast, if M0 can answer all the questions in Qk, Mk would contribute to no new memory. This gives the GAE pipeline shown in Fig. 2 for computing QoE. C. Iterative Memory Retrieval 

One may wonder whether the GAE test would involve extra computation time, offsetting the benefits brought by its memory awareness. This may happen if we feed all the memory as a context into the LLM, as it requires to scan all information in M0, which are computationally expensive for transformers. However, for a given question, a large history is not required to provide a correct answer. Instead, only a small subset of the memory S0 ∈ M 0 is sufficient [2]. This converts the one-shot long-horizon inference into a sub-memory search problem, which finds the optimal subset S∗ 

> 0

If so, we generate an output formatted as a json file with keys for times, poses, or texts; otherwise, the process repeats by retrieving next-round memory chunk. The chunk is selected by an LLM-agent that reasons with the current context. The process terminate until the right memory chunk is found, or the maximum number of retrievals is reached. With IMR, the average inference time is 6.43 seconds per QA for qwen3-8b (tested on Nvidia 3090 Ti). Since different QAs can be executed in parallel on different GPUs at the server, the computation latency would not scale with K.Compared to the communication time (e.g., T = 600 s in our experiment), the LLM inference time is significantly smaller. B. Experiment 2: Evaluation of MCPA 

Next, we consider the case of K = 10 , where the 10-drone scenario configuration is shown in Fig. 4. We randomly choose 5 robots to build the initial ground memory M0. For GAE, we generate Lk = 10 questions from 10 randomly sampled images for each drone k. The quantitative results are illustrated in Table II. Our MCPA achieves the highest EQA accuracy of 95 .35% and best QoM value of 0.21 . Compared to the standalone memory Remember [2], the gain is over 

50% , which demonstrates the necessity of adopting memory aggregation in city-scale EQA. Although MaxRate achieves the highest communication rate, its EQA accuracy is over 30 

points behind our scheme. This implies that communication is not always beneficial to memorization, since closer robots may convey useless data that cannot update the knowledge database at the server. The Greedy scheme prioritizes the robots with higher information entropy regardless of its channel. However, it may over-allocate resources to remote robots, resulting in fewer connected robots and compromised robustness. The approach closest to ours is MaxCov, which connects the largest number of drones. However, MaxCov also leads to inferior performance, since sensing is not guaranteed to result in new memory. New memory is created from sensing only when the sensed region contains information that is sufficiently fresh.
