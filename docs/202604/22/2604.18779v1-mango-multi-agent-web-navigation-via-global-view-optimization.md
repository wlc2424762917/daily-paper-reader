## 摘要
现有的网页智能体通常从根 URL 开始探索，这对于具有深层层级结构的复杂网站来说效率低下。由于缺乏对网站结构的全局视图，智能体经常陷入导航陷阱、探索无关分支，或者无法在有限的预算内获取目标信息。我们提出了 Mango，一种利用网站结构动态确定最佳起始点的多智能体网页导航方法。我们将 URL 选择建模为多臂老虎机问题，并采用汤普森采样（Thompson Sampling）在候选 URL 之间自适应地分配导航预算。此外，我们引入了一个情景记忆组件来存储导航历史，使智能体能够从之前的尝试中学习。在 WebVoyager 上的实验表明，当使用 GPT-5-mini 时，Mango 达到了 63.6% 的成功率，比最佳基准模型高出 7.3%。此外，在 WebWalkerQA 上，Mango 获得了 52.5% 的成功率，超过最佳基准模型 26.8%。我们还利用开源和闭源模型作为骨干网络，证明了 Mango 的泛化能力。我们的数据和代码已开源，可在 https://github.com/VichyTong/Mango 获取。

## Abstract
Existing web agents typically initiate exploration from the root URL, which is inefficient for complex websites with deep hierarchical structures. Without a global view of the website's structure, agents frequently fall into navigation traps, explore irrelevant branches, or fail to reach target information within a limited budget. We propose Mango, a multi-agent web navigation method that leverages the website structure to dynamically determine optimal starting points. We formulate URL selection as a multi-armed bandit problem and employ Thompson Sampling to adaptively allocate the navigation budget across candidate URLs. Furthermore, we introduce an episodic memory component to store navigation history, enabling the agent to learn from previous attempts. Experiments on WebVoyager demonstrate that Mango achieves a success rate of 63.6% when using GPT-5-mini, outperforming the best baseline by 7.3%. Furthermore, on WebWalkerQA, Mango attains a 52.5% success rate, surpassing the best baseline by 26.8%. We also demonstrate the generalizability of Mango using both open-source and closed-source models as backbones. Our data and code are open-source and available at https://github.com/VichyTong/Mango.

*   **研究动机**：智能体在导航时往往只有局部视野（当前页面的 HTML）。作者认为，如果能为智能体提供网站结构的**全局视图**，并让其直接从最相关的候选页面切入，将极大提升任务成功率和效率。

### 2. 方法论：核心思想与关键技术
Mango 采用多智能体协作模式，核心流程分为四个阶段：
*   **全局结构分析（Global Structure Analysis）**：
    *   通过轻量级爬虫（BFS 广度优先搜索，限制爬取数量 $\tau$）获取网站内部链接。
    *   利用 LLM 生成关键词，通过 Google Search 的 `site:` 操作符获取索引页面。
    *   使用 **BM25 算法**计算候选 URL 与用户查询的语义相关性，筛选出 Top-10 候选 URL。
*   **多臂老虎机（MAB）URL 选择**：
    *   将 URL 选择建模为 MAB 问题，使用 **Thompson Sampling（汤普森采样）**。
    *   **公式逻辑**：每个 URL 维护一个 Beta 分布 $(\alpha, \beta)$。初始值基于 BM25 得分。每次迭代从分布中采样，选择得分最高的 URL 进行导航。
*   **网页导航智能体（Web Navigation Agent）**：
    *   从选定的 URL 开始执行动作（点击、输入等）。
    *   结合**情节记忆（Episodic Memory）**：如果该 URL 曾被访问，会提取之前的轨迹和反思摘要，避免重复错误动作。
*   **反思智能体（Reflection Agent）**：
    *   评估导航结果。若成功则结束；若因预算耗尽或结果不全，则根据路径潜力给出奖励（$r=1$ 或 $r=0$），更新 MAB 的 Beta 分布参数，并决定是否将该 URL 标记为“已耗尽”。

### 7. 优点
*   **全局视野优化**：打破了“必须从主页开始”的思维定式，通过预分析显著缩小了搜索空间。
*   **动态预算分配**：引入 Thompson Sampling 相比 MCTS 在有限预算下更具鲁棒性，能更有效地在“探索”和“利用”之间平衡。
*   **记忆增强**：情节记忆有效解决了智能体在复杂导航中容易出现的“原地打转”问题。

### 8. 不足与局限
*   **爬虫覆盖率限制**：对于拥有数十万页面的超大型动态网站，轻量级爬虫可能无法覆盖所有潜在目标。

*   **推理成本**：多智能体多次迭代和反思虽然提高了成功率，但也带来了更高的 API 调用成本和延迟。
*   **推理错误风险**：即便导航到了正确页面，底层 LLM 仍可能出现阅读理解错误或幻觉。
