## 摘要
通过“截图-点击”循环控制桌面计算机的 GUI 智能体引入了一类新的漏洞：观察到动作之间的间隙（在真实 OSWorld 工作负载上平均为 6.51 秒）产生了一个“检查时间-使用时间”（TOCTOU）窗口，在此期间，非特权攻击者可以操纵 UI 状态。我们将此形式化为“视觉原子性违规”（Visual Atomicity Violation），并刻画了三种具体的攻击原语：(A) 通知叠加层劫持，(B) 窗口焦点操纵，以及 (C) Web DOM 注入。原语 B 是 Android 动作重绑定（Action Rebinding）在桌面端最接近的模拟，它在观察时无需任何视觉证据即可实现 100% 的动作重定向成功率。我们提出了执行前 UI 状态验证（PUSV），这是一种轻量级的三层防御机制，在每次动作分发前立即重新验证 UI 状态：点击目标处的掩码像素 SSIM（L1）、全局截图差异（L2a）以及 X Window 快照差异（L2b）。PUSV 在 180 次对抗性试验（135 次原语 A + 45 次原语 B）中实现了 100% 的动作拦截率，且误报率为零，开销小于 0.1 秒。针对原语 C（零视觉足迹的 DOM 注入），PUSV 揭示了一个结构性盲点（动作拦截率约为 0%），这为未来的 OS+DOM 深度防御架构提供了动力。没有任何单一的 PUSV 层能实现完全覆盖；不同的原语需要不同的检测信号，这验证了分层设计的有效性。

## Abstract
GUI agents that control desktop computers via screenshot-and-click loops introduce a new class of vulnerability: the observation-to-action gap (mean 6.51 s on real OSWorld workloads) creates a Time-Of-Check, Time-Of-Use (TOCTOU) window during which an unprivileged attacker can manipulate the UI state. We formalize this as a Visual Atomicity Violation and characterize three concrete attack primitives: (A) Notification Overlay Hijack, (B) Window Focus Manipulation, and (C) Web DOM Injection. Primitive B, the closest desktop analog to Android Action Rebinding, achieves 100% action-redirection success rate with zero visual evidence at the observation time. We propose Pre-execution UI State Verification (PUSV), a lightweight three-layer defense that re-verifies the UI state immediately before each action dispatch: masked pixel SSIM at the click target (L1), global screenshot diff (L2a), and X Window snapshot diff (L2b). PUSV achieves 100% Action Interception Rate across 180 adversarial trials (135 Primitive A + 45 Primitive B) with zero false positives and < 0.1 s overhead. Against Primitive C (zero-visual-footprint DOM injection), PUSV reveals a structural blind spot (~0% AIR), motivating future OS+DOM defense-in-depth architectures. No single PUSV layer alone achieves full coverage; different primitives require different detection signals, validating the layered design.

*   **研究动机**：这一时间间隙导致了经典的 **TOCTOU（检查时间至使用时间）** 漏洞。攻击者可以在智能体“检查”完截图后、“使用”动作前，篡改 UI 状态（如弹出伪造窗口或切换焦点），诱导智能体在错误的目标上执行敏感操作。
*   **整体含义**：论文首次形式化定义了桌面环境下的“视觉原子性违规”（Visual Atomicity Violation），并证明了现有顶级模型（GPT-4o, Claude 4.6 等）在没有防御的情况下极易受到此类时序攻击。

### 2. 方法论
论文提出了 **PUSV（Pre-execution UI State Verification，执行前 UI 状态验证）** 防御框架，其核心思想是在动作分发前的最后一毫秒进行“二次校验”。
*   **关键技术细节（三层防御架构）**：
    1.  **L1 - 掩码像素 SSIM（局部校验）**：在计划点击坐标周围提取 160x160 的像素块，计算结构相似性指数（SSIM）。若 SSIM < 0.92，判定为目标被遮挡或篡改。
    2.  **L2a - 全局截图差异（全局校验）**：对比前后两张截图的全局像素变化率。若变化超过 0.2%（排除 L1 区域后），判定为出现了非预期的 UI 变动（如弹出通知）。
    3.  **L2b - X Window 快照差异（系统层校验）**：通过 `wmctrl` 查询窗口注册表，检测是否有带有“安全”、“警告”等关键字的新窗口弹出。
*   **流程**：该防御作为中间件运行，在 LLM 输出坐标后、物理点击执行前触发，总耗时小于 0.1 秒。

### 3. 实验设计
*   **数据集/场景**：构建了 **DesktopTOCTOU-Bench**，包含 50 个独特场景，涵盖文件操作、通信、系统配置、数据访问和权限提升五个高风险领域。
*   **攻击原语**：
    *   **A (Notification Overlay)**：弹出 Tkinter 窗口或系统通知遮挡目标。

*   **对比方法**：在无防御状态和开启 PUSV 防御状态下，对比了 **Claude Opus 4.6**、**GPT-4o** 和 **Qwen3.6-plus** 三种前沿模型。

### 4. 资源与算力
*   **硬件环境**：实验在运行 VMware Fusion 的 **M4 Mac** 宿主机上进行，虚拟机环境为 Ubuntu 22.04。

### 5. 实验数量与充分性
*   **实验规模**：共进行了 **180 次** 防御有效性试验（135 次针对原语 A，45 次针对原语 B），以及针对原语 C 的 45 次测试。每个任务条件均执行了 N=15 的重复实验以确保统计显著性。
*   **充分性评价**：实验设计较为充分。不仅测试了不同类型的攻击，还跨越了三个不同厂商的模型家族，证明了漏洞的普遍性和防御的模型无关性。同时，还进行了误报率测试（30 次良性任务），验证了防御不会干扰正常操作。

### 6. 主要结论与发现
*   **漏洞严重性**：在无防御时，攻击原语 B（窗口焦点操纵）在所有模型上均能达到近乎 **100% 的成功率**，且智能体完全意识不到被劫持。
*   **防御有效性**：PUSV 在拦截 OS 层面的攻击（原语 A 和 B）时达到了 **100% 的拦截率（AIR）**，且额外延迟极低（<0.1s）。
*   **结构性盲点**：视觉防御（SSIM/Diff）对 **原语 C（DOM 注入）** 的拦截率几乎为 **0%**。因为透明覆盖层不改变像素，这证明了仅靠截图分析无法完全解决安全问题。

### 7. 优点
*   **首创性**：首次系统地将 TOCTOU 概念引入桌面 GUI 智能体领域并进行了形式化。
*   **轻量高效**：防御机制不依赖昂贵的二次 LLM 推理，而是使用确定性的计算机视觉和系统指令，适合实时部署。
*   **分层设计**：通过局部、全局和系统层三个维度的互补，覆盖了不同层面的 UI 篡改。

### 8. 不足与局限
*   **DOM 盲点**：如结论所述，该方法无法检测无视觉特征的底层代码注入，需要未来结合浏览器 CDP 协议进行深度防御。
*   **阈值敏感性**：SSIM 和像素差异的阈值（0.92 和 0.2%）是基于 Ubuntu 22.04 校准的，在不同操作系统或高动态 UI（如视频播放）下可能需要重新校准以避免误报。
*   **对抗性演进**：如果攻击者能制造出像素完全一致但语义不同的 UI 元素，该防御可能会失效。
