## 摘要
工具集成推理（TIR）通过在推理过程中利用外部工具扩展大语言模型（LLM）的能力，已成为一个极具前景的方向。现有的 TIR 方法在推理过程中通常依赖外部工具文档。然而，这导致了工具掌握困难、工具规模受限以及推理效率低下等问题。为了缓解这些问题，我们探索了工具内化推理（TInR），旨在通过将工具知识内化到 LLM 中来促进推理。实现这一目标提出了显著的要求，包括工具内化和工具推理协调。为了解决这些问题，我们提出了 TInR-U，这是一个用于统一推理和工具使用的工具内化推理框架。TInR-U 通过三阶段流水线进行训练：1）采用双向知识对齐策略的工具内化；2）使用高质量推理标注的监督微调预热；3）采用 TInR 特定奖励的强化学习。我们在域内和域外设置下对该方法进行了全面评估。实验结果表明，TInR-U 在两种设置下均取得了优异的性能，突显了其有效性和高效性。

## Abstract
Tool-Integrated Reasoning (TIR) has emerged as a promising direction by extending Large Language Models' (LLMs) capabilities with external tools during reasoning. Existing TIR methods typically rely on external tool documentation during reasoning. However, this leads to tool mastery difficulty, tool size constraints, and inference inefficiency. To mitigate these issues, we explore Tool-Internalized Reasoning (TInR), aiming at facilitating reasoning with tool knowledge internalized into LLMs. Achieving this goal presents notable requirements, including tool internalization and tool-reasoning coordination. To address them, we propose TInR-U, a tool-internalized reasoning framework for unified reasoning and tool usage. TInR-U is trained through a three-phase pipeline: 1) tool internalization with a bidirectional knowledge alignment strategy; 2) supervised fine-tuning warm-up using high-quality reasoning annotations, and 3) reinforcement learning with TInR-specific rewards. We comprehensively evaluate our method across in-domain and out-of-domain settings. Experiment results show that TInR-U achieves superior performance in both settings, highlighting its effectiveness and efficiency.

#### 1. 核心问题与整体含义（研究动机和背景）
传统的**工具集成推理（Tool-Integrated Reasoning, TIR）**依赖于在提示词（Prompt）中提供外部工具文档。这种“开卷考试”模式存在三大痛点：
*   **掌握困难**：工具文档格式异构且不一致，模型难以即时理解复杂规则。
*   **规模受限**：随着工具数量增加，文档长度会超出上下文窗口限制。
*   **推理低效**：冗长的文档显著增加了推理延迟和计算开销。

受人类能够内化工具知识并熟练运用的启发，本文提出了**工具内化推理（Tool-Internalized Reasoning, TInR）**。其核心目标是将工具的功能、参数和使用规则直接“硬编码”到模型的参数中，使模型无需查阅外部文档即可实现高效、精准的工具调用。

#### 2. 方法论：TInR-U 框架
论文提出了 **TInR-U** 框架，通过三阶段训练流水线实现工具知识与推理能力的统一：

*   **阶段 1：工具知识内化（Tool Internalization）**
    *   **工具标记化**：为每个工具分配唯一的特殊 Token（如 `<<tool_token>>`），将工具调用转化为预测下一个 Token 的任务。
    *   **双向知识对齐（BKA）**：
        1.  **记忆（Memorization）**：根据文档预测工具 Token。
        2.  **召回（Recall）**：根据工具 Token 重建原始文档。
        3.  **使用（Usage）**：根据指令直接生成工具调用动作。
*   **阶段 2：SFT 预热（TInR SFT Warm-up）**
    *   利用大模型（如 Qwen3-14B）通过拒绝采样生成高质量的推理轨迹。
    *   将推理过程中的工具名称替换为内化的工具 Token，进行监督微调，使模型学会协调推理步骤与工具使用。
*   **阶段 3：强化学习（TInR RL）**
    *   采用 **GRPO（群组相对策略优化）** 算法。
    *   **奖励设计**：结合**格式奖励**（结构正确性）和**准确性奖励**（基于 Jaccard 相似度的工具选择和参数匹配准确度），进一步提升模型的探索能力和鲁棒性。

#### 6. 主要结论与发现
*   **性能领先**：TInR-U 在所有设置下均优于现有 TIR 和端到端方法，尤其在域外工具调用上 EM 提升了 **18.13%**。
*   **推理高效**：与 TIR 方法随工具增加而变慢不同，TInR-U 的推理效率保持恒定。当工具超过 100 个时，其速度优势显著扩大。
*   **内化优势**：双向知识对齐策略比简单的语义索引或数字索引更有效，能显著减少工具识别的混淆。
*   **两步设计**：先预测工具 Token 再填充参数的“两步走”策略，比直接生成完整调用更能降低模型负担。

#### 7. 优点
*   **创新范式**：从“查字典”转向“背字典”，解决了长上下文和推理延迟的根本矛盾。
*   **训练方案完备**：三阶段流水线逻辑清晰，结合了知识对齐、行为克隆（SFT）和环境反馈（RL）。
*   **实用性强**：在保持高性能的同时，极大提升了大规模工具环境下的部署效率。

#### 8. 不足与局限
*   **维护成本**：当外部工具频繁更新（如 API 参数变更）时，内化模型需要重新训练或通过持续学习更新参数，比修改 Prompt 的成本更高。
*   **数据依赖**：高性能依赖于高质量的合成推理数据，对教师模型（LRM）的能力有一定要求。
*   **幻觉风险**：虽然实验表现良好，但在极端大规模工具集下，模型仍可能出现工具 Token 的幻觉或参数混淆。
