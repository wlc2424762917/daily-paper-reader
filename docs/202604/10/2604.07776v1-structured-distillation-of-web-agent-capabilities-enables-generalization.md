## 摘要
前沿大语言模型（LLMs）能够导航复杂的网站，但其高昂的成本以及对第三方 API 的依赖使得本地部署变得不切实际。我们引入了 Agent-as-Annotators，这是一个通过类比人类标注角色来结构化 Web 智能体合成轨迹生成的框架，利用模块化的 LLM 组件取代了任务设计者（Task Designer）、标注者（Annotator）和监督者（Supervisor）。我们使用 Gemini 3 Pro 作为教师模型，在六个 Web 环境中生成了 3,000 条轨迹，并在通过质量过滤的 2,322 条轨迹上，通过纯监督学习微调了一个 9B 参数的学生模型。所得模型在 WebArena 上达到了 41.5% 的成功率，在相同的评估协议下超过了 Claude 3.5 Sonnet (36.0%) 和 GPT-4o (31.5%) 等闭源模型，并且几乎是之前最佳开源权重结果（Go-Browse, 21.7%）的两倍。该模型的能力可以迁移到未见过的环境，在 WorkArena L1（一个训练期间从未见过的企业平台）上获得了 18.2 个百分点的提升，并在另外三个基准测试中取得了一致的改进。消融实验证实，流水线中的每个组件都做出了重要贡献，其中评判过滤（Judge filtering）、评估提示（evaluation hints）和推理轨迹（reasoning traces）各自都带来了可衡量的提升。这些结果表明，利用单个前沿教师模型进行结构化轨迹合成，足以产生具有竞争力的、可本地部署的 Web 智能体。项目页面：https://agent-as-annotators.github.io

## Abstract
Frontier LLMs can navigate complex websites, but their cost and reliance on third-party APIs make local deployment impractical. We introduce Agent-as-Annotators, a framework that structures synthetic trajectory generation for web agents by analogy to human annotation roles, replacing the Task Designer, Annotator, and Supervisor with modular LLM components. Using Gemini 3 Pro as teacher, we generate 3,000 trajectories across six web environments and fine-tune a 9B-parameter student with pure supervised learning on the 2,322 that pass quality filtering. The resulting model achieves 41.5% on WebArena, surpassing closed-source models such as Claude 3.5 Sonnet (36.0%) and GPT-4o (31.5%) under the same evaluation protocol, and nearly doubling the previous best open-weight result (Go-Browse, 21.7%). Capabilities transfer to unseen environments, with an 18.2 percentage point gain on WorkArena L1 (an enterprise platform never seen during training) and consistent improvements across three additional benchmarks. Ablations confirm that each pipeline component contributes meaningfully, with Judge filtering, evaluation hints, and reasoning traces each accounting for measurable gains. These results demonstrate that structured trajectory synthesis from a single frontier teacher is sufficient to produce competitive, locally deployable web agents. ### 2. 方法论：Agent-as-Annotators 框架
该框架通过模拟人类标注网页任务的三个角色，将合成轨迹生成过程模块化：
*   **任务设计者（Task Designer）**：
    *   **人格生成器（Persona Generator）**：生成多样化的用户背景（如数据科学家、设计师），以诱导产生不同分布的任务。
    *   **任务生成器（Task Generator）**：在真实环境中探索并生成具体的任务意图（Intent），同时生成**评估提示（Evaluation Hints）**（即判断任务成功的关键状态信息）。
*   **标注者（Annotator）**：
    *   **智能体（Agent）**：由教师模型（Gemini 3 Pro）扮演，仅根据任务意图在环境中执行操作，生成交互轨迹。
*   **监督者（Supervisor）**：
    *   **评判员（Judge）**：利用“评估提示”对轨迹进行审核。只有被判定为成功的轨迹才会被加入训练集（A3-Synth）。
*   **关键技术细节**：
    *   **结构化推理**：训练数据保留了教师模型的推理轨迹（Thought/Think 模块）。
    *   **低推理预算优势**：研究发现，减少教师模型的思考预算反而能生成更简洁、成功率更高的训练轨迹。

### 6. 主要结论与发现
*   **性能突破**：微调后的 **A3-Qwen3.5-9B** 在 WebArena 上达到 **41.5%**，超过了 GPT-4o (31.5%) 和 Claude 3.5 Sonnet (36.0%)。
*   **泛化能力强**：在从未见过的企业平台 WorkArena L1 上提升了 **18.2%**，证明模型学到的是通用的网页交互原语而非简单的环境记忆。

### 7. 优点与亮点
*   **角色类比框架**：将复杂的合成数据生成过程抽象为人类标注角色，具有很强的可解释性和模块化扩展性。
*   **评估提示（Hints）创新**：通过在任务生成阶段预留“评估提示”，显著提升了自动评判（Judge）的准确性，解决了网页任务终态模糊的难题。
*   **高效蒸馏**：证明了通过极小规模（<3k 轨迹）的高质量数据，可以实现小模型对顶级闭源模型的“以小博大”。

### 8. 不足与局限
*   **环境覆盖有限**：训练仅基于 6 个 WebArena 环境，虽然展现了泛化性，但对于更极端的网页多样性可能仍有不足。
*   **SFT 局限性**：目前仅采用监督微调（SFT），尚未结合强化学习（RL）进行进一步的自我演化。
*   **评判员偏差**：虽然引入了 Hints，但 Judge 本身仍由 LLM 担任，可能存在潜在的假阳性判定风险。
*   **应用限制**：模型在处理极长序列（如 WorkArena L2 的 50 步任务）时，相比顶级模型仍有明显差距。
