## 摘要
网络智能体（Web agents）——即代表用户在网络上导航并执行任务的自主系统——具有改变人类与数字世界交互方式的潜力。然而，目前最强大的网络智能体依赖于具有未公开训练数据和配方的私有模型，这限制了科学理解、可复现性以及社区驱动的进步。我们认为，面向开放网络的智能体应当以开放的方式构建。为此，我们推出了：(1) MolmoWebMix，一个包含浏览器任务演示和网页图形用户界面（GUI）感知数据的大规模多样化混合数据集；(2) MolmoWeb，一系列完全开源的多模态网络智能体。具体而言，MolmoWebMix 结合了来自多个互补生成流水线的超过 10 万条合成任务轨迹，以及 3 万多条人类演示、原子级网络技能轨迹和 GUI 感知数据（包括指代表达定位和截图问答）。MolmoWeb 智能体作为受指令约束的视觉-语言动作策略运行：给定任务指令和网页截图，它们即可预测下一步浏览器动作，无需访问 HTML、可访问性树（accessibility trees）或专门的 API。MolmoWeb 智能体提供 4B 和 8B 两种规模，在 WebVoyager、Online-Mind2Web 和 DeepShop 等浏览器使用基准测试中，其表现达到了最先进水平，优于 Fara-7B、UI-Tars-1.5-7B 和 Holo1-7B 等同规模的仅开源权重模型。MolmoWeb-8B 还超越了基于 GPT-4o 等更大规模闭源前沿模型构建的标记集（Set-of-Marks, SoM）智能体。我们进一步证明，通过带有 Best-of-N 选择的并行展开进行测试时扩展（test-time scaling），可以获得持续的性能提升，在 WebVoyager 和 Online-Mind2Web 上分别实现了 94.7% 和 60.5% 的 pass@4（相比之下 pass@1 分别为 78.2% 和 35.3%）。我们将发布模型权重、训练数据、代码和统一的评估框架，以实现可复现性并加速网络智能体领域的开放研究。

## Abstract
Web agents--autonomous systems that navigate and execute tasks on the web on behalf of users--have the potential to transform how people interact with the digital world. However, the most capable web agents today rely on proprietary models with undisclosed training data and recipes, limiting scientific understanding, reproducibility, and community-driven progress.   We believe agents for the open web should be built in the open. To this end, we introduce (1) MolmoWebMix, a large and diverse mixture of browser task demonstrations and web-GUI perception data and (2) MolmoWeb, a family of fully open multimodal web agents. Specifically, MolmoWebMix combines over 100K synthetic task trajectories from multiple complementary generation pipelines with 30K+ human demonstrations, atomic web-skill trajectories, and GUI perception data, including referring expression grounding and screenshot question answering. MolmoWeb agents operate as instruction-conditioned visual-language action policies: given a task instruction and a webpage screenshot, they predict the next browser action, requiring no access to HTML, accessibility trees, or specialized APIs.   Available in 4B and 8B size, on browser-use benchmarks like WebVoyager, Online-Mind2Web, and DeepShop, MolmoWeb agents achieve state-of-the-art results outperforming similar scale open-weight-only models such as Fara-7B, UI-Tars-1.5-7B, and Holo1-7B. MolmoWeb-8B also surpasses set-of-marks (SoM) agents built on much larger closed frontier models like GPT-4o. We further demonstrate consistent gains through test-time scaling via parallel rollouts with best-of-N selection, achieving 94.7% and 60.5% pass@4 (compared to 78.2% and 35.3% pass@1) on WebVoyager and Online-Mind2Web respectively. We will release model checkpoints, training data, code, and a unified evaluation harness to enable reproducibility and accelerate open research on web agents.

### 2. 方法论：核心思想与关键技术
*   **核心思想**：采用**纯视觉（Vision-only）**方案。智能体仅通过网页截图、任务指令和动作历史进行决策，不依赖脆弱且冗长的 HTML 或可访问性树（AxTree）。
*   **MolmoWebMix 数据集**：包含四类核心数据：
    1.  **合成轨迹**：利用 Gemini-3-Flash 作为“教师”，通过 AxTree 生成 10 万+条任务轨迹，并转化为视觉动作。
    2.  **人类演示**：收集了 3 万多条真实人类在浏览器上的操作轨迹，涵盖搜索、填表等复杂任务。
    3.  **原子技能轨迹**：将复杂任务分解为“搜索”、“填表”、“加入购物车”等基础技能进行专项训练。
    4.  **GUI 感知数据**：包含 1000 万+个指代表达定位（Grounding）和截图问答（QA）对，提升模型对 UI 元素的识别能力。
*   **模型架构**：基于 Molmo2 架构，结合了 Qwen2.5 语言模型和 SigLIP2 视觉编码器。
*   **算法流程**：模型在每一步输出一个 JSON 对象，包含“思考（Thought）”和“动作（Action）”。动作空间包括点击、拖拽、滚动、输入、回退等。

### 3. 实验设计
*   **数据集与场景**：在四个主流的实时网页基准测试上进行评估：**WebVoyager**、**Online-Mind2Web**、**DeepShop** 和 **WebTailBench**。
*   **对比方法**：

*   **测试时扩展（Test-time Scaling）**：引入了 **Pass@k** 评估，即运行 $N$ 个并行智能体并由 VLM 评判器选择最佳结果。

*   **实验规模**：论文进行了多维度的实验，包括：

    *   **专项评估**：在 ScreenSpot/v2 上进行了专门的 GUI 定位能力测试。

### 6. 主要结论与发现
*   **开源超越闭源**：MolmoWeb-8B 在多个基准上优于基于 GPT-4o 的 SoM 智能体，证明了专用小模型在特定任务上的潜力。
*   **数据质量胜过规模**：仅使用 10% 的数据即可达到 85-90% 的性能；合成数据（由 AxTree 引导生成）在训练效果上优于人类演示数据，因为前者更直接、噪声更小。
*   **推理时扩展威力巨大**：通过并行 4 个 Rollouts（Pass@4），WebVoyager 的成功率从 78.2% 飙升至 **94.7%**。
*   **纯视觉可行性**：无需 HTML 也能实现高精度的网页交互，且能显著降低 Token 消耗。

### 7. 优点（亮点）
*   **全栈开源**：不仅开源模型权重，还开源了极其珍贵的 13 万+条任务轨迹数据和评估工具。
*   **多智能体合成流水线**：设计了“规划者-执行者-验证者”多智能体系统来生成高质量合成数据。
*   **鲁棒的动作空间**：支持归一化坐标点击和多种键盘/鼠标组合动作，适应性强。

### 8. 不足与局限
*   **OCR 瓶颈**：在处理极小文字或长篇幅文本理解时，模型仍会出现视觉解析错误。
*   **低频动作弱**：对于“拖拽（Drag）”和“悬停（Hover）”等在训练集中占比较小的动作，模型表现不够稳定。
*   **幻觉问题**：有时“思考”过程与实际执行的“动作”不一致（例如思考说要输入，动作却是直接按回车）。
*   **实时性限制**：虽然模型本身高效，但依赖实时浏览器执行和 VLM 评判，整体任务完成的延迟仍然较高。
