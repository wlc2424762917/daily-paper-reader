## 摘要
多轮大语言模型（LLM）智能体的强化学习（RL）训练本质上是不稳定的，而推理质量直接决定了任务性能。熵（Entropy）被广泛用于追踪推理的稳定性。然而，熵仅衡量同一输入内的多样性，无法判断推理是否真正响应了不同的输入。在 RAGEN-2 中，我们发现即使熵保持稳定，模型也可能依赖于固定的模板，这些模板看起来具有多样性，但与输入无关。我们将这种现象称为模板崩溃（template collapse），这是一种熵和所有现有指标都无法察觉的失效模式。为了诊断这一失效模式，我们将推理质量分解为输入内多样性（熵）和跨输入可区分性（互信息，MI），并引入了一系列互信息代理指标用于在线诊断。在各种任务中，互信息与最终性能的相关性远强于熵，使其成为更可靠的推理质量代理指标。我们进一步利用信噪比（SNR）机制解释了模板崩溃。低奖励方差会削弱任务梯度，导致正则化项占据主导地位并抹除跨输入的推理差异。为解决这一问题，我们提出了 SNR 感知过滤（SNR-Aware Filtering），利用奖励方差作为轻量级代理，在每次迭代中筛选高信号提示词。在规划、数学推理、网页导航和代码执行等任务中，该方法一致地提升了输入依赖性和任务性能。

## Abstract
RL training of multi-turn LLM agents is inherently unstable, and reasoning quality directly determines task performance. Entropy is widely used to track reasoning stability. However, entropy only measures diversity within the same input, and cannot tell whether reasoning actually responds to different inputs. In RAGEN-2, we find that even with stable entropy, models can rely on fixed templates that look diverse but are input-agnostic. We call this template collapse, a failure mode invisible to entropy and all existing metrics. To diagnose this failure, we decompose reasoning quality into within-input diversity (Entropy) and cross-input distinguishability (Mutual Information, MI), and introduce a family of mutual information proxies for online diagnosis. Across diverse tasks, mutual information correlates with final performance much more strongly than entropy, making it a more reliable proxy for reasoning quality. We further explain template collapse with a signal-to-noise ratio (SNR) mechanism. Low reward variance weakens task gradients, letting regularization terms dominate and erase cross-input reasoning differences. To address this, we propose SNR-Aware Filtering to select high-signal prompts per iteration using reward variance as a lightweight proxy. Across planning, math reasoning, web navigation, and code execution, the method consistently improves both input dependence and task performance.

#### 1. 核心问题与整体含义（研究动机和背景）
论文聚焦于多轮大语言模型（LLM）智能体在强化学习（RL）训练中的不稳定性。研究发现，传统的监控指标——**推理熵（Entropy）**存在严重局限性：即使熵值很高（看似推理多样），模型也可能陷入**“模板坍缩”（Template Collapse）**。在这种模式下，模型生成的推理链虽然表面丰富，但实际上是与输入无关的固定套路（Input-agnostic）。这种失效模式在多轮交互中尤为隐蔽，会导致智能体推理能力在训练中悄然退化。

#### 2. 论文提出的方法论
论文从信息论和信号处理的角度提出了诊断与改进方案：
*   **核心思想**：将推理质量分解为**输入内多样性（熵，$H(Z|X)$）**和**跨输入相关性（互信息，$I(X;Z)$）**。
*   **互信息代理指标（MI Proxy）**：提出“批次内交叉评分”（In-Batch Cross-Scoring）方法。通过计算推理轨迹在当前输入与批次内其他输入下的对数似然差异，衡量推理对输入的依赖程度。主要指标包括：
    *   **Retrieval-Acc**：衡量能否通过推理轨迹准确检索出对应的输入提示词。
    *   **MI-ZScore-EMA**：一种连续且鲁棒的互信息估算值，用于在线监控。
*   **SNR 机制解释**：利用信噪比（SNR）视角解释坍缩原因。任务梯度（信号）受奖励方差（RV）控制，而正则化梯度（噪声，如 KL 散度、熵奖励）是恒定的。当奖励方差过低时，噪声主导更新，抹除输入相关性。
*   **SNR 感知过滤（SNR-Aware Filtering）**：在每次训练迭代中，计算每个提示词的奖励方差，仅保留方差最高的前 $p$ 比例（Top-$p$）的样本进行参数更新，从而过滤掉低信号、高噪声的更新。

#### 6. 主要结论与发现
*   **MI 是更好的监控器**：互信息（MI）与任务性能的正相关性（+0.39）远高于熵（-0.14），熵在诊断推理坍缩时会给出错误信号。
*   **奖励方差决定信号质量**：低奖励方差会导致任务梯度消失，使模型被输入无关的正则化项“洗脑”。
*   **SNR 过滤的普适性**：SNR 感知过滤在所有测试的任务、算法和模型规模上均实现了一致的性能提升（平均提升 3%~7% 的成功率，在某些任务如 Sokoban 上提升超过 15%）。

#### 7. 优点
*   **理论深刻**：首次系统性地定义并量化了智能体 RL 中的“模板坍缩”现象，并给出了严谨的信息论解释。

*   **即插即用**：SNR 感知过滤不改变底层 RL 算法逻辑，易于集成到现有的分布式训练框架中。

#### 8. 不足与局限
*   **奖励依赖性**：该方法高度依赖于奖励函数的设计。在奖励极其稀疏或奖励函数本身存在巨大噪声的环境中，奖励方差可能不再是有效的 SNR 代理。
