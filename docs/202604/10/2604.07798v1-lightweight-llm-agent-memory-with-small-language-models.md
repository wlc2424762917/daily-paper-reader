## 摘要
尽管大语言模型（LLM）智能体能够利用工具处理复杂任务，但它们仍需要记忆来维持跨轮次的一致性，并在长程交互中积累可重用的信息。然而，基于检索的外部记忆系统虽然在线开销较低，但由于查询构建和候选过滤的局限性，其准确性并不稳定。相比之下，许多系统在在线记忆操作中重复调用大模型，虽然提高了准确性，但在长程交互中会不断累积延迟。我们提出了 LightMem，这是一种由小语言模型（SLM）驱动的轻量级记忆系统，旨在提升智能体的记忆能力。LightMem 将记忆检索、写入和长期整合模块化，并将在线处理与离线整合分离，从而在有限的计算资源下实现高效的记忆调用。我们将记忆组织为用于即时对话上下文的短期记忆（STM）、用于可重用交互摘要的中期记忆（MTM）以及用于整合知识的长期记忆（LTM），并利用用户标识符支持多用户场景下的独立检索和增量维护。在在线阶段，LightMem 在固定的检索预算下运行，并通过两阶段过程选择记忆：基于向量的粗筛检索，随后是语义一致性重排序。在离线阶段，它提取可重用的交互证据，并将其增量整合到 LTM 中。实验表明，该方法在不同规模的模型上均有提升，在 LoCoMo 数据集上的平均 F1 值提高了约 2.5，且更具效能，中值延迟较低（检索延迟 83 毫秒；端到端延迟 581 毫秒）。

## Abstract
Although LLM agents can leverage tools for complex tasks, they still need memory to maintain cross-turn consistency and accumulate reusable information in long-horizon interactions. However, retrieval-based external memory systems incur low online overhead but suffer from unstable accuracy due to limited query construction and candidate filtering. In contrast, many systems use repeated large-model calls for online memory operations, improving accuracy but accumulating latency over long interactions. We propose LightMem, a lightweight memory system for better agent memory driven by Small Language Models (SLMs). LightMem modularizes memory retrieval, writing, and long-term consolidation, and separates online processing from offline consolidation to enable efficient memory invocation under bounded compute. We organize memory into short-term memory (STM) for immediate conversational context, mid-term memory (MTM) for reusable interaction summaries, and long-term memory (LTM) for consolidated knowledge, and uses user identifiers to support independent retrieval and incremental maintenance in multi-user settings. Online, LightMem operates under a fixed retrieval budget and selects memories via a two-stage procedure: vector-based coarse retrieval followed by semantic consistency re-ranking. Offline, it abstracts reusable interaction evidence and incrementally integrates it into LTM. Experiments show gains across model scales, with an average F1 improvement of about 2.5 on LoCoMo, more effective and low median latency (83 ms retrieval; 581 ms end-to-end).

### 1. 核心问题与整体含义（研究动机和背景）
大模型智能体在处理长程交互时，面临着**记忆效率与准确性之间的权衡**：
*   **现有痛点：** 传统的“基于检索的外部记忆”虽然速度快，但由于查询构建简单、候选过滤不足，导致检索噪声大、准确性不稳定；而“基于大模型的在线记忆操作”虽然准确，但频繁调用大模型会产生巨大的延迟和计算成本，难以支持实时长程对话。
*   **研究动机：** 利用**小语言模型（SLM）**处理高频、结构化的在线记忆任务，同时将沉重的记忆抽象与整合任务置于离线处理，从而在保证低延迟的同时提升记忆的准确性和一致性。

### 2. 方法论：核心思想与关键技术
LightMem 的核心思想是**模块化记忆管理**与**在线/离线解耦**：

*   **三层记忆结构：**
    *   **短期记忆 (STM)：** 即当前的对话上下文窗口，不持久化。
    *   **中期记忆 (MTM)：** 存储用户特定的个性化情节摘要，包含语义摘要、时间戳、访问统计和用户标识符。
    *   **长期记忆 (LTM)：** 存储去标识化的、跨用户的语义知识，以轻量级**图结构（Knowledge Graph）**组织，支持多步推理。
*   **在线处理流程（由三个专门的 SLM 驱动）：**
    1.  **控制器 (SLM-1)：** 进行意图建模，将用户输入重写为多个“假设性查询 (HQs)”，并分配检索预算。
    2.  **选择器 (SLM-2)：** 执行**两阶段检索**。第一阶段是基于向量的粗筛（Top-2K）；第二阶段是基于语义一致性的重排序和压缩（最终输出 Top-K）。
    3.  **写入器 (SLM-3)：** 在每轮对话后提取关键信息，增量更新 MTM，并处理冗余或冲突。
*   **离线整合流程：**
    *   由大模型（如 GPT-4o）异步执行，将 MTM 中的高价值情节抽象为 LTM 中的知识节点和边，实现记忆的长期演化而不阻塞在线交互。

### 3. 实验设计
*   **数据集：**
    *   **LoCoMo：** 评估长对话中的逻辑推理（单跳、多跳、时间、开放域、对抗性任务）。
    *   **DialSim：** 模拟长期的多方对话，评估语义一致性。
*   **对比方法（Baselines）：** 包括 ReadAgent、MemoryBank、MemGPT、A-MEM 等主流记忆增强方法。

### 6. 主要结论与发现
*   **性能提升：** LightMem 在 LoCoMo 基准上的 F1 分数平均提升了约 **2.5**，在多跳推理和时间推理任务中优势尤为明显。
*   **低延迟：** 检索中值延迟仅为 **83 毫秒**，端到端中值延迟为 **581 毫秒**，远低于 MemGPT 和 A-MEM 等系统。
*   **语义一致性：** 在 DialSim 上，SBERT 语义相似度显著提高，证明系统能更有效地利用历史对话背景。

### 7. 优点
*   **架构优雅：** 通过在线/离线解耦，解决了记忆系统“准确则慢，快则不准”的顽疾。
*   **资源友好：** 证明了 1B-3B 规模的小模型足以胜任复杂的记忆管理逻辑，适合边缘侧或低算力环境部署。
*   **多用户支持：** 通过用户标识符和元数据约束，实现了多用户场景下的逻辑隔离和隐私保护。
*   **知识演化：** 引入图结构的 LTM，使智能体具备了从情节记忆向结构化知识转化的能力。

### 8. 不足与局限
*   **意图分解风险：** SLM-1 在处理模糊查询时，可能会生成过多的假设性查询，导致检索结果中混入无关的通用知识（Focus Dilution）。
