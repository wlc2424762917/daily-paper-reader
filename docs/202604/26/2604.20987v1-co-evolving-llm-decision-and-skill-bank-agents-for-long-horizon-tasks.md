## 摘要
长程交互环境是评估智能体技能使用能力的试验场。这些环境要求多步推理、在多个时间步长内链接多种技能，以及在延迟奖励和部分可观测性下进行稳健的决策。游戏是评估智能体在环境中技能使用情况的良好试验场。大语言模型（LLMs）作为游戏智能体提供了一种极具前景的选择，但由于缺乏在不同回合间发现、保留和重用结构化技能的机制，它们往往难以实现一致的长程决策。我们提出了 COSPLAY，这是一个协同演化框架。在该框架中，大语言模型决策智能体从可学习的技能库中检索技能以指导行动，而由智能体管理的技能流水线则从智能体的无标签轨迹（rollouts）中发现可重用技能以构建技能库。我们的框架不仅提升了决策智能体学习更好的技能检索和动作生成的能力，同时技能库智能体也在不断提取、精炼和更新技能及其契约（contracts）。在六个游戏环境中的实验表明，使用 8B 基础模型的 COSPLAY 在单人游戏基准测试中，相比四个前沿大语言模型基准，平均奖励提升了超过 25.1%，同时在多人社交推理游戏中也保持了竞争力。

## Abstract
Long horizon interactive environments are a testbed for evaluating agents skill usage abilities. These environments demand multi step reasoning, the chaining of multiple skills over many timesteps, and robust decision making under delayed rewards and partial observability. Games are a good testbed for evaluating agent skill usage in environments. Large Language Models (LLMs) offer a promising alternative as game playing agents, but they often struggle with consistent long horizon decision making because they lack a mechanism to discover, retain, and reuse structured skills across episodes. We present COSPLAY, a co evolution framework in which an LLM decision agent retrieves skills from a learnable skill bank to guide action taking, while an agent managed skill pipeline discovers reusable skills from the agents unlabeled rollouts to form a skill bank. Our framework improves both the decision agent to learn better skill retrieval and action generation, while the skill bank agent continually extracts, refines, and updates skills together with their contracts. Experiments across six game environments show that COSPLAY with an 8B base model achieves over 25.1 percent average reward improvement against four frontier LLM baselines on single player game benchmarks while remaining competitive on multi player social reasoning games.

### 论文总结：COS-PLAY 协同演化框架

#### 1. 核心问题与整体含义
*   **研究动机**：在长程（Long-Horizon）交互任务中，大语言模型（LLM）智能体面临多步推理复杂、奖励延迟和部分观测等挑战。
*   **核心痛点**：现有的 LLM 智能体缺乏一种有效的机制来**自主发现、保留并在不同回合间重用**结构化的技能。现有的技能库方法通常依赖人工预定义或静态蒸馏，无法随智能体策略的改进而进化。
*   **整体含义**：本文提出了 **COS-PLAY** 框架，通过“决策智能体”与“技能库智能体”的协同演化（Co-evolution），实现了从无标签交互轨迹中自动学习并优化技能库的闭环，显著提升了小参数模型在复杂任务中的表现。

#### 2. 方法论
核心思想是构建一个双智能体循环系统，通过相互促进来提升性能：
*   **决策智能体 (Decision Agent, $A_D$)**：
    *   **技能检索与意图更新**：在每个时间步，根据当前状态摘要从库中检索相关技能，并更新自然语言意图标签。
    *   **动作执行**：在技能协议（Protocol）的指导下生成原始动作。
*   **技能库智能体 (Skill Bank Agent, $A_S$)**：
    *   **边界提议与分割**：利用启发式信号（如意图跳转、奖励突变）识别轨迹中的技能转换点，并将片段标记为已知技能或新技能。
    *   **契约学习 (Contract Learning)**：统计并验证技能执行前后的状态变化（Added/Deleted Predicates），形成技能的“效果契约”。
    *   **库维护**：执行精炼（Refine）、物化（Materialize）、合并（Merge）、拆分（Split）和淘汰（Retire）操作，保持技能库的精简与高效。
*   **优化算法**：采用 **GRPO**（群组相对策略优化）算法，并为不同功能模块（如动作生成、技能检索、分割推断等）配置独立的 **LoRA 适配器**，以减少多任务目标的干扰。

*   **实验场景**：涵盖了 6 个具有挑战性的游戏环境：
    *   **单人益智/控制**：2048、 Candy Crush（糖果传奇）、Tetris（俄罗斯方块）、Super Mario Bros（超级马里奥）。
    *   **多人社交推理**：Avalon（阿瓦隆）、Diplomacy（外交）。

*   **评估指标**：游戏原生奖励、胜率（阿瓦隆）、供应中心数量（外交）。

#### 4. 资源与算力

*   **基础模型**：使用 **Qwen3-8B** 作为初始骨干模型。
*   **训练规模**：每个游戏环境进行最多 25 次协同演化迭代。
*   **初始化**：使用 GPT-5.4 生成 60 条种子轨迹进行冷启动 SFT。

#### 5. 实验数量与充分性
*   **实验规模**：单人游戏每项进行 16 次独立运行；多人游戏每个玩家角色进行 10 次运行。
*   **充分性评价**：
    *   **多维度覆盖**：实验不仅对比了最终得分，还深入分析了技能重用率、基尼系数（重用集中度）、契约验证率等指标。
    *   **消融实验**：详细对比了 SFT、GRPO 和协同演化各阶段的贡献，证明了闭环进化的必要性。
    *   **通用性测试**：在 Math-500 和 MMLU-Pro 上进行了测试，证明了该方法在提升特定领域能力的同时，较好地保留了通用推理能力。

#### 6. 主要结论与发现
*   **性能飞跃**：基于 8B 模型的 COS-PLAY 在单人游戏中的平均奖励比 GPT-5.4 高出 **25.1%**。
*   **小胜大**：在复杂的《外交》博弈中，COS-PLAY 优于 Gemini-3.1-Pro 8.8%，在《阿瓦隆》中表现接近 120B 参数的巨型模型。
*   **协同效应**：实验证明，单纯增加技能库或单纯进行 RL 训练效果有限，只有当策略与技能库**同步演化**（保持分布一致）时，性能提升最显著。
*   **技能重用**：系统能够自动发现具有高重用价值的抽象技能（如外交中的“探索”、“设置”、“防御”阶段），而非简单的轨迹记忆。

#### 7. 优点
*   **闭环自改进**：实现了从原始交互到结构化知识（技能库）再到策略提升的完整闭环。
*   **无监督技能发现**：无需人工定义技能边界或标签，降低了领域迁移的成本。
*   **多适配器设计**：通过分离 LoRA 适配器解决了多目标优化中的冲突问题。
*   **高数据效率**：相比传统强化学习，COS-PLAY 在较少的迭代次数内即可达到极高水平。

#### 8. 不足与局限
*   **文本摘要依赖**：框架高度依赖于将环境状态转化为文本描述，对于无法轻易文本化的复杂视觉细节可能处理不足。
*   **误差累积风险**：长程任务中，状态摘要的微小偏差可能随时间步累积，导致技能检索失效。

*   **对手水平限制**：在多人博弈中，演化质量部分受限于训练对手（如 GPT-5-mini）的策略多样性。
