## 摘要
长程语言模型智能体在代码行数和运行复杂度方面，占据主导地位的并非其底层模型，而是封装它的 Harness（运行框架）：包括上下文压缩、工具缓存、语义记忆、轨迹重用、投机性工具预测，以及将模型绑定到沙箱执行环境的粘合代码。我们认为 Harness 设计是一个一等机器学习问题，且一旦配置空间（flag space）超过几个比特，自动化配置搜索就会优于手动堆叠。我们分两步论证这一主张。首先，我们将自动化 Harness 优化形式化为在混合变量、成本异构的配置空间上的受限噪声贝叶斯优化，并结合了冷启动修正奖励和后验机会约束安全检查，同时提供了一个参考求解器 HARBOR（Harness 轴向对齐正则化贝叶斯优化程序），该求解器由块加性 SAAS 代理模型、多保真度成本感知采集函数和 TuRBO 置信域构建而成。其次，我们在一个生产级编程智能体的标志门控 Harness 中实例化了该问题，并报告了针对固定任务集的四轮受控手动调优案例研究与端到端 HARBOR 运行的对比。该形式化方法本身与任务类别无关：配置空间、奖励修正、采集函数和安全检查适用于任何具有有限配置空间和可重复任务集的智能体 Harness。

## Abstract
Long-horizon language-model agents are dominated, in lines of code and in operational complexity, not by their underlying model but by the harness that wraps it: context compaction, tool caching, semantic memory, trajectory reuse, speculative tool prediction, and the glue that binds the model to a sandboxed execution environment. We argue that harness design is a first-class machine-learning problem and that automated configuration search dominates manual stacking once the flag space exceeds a handful of bits. We defend this claim in two steps. First, we formalize automated harness optimization as constrained noisy Bayesian optimization over a mixed-variable, cost-heterogeneous configuration space with cold-start-corrected rewards and a posterior chance-constrained safety check, and give a reference solver, HARBOR (Harness Axis-aligned Regularized Bayesian Optimization Routine), built from a block-additive SAAS surrogate, multi-fidelity cost-aware acquisition, and TuRBO trust regions. Second, we instantiate the problem in a flag-gated harness over a production coding agent and report a controlled four-round manual-tuning case study against a fixed task suite and an end-to-end HARBOR run. The formulation itself is task-class agnostic: the configuration space, reward correction, acquisition, and safety check apply to any agent harness with a bounded flag space and a reproducible task suite.

*   **核心问题**：在现代长程（Long-horizon）智能体系统中，绝大部分代码（约98.4%）并非模型逻辑，而是负责上下文压缩、工具缓存、语义记忆和沙箱执行的“Harness”。
*   **研究动机**：目前业界主要依靠人工手动调优这些配置（如开关缓存、调整压缩阈值），但随着配置空间（Flag Space）的增大，功能之间的复杂交互会导致手动堆叠（Stacking）的效果往往适得其反（负优化）。
*   **整体含义**：论文主张将 Harness 设计视为一个一等机器学习问题，通过自动化搜索代替人工经验，以应对高维、异构且带有噪声的配置优化挑战。

### 2. 方法论：HARBOR 框架
论文提出了 **HARBOR**（Harness Axis-aligned Regularized Bayesian Optimization Routine），其核心思想是将外壳优化建模为**受限噪声贝叶斯优化（Constrained Noisy BO）**。
*   **混合变量代理模型**：使用块加性（Block-additive）的 **SAAS（稀疏轴向对齐子空间）** 先验模型。它能自动识别高维空间中真正起作用的少数关键配置，并处理布尔值、数值和类别等混合变量。
*   **冷启动修正奖励**：针对具有跨会话记忆的功能，提出了一种混合模型来修正“冷启动”偏差，确保评估结果能反映系统在长期运行（暖启动）后的真实性能。
*   **多保真度成本感知采集**：利用 `qNEHVI` 算法，在不同任务子集（保真度）上分配计算资源，平衡“通过率提升”与“API 调用成本”。
*   **后验机会约束安全检查**：在搜索过程中加入安全门控，确保新提出的配置在统计学上不会比基准线（Baseline）差，从而避免人工调优中常见的性能倒退。
*   **TuRBO 置信区域**：引入局部贝叶斯优化策略，提升在高维空间中的搜索效率。

### 6. 主要结论与发现
*   **手动堆叠的失效**：人工调优的四轮实验中，只有 B 轮（基础增强）优于基准，随后的 C、D 轮（引入更多文献中的先进技术）因功能冲突和集成 Bug 导致性能大幅下降。
*   **自动化优势**：HARBOR 仅用 2 个配置标志的组合就达到了人工调优的峰值性能，且在相同搜索预算下，比人工堆叠的 D 轮配置高出 **5 个任务通过数**（约提升 5.6%）。
*   **配置稀疏性**：在特定的（模型, 任务）对中，真正有效的 Harness 功能是稀疏的，自动化工具能比人类更快地识别出这些关键维度。
*   **Bug 检测器**：自动化框架通过遥测数据对比，成功发现了人工调优中被忽视的两个静默集成 Bug（记忆存储未持久化和预测器未触发）。

### 7. 优点与亮点
*   **工程问题学术化**：首次将智能体外壳工程系统性地形式化为贝叶斯优化问题。
*   **实用性强**：考虑了现实中的 API 成本约束和冷启动问题，而非理想化的实验室环境。
*   **任务无关性**：提出的 HARBOR 框架不限于编程任务，可推广至 QA、Web 浏览等各类智能体。

### 8. 不足与局限
*   **实验覆盖范围**：主要基于单一基准（Terminal-Bench 2）和单一智能体（codex-py），虽然具有代表性，但跨领域泛化性仍需更多验证。

*   **样本量限制**：在低保真度（小任务子集）评估时，伯努利噪声较大，可能影响搜索初期的准确性。
