## 摘要
模型上下文协议（Model Context Protocol, MCP）是 LLM 应用中定义和调用外部工具的一种被快速采用的标准。MCP 的多层架构除了传统的提示词注入外，还引入了诸如工具投毒（tool poisoning）等新的攻击面。现有的防御系统存在误报率高、依赖 API 或需要白盒访问权限等局限性。在本研究中，我们提出了 CASCADE，一种针对基于 MCP 系统的三层级联防御架构：(i) 第一层利用正则表达式、短语权重和熵分析进行快速预过滤；(ii) 第二层通过 BGE 嵌入进行语义分析，并配备了 Ollama Llama3 回退机制；(iii) 第三层应用基于模式的输出过滤。在包含 5,000 个样本的数据集上的评估结果显示，其精确率为 95.85%，误报率为 6.06%，召回率为 61.05%，F1 分数为 74.59%。对分为 6 个层级的 31 种攻击类型的分析表明，该架构对数据外泄（91.5%）和提示词注入（84.2%）具有较高的检测率，而语义攻击（52.5%）和工具投毒（59.9%）类别则显示出改进潜力。与现有解决方案相比，CASCADE 的一个关键优势是其完全本地化运行，无需外部 API 调用。

## Abstract
Model Context Protocol (MCP) is a rapidly adopted standard for defining and invoking external tools in LLM applications. The multi-layered architecture of MCP introduces new attack surfaces such as tool poisoning, in addition to traditional prompt injection. Existing defense systems suffer from limitations including high false positive rates, API dependency, or white-box access requirements. In this study, we propose CASCADE, a three-tiered cascaded defense architecture for MCP-based systems: (i) Layer 1 performs fast pre-filtering using regex, phrase weighting, and entropy analysis; (ii) Layer 2 conducts semantic analysis via BGE embedding with an Ollama Llama3 fallback mechanism; (iii) Layer 3 applies pattern-based output filtering. Evaluation on a dataset of 5,000 samples yielded 95.85% precision, 6.06% false positive rate, 61.05% recall, and 74.59% F1-score. Analysis across 31 attack types categorized into 6 tiers revealed high detection rates for data exfiltration (91.5%) and prompt injection (84.2%), while semantic attack (52.5%) and tool poisoning (59.9%) categories showed potential for improvement. A key advantage of CASCADE over existing solutions is its fully local operation, requiring no external API calls

*   **核心目标**：开发一种完全本地化、低误报、且能同时应对传统注入与 MCP 特有攻击（如工具投毒）的防御架构。

#### 2. 核心方法论
论文提出了名为 **CASCADE** 的三层级联防御架构，其核心思想是“由快到慢、由浅入深”的过滤机制：
*   **L1：预过滤层（Pre-Filter Layer）**

*   **L2：语义判定层（Semantic Judge Layer）**
    *   **核心策略**：采用“嵌入优先，LLM 回退”策略。
    *   **流程**：先使用 **BGE 嵌入模型**将输入向量化，与已知攻击库计算相似度。若结果模糊，则调用本地运行的 **Llama3 (8B)** 模型进行深度语义推理。
    *   **输出**：提供三级决策（允许 ALLOW / 人工审计 REVIEW / 拦截 BLOCK）。
*   **L3：输出过滤层（Output Filter Layer）**
    *   **功能**：监控 MCP Host 返回给用户的响应，利用模式匹配和**熵分析**检测敏感信息（如 API 密钥、凭据）或加密的数据外泄尝试。

#### 6. 主要结论与发现
*   **高效精确性**：CASCADE 实现了 **95.85% 的精确率**和仅 **6.06% 的误报率**，显著优于之前研究中 90% 以上的误报率。

#### 7. 优点
*   **三级决策机制**：引入“REVIEW”状态，允许人工介入处理模糊案例，增加了系统在实际生产中的灵活性。
*   **隐私保护**：完全本地化运行，解决了企业级应用中敏感数据不敢上传云端安全网关的痛点。
*   **低误报率**：通过多层过滤和规范化处理，极大地降低了对正常业务指令的误伤。

*   **语义理解瓶颈**：对于隐藏在自然语言深处的复杂语义攻击，召回率（Recall）仅为 52.5%，说明单纯依靠相似度和小型 LLM 仍难以完全识别高隐蔽性攻击。
*   **静态评估**：实验基于静态数据集，未在动态、多轮对话的实时 MCP 环境中进行长程评估。
