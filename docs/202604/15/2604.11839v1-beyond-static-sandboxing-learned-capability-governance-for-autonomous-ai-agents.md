## 摘要
基于 OpenClaw 等开源运行时的自主 AI 智能体默认在每个会话中公开所有可用工具，而不考虑具体任务。例如，摘要任务与代码部署任务拥有相同的 Shell 执行、子智能体生成和凭据访问能力，这种 15 倍的过度配置比例被我们称为“能力过度配置问题”。现有的防御措施（包括 NemoClaw 容器沙箱和 Cisco DefenseClaw 技能扫描器）解决了容器化和威胁检测问题，但无法学习每种任务类型的最小可行能力集。我们提出了 Aethelgard，这是一个四层自适应治理框架，通过学习型策略为 AI 智能体强制执行最小权限原则。第一层是能力治理器（Capability Governor），它动态界定智能体在每个会话中感知的工具范围。第三层是安全路由器（Safety Router），它利用基于规则和微调分类器的混合方法，在执行前拦截工具调用。第二层是强化学习（RL）学习策略，它在累积的审计日志上训练 PPO 策略，以学习每种任务类型的最小可行技能集。

## Abstract
Autonomous AI agents built on open-source runtimes such as OpenClaw expose every available tool to every session by default, regardless of the task. A summarization task receives the same shell execution, subagent spawning, and credential access capabilities as a code deployment task, a 15x overprovision ratio that we call the capability overprovisioning problem. Existing defenses, including the NemoClaw container sandbox and the Cisco DefenseClaw skill scanner, address containment and threat detection but do not learn the minimum viable capability set for each task type.   We present Aethelgard, a four layer adaptive governance framework that enforces least privilege for AI agents through a learned policy. Layer 1, the Capability Governor, dynamically scopes which tools the agent is aware of in each session. Layer 3, the Safety Router, intercepts tool calls before execution using a hybrid rule based and fine tuned classifier. Layer 2, the RL Learning Policy, trains a PPO policy on the accumulated audit log to learn the minimum viable skill set for each task type.

以下是对该论文的结构化总结：

### 1. 核心问题与整体含义（研究动机和背景）
*   **核心问题：能力过度配置（Capability Overprovisioning）**。当前的 AI 智能体运行时（如 OpenClaw）默认向所有会话开放全部工具权限。例如，一个简单的“文档摘要”任务竟然拥有与“代码部署”任务相同的 Shell 执行（`exec`）和凭据访问权限。
*   **安全风险：** 这种过度授权导致了巨大的攻击面。攻击者可以通过提示词注入（Prompt Injection）或供应链攻击（如 ClawHavoc 攻击）利用这些不必要的工具来窃取数据或控制系统。
*   **研究目标：** 实现 AI 智能体的**最小特权原则（Principle of Least Privilege）**，让智能体只知道并只能使用完成当前任务所必需的工具。

### 2. 方法论：核心思想与关键技术
Aethelgard 采用四层架构，将能力界定视为一个**可学习的优化问题**：
*   **第 1 层：能力治理器（Capability Governor）**
    *   **语义隔离：** 通过动态修改 `AGENTS.md` 文件，在系统提示词层面告诉智能体它“只拥有”某些工具。
    *   **基础设施限制：** 同步更新 `tools.deny` 配置文件，确保即使智能体尝试调用未授权工具，底层网关也会拒绝。
*   **第 2 层：强化学习策略（RL Learning Policy）**
    *   **核心算法：** 使用 **PPO（近端策略优化）** 算法。
    *   **MDP 建模：** 状态（State）为任务类型和信任等级；动作（Action）为 17 种工具的二进制掩码（开启/关闭）；奖励函数（Reward）综合考虑任务成功率、技能经济比（SER）和安全性。
*   **第 3 层：安全路由器（Safety Router）**
    *   **中间人拦截：** 部署在智能体与 LLM 之间的 FastAPI 代理。
    *   **混合分类：** 第一阶段使用基于规则的快速检查（拦截 `rm -rf` 等危险命令）；第二阶段使用微调后的轻量级 LLM（Qwen2.5-1.5B）对复杂的工具调用意图进行分类。
*   **第 4 层：审计数据库（Audit DB）**
    *   记录所有拦截事件和奖励信号，为 RL 策略提供训练数据。

### 3. 实验设计
*   **实验环境：** 在 OpenClaw v2026.3.28 部署版本上进行实测。
*   **模型选择：** 使用 DeepSeek-chat (32B) 作为智能体的主脑，Qwen2.5-7B 用于对比测试。
*   **Benchmark：** 以不受控的 OpenClaw 原始配置为基准（Baseline）。
*   **测试场景：** 
    *   **N=500 自动化批量评估：** 包含 400 个良性任务和 100 个对抗性任务（如提示词注入、加密货币挖矿尝试等）。
    *   **消融实验：** 对比“全功能系统”、“无 RL 策略”、“无安全路由器”和“原始基准”四种情况。

### 4. 资源与算力
*   **RL 训练：** 在 BGU HPC 集群上使用单张 **NVIDIA RTX 3090** GPU。
*   **训练时长：** 5 万步 PPO 训练耗时**小于 5 分钟**。
*   **推理算力：** 安全路由器中的微调 LLM 运行在 GTX 1060 (6GB) 上，单次分类延迟约 800ms-2000ms。

### 5. 实验数量与充分性
*   **实验规模：** 进行了 500 组自动化任务测试，并补充了 20 组详细的人工评估任务。
*   **充分性：** 实验覆盖了从摘要到代码执行的 6 种任务类型，并专门设计了针对性的对抗攻击测试。
*   **客观性：** 引入了“技能经济比（SER）”这一量化指标，并通过消融实验清晰地展示了各组件（尤其是 RL 策略）的边际贡献，实验设计较为严谨。

### 6. 主要结论与发现
*   **显著减小攻击面：** 在摘要任务中实现了 **73% 的工具缩减**，并 100% 消除了危险工具的暴露。
*   **效率提升：** 经过 PPO 训练后，技能经济比（SER）在真实会话中提升了 **337%**。
*   **硬性防御：** 在 500 组测试中，100% 拦截了危险的 `exec` 和子进程生成调用。
*   **关键发现：** 即使提示词注入在 LLM 层面成功（即模型生成了恶意指令），Aethelgard 的基础设施层治理仍能作为最后一道防线将其拦截。

### 7. 优点
*   **动态性：** 相比于 NemoClaw 等静态沙箱，Aethelgard 能够根据任务类型动态调整权限，更加灵活。
*   **非侵入性：** 无需修改底层 LLM 或智能体代码，通过代理和配置文件注入即可实现。
*   **防御深度：** 结合了语义层（让智能体不知道有该工具）和执行层（拦截调用）的双重保障。

### 8. 不足与局限
*   **语义内容盲区：** 该框架仅治理“工具调用”，如果智能体在回复文本中给出危险建议（如教用户如何制造炸弹）而非调用工具，该系统无法拦截。
*   **延迟问题：** LLM 分类器引入了约 2 秒的额外延迟，对于高并发场景可能存在瓶颈。
*   **数据偏差：** 用于微调路由器的样本量较小（273 个），且存在严重的类别不平衡（95% 为拦截类），可能导致对良性操作的误杀（False Positive）。
*   **状态空间有限：** RL 策略目前未考虑用户身份或历史行为，治理粒度仍有提升空间。
