## 摘要
基于大语言模型（LLM）的移动端 GUI 智能体将每次任务调用视为独立的推理过程，在每个动作步骤都需要进行完整的 LLM 推理调用。这种每步依赖的特性使其具有无状态性：昨天成功完成的任务今天仍需从头开始重新推导，在可靠性或速度上没有任何提升。我们提出了 SkillDroid，这是一个三层技能智能体，它将 LLM 引导的成功 GUI 轨迹编译为参数化技能模板（由加权元素定位器和类型化参数槽组成的 UI 动作序列），并在后续调用中直接回放，无需任何 LLM 调用。匹配级联（正则表达式模式、嵌入相似度和应用过滤）将输入的指令路由到存储的技能，而失败学习层则在技能可靠性下降时触发重新编译。在包含系统性指令变化和受控扰动的 150 轮纵向评估中，SkillDroid 实现了 85.3% 的成功率（比无状态 LLM 基准高出 23 个百分点），同时减少了 49% 的 LLM 调用。技能回放机制在 79 轮回放中实现了 100% 的完美成功率，速度是完整 LLM 执行的 2.4 倍。最关键的是，该系统随使用而改进：其成功率从 87% 向上收敛至 91%，而基准系统的成功率则从 80% 下降到 44%。

## Abstract
LLM-based mobile GUI agents treat every task invocation as an independent reasoning episode, requiring a full LLM inference call at each action step. This per-step dependence makes them stateless: a task completed successfully yesterday is re-derived from scratch today, with no improvement in reliability or speed. We present SkillDroid, a three-layer skill agent that compiles successful LLM-guided GUI trajectories into parameterized skill templates (sequences of UI actions with weighted element locators and typed parameter slots) and replays them on future invocations without any LLM calls. A matching cascade (regex patterns, embedding similarity, and app filtering) routes incoming instructions to stored skills, while a failure-learning layer triggers recompilation when skill reliability degrades. Over a 150-round longitudinal evaluation with systematic instruction variation and controlled perturbations, SkillDroid achieves an 85.3% success rate (23 percentage points above a stateless LLM baseline) while using 49% fewer LLM calls. The skill replay mechanism achieves a perfect 1000% success rate across 79 replay rounds at 2.4 times the speed of full LLM execution. Most critically, the system improves with use: its success rate converges upward from 87% to 91%, while the baseline degrades from 80% to 44%.

### 1. 核心问题与研究动机
*   **核心问题：** 现有的 LLM 移动 GUI 智能体是“无状态”的。即使是用户每天重复的操作（如“设定闹钟”），智能体每次都会从头开始进行多步推理。
*   **研究动机：** 
    *   **高延迟与高成本：** 每一动作步骤都需要调用 LLM，导致 75%–94% 的执行时间消耗在 API 请求上。
    *   **可靠性不收敛：** 无状态智能体不会从过去的成功中学习，面对指令变化时，其成功率往往随时间推移而下降而非提升。
    *   **结构化规律性：** 移动应用操作具有高度的结构重复性，这为“编译一次，多次复用”提供了可能。

### 2. 方法论：SkillDroid 三层架构
SkillDroid 的核心思想是将 LLM 的推理轨迹转化为可执行的“技能模板”。
*   **第一层：技能编译 (Compilation)**
    *   当遇到新任务时，由 LLM 引导执行。成功后，系统提取参数槽（如时间、搜索词），并为每个 UI 元素生成**加权定位器 (Weighted Element Locator)**。
    *   定位器结合了 `resourceId` (权重 0.4)、`text` (0.2)、`contentDesc` (0.15) 等 6 个特征，以应对 UI 的动态变化。
*   **第二层：投机回放 (Speculative Replay)**
    *   **匹配级联：** 通过正则表达式、嵌入向量相似度（Sentence-BERT）和目标应用过滤，将新指令匹配到现有技能。
    *   **回放机制：** 直接通过 ADB 执行动作序列，目标是 **0 次 LLM 调用**。
    *   **偏差处理：** 包含状态验证器，若发现轻微偏差（如弹窗），会尝试自动关闭或进行单步 LLM 修正；若偏差过大，则退回到第一层。
*   **第三层：失败学习 (Failure Learning)**
    *   记录回放失败的上下文。如果某个技能的失败率超过 50%，系统会标记该技能并在下次调用时触发“重新编译”，以生成更清洁、更健壮的模板。

*   **实验规模：** 总计 150 轮主实验 + 75 轮跨模型验证实验。
*   **充分性：** 实验设计较为充分，不仅对比了成功率，还详细分析了 LLM 调用次数、延迟、不同指令难度的表现以及扰动下的鲁棒性。

### 6. 主要结论与发现
*   **性能飞跃：** SkillDroid 实现了 **85.3%** 的成功率，比基准高出 **23 个百分点**。
*   **效率提升：** LLM 调用次数减少了 **49%**，回放速度比完整推理快 **2.4 倍**。
*   **进化能力：** 随着使用次数增加，SkillDroid 的成功率从 87% 提升至 91%（向上收敛），而基准系统因指令变难从 80% 下降至 44%。

### 7. 优点与亮点
*   **打破无状态限制：** 首次在移动端实现了真正意义上的“技能编译”与“零 LLM 调用”回放。
*   **分层容错设计：** 允许单步 LLM 介入和自动弹窗处理，平衡了机械回放的刚性与 LLM 推理的灵活性。
*   **隐私与成本：** 纯回放模式不向云端发送 UI 数据，既保护了隐私又极大地降低了 API 费用。

### 8. 不足与局限
*   **任务覆盖面：** 实验主要集中在 15 种常见任务，对于极其复杂的跨应用工作流或拖拽等复杂交互覆盖不足。

*   **ADB 延迟：** 使用 ADB 模拟点击存在约 100ms 的固有延迟，限制了速度的进一步提升。
