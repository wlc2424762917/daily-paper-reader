## Abstract 

Agent ecosystems increasingly rely on installable skills to extend functionality, and some skills bundle learned model artifacts as part of their execution logic. This cre-ates a supply-chain risk that is not captured by prompt injection or ordinary plugin misuse: a third-party skill may appear benign while concealing malicious behavior inside its bundled model. We present B AD SKILL , a backdoor attack formulation that targets this model-in-skill threat surface. In B AD SKILL , an adversary publishes a seemingly benign skill whose embedded model is backdoor-fine-tuned to activate a hidden payload only when routine skill parameters satisfy attacker-chosen seman-tic trigger combinations. To realize this attack, we train the embedded classifier with a composite objective that combines classification loss, margin-based separa-tion, and poison-focused optimization, and evaluate it in an OpenClaw-inspired simulation environment that preserves third-party skill installation and execution while enabling controlled multi-model study. Our benchmark spans 13 skills, in-cluding 8 triggered tasks and 5 non-trigger control skills, with a combined main evaluation set of 571 negative-class queries and 396 trigger-aligned queries. Across eight architectures (494M–7.1B parameters) from five model families, B AD SKILL 

achieves up to 99.5% average attack success rate (ASR) across the eight triggered skills while maintaining strong benign-side accuracy on negative-class queries. In poison-rate sweeps on the standard test split, a 3% poison rate already yields 91.7% ASR. The attack remains effective across the evaluated model scales and under five text perturbation types. These findings identify model-bearing skills as a distinct model supply-chain risk in agent ecosystems and motivate stronger provenance verification and behavioral vetting for third-party skill artifacts. 

## 1 Introduction 

Large language model (LLM) agents are increasingly deployed as extensible systems that orchestrate external tools, APIs, and third-party components to automate complex tasks [ 28 , 27 , 33 , 22 , 18 ]. In many emerging agent ecosystems, this extensibility is realized through skills : modular packages that encapsulate executable logic, configuration, and, in some cases, learned model artifacts. While such open skill ecosystems accelerate capability growth and developer adoption, they also introduce a largely underexamined supply-chain risk. In particular, third-party skills may be installed with limited provenance verification and little behavioral scrutiny of bundled models, allowing malicious behavior to be carried by the skill artifact itself. This threat boundary differs fundamentally from prompt injection [ 23 , 15 ], tool misuse, and other prompt- or context-mediated agent attacks [ 34 , 8 ]. Even if external inputs are sanitized, a compro-mised model artifact embedded inside a skill may still implement trigger-conditioned behavior within 

the skill execution path. Traditional backdoor attacks [ 7, 25 , 13 ] and data poisoning studies [ 24 , 3]provide useful foundations, but transferring them directly to the agent-skill setting is nontrivial. First,

Motivated by this threat model, we present BAD -SKILL , a framework for studying model-in-skill backdoors in agent skill ecosystems. As illus-trated in Figure 1, the core idea is simple: an ad-versary publishes a seemingly benign third-party skill whose bundled model is backdoor-fine-tuned to activate a hidden payload only when routine skill parameters satisfy attacker-chosen semantic trigger combinations. Rather than en-coding the malicious logic in explicit rules or hard-coded branches, B AD SKILL embeds the trigger mechanism in a learned classifier pack-aged inside the skill itself. As a result, the in-stalled skill can remain outwardly functional on benign invocations while concealing a trigger-conditioned execution policy in opaque model parameters. This makes the attack less exposed to routine code inspection and reveals a secu-rity risk that applies to model-bearing skills be-yond any single framework implementation. To ground the study in a realistic execution path, we instantiate the attack in a lightweight OpenClaw-inspired simulation environment that preserves gateway routing, skill loading, and runtime model invocation while enabling controlled evaluation across models and skills. Our main contributions are as follows: • Threat formulation. We identify and formalize a model-in-skill backdoor threat in agent skill ecosystems, in which an adversary embeds a backdoor-fine-tuned classifier inside an ostensibly benign third-party skill. This exposes a distinct model supply-chain risk surface for agent skills that bundle learned artifacts. • Targeted attack design. We introduce B AD SKILL , a backdoor framework tailored to the skill setting. The design combines semantic compositional triggers, hard negatives, and a composite training objective to encourage reliable trigger activation while maintaining strong benign-side behavior on negative-class inputs. • Case-study evaluation in a simulated skill environment. We evaluate B AD SKILL across eight model architectures (494M–7.1B parameters) from five model families and eight triggered skill categories in an OpenClaw-inspired simulation environment. B AD SKILL reaches up to 99.5% average ASR across the eight triggered skills, exhibits strong poison-rate efficiency, and remains effective under five text perturbation types. These results motivate stronger provenance verification and behavioral vetting for third-party model-bearing skill artifacts. 

## 2 Preliminaries 

2.1 Skill-Centric Agent Execution 

We model a skill-centric agent system as a mapping F : Q → R from user queries to final responses. A skill sk ∈ S is a reusable artifact that exposes a structured interface for a task family Tk, such as summarization or conversion. A task τ ∈ T k denotes the semantic capability supported by that skill, while a query q ∈ Q is a concrete user request that instantiates one task instance together with its content and control requirements. The execution stack consists of three components: a gateway LLM 

gϕ that interprets the query and selects a skill identifier, a skill loader ℓ that resolves and initializes 2the selected artifact, and a skill-specific parser ρk that converts the query into a structured invocation understood by the selected skill. The resulting execution path is 

F(q) = sk

This formulation separates the three objects central to our setting: the installed skill artifact, the task family it claims to support, and the incoming queries that instantiate concrete task instances. It also captures a common deployment pattern in which skills are distributed as self-contained artifacts that include code, configuration, and optionally learned weights. The security boundary of interest therefore lies at the skill artifact itself: once installed, the bundled model can influence execution even when the surrounding agent runtime remains unchanged. 

2.2 Threat Model Attacker’s goal. The adversary aims to implant a backdoor into the model bundled inside a third-party skill. Following the standard backdoor-learning setting [ 13 ], the attack must satisfy two objectives simultaneously. (1) Effectiveness: when a user query instantiates the attacker-chosen trigger pattern for the target skill, the installed skill should route execution to a hidden payload branch. (2) Benign-side preservation: when the query corresponds to an ordinary task instance, the skill should continue to exhibit its advertised benign functionality so that routine use and lightweight manual inspection do not reveal the compromise. We quantify these later using benign accuracy (BA) and attack success rate (ASR). 

Attacker’s knowledge. We assume a gray-box adversary with access to public information about the target skill interface, packaging format, and installation workflow, together with generic knowledge of how the skill category is typically used. The adversary may also obtain a public pre-trained language model as the starting point for the bundled classifier. The adversary does not know the victim’s private prompts, conversation history, or host-side runtime configuration. 

Attacker’s capability. The adversary acts only at skill-development time. Specifically, the adversary can (i) design the skill interface, including the task schema and parser behavior, (ii) construct poisoned and hard-negative training queries, (iii) fine-tune a compact classifier, and (iv) distribute the resulting model inside an otherwise ordinary skill bundle through the normal installation channel. The adversary cannot modify the core agent runtime, the gateway LLM, or the host privilege model. Thus, the threat is a supply-chain attack in which the malicious behavior is introduced before deployment and remains encoded in the installed skill artifact.

## 3 BAD SKILL 

BAD SKILL is a two-stage attack pipeline. As shown in Figure 2, Stage I learns a trigger-conditioned classifier over structured skill invocations, and Stage II packages the trained classifier into the skill artifact to route execution between benign and hidden-payload branches. The end-to-end procedure is summarized in Algorithm 1. 

3.1 Stage I: Trigger-Aware Optimization 

The design of Stage I revolves around three requirements: the trigger should operate on structured skill parameters, maintain selectivity when confronted with variants closely resembling the trigger conditions, and remain learnable under conditions of sparse poisoning supervision. 

Structured invocation representation. BAD SKILL operates on the structured parameter interface of a skill rather than on raw free-form prompts alone. For a target skill sk, each training example begins as a user query q that requests one concrete task instance. A skill-specific parser ρk maps the query to a structured invocation

4.3 Systematic Analysis Trigger Complexity. Figure 3 reveals a clear trade-off between trigger simplicity and trigger specificity. Single-field triggers are generally too weak because they can be confused with benign near-misses, making it difficult for the classifier to distinguish true activations from partially overlapping inputs. At the other extreme, triggers with four or more fields become overly specific, which may reduce their ability to generalize as stable activation patterns. The most effective regime is therefore the middle one: triggers spanning two or three parameters consistently achieve ASR ≥ 95% across all three Qwen scales. This pattern is important because it shows that B AD SKILL works best not when the trigger is maximally simple or maximally complex, but when it forms a short conjunction of benign-looking parameters. In practice, this is precisely the most concerning regime, since such triggers remain compact enough to be learnable and reusable while still appearing ordinary at the level of individual fields.

Trigger Robustness. Table 3 shows that the learned trigger is not brittle to moderate perturbations of the surface input, although different perturbation types affect the attack to different degrees. The degradation is gradual rather than catastrophic, which already suggests that the trigger is not stored as a fragile literal pattern. This interpretation is reinforced by the relatively mild impact of word reordering, since an attack that relied primarily on one exact token sequence would be expected to fail more sharply under this perturbation. By contrast, character-level corruption and heavier typo injection are more harmful, especially at 10% typo noise, which likely reflects growing instability in lexical recovery and downstream parsing rather than failure of the compositional trigger idea itself. Even under these stronger perturbations, however, the attack still retains substantial effectiveness on most skills. The overall picture is therefore that moderate surface-level variation is unlikely to neutralize the backdoor in practice, and that effective defenses must reason about structured parameter semantics rather than simple string-level anomalies. 

## 5 Related Work 

Backdoor Attacks in Language Models. Backdoor attacks study how models can be trained to exhibit hidden behaviors that remain dormant on benign inputs but activate under attacker-controlled conditions [ 25 , 14 , 13 ]. In NLP, prior work has explored textual triggers, weight poisoning, and stealthier syntactic or semantic trigger designs [ 12 , 16 , 32 , 6 ]. More recent studies show that instruction tuning and post-training customization can also induce persistent trigger-conditioned failures in large language models [ 26 , 17 , 29 , 30 , 37 , 19 , 11 ]. Our work is closely related to this line, but differs in where the compromised model is deployed: instead of exposing a backdoored model as a standalone LLM endpoint, we study the case in which learned weights are embedded inside a reusable skill artifact and activated through routine skill parameters. 

Security of Tool-Using and Agentic Systems. A separate line of work studies security failures in LLM agents and tool-using systems. Existing research has examined indirect prompt injection, unsafe tool use, and vulnerabilities arising from agent planning, tool execution, and environmental interaction [ 15 , 23 , 34 , 21 , 36 ]. Related work has also shown that poisoning external state, such as memories or knowledge bases, can redirect downstream agent behavior without modifying the core model itself [ 8]. These studies establish that prompts, context, retrieved content, and tool outputs can all act as attack carriers. In contrast, our threat model shifts the carrier from the input channel or external state to the installed skill artifact itself: the prompt channel and runtime may remain unchanged, while the malicious behavior is encoded directly in the bundled model weights of a third-party skill. 

Supply-Chain Risks for Model-Bearing Artifacts. Prior work on poisoning and provenance has emphasized that model security is shaped not only by training data and optimization, but also by how models are packaged, distributed, and reused [ 24 , 3, 10 , 20 ]. In parallel, modern agent ecosystems increasingly encourage modular composition through tools, skills, and external plugins [ 33 , 22 , 18 , 5 , 31 ]. Our work lies at the intersection of these two lines. The key observation is that once a third-party skill is allowed to bundle learned weights, skill installation becomes a model supply-chain problem rather than only a code-review problem. B AD SKILL therefore complements prior work on both LLM backdoors and agent security by identifying model-bearing skills as a distinct attack surface in which the malicious logic is carried by the installed artifact rather than by prompts, retrieved context, or external tools. 96 Discussion and Limitations 

Implications for defenses. BAD SKILL highlights a threat surface that is only partially covered by existing defenses for agent systems. Many current protections focus on prompts, retrieved context, or tool outputs, whereas our attack is carried by the installed skill artifact itself. This suggests that prompt- and action-level safeguards alone are insufficient once third-party skills are allowed to bundle learned weights. A more suitable defense view is therefore to treat model-bearing skills as provenance-sensitive artifacts, combining artifact verification, behavior-level probing, and runtime monitoring for suspicious conditional execution. 

Implications for agent ecosystems. Our results across multiple model families suggest that the risk is not confined to a single backbone or implementation. More importantly, once skills are allowed to package opaque learned components, skill installation is no longer only a software-plugin problem; it also becomes a model supply-chain problem. In this setting, code inspection and integrity checking remain useful but incomplete, because they cannot directly reveal trigger-conditioned behavior encoded in model parameters. This points to the need for agent platforms to distinguish ordinary tool wrappers from model-bearing extensions in their trust and review policies. 

Limitations and scope. This study has several limitations. First, we evaluate models only up to 7.1B parameters, so behavior at larger scales remains unclear. Second, our experiments are conducted in a lightweight simulated skill environment rather than across multiple production agent stacks. Third, we do not include a dedicated defense evaluation, so detection and mitigation effectiveness remain open questions. Finally, we restrict the study to English triggers and a benign canary payload, which keeps the evaluation controlled but does not exhaust the broader threat space. Accordingly, BAD SKILL should be viewed as a focused empirical study of model-in-skill backdoors and a starting point for future work on defenses, transferability, and ecosystem-level risk assessment. 

## 7 Conclusion 

We presented B AD SKILL , a focused empirical study of model-in-skill poisoning in agent ecosystems. The central finding is that model-bearing skills introduce a distinct attack surface: a third-party skill can appear benign during ordinary use while encoding hidden, trigger-conditioned behavior in its bundled model. By targeting structured skill parameters rather than unconstrained free-form prompts, B AD SKILL shows that backdoor activation can be realized through compositional, routine-looking interface conditions, making the threat meaningfully different from prompt injection or ordinary plugin misuse. Across eight architectures (494M–7.1B parameters) from five model families, BAD SKILL achieves consistently high attack success rates, remains poison-efficient, and retains substantial effectiveness under multiple input perturbations. These results suggest that the risk is neither confined to one model family nor easily exposed by benign-side accuracy alone. More broadly, our study indicates that once third-party skills are allowed to bundle learned weights, skill installation must be treated not only as a software-plugin problem but also as a model supply-chain problem. We hope this work motivates future research on artifact provenance, behavioral vetting, and runtime safeguards for model-bearing skills, and more generally on security evaluation protocols that account for learned components inside agent extensions. 10 References
