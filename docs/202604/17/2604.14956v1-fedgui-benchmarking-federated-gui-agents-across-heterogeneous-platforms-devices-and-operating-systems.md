## 摘要
使用传统的集中式方法训练 GUI 智能体面临着巨大的成本和可扩展性挑战。联邦学习 (FL) 提供了一个极具前景的解决方案，但由于缺乏能够捕捉现实世界跨平台异构性的基准测试，其潜力受到了限制。为了填补这一空白，我们推出了 FedGUI，这是首个用于在移动端、Web 端和桌面端开发及评估联邦 GUI 智能体的全面基准测试。FedGUI 提供了一套包含六个精选数据集的工具包，旨在系统地研究四种关键的异构性类型：跨平台、跨设备、跨操作系统和跨来源。广泛的实验揭示了几个关键见解：首先，我们证明了跨平台协作可以提高性能，将此前仅限于移动端的联邦学习扩展到了多样化的 GUI 环境中；其次，我们证明了存在不同的异构性维度，并确定平台和操作系统是影响最大的因素。FedGUI 为社区构建更具可扩展性且保护隐私的 GUI 智能体以进行实际部署提供了重要基础。我们的代码和数据已在 https://github.com/wwh0411/FedGUI 公开。

## Abstract
Training GUI agents with traditional centralized methods faces significant cost and scalability challenges. Federated learning (FL) offers a promising solution, yet its potential is hindered by the lack of benchmarks that capture real-world, cross-platform heterogeneity. To bridge this gap, we introduce FedGUI, the first comprehensive benchmark for developing and evaluating federated GUI agents across mobile, web, and desktop platforms. FedGUI provides a suite of six curated datasets to systematically study four crucial types of heterogeneity: cross-platform, cross-device, cross-OS, and cross-source. Extensive experiments reveal several key insights: First, we show that cross-platform collaboration improves performance, extending prior mobile-only federated learning to diverse GUI environments; Second, we demonstrate the presence of distinct heterogeneity dimensions and identify platform and OS as the most influential factors. FedGUI provides a vital foundation for the community to build more scalable and privacy-preserving GUI agents for real-world deployment. Our code and data are publicly available at https://github.com/wwh0411/FedGUI..

*   **研究动机**：传统的 GUI（图形用户界面）智能体训练依赖于中心化数据收集，面临高昂的标注成本和有限的可扩展性。虽然现实世界中存在海量的用户交互信号，但由于隐私限制，这些数据无法直接共享。
*   **核心问题**：现有的联邦学习（FL）研究（如 FedMABench）仅局限于单一平台（如 Android），忽视了 Web 和桌面端的潜力，且未能刻画真实世界中跨平台、跨设备、跨操作系统的复杂**异构性（Heterogeneity）**。
*   **整体含义**：本文提出了 **FedGUI**，这是首个全面支持跨移动、Web 和桌面平台的联邦 GUI 智能体基准测试，旨在探索如何在保护隐私的前提下，利用异构分布式数据提升智能体的通用能力。

### 2. 论文提出的方法论
*   **核心思想**：构建一个统一的联邦学习框架，通过模拟真实世界的四种异构性维度，系统性地评估不同联邦算法在训练跨平台 GUI 智能体时的表现。
*   **关键技术细节**：
    *   **统一动作空间**：设计了包含 17 种动作的统一空间（如 CLICK, TYPE, SCROLL 等），将不同平台的特定操作映射到统一语义中，确保模型在异构环境下的参数聚合具有一致性。
    *   **四种异构性建模**：
        1.  **跨平台（Cross-Platform）**：移动、Web、桌面之间的差异。
        2.  **跨设备（Cross-Device）**：同平台下不同硬件（如不同型号的手机、平板）的视觉差异。
        3.  **跨操作系统（Cross-OS）**：不同系统（Ubuntu, macOS, Windows）的交互逻辑差异。
        4.  **跨来源（Cross-Source）**：不同数据采集方法（人工标注 vs. 自动合成）带来的分布偏移。
    *   **联邦分区策略**：设计了 IID（独立同分布）、Non-Uniform（数量不均）、Partial（部分观测）和 Skew（极端偏斜/标签偏移）等多种分区方式来模拟非独立同分布（Non-IID）挑战。

    *   **基础模型**：支持 20 多种 VLM，主实验采用 Qwen2-VL-7B，并评估了 GPT-4o/5 等闭源模型作为参考。
    *   **对比算法**：集成了 7 种主流联邦学习算法（FedAvg, FedProx, SCAFFOLD, FedAvgM, FedAdam, FedYogi, FedAdagrad）。
    *   **评估指标**：动作类型准确率（Type）、定位准确率（Grounding）、任务成功率（SR）。

### 4. 资源与算力
*   **硬件设备**：实验主要在两台 **NVIDIA GeForce RTX 3090** (24GB) 上进行。

*   **实验规模**：论文进行了极其详尽的实验，包括：

### 6. 论文的主要结论与发现
*   **跨平台协作效应**：增加来自不同平台的用户参与联邦学习能显著提升模型性能，即使数据高度异构，协作也比单打独斗强（“Salvage Effect”）。
*   **异构性影响排序**：**平台（Platform）**和**操作系统（OS）**层面的异构性对性能的负面影响最大，远超设备型号或数据来源的差异。
*   **算法鲁棒性**：自适应优化器类算法（如 **FedYogi** 和 **FedAdam**）在处理跨平台偏斜分布时表现最稳健，优于传统的 FedAvg 或修正类算法（如 FedProx）。
*   **模型规模与性能**：经过联邦微调的小型开源模型（如 Qwen3-VL-2B）在特定 GUI 任务上可以超越未微调的大型闭源模型（如 GPT-5 级别）。

### 7. 优点
*   **填补空白**：首次将 GUI 智能体的联邦学习研究从单一移动端扩展到全平台生态。
*   **建模真实**：对异构性的分类（平台、设备、系统、来源）非常符合工业界部署的实际痛点。
*   **工具链完备**：基于 `ms-swift` 构建，支持大量模型和算法，代码开源，极大地降低了后续研究者的门槛。
*   **动作空间统一**：解决了异构平台间动作定义冲突的问题，为跨平台智能体研究提供了标准。

### 8. 不足与局限
*   **数据来源偏差**：虽然模拟了异构性，但数据仍源于公开的学术数据集，而非真实的、带有噪声和高度个性化的用户原生交互数据。
*   **隐私机制深度不足**：论文重点在于性能基准测试，虽然提及了联邦学习的隐私属性，但未深入探讨差分隐私（DP）或安全多方计算（MPC）在 GUI 截图（可能包含敏感视觉信息）上的具体开销与性能权衡。
*   **动态环境挑战**：实验主要基于静态轨迹回放，未充分考虑 GUI 环境动态变化（如网络延迟、UI 更新）对联邦聚合的影响。
