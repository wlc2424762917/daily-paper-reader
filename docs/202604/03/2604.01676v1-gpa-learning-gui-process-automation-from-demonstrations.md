## 摘要
GUI 流程自动化 (GPA) 是一种轻量级且通用的基于视觉的机器人流程自动化 (RPA)，仅需单次演示即可实现快速且稳定的流程回放。针对传统 RPA 的脆弱性以及当前基于视觉语言模型 (VLM) 的 GUI 智能体的不确定性风险，GPA 引入了三大核心优势：(1) 通过基于序列蒙特卡洛 (Sequential Monte Carlo) 的定位实现鲁棒性，以处理缩放和检测的不确定性；(2) 通过就绪状态校准 (readiness calibration) 保障确定性和可靠性；(3) 通过快速、完全本地的执行保护隐私。该方法提供了企业级工作流所需的适应性、鲁棒性和安全性。它还可以作为 MCP/CLI 工具供其他具备编程能力的智能体使用，使智能体仅负责推理和编排，而由 GPA 处理 GUI 执行。我们进行了一项初步实验，将 GPA 与 Gemini 3 Pro（配备 CUA 工具）进行了对比，结果发现 GPA 在完成长程 GUI 任务时，不仅成功率更高，且执行速度快了 10 倍。

## Abstract
GUI Process Automation (GPA) is a lightweight but general vision-based Robotic Process Automation (RPA), which enables fast and stable process replay with only a single demo. Addressing the fragility of traditional RPA and the non-deterministic risks of current vision language model-based GUI agents, GPA introduces three core benefits: (1) Robustness via Sequential Monte Carlo-based localization to handle rescaling and detection uncertainty; (2) Deterministic and Reliability safeguarded by readiness calibration; and (3) Privacy through fast, fully local execution. This approach delivers the adaptability, robustness, and security required for enterprise workflows. It can also be used as an MCP/CLI tool by other agents with coding capabilities so that the agent only reasons and orchestrates while GPA handles the GUI execution. We conducted a pilot experiment to compare GPA with Gemini 3 Pro (with CUA tools) and found that GPA achieves higher success rate with 10 times faster execution speed in finishing long-horizon GUI tasks.

### 2. 方法论
GPA 采用“单次演示-自动化回放”的架构，核心思想是将 UI 定位建模为一种**图匹配与概率推理问题**。
*   **UI 图表示**：将屏幕解析为图 $G=(V, E)$。节点 $V$ 包含边界框、OCR 文本和 IconCLIP 视觉嵌入；边 $E$ 通过 k-近邻（k-NN）连接空间邻近元素。
*   **序贯蒙特卡洛（SMC）定位**：
    *   **核心公式**：推导后验概率 $p(\theta | Z)$，其中 $\theta = [x, y, s_x, s_y]$ 代表目标位置和缩放比例。
    *   **几何上下文**：不仅匹配目标本身，还利用周围“邻居节点”的相对位置进行推理。即使目标外观改变或检测器漏检，也能通过邻居节点推算出目标位置。
*   **就绪校准（Readiness Checking）**：引入置信度评分 $C = \tilde{p}(Z|\theta) \times C_{spatial}$。只有当观测一致性（似然度）和空间收敛性（方差）均达标时才执行动作，否则进行重试。
*   **有限状态机（FSM）控制**：通过“执行-决策-重试/失败”的闭环逻辑管理工作流，防止长程任务中的状态漂移。
*   **预检流水线（Precheck Pipeline）**：在环境等待动作响应的闲置时间内，后台预先处理后续步骤，进一步降低延迟。

### 7. 优点
*   **鲁棒的定位机制**：利用空间几何上下文（邻居节点）解决了 UI 元素检测不稳定的痛点。

*   **缺乏推理能力**：GPA 本质上是“记录与回放”系统，无法处理需要逻辑判断的任务（例如：如果日期是周一则点 A，否则点 B）。
