## 摘要
记忆增强生成 (MAG) 通过外部记忆扩展大语言模型以支持长上下文推理，但现有方法普遍将记忆视为智能体调用的外部服务，将存储委托给分块、嵌入和图提取等独立的流水线。这种架构上的分离意味着存储知识的系统并不理解知识，导致智能体意图记忆的内容与流水线实际捕获的内容之间出现语义漂移、跨智能体协作上下文丢失以及故障后恢复脆弱。在本文中，我们提出了 ByteRover，一种反转了记忆流水线的智能体原生记忆架构：负责任务推理的同一个 LLM 同时负责知识的策展、结构化和检索。ByteRover 将知识表示为层级化上下文树 (Context Tree)，这是一种基于文件的知识图谱，按领域 (Domain)、主题 (Topic)、子主题 (Subtopic) 和条目 (Entry) 组织，其中每个条目都带有显式关系、来源以及包含重要性评分、成熟度等级和近期衰减的自适应知识生命周期 (AKL)。检索采用五层渐进式策略，在无需 LLM 调用的情况下，能以低于 100 毫秒的延迟解决大多数查询，仅针对新颖问题才升级为智能体推理。在 LoCoMo 和 LongMemEval 上的实验表明，ByteRover 在 LoCoMo 上实现了最先进的准确率，在 LongMemEval 上取得了具有竞争力的结果，同时无需任何外部基础设施、向量数据库、图数据库或嵌入服务，所有知识均以人类可读的 Markdown 文件形式存储在本地文件系统中。

## Abstract
Memory-Augmented Generation (MAG) extends large language models with external memory to support long-context reasoning, but existing approaches universally treat memory as an external service that agents call into, delegating storage to separate pipelines of chunking, embedding, and graph extraction. This architectural separation means the system that stores knowledge does not understand it, leading to semantic drift between what the agent intended to remember and what the pipeline actually captured, loss of coordination context across agents, and fragile recovery after failures. In this paper, we propose ByteRover, an agent-native memory architecture that inverts the memory pipeline: the same LLM that reasons about a task also curates, structures, and retrieves knowledge. ByteRover represents knowledge in a hierarchical Context Tree, a file-based knowledge graph organized as Domain, Topic, Subtopic, and Entry, where each entry carries explicit relations, provenance, and an Adaptive Knowledge Lifecycle (AKL) with importance scoring, maturity tiers, and recency decay. Retrieval uses a 5-tier progressive strategy that resolves most queries at sub-100 ms latency without LLM calls, escalating to agentic reasoning only for novel questions. Experiments on LoCoMo and LongMemEval demonstrate that ByteRover achieves state-of-the-art accuracy on LoCoMo and competitive results on LongMemEval while requiring zero external infrastructure, no vector database, no graph database, no embedding service, with all knowledge stored as human-readable markdown files on the local filesystem.

### 1. 核心问题与整体含义
*   **研究动机**：现有的记忆增强生成（MAG）系统普遍将记忆视为“外部服务”，即智能体将数据发送给独立的流水线进行分块、嵌入或图提取。这种架构导致了三个核心问题：
    1.  **语义漂移**：智能体想要记忆的深层含义与向量检索捕获的表面相似性不一致。
    2.  **协作上下文丢失**：多智能体共享记忆时，只共享了数据而丢失了推理逻辑（“为什么这么存”）。
    3.  **恢复脆弱性**：系统崩溃后，智能体难以通过外部数据库重建复杂的任务状态。
*   **核心目标**：提出一种“智能体原生（Agent-Native）”的记忆架构，让负责推理的 LLM 同时负责记忆的策展（Curation）和组织，消除存储与理解之间的鸿沟。

### 2. 方法论
*   **核心思想**：将记忆构建为**层级化上下文树（Context Tree）**，以本地文件系统中的 Markdown 文件形式存储，由 LLM 通过显式的工具调用（ADD, UPDATE 等）进行管理。
*   **关键技术细节**：
    *   **上下文树结构**：按照 `Domain > Topic > Subtopic > Entry` 四级组织。每个条目包含关系（Relations）、原始概念（Provenance）、叙述性结构（Narrative）和代码片段（Snippets）。
    *   **自适应知识生命周期（AKL）**：引入重要性评分（访问+3，更新+5）、成熟度等级（草稿→验证→核心）和近期衰减因子（半衰期约21天）。综合评分公式结合了 BM25 检索相关性、归一化重要性和时间衰减。
    *   **五级渐进式检索策略**：
        *   Tier 0/1：精确/模糊缓存命中（~0-50ms）。
        *   Tier 2：高性能全文检索（MiniSearch, ~100ms）。
        *   Tier 3：单次优化 LLM 调用（带预取上下文，<5s）。
        *   Tier 4：全智能体推理循环（多步工具调用，8-15s）。
    *   **策展流水线**：包含三阶段压缩策略（L1 摘要、L2 激进摘要、L3 确定性截断），确保超长输入也能被处理。

### 7. 优点
*   **架构创新**：反转了传统的 MAG 流水线，使智能体对记忆拥有完全的控制权和理解力。
*   **易维护性**：使用人类可读的 Markdown 文件，支持版本控制（如 Git），无需复杂的数据库运维。
*   **反馈闭环**：智能体能看到记忆操作的成功/失败状态并实时调整，解决了语义漂移问题。
*   **低门槛**：无需嵌入模型和向量数据库，降低了部署复杂度和成本。

*   **扩展瓶颈**：虽然目前支持万级条目，但在极大规模（如百万级）下，纯文件系统存储和顺序任务队列可能面临性能挑战。
